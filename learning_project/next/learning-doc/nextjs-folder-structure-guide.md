# Next.js Folder Structure Guide - Complete Beginner's Guide

## 🏗️ What is Next.js?

Next.js is a React framework that makes building websites easier. Think of it as React with superpowers! It adds features like automatic page routing, server-side rendering, and easy deployment.

## 📁 Your Next.js Project Structure Explained

### 🏠 Root Level (Main Directory)

This is the foundation of your project. Contains all the configuration files that tell your project how to work.

#### Key Files:

**`package.json` - Your Project's Shopping List**
```json
{
  "name": "next",
  "version": "0.1.0",
  "scripts": {
    "dev": "next dev",     // Command to start development server
    "build": "next build", // Command to build for production
    "start": "next start"  // Command to start production server
  }
}
```
- **Purpose**: Lists all the tools (libraries) your project uses
- **Key Commands**:
  - `npm run dev` - Starts your development server
  - `npm run build` - Prepares your app for production
  - `npm run start` - Runs your production app

**`next.config.ts` - Your Project's Settings Manual**
- **Purpose**: Tells Next.js how to behave
- **Common Settings**: Image optimization, environment variables, custom webpack config

**`tsconfig.json` - TypeScript Configuration**
- **Purpose**: Helps catch errors before they happen
- **Benefit**: Better code suggestions and error detection

**`public/` - Your Public Storage Room**
```
public/
├── file.svg
├── globe.svg
├── next.svg
├── vercel.svg
└── window.svg
```
- **Purpose**: Contains files that can be accessed directly by users
- **Examples**: Images, icons, fonts, PDFs
- **URL Access**: `public/logo.png` can be accessed as `yoursite.com/logo.png`

### 📂 app/ - Your Main Application (The Living Room)

This is where all your website pages live. Next.js uses "file-based routing" - meaning the file names create your website URLs.

#### Key Files in app/:

**`layout.tsx` - Your House Blueprint**
```tsx
import type { Metadata } from "next";
import { Geist, Geist_Mono } from "next/font/google";
import "./globals.css";

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en">
      <body
        className={`${geistSans.variable} ${geistMono.variable} antialiased`}
      >
        {children}
      </body>
    </html>
  );
}
```
- **Purpose**: Main structure that wraps ALL pages
- **Contains**: Things that appear on every page (header, footer, navigation)
- **Think of it as**: The walls and roof of your house
- **Key Concept**: The `{children}` prop is where your page content goes

**`page.tsx` - Your Home Page**
```tsx
export default function Home() {
  return (
    <h1>Home</h1>
  )
}
```
- **Purpose**: Your website's main page
- **URL**: `yoursite.com/` (root URL)
- **Structure**: A React component that returns JSX

**`globals.css` - Your House Paint and Decorations**
```css
@import "tailwindcss";

:root {
  --background: #ffffff;
  --foreground: #171717;
}

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --font-sans: var(--font-geist-sans);
  --font-mono: var(--font-geist-mono);
}

@media (prefers-color-scheme: dark) {
  :root {
    --background: #0a0a0a;
    --foreground: #ededed;
  }
}

body {
  background: var(--background);
  color: var(--foreground);
  font-family: Arial, Helvetica, sans-serif;
}
```
- **Purpose**: Global styles that apply to your entire website
- **Features**: Colors, fonts, basic layout styles, dark mode support
- **Tailwind CSS**: Utility-first CSS framework for easy styling

## 🛣️ How Routing Works (File-Based Routing)

Next.js uses file names to create URLs automatically. No manual configuration needed!

### Basic Routing Examples:

| File Path | URL | Description |
|-----------|-----|-------------|
| `app/page.tsx` | `yoursite.com/` | Homepage |
| `app/about/page.tsx` | `yoursite.com/about` | About page |
| `app/contact/page.tsx` | `yoursite.com/contact` | Contact page |
| `app/blog/page.tsx` | `yoursite.com/blog` | Blog page |

### Creating a New Page - Step by Step:

1. **Create a new folder** inside `app/`
   ```
   app/
   ├── about/     ← New folder
   ├── contact/
   └── page.tsx
   ```

2. **Create a page.tsx file** inside that folder
   ```
   app/
   ├── about/
   │   └── page.tsx    ← New file
   ├── contact/
   └── page.tsx
   ```

3. **Add content to your new page**
   ```tsx
   // app/about/page.tsx
   export default function About() {
     return (
       <div>
         <h1>About Us</h1>
         <p>This is our about page!</p>
       </div>
     )
   }
   ```

4. **Visit your new page** at `yoursite.com/about`

## 🎨 Special Folders and Features

### `app/api/` - Your Backroom Operations
```
app/
├── api/
│   ├── users/
│   │   └── route.tsx
│   └── posts/
│       └── route.tsx
├── about/
└── page.tsx
```
- **Purpose**: Contains API endpoints (backend functions)
- **Example**: `app/api/users/route.tsx` creates an API at `/api/users`
- **Use Case**: Database operations, external API calls, form submissions

### `app/(auth)/` - Route Groups
```
app/
├── (auth)/
│   ├── login/
│   │   └── page.tsx
│   └── register/
│       └── page.tsx
├── about/
└── page.tsx
```
- **Purpose**: Organize routes without affecting URL structure
- **URL**: `/login` and `/register` (parentheses don't appear in URL)
- **Use Case**: Group related pages together

### Dynamic Routes
```
app/
├── posts/
│   ├── page.tsx          → /posts
│   └── [id]/
│       └── page.tsx      → /posts/1, /posts/2, /posts/any-id
├── users/
│   └── [username]/
│       └── page.tsx      → /users/john, /users/mary
└── page.tsx
```
- **Purpose**: Create pages that handle dynamic data
- **Syntax**: Use square brackets `[parameter]` for dynamic parts
- **Example**: Blog posts, user profiles, product pages

## 🧩 Understanding the Layout System

### How Layouts Work:
```
layout.tsx (Root Layout)
├── Header (appears on all pages)
├── {children} (Page content changes here)
│   ├── page.tsx (Home page content)
│   ├── about/page.tsx (About page content)
│   └── contact/page.tsx (Contact page content)
└── Footer (appears on all pages)
```

### Creating a Custom Layout:
```tsx
// app/about/layout.tsx
export default function AboutLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <div>
      <h2>About Section</h2>
      <nav>
        <a href="/about/team">Our Team</a>
        <a href="/about/history">Our History</a>
      </nav>
      {children} {/* This is where page.tsx content goes */}
    </div>
  );
}
```

## 🚨 Common Mistakes & How to Fix Them

### 1. Empty Return Statement
**❌ Wrong:**
```tsx
function Home() {
  return(
    // Nothing here!
  )
}
```

**✅ Correct:**
```tsx
export default function Home() {
  return (
    <div>
      <h1>Welcome to Next.js!</h1>
      <p>This is your homepage.</p>
    </div>
  )
}
```

**Problem**: Empty return statements cause TypeScript errors
**Solution**: Always return some JSX content inside the return statement

### 2. Missing Export Statement
**❌ Wrong:**
```tsx
function Home() {
  return <h1>Home</h1>
}
// Forgot to export!
```

**✅ Correct:**
```tsx
export default function Home() {
  return <h1>Home</h1>
}
```

**Problem**: Page won't render without export
**Solution**: Always use `export default` for page components

### 3. Wrong File Extension
**❌ Wrong:**
- `page.js` (JavaScript)
- `page.ts` (TypeScript without JSX)

**✅ Correct:**
- `page.tsx` (TypeScript with JSX)

**Problem**: Next.js expects `.tsx` files for React components
**Solution**: Always use `.tsx` extension for page files

## 💡 Key Concepts Summary

### 1. File-Based Routing
- **File names = URLs**
- No manual route configuration needed
- Automatic page creation

### 2. Layout System
- **`layout.tsx`** wraps all pages
- **`{children}`** prop contains page content
- Can create nested layouts

### 3. Components
- Each `page.tsx` is a React component
- Returns JSX (HTML-like syntax)
- Can use React hooks and state

### 4. Automatic Features
- **Hot Reload**: Changes appear instantly
- **Image Optimization**: Automatic image optimization
- **Code Splitting**: Only loads necessary code
- **SEO**: Built-in SEO features
