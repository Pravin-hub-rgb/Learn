# Doc 1.1 — Pehla App Banao + Emulator Pe Chalao

## Abhi Tak Kya Hua

1.0 mein samjha — React Native kya hai, Expo kya hai, Metro kya hai, Emulator kya hai.

Ab practically karte hain — ek naya project banayenge aur emulator pe chalayenge.
Sirf yeh — koi component nahi, koi styling nahi. Bas "kaam karra hai" confirm karna hai.

---

## Humara Goal Is Doc Mein

Do kaam:
1. Ek naya Expo project banana
2. Woh emulator pe chalana — screen pe kuch dikhna chahiye

Bas itna. Chalo karte hain.

---

## Step 1 — Project Banana

Terminal kholo. Apna projects folder open karo — jahan bhi rakhte ho apna kaam.

Yeh command run karo:

```bash
npx create-expo-app@latest RNSeekhna --template blank-typescript
```

**Ruko — yeh tod ke samjhte hain:**

`npx` — yeh Node ka ek tool hai. Matlab — "yeh package download kar aur seedha run kar, pehle install mat kar globally."

`create-expo-app@latest` — yeh Expo ka official project-creator tool hai. `@latest` matlab — sabse nayi version use karo.

`RNSeekhna` — tera project ka naam. Kuch bhi rakh sakta tha — maine yeh rakha.

`--template blank-typescript` — yeh bata raha hai ki kaun sa "starting point" chahiye.

**Template kya hota hai?**

Socho ghar banana hai. Do options hain:
- **Khali plot** — sab kuch khud banao
- **Ready-made flat** — basic structure already hai, tune sirf apna saman rakhna hai

`blank-typescript` ek ready-made khali flat hai — TypeScript already configured, koi extra cheez nahi. Bilkul saaf.

---

**Command run karo — yeh dikhega:**

```
✔ What is your app named? › RNSeekhna
✔ Downloaded and extracted project files.
✔ Installed dependencies.

✅ Your project is ready!
```

Pehli baar thoda time lagega — internet se files download ho rahi hain. **1-2 minute wait karo.**

## Step 3 — Andar Kya Hai? Dekho

VS Code mein left side pe yeh files dikhenge:

```
RNSeekhna/
├── assets/
│   ├── adaptive-icon.png
│   ├── favicon.png
│   ├── icon.png
│   └── splash-icon.png
├── .gitignore
├── App.tsx         ← ★ yahi sabse important file hai abhi
├── app.json
├── package.json
├── tsconfig.json
└── package-lock.json
```

**Har file kya kaam karti hai:**

`App.tsx` — **yahan se app shuru hoti hai.** Jaise Next.js mein `pages/index.tsx` hota tha — entry point. Abhi sirf yahi file matter karti hai.

`app.json` — app ki settings. Naam, icon, splash screen — sab yahan define hota hai.

`package.json` — tune Next.js mein dekha hoga — dependencies ki list. Same cheez yahan bhi.

`tsconfig.json` — TypeScript ki settings. Tune Next.js mein bhi dekha hoga. Same.

`assets/` — images folder. Icons, splash screen wagera.

## Step 4 — App.tsx Kholo Aur Dekho

`App.tsx` file kholo. Yeh dikhega:

```tsx
import { StatusBar } from 'expo-status-bar';
import { StyleSheet, Text, View } from 'react-native';

export default function App() {
  return (
    <View style={styles.container}>
      <Text>Open up App.tsx to start working on your app!</Text>
      <StatusBar style="auto" />
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  },
});
```

**Abhi isko mat chhedo. Sirf dekho.**

Next.js wala component yaad hai? Waise hi hai yeh — ek function jo JSX return karta hai.

Kuch cheezein alag dikhenge — `View`, `Text`, `StyleSheet` — yeh sab Next.js mein nahi the. Inko baad ki docs mein detail mein samjhenge. Abhi focus sirf ek hai — **yeh emulator pe chalana.**

## Step 5 — Emulator Start Karo

**Naya terminal window kholo** — VS Code mein `Ctrl + Shift + ` `` backtick press karo ya new terminal open karo.

Yeh command run karo:

```bash
emulator -avd Medium_Phone_API_36.1
```

**Yeh kya ho raha hai?**

`emulator` — Android Studio ka tool hai jo virtual phone start karta hai.

`-avd` — matlab "Android Virtual Device" — kaunsa virtual phone start karna hai.

`Medium_Phone_API_36.1` — tune pehle `emulator -list-avds` mein yeh naam dekha tha — yahi tera virtual phone ka naam hai.

**Ek Android phone ki window khulegi tere computer pe.**

Pehli baar boot hone mein **1-2 minute lagenge.** Lock screen dikhne tak wait karo — jaldi mat karo.

---

## Step 6 — Expo Dev Server Start Karo

Emulator chal raha hai — ab **pehle wale terminal** mein wapas jao (jahan `cd RNSeekhna` kiya tha).

Yeh run karo:

```bash
npx expo start
```

**Kya ho raha hai yeh?**

Yaad hai 1.0 mein Metro Bundler ka flow dikhaya tha?

```
Tera Code → Metro Bundler → Emulator/Phone
```

`npx expo start` wahi Metro Bundler start karta hai. Tera code leta hai, bundle karta hai, aur emulator ko bhejta hai.

**Terminal mein yeh dikhega:**

```
Starting project at C:\Users\Pravin\Desktop\main\Learn\RNSeekhna
...
› Metro waiting on exp://192.168.x.x:8081

› Press a │ open Android
› Press w │ open web
› Press r │ reload app
```

**Ab `a` press karo keyboard pe.**

Expo detect karega ki emulator chal raha hai — aur app automatically wahan install karke kholega.

Pehli baar **1-2 minute** lagenge — bundle ban raha hai.

## Step 7 — Ek Change Karo, Verify Karo

Ab ek chhoti si cheez karte hain — confirm karne ke liye ki live reload kaam karra hai.

`App.tsx` mein yeh line dhundo:

```tsx
<Text>Open up App.tsx to start working on your app!</Text>
```

Sirf text change karo:

```tsx
<Text>Mera pehla React Native app!</Text>
```

**`Ctrl + S` press karo — file save karo.**

Emulator pe dekho — **automatically update ho jayega.** Manually reload nahi karna padega.

**Yeh feature hai Fast Refresh** — Next.js ke HMR jaisa hi. Save karo, screen update ho jaati hai.

---

## Step 8 — console.log Kahan Dikhta Hai?

Next.js mein jab `console.log` likhte the — browser ke DevTools mein dikhta tha. `F12` dabao, Console tab — wahan.

React Native mein **browser nahi hai.**

Toh `console.log` kahan dikhega?

**Terminal mein — wahi terminal jahan `npx expo start` chal raha hai.**

Try karo. `App.tsx` mein yeh add karo:

```tsx
export default function App() {
  console.log('Yeh terminal mein dikhega!')   // ← yeh line add karo

  return (
    <View style={styles.container}>
      <Text>Mera pehla React Native app!</Text>
      <StatusBar style="auto" />
    </View>
  );
}
```

Save karo — `Ctrl + S`.

Ab terminal mein dekho — yeh dikhega:

```
LOG  Yeh terminal mein dikhega!
```

**Yahi tera debugging tool hai poori React Native journey mein.** Jab bhi kuch samajh nahi aaye — `console.log` karo, terminal mein dekho.

## Warnings Dikhenge — Normal Hain

Terminal mein yeh type ki cheezein dikh sakti hain:

```
npm warn Unknown env config "versio"
npm warn Unknown env config "versiongit"
```

Yeh **warnings** hain — **errors nahi.** Kaam pe koi effect nahi. Ignore karo.

`warn` aur `error` mein fark hota hai — `error` matlab kuch toot gaya, `warn` matlab "yeh cheez theek karo kabhi" — abhi nahi.
