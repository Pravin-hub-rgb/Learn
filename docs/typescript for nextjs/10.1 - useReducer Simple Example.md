### Step 3: Reducer function

✅ **Ab aage hi likho**
```tsx
function reducer(state: State, action: Action): State {
  switch(action.type) {
    case "TOGGLE_THEME":
      return {
        ...state,
        theme: state.theme === "dark" ? "light" : "dark"
      }

    case "SET_FONT_SIZE":
      return {
        ...state,
        fontSize: action.payload
      }

    default:
      return state;
  }
}
```

✅ **Is code mein kya ho raha hai poora logic step by step:**
1.  Pehle reducer ko 2 cheeze receive karta hai: `state` (jo abhi ka state hai) aur `action` (jo order bheja hai)
2.  Phir `switch(action.type)` se woh check karta hai ki konsa order aaya hai
3.  Agar `case "TOGGLE_THEME": order aaya toh:
    - Pehle purane state ko copy karta hai `...state` se`
    - Sirf theme ko toggle karta hai dark <-> light
    - Naya state return karta hai
4.  Agar `case "SET_FONT_SIZE": order aaya toh:
    - Pehle purane state ko copy karta hai
    - Sirf fontSize ko action ke payload se set karta hai
    - Naya state return karta hai
5.  Agar koi aur kuch aaya toh purana state hi return karta hai

✅ Yeh hi poora logic hai.
✅ Sab state changes ek hi jagah hai.
✅ Sirf fixed tareeke se hi change ho sakta hai.
✅ Yeh naukar sirf apne fixed order hi manta hai. Koi aur order sunega hi nahi.
