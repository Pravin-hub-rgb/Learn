# Doc 2.3 — Cookies padhna proxy mein

## Abhi Tak Kya Hua

2.2 mein headers padhna seekha — `request.headers.get()` se koi bhi header nikala. Ab ek special cheez hai — **cookies** — technically headers ke andar hi hoti hain — par Next.js ne cookies ke liye alag aur easy API di hai.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Cookies proxy mein kaise padhte hain — `request.cookies` API — aur yeh practically kab use hota hai.

---

## Cookie headers mein hi hoti hai — par Next.js ne easy banaya

Technically cookie ek header hai — `Cookie` naam ki:

```
Cookie: session=abc123; theme=dark; language=en
```

Headers wali approach se padhna:

```ts
const cookieHeader = request.headers.get("cookie")
// → "session=abc123; theme=dark; language=en"
// phir khud parse karo — complicated
```

Khud parse karna mushkil hai — Next.js ne special API di hai:

```ts
request.cookies.get("session")
// → { name: "session", value: "abc123" }
```

Directly koi bhi cookie naam se nikalo — string parse karne ki zaroorat nahi.

---

## Step 1 — cookie padhna

`proxy.ts` update karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const sessionCookie = request.cookies.get("session")
  console.log("Session cookie:", sessionCookie)
  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`request.cookies.get("session")` — `get` method hai — cookie ka naam do — object milta hai.

Browser mein manually cookie set karo — DevTools:

```
F12 → Application → Cookies → localhost:3000 → + (add)
Name: session
Value: abc123
```

Ab visit karo:

```
http://localhost:3000/
```

Terminal mein:

```
Session cookie: { name: 'session', value: 'abc123' }
```

Cookie nahi set ki toh:

```
Session cookie: undefined
```

`undefined` — cookie thi hi nahi. ✅

---

## Step 2 — cookie ki value nikalo

`get()` ek object return karta hai — `{ name, value }` — sirf value chahiye toh:

```ts
const sessionCookie = request.cookies.get("session")
const sessionValue = sessionCookie?.value

console.log("Session value:", sessionValue)
```

`sessionCookie?.value` — `?.` matlab "agar `sessionCookie` exist karta hai tabhi `.value` lo — warna `undefined` do." Zarooori hai kyunki agar cookie nahi hai — `sessionCookie` `undefined` hoga — aur `undefined.value` error dega.

Terminal mein:

```
Session value: abc123
```

---

## Step 3 — cookie exist karti hai ya nahi check karo

`has()` method — cookie hai ya nahi — `true` ya `false`:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const hasSession = request.cookies.has("session")
  console.log("Session cookie hai?", hasSession)
  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Cookie set hai toh:

```
Session cookie hai? true
```

Cookie nahi hai toh:

```
Session cookie hai? false
```

---

## Practical use — cookie se login check karna

Yahi aage kaam aayega — protected routes mein — cookie check karke decide karenge user logged in hai ya nahi:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const hasSession = request.cookies.has("session")

  if (!hasSession) {
    console.log("Session nahi hai — login pe bhejo")
  } else {
    console.log("Session hai — aage jaane do")
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`!hasSession` — `!` matlab "ulta" — agar `hasSession` `false` hai — `!false` = `true` — toh andar jaayega.

Cookie ke saath:

```
Session hai — aage jaane do
```

Cookie ke bina:

```
Session nahi hai — login pe bhejo
```

Abhi sirf `console.log` kiya — redirect karna Phase 3 mein karenge jab `NextResponse.redirect()` seekhenge.

---

## Summary

- Cookies technically headers mein hoti hain — par Next.js ne alag API di
- `request.cookies.get("naam")` — cookie ka object milta hai — `{ name, value }`
- `request.cookies.has("naam")` — cookie hai ya nahi — `true`/`false`
- `?.value` — safely value nikalo — agar cookie nahi toh `undefined`

## Agla Step

**Phase 3** mein — response banana seekhenge — `NextResponse` ke saath kya kya kar sakte hain — redirect, rewrite, headers add karna.
