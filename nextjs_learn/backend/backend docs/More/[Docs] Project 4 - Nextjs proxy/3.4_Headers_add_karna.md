# Doc 3.4 — Response Headers — proxy se page ko information dena

## Abhi Tak Kya Hua

3.3 mein `rewrite()` seekha — URL same rakhte hue alag content dena. Abhi tak `NextResponse` se teen kaam seekhe — `next()`, `redirect()`, `rewrite()`. Ab ek aur kaam — response ke saath **extra information bhejna** — headers ke through. Aaj yeh samjhenge.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Pehle ek real problem se shuru karenge — phir dekhenge ki proxy headers se us problem ko kaise solve karta hai.

---

## Problem — layout ko pata nahi user kahan hai

Socho tera `layout.tsx` hai — usme navigation bar hai — active link highlight karna hai — jis page pe user hai woh link bold dikhe.

```
/dashboard pe hai → Dashboard link bold
/profile pe hai   → Profile link bold
```

Par `layout.tsx` Server Component hai — Server Component mein `window.location` nahi hota — woh browser ki cheez hai. `usePathname()` hook bhi nahi — woh Client Component mein hota hai.

Toh kaise pata chalega?

---

## Proxy ko toh pata hai

`request.nextUrl.pathname` se proxy ko pata hai user kahan ja raha hai — yeh Phase 2 mein seekha tha.

Toh idea yeh hai — proxy woh information **response ke saath bhej de** — layout padh le.

Yeh kaam hota hai **response headers** se.

2.2 mein **request headers** dekhe the — browser server ko extra info bhejta hai. Ab **ulta** — server browser ko extra info bhejega — **response headers** mein.

---

## `response.headers.set()` — kya hota hai, kaise kaam karta hai

Abhi tak `NextResponse.next()` seedha return karte the. Par response ko save karke usmein header add kar sakte hain.

`response.headers` — response ka ek object hai — isme headers hain.

`.set("naam", "value")` — `.set()` method hai — do arguments leta hai:
- Pehla — header ka naam — string
- Doosra — header ki value — string

Pehle sirf dekho — response mein kya headers already hain — `console.log` karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const response = NextResponse.next()
  console.log(response.headers)
  return response
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`const response = NextResponse.next()` — response seedha return karne ki jagah variable mein save kiya — taaki modify kar sakein.

Terminal mein kuch aisa dikhega:

```
Headers { 'x-powered-by': 'Next.js', ... }
```

Response mein already kuch headers hain. Hum isme apna add karenge.

---

## Step 1 — apna header add karo

Ab `.set()` use karo — ek test header add karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const response = NextResponse.next()
  response.headers.set("x-hello", "world")
  return response
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`response.headers.set("x-hello", "world")` — `"x-hello"` naam ka header add kiya — value `"world"`.

`"x-"` prefix — convention hai custom headers ke liye — matlab "yeh browser ya Next.js ka standard header nahi — humara apna custom hai."

DevTools mein verify karo:

```
http://localhost:3000/
F12 → Network → localhost pe click → Headers tab → Response Headers
```

`x-hello: world` dikh raha hai? ✅ — Proxy ne header add kiya.

---

## Step 2 — pathname daalo — hardcoded nahi

Abhi `"world"` hardcode kiya — actual pathname chahiye:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const response = NextResponse.next()
  response.headers.set("x-pathname", request.nextUrl.pathname)
  return response
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Alag alag pages visit karo — DevTools mein Response Headers dekho:

```
/dashboard visit → x-pathname: /dashboard ✅
/about visit     → x-pathname: /about ✅
```

---

## Step 3 — layout mein yeh header padho

Ab proxy ne header set kiya — layout mein kaise milega?

Next.js mein ek function hai — `headers()` — jo `"next/headers"` package se aata hai. Yeh Server Components mein use hota hai — request ke saath aaye **saare headers** milte hain — proxy wale bhi.

Pehle import karo — zaroorat ab padi:

```tsx
import { headers } from "next/headers"
```

Ab `layout.tsx` mein use karo — pehle sirf `console.log`:

```tsx
import { headers } from "next/headers"

export default async function RootLayout({ children }: { children: React.ReactNode }) {
  const headersList = await headers()
  const pathname = headersList.get("x-pathname")
  console.log("Pathname:", pathname)

  return (
    <html>
      <body>{children}</body>
    </html>
  )
}
```

`await headers()` — `headers()` async function hai — thoda time leta hai — `await` lagana zaroori. Isliye function `async` bana.

`headersList.get("x-pathname")` — `.get()` method — header ka naam do — value milti hai. Proxy ne jo set kiya wahi milega.

Terminal mein dekho koi bhi page visit karke:

```
Pathname: /dashboard
```

Proxy ne set kiya — layout ne padha — connection complete. ✅

---

## Summary

- Response headers — proxy page ko extra information bhejta hai
- `const response = NextResponse.next()` — response save karo — modify karne ke liye
- `response.headers.set("naam", "value")` — header add karo — do arguments
- `"x-"` prefix — custom headers ka convention
- `headers()` from `"next/headers"` — Server Component mein proxy ke set kiye headers padho
- `.get("naam")` — specific header ki value nikalo
