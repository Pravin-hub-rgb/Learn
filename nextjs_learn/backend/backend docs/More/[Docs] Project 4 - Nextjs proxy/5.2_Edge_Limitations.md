# Doc 5.2 — Edge Limitations — kya kaam nahi karta

## Abhi Tak Kya Hua

5.1 mein Edge Runtime samjha — proxy wahan chalta hai — user ke paas — fast. Par kuch cheezein Edge mein kaam nahi karti. Aaj practically dekhenge.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Edge limitations practically dekhenge — error aane do — phir fix karenge — aur alternatives samjhenge.

---

## Limitation 1 — Database direct connect nahi ho sakta

Socho proxy mein database se user verify karna hai:

```ts
import { PrismaClient } from "@prisma/client"

const prisma = new PrismaClient()

export function proxy(request: NextRequest) {
  const user = await prisma.user.findUnique({ where: { id: "123" } })
  // ...
}
```

Yeh kaam nahi karega — error aayega:

```
Error: PrismaClient is unable to run in this environment.
The "edge" runtime does not support Node.js built-in modules.
```

**Kyun?** — Prisma ke andar Node.js native modules hain — Edge mein nahi hote.

**Solution** — database check proxy mein mat karo. Proxy sirf cookie check kare — detailed auth page mein karo.

---

## Limitation 2 — `bcrypt` kaam nahi karta

Password verify karne ke liye `bcrypt` use karte hain:

```ts
import bcrypt from "bcrypt"

export function proxy(request: NextRequest) {
  const isValid = await bcrypt.compare(password, hash) // ❌
}
```

Error:

```
Error: The edge runtime does not support Node.js 'crypto' module.
```

**Solution** — `bcrypt` ki jagah `bcryptjs` use karo — pure JavaScript implementation:

```ts
import bcryptjs from "bcryptjs" // ✅ Edge mein kaam karta hai
```

Ya better — password verification proxy mein mat karo — API route mein karo.

---

## Limitation 3 — `jose` vs `jsonwebtoken`

JWT verify karna common hai proxy mein. Do popular libraries hain:

```ts
// ❌ jsonwebtoken — Node.js crypto use karta hai
import jwt from "jsonwebtoken"
jwt.verify(token, secret)

// ✅ jose — pure JS — Edge compatible
import { jwtVerify } from "jose"
await jwtVerify(token, secret)
```

`jose` — specifically Edge ke liye banaya gaya hai — same kaam karta hai JWT ke liye — par Edge mein chalata hai.

---

## Limitation 4 — file system nahi

```ts
// ❌ Edge mein nahi
import fs from "fs"
const data = fs.readFileSync("./config.json")
```

Error:

```
Module not found: Can't resolve 'fs'
```

**Solution** — file data `next.config.js` mein ya environment variables mein rakho.

---

## Practically kab problem aati hai

Proxy mein yeh patterns avoid karo:

```ts
// ❌ Avoid
import { PrismaClient } from "@prisma/client"
import bcrypt from "bcrypt"
import jwt from "jsonwebtoken"
import fs from "fs"
import path from "path"

// ✅ Use these instead
import { jwtVerify } from "jose"          // JWT
import bcryptjs from "bcryptjs"           // password hash (agar zaroorat ho)
```

**Golden rule — proxy mein sirf yeh karo:**
- Cookie/header padho
- Simple condition check karo
- Redirect/rewrite karo

Heavy logic — API routes ya Server Components mein karo.

---

## Node.js runtime option — agar Edge kaam na kare

Agar kisi wajah se Edge limitations bahut zyada problem kar rahein — proxy ko Node.js runtime pe chala sakte hain:

```ts
export const runtime = "nodejs"  // proxy file mein add karo
```

Par yeh Edge ka speed advantage khatam kar deta hai — use karo tabhi jab zaroorat ho.

---

## Summary

- Database clients (Prisma) — Edge mein nahi — Node.js modules use karte hain
- `bcrypt` → `bcryptjs` — Edge compatible alternative
- `jsonwebtoken` → `jose` — Edge compatible JWT library
- File system — Edge mein nahi
- Proxy mein sirf cookie check + redirect — heavy logic bahar rakho
- `export const runtime = "nodejs"` — agar zaroorat ho Node.js runtime pe le jaao
