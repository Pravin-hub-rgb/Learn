## File kahan banate hain?

Pehle yeh clear karo — `proxy.ts` kahan banana hai.

Project structure kuch aisa hoga abhi:

```
my-app/
  app/
    page.tsx
    layout.tsx
  public/
  package.json
  next.config.ts
```

`proxy.ts` banana hai **project root** mein — matlab `app` folder ke bahar — `next.config.ts` ke saath wali jagah:

```
my-app/
  app/
    page.tsx
    layout.tsx
  proxy.ts        ← yahan banana hai
  public/
  package.json
  next.config.ts
```

**Kyun root mein?** — Next.js specifically yahan dhundhta hai — andar kahi bhi nahi chalega. Yeh Next.js ka convention hai.

## Step 1 — `proxy.ts` banao

Ab hum ek simple file banaenge — bas check karenge ki yeh work kar raha hai ki nahi.

**Pehle Request aur Response kya hota hai samjho:**

- **Request** — browser se server pe jaane wali cheez (URL, headers, data)
- **Response** — server se browser pe wapas jaane wali cheez (HTML, JSON, status)

**NextRequest** — Next.js ka ek object jo request ki saari information deta hai
**NextResponse** — Next.js ka ek class jo response banana mein help karti hai

Pehle ek file banao — `proxy.ts` — project root mein.

Pehle bas ek line likho:

```ts
console.log("proxy chala!")
```

Ab test karo — dev server chalao aur koi bhi URL visit karo.

**Error aayega** — "proxy function not found" ya kuch aisa!

**Kyun error aaya?** Next.js ko pata nahi chala ki kaun sa function run karna hai. `export default` karna padta hai.

Ab likho:

```ts
export default function proxy() {
  console.log("proxy chala!")
}
```

Ab test karo — dekho chal raha hai ya nahi.

**Chal gaya!** Terminal mein "proxy chala!" print ho raha hai.

**Itne saare logs kyun?** Next.js ke internal logs hain — GET / 200 in 43ms etc. — yeh proxy ke logs nahi hain.

Ab thoda detail mein karna hai — request ka path bhi print karna hai.

**Ab hum request ka path print karna chahenge:**

**Iske liye humare paas `request.nextUrl.pathname` hai** — request ke andar Next.js ek special object deta hai `nextUrl` jiska ek property hai `pathname` jo URL ka path deta hai.

```ts
export default function proxy(request) {
  console.log("proxy chala!", request.nextUrl.pathname)
}
```

Ab test karo — dekhoge ki konsi URL pe request aayi hai.

**Ab import add karo:**

```ts
import { NextRequest } from "next/server"

export default function proxy(request: NextRequest) {
  console.log("proxy chala!", request.nextUrl.pathname)
}
```

Ab test karo — dekhoge ki konsi URL pe request aayi hai.

**Terminal mein dekho** — dev server chala rahe ho? Agar nahi:

```bash
npm run dev
```

Ab browser mein koi bhi URL visit karo:

```
http://localhost:3000
```

Terminal mein dekho — kuch aisa print hoga:

```
Proxy chal raha hai: /
Proxy chal raha hai: /_next/static/...
Proxy chal raha hai: /_next/static/...
```

**Dekha?** — Sirf `/` visit kiya — par kai baar print hua. Kyunki Next.js static files bhi load karta hai — CSS, JS — woh bhi requests hain — proxy unpe bhi chala.

Yahi reason hai ki proxy mein zyada kaam mat karo — har choti cheez pe chalta hai.

## Step 3 — `NextResponse.next()` — "kuch mat karo — aage jaane do"

Abhi proxy sirf log kar raha hai — lekin request ko aage kyun nahi bhej raha? Browser mein page load hua ya nahi?

**Check karo** — agar page load nahi hua — toh problem yeh hai ki proxy ne request rok li, lekin aage nahi bheja.

**Solution** — `NextResponse.next()` use karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export default function proxy(request: NextRequest) {
  console.log("proxy chala!", request.nextUrl.pathname)
  
  // Yeh line zaroori hai — request ko aage bhejo
  return NextResponse.next()
}
```

**Kyun zaroori hai?**
- Proxy middleware har request pe chalta hai — aur uske baad kuch nahi hota
- `NextResponse.next()` batata hai — "Maine check kar liya, ab aage bhejo request"
- Bina yeh line ke — request hang ho jayegi — page load nahi hoga

**Test karo** — ab page load hoga. Terminal mein log dikhega, aur page bhi dikhega.

**Real-life example** — Security guard ki tarah:
- Guard tumhe dekhta hai (proxy chala)
- Phir kehta hai "proceed" (`NextResponse.next()`)
- Tum andar jaate ho (page load hota hai)

Yeh sabse common pattern hai — check karo, phir aage bhejo.

## Step 4 — `config` export karo — `matcher` lagao

Abhi proxy **har** request pe chal raha hai — static files pe bhi. Yeh sahi nahi — static files pe kuch check nahi karna hota.

`matcher` — Next.js ko batata hai ki **kaunse routes pe proxy chale**.

`config` naam ka ek object export karo — file ke neeche:

```ts
export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico|.*\\.svg).*)"],
}
```

`export const config` — yeh Next.js ko configuration deta hai — `matcher` key se batate hain kahan chalao proxy.

`"/((?!_next/static|_next/image|favicon.ico|.*\\.svg).*)"` — yeh ek **regex** hai — regular expression — ek pattern jo paths match karta hai.

Abhi full regex samjhne ki zaroorat nahi — bas itna samjho:

```
/((?!_next/static|_next/image|favicon.ico|.*\\.svg).*)
         ↑
   "in paths ko CHHOD DO"
   _next/static → Next.js ki JS/CSS files
   _next/image  → optimized images
   favicon.ico  → browser icon
   .*\\.svg     → SVG files (koi bhi cheez + .svg extension)
```

**Kyun `.*\\.svg`?**
- `.*` → zero ya zyada koi bhi characters
- `\\.` → dot ko escape karna (regex mein dot special character hai)
- `svg` → SVG extension

Matlab — "in teeno ko chhod ke baaki sab pe proxy chalao."

Ab terminal dekho — visit karo:

```
http://localhost:3000
```

Ab sirf yeh dikhega:

```
Proxy chal raha hai: /
```

Static files wali lines gayi? ✅ — Matcher kaam kar raha hai.

## Step 5 — verify karo alag routes pe

Naye pages banao — dekho proxy unpe bhi chalta hai:

`app/about/page.tsx` banao:

```tsx
export default function About() {
  return <h1>About Page</h1>
}
```

`app/dashboard/page.tsx` banao:

```tsx
export default function Dashboard() {
  return <h1>Dashboard</h1>
}
```

Ab dono visit karo:

```
http://localhost:3000/about
http://localhost:3000/dashboard
```

Terminal mein:

```
Proxy chal raha hai: /about
Proxy chal raha hai: /dashboard
```

Dono pe chala? ✅ — Proxy sab routes cover kar raha hai — tune sirf ek jagah likha.

## Summary

- `proxy.ts` — project root mein banana hai — `app` folder ke bahar
- `proxy()` function — har request pe chalta hai — `NextRequest` milta hai
- `NextResponse.next()` — "kuch mat karo — aage jaane do"
- `config.matcher` — kaunse routes pe proxy chale — static files exclude karo
- `request.nextUrl.pathname` — request ka URL path
