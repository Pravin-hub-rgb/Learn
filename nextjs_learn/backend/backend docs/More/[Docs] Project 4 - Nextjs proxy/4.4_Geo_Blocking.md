# Doc 4.4 — Geo Blocking — country se access rokna

## Abhi Tak Kya Hua

4.3 mein rate limiting implement ki — IP se request count track kiya. Ab ek aur scenario — **geo blocking** — specific countries se access rokna.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Geo blocking kya hai samjhenge — Vercel headers se country kaise padhte hain — aur implement karenge. Locally test nahi ho sakta — honestly batayenge kyun aur kaise verify karein.

---

## Geo blocking kyun chahiye?

Kuch apps specific countries mein available nahi hote — legal reasons, compliance, licensing. Ya kuch countries se zyada spam/attack aata hai — unhe block karo.

Example:

```
App sirf India aur US mein available hai
Koi Russia se aaya → 403 Forbidden
```

---

## Country kaise pata chalti hai?

User ka IP address hota hai — IP address se country pata chalti hai — yeh process hai **IP Geolocation**.

Khud karna complicated hai — IP databases download karne padte hain — update rakhne padte hain.

**Vercel** — ek header automatically add karta hai:

```
x-vercel-ip-country: IN
```

`IN` — India ka country code. `US` — United States. `GB` — United Kingdom. ISO 3166-1 alpha-2 standard hai yeh.

Bina kuch install kiye — Vercel pe deploy karo — yeh header automatically milega.

---

## Locally test nahi ho sakta — kyun?

Yeh header sirf **Vercel** add karta hai — localhost pe yeh header nahi hoga — `null` milega.

Toh locally test karne ke liye manually header simulate karenge — DevTools se nahi hoga — isilye curl command use karenge.

---

## Step 1 — geo blocking implement karo

`proxy.ts`:

```ts
import { NextRequest, NextResponse } from "next/server"

const ALLOWED_COUNTRIES = ["IN", "US", "GB"]

export function proxy(request: NextRequest) {
  const country = request.headers.get("x-vercel-ip-country")

  if (country && !ALLOWED_COUNTRIES.includes(country)) {
    return new NextResponse("This service is not available in your region.", {
      status: 403,
    })
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`ALLOWED_COUNTRIES` — allowed countries ka array.

`request.headers.get("x-vercel-ip-country")` — Vercel ka country header padha.

`country && !ALLOWED_COUNTRIES.includes(country)` — do conditions:
- `country` — header aaya? Locally `null` hoga — toh `null && ...` = `false` — block nahi hoga locally
- `!ALLOWED_COUNTRIES.includes(country)` — country allowed list mein nahi hai?

`.includes()` — array mein yeh value hai? — `true`/`false`. `!` — ulta karo — "nahi hai?"

`403` — HTTP status code — "Forbidden" — access nahi.

## Step 2 — locally simulate karo

Locally test karne ke liye — apne code mein temporarily hardcode karo country:

```ts
import { NextRequest, NextResponse } from "next/server"

const ALLOWED_COUNTRIES = ["IN", "US", "GB"]

export function proxy(request: NextRequest) {
  const country = request.headers.get("x-vercel-ip-country") ?? "RU" // test ke liye RU hardcode

  if (!ALLOWED_COUNTRIES.includes(country)) {
    return new NextResponse("This service is not available in your region.", {
      status: 403,
    })
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`?? "RU"` — locally header `null` aata hai — `null ?? "RU"` = `"RU"` — Russia simulate kiya.

Visit karo:

```
http://localhost:3000/
```

"This service is not available in your region." dikh raha hai? ✅

Ab `"RU"` ko `"IN"` karo — allowed country:

```ts
const country = request.headers.get("x-vercel-ip-country") ?? "IN"
```

Page normally khulja raha hai? ✅

Test ho gaya — ab yeh hardcoded value hata do — production mein Vercel khud header dega:

```ts
const country = request.headers.get("x-vercel-ip-country")
```

## Step 3 — blocked users ko proper page dikhao

Sirf text return karna ugly lagta hai — ek proper page banana:

`app/blocked/page.tsx` banao:

```tsx
export default function BlockedPage() {
  return (
    <main>
      <h1>Access Restricted</h1>
      <p>This service is not available in your region.</p>
    </main>
  )
}
```

`proxy.ts` mein redirect karo is page pe — redirect ki jagah rewrite — URL same dikhega:

```ts
import { NextRequest, NextResponse } from "next/server"

const ALLOWED_COUNTRIES = ["IN", "US", "GB"]

export function proxy(request: NextRequest) {
  const country = request.headers.get("x-vercel-ip-country")

  if (country && !ALLOWED_COUNTRIES.includes(country)) {
    return NextResponse.rewrite(new URL("/blocked", request.url))
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico|blocked).*)"],
}
```

`rewrite()` use kiya — URL bar mein `/dashboard` hi dikhega — par blocked page ka content aayega. User confuse nahi hoga.

Matcher mein `blocked` add kiya — taaki blocked page pe proxy na chale — infinite loop se bachao.

## Summary

- Geo blocking — specific countries se access rokna
- Vercel automatically `x-vercel-ip-country` header deta hai
- Locally `null` aata hai — `??` se test country hardcode karo
- `.includes()` — array mein value hai?
- `403` — Forbidden status code
- `rewrite()` better hai redirect se — URL same dikhta hai
