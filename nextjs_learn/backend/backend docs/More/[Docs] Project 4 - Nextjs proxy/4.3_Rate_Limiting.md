## Problem — koi ek banda bahut zyada requests bhej raha hai

Socho tera app chal raha hai — ek banda script likh ke har second 500 requests bhej raha hai:

```
/api/search → 500 baar per second
/api/search → 500 baar per second
...
```

Server overload ho jaata hai — real users ko response nahi milta — app slow ya crash.

Ya login form pe koi **brute force** kar raha hai — har second alag password try kar raha hai automatically.

**Rate limiting** — ek rule — "ek IP se ek minute mein sirf 10 requests — 11vi pe rok do."

---

## Data kahan rakhenge — module level object

Pehle yeh samjho — kitni requests aayi yeh **yaad kahan rakhenge?**

Proxy function har request pe call hota hai — function ke andar variables har baar fresh hote hain. Toh count function ke andar rakhne se kaam nahi chalega.

**Module level** pe rakhna hoga — matlab function ke bahar — file ke upar. Yeh ek baar initialize hota hai — function bar bar call hone pe bhi yeh wahi object rehta hai.

```ts
const requestCounts = {}  // ← module level — function ke bahar

export function proxy(request: NextRequest) {
  // yahan se requestCounts access hota hai
  // har call mein fresh nahi hota
}
```

## Step 1 — IP padho aur log karo

Pehle sirf IP log karo aur khali object dikho — dekhte hain kaise kaam karega:

```ts
import { NextRequest, NextResponse } from "next/server"

const requestCounts: Record<string, { count: number; resetTime: number }> = {}

export function proxy(request: NextRequest) {
  const ip = request.headers.get("x-forwarded-for") ?? "unknown"

  console.log("IP:", ip)
  console.log("requestCounts abhi:", requestCounts)

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`Record<string, { count: number; resetTime: number }>` — TypeScript type hai. Iska matlab samjho pehle ek simple example se:

```ts
// Record<KeyType, ValueType>
// → ek object jisme keys KeyType ki hain aur values ValueType ki

const ages: Record<string, number> = {
  "Pravin": 25,
  "Rahul": 30,
}
```

Hamare case mein — keys strings hain (IP addresses) — values `{ count, resetTime }` objects hain.

Visit karo — terminal mein:

```
IP: unknown
requestCounts abhi: {}
```

Object khali hai — koi count nahi abhi. ✅

Chalo step-by-step samjhte hain:

### 1. Pehle samjho: "Record" kya hai?

**Record** ek **utility type** hai TypeScript mein. Ye basically ek **object type** define karne ka shortcut hai.

### 2. Simple example se shuru karte hain:

```typescript
// Normal object type define karna
type User = {
  name: string;
  age: number;
}

// Ab agar humein multiple users store karna ho
type Users = {
  "john": User,
  "jane": User,
  "bob": User
}
```

### 3. Ab Record ka use:

```typescript
// Record ka use karke same cheez ko aise likh sakte hain:
type Users = Record<string, User>
```

### 4. Record<string, User> ka matlab:

- `string` = keys honge string type ki (jaise "john", "jane", "bob")
- `User` = values honge User type ke

### 5. Tere code mein:

```typescript
Record<string, { count: number; resetTime: number }>
```

**Isko break karte hain:**

- `Record` = utility type
- `<string, { count: number; resetTime: number }>` = type parameters

### 6. Type parameters ka matlab:

- `string` = keys honge string type ki (IP addresses)
- `{ count: number; resetTime: number }` = values honge objects jinke andar 2 properties hongi

### 7. Equivalent normal type:

```typescript
type RequestCount = {
  count: number;
  resetTime: number;
}

type RequestCounts = Record<string, RequestCount>;
```

### 8. Practical example:

```typescript
const requestCounts: Record<string, { count: number; resetTime: number }> = {
  "192.168.1.1": { count: 5, resetTime: 1234567890 },
  "127.0.0.1": { count: 2, resetTime: 1234567891 }
}
```

# Step 2 — Har Request Pe Count Badhao

## Pehle Samjho — Hum Kya Kar Rahe Hain?

Ab hum har request pe count badhane wale hain. Yeh step rate limiting ka core hai.

### **Basic Concept:**
- Har IP address ke liye ek counter rakhega
- Har request aane pe us IP ka count 1 badhega
- Count track karega ki ek minute mein kitni requests aayi

### **Logic Samjho:**
1. **Pehli baar request aaye** → Naya record banao → count = 1
2. **Doosri ya zyada baar request aaye** → Purana record update karo → count++

### **Time Tracking:**
- `resetTime` = Kab tak ka window active hai
- `WINDOW_MS` = Window ka size (1 minute)
- Abhi se 1 minute baad tak ka count track hoga

**⚠️ Important:** Abhi hum sirf count badha rahe hain. Window expire check aur reset Step 3 mein karenge.

## Real Life Scenarios

### **Scenario 1: Naya User Aata Hai**
```
Time: 12:00:00 PM
IP: 192.168.1.1
Request: GET /api/search
Action: Pehli baar request → Naya record banao
Record: { count: 1, resetTime: 12:01:00 PM }
```

### **Scenario 2: Same User Dobara Aata Hai (1 Minute ke Andar)**
```
Time: 12:00:30 PM (30 seconds baad)
IP: 192.168.1.1
Request: GET /api/search
Action: Pehle se record hai → Count badhao
Record: { count: 2, resetTime: 12:01:00 PM }
```

### **Scenario 3: 1 Minute Baad User Aata Hai**
```
Time: 12:01:30 PM (1 minute 30 seconds baad)
IP: 192.168.1.1
Request: GET /api/search
Action: Window expire ho gaya → Fresh start (Step 3 mein)
Record: { count: 1, resetTime: 12:02:30 PM }
```

## Ab Code Samjho

```typescript
import { NextRequest, NextResponse } from "next/server"

const WINDOW_MS = 60 * 1000  // 1 minute — 60 seconds * 1000 milliseconds
const requestCounts: Record<string, { count: number; resetTime: number }> = {}

export function proxy(request: NextRequest) {
  const ip = request.headers.get("x-forwarded-for") ?? "unknown"
  const now = Date.now()

  // Logic: Pehli baar ya repeat request?
  if (!requestCounts[ip]) {
    // Pehli baar request aa rahi hai is IP se
    // Naya record banao
    requestCounts[ip] = { 
      count: 1,                    // Count shuru karo 1 se
      resetTime: now + WINDOW_MS   // 1 minute baad reset hoga
    }
  } else {
    // Pehle se record hai — count badhao
    requestCounts[ip].count++
  }

  console.log(`IP: ${ip} — Count: ${requestCounts[ip].count}`)

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

## Code Explanation

### **Why Milliseconds?**

**Q: `Date.now()` milliseconds mein kyu deta hai?**
**A:** Computers time ko milliseconds mein track karte hain - exact precision ke liye
- `Date.now()` = 1677614400000 (exact time)
- `new Date()` = "2023-03-01T12:00:00.000Z" (human readable)

**Q: `WINDOW_MS` milliseconds mein kyu define karna?**
**A:** Comparison ke liye dono ko same unit mein hona chahiye
- `now` = milliseconds
- `WINDOW_MS` = milliseconds
- `now + WINDOW_MS` = milliseconds
- Easy comparison: `now > resetTime`

### **Logic Check:**

```typescript
if (!requestCounts[ip]) {
  // Pehli baar request aa rahi hai
  requestCounts[ip] = { count: 1, resetTime: now + WINDOW_MS }
} else {
  // Pehle se record hai
  requestCounts[ip].count++
}
```

**`!requestCounts[ip]` kya check kar raha hai?**
- `requestCounts[ip]` = undefined agar pehli baar ho
- `!undefined` = true
- Matlab: agar pehli baar request ho toh naya record banao

**`resetTime: now + WINDOW_MS` kya hai?**
- `now` = Abhi ka time (milliseconds)
- `WINDOW_MS` = 1 minute (60,000ms)
- `now + WINDOW_MS` = 1 minute baad ka time
- Is time tak count track hoga

**`count++` kyu kar rahe hain?**
- `count++` = count ko 1 badhao
- Har request pe count increment karo

# Step 3 — 1 Minute Baad Count Reset Karna

## Pehle Samjho — Hum Kya Kar Rahe Hain?

Ab hum window expire check karenge aur count reset karenge. Yeh step rate limiting ka most important part hai.

### **Basic Concept:**
- Har IP ke liye ek time window hai (1 minute)
- Window expire hone pe count reset ho jata hai
- Fresh start hota hai - naya 1 minute ka cycle shuru

### **Logic Samjho:**
1. **Pehli baar request aaye** → Naya record banao → count = 1
2. **Window expire ho gaya** → Fresh start → count = 1
3. **Window active hai** → Count badhao → count++

### **Time Tracking:**
- `resetTime` = Kab tak ka window active hai
- `now > resetTime` = Window expire ho gaya
- `now <= resetTime` = Window abhi active hai

## Real Life Scenarios

### **Scenario 1: Window Expire Ho Gaya**
```
Time: 12:01:30 PM (1 minute 30 seconds baad)
IP: 192.168.1.1
Previous Record: { count: 8, resetTime: 12:01:00 PM }
Action: Window expire → Fresh start
New Record: { count: 1, resetTime: 12:02:30 PM }
```

### **Scenario 2: Window Abhi Active Hai**
```
Time: 12:00:45 PM (45 seconds baad)
IP: 192.168.1.1
Previous Record: { count: 3, resetTime: 12:01:00 PM }
Action: Window active → Count badhao
New Record: { count: 4, resetTime: 12:01:00 PM }
```

### **Scenario 3: Naya User Aata Hai**
```
Time: 12:02:00 PM
IP: 10.0.0.1
Previous Record: None
Action: Pehli baar → Naya record
New Record: { count: 1, resetTime: 12:03:00 PM }
```

### **Pehle Step 2 ka Final Code Yaad Karo**

**Ab hum isme window expire check add karenge.**

### **Kyun Window Expire Check Chahiye?**

**Problem:** Step 2 mein count sirf badhta hai, kabhi reset nahi hota. 1 minute baad bhi count badhna continue karta hai.

**Solution:** Har request pe check karna ki:
- Kya pehli baar request aa rahi hai? (Step 2)
- YA kya window expire ho gaya hai? (Step 3)

### **Window Expire Check Kaise Kare?**

Humko check karna hai ki current time (`now`) reset time (`resetTime`) se zyada hai ya nahi.

**Syntax Explanation:**
- `now > requestCounts[ip].resetTime` - yeh condition check karegi ki window expire ho gaya hai ya nahi
- `>` - greater than operator - compare karta hai dono values ko
- `now` - current time in milliseconds
- `requestCounts[ip].resetTime` - us IP ke liye set kiya gaya reset time

**Ab hum `if` condition mein ek aur condition add karenge:**

```typescript
if (!requestCounts[ip] || now > requestCounts[ip].resetTime) {
  // Pehli baar ya window expire — fresh start
  requestCounts[ip] = { count: 1, resetTime: now + WINDOW_MS }
} else {
  // Window active hai — count badhao
  requestCounts[ip].count++
}
```

**Explanation of `||` operator:**
- `||` - OR operator - dono conditions mein se koi bhi true ho toh if block execute hoga
- `!requestCounts[ip]` - agar pehli baar request ho (Step 2 ki condition)
- `now > requestCounts[ip].resetTime` - agar window expire ho gaya ho (Step 3 ki nayi condition)

## Step 3.2 — Console Log Mein Window Status Dikhana

### **Window Status Kaise Check Kare?**

Humko console mein dikhana hai ki window active hai ya expired.

**Syntax Explanation:**
- `now <= requestCounts[ip].resetTime` - yeh check karega ki window abhi active hai ya nahi
- `<=` - less than or equal to operator
- Agar current time reset time se kam ya barabar hai toh window active hai

**Ternary Operator Use Karna:**
```typescript
now <= requestCounts[ip].resetTime ? 'Active' : 'Expired'
```

**Ternary operator syntax:**
- `condition ? value_if_true : value_if_false`
- Agar condition true hai toh 'Active' return hoga, nahi toh 'Expired'

**Ab hum console.log mein window status add karenge:**

```typescript
console.log(`IP: ${ip} — Count: ${requestCounts[ip].count} — Window: ${now <= requestCounts[ip].resetTime ? 'Active' : 'Expired'}`)
```

## Step 4 — limit cross hone pe block karo

Ab limit define karo aur block karo:

```ts
import { NextRequest, NextResponse } from "next/server"

const RATE_LIMIT = 10
const WINDOW_MS = 60 * 1000
const requestCounts: Record<string, { count: number; resetTime: number }> = {}

export function proxy(request: NextRequest) {
  const ip = request.headers.get("x-forwarded-for") ?? "unknown"
  const now = Date.now()

  if (!requestCounts[ip] || now > requestCounts[ip].resetTime) {
    requestCounts[ip] = { count: 1, resetTime: now + WINDOW_MS }
  } else {
    requestCounts[ip].count++
  }

  if (requestCounts[ip].count > RATE_LIMIT) {
    return new NextResponse("Too Many Requests", { status: 429 })
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`RATE_LIMIT = 10` — 10 requests per minute allow.

`new NextResponse("Too Many Requests", { status: 429 })` — custom response banana. `new NextResponse()` do arguments leta hai:
- Pehla — response body — string
- Doosra — options object — `status` key se HTTP status code

`429` — HTTP ka official "Too Many Requests" code hai.

## Ek important limitation — honestly

Yeh in-memory solution hai — ek JavaScript object mein count rakh rahe hain.

Production mein Vercel multiple server instances chalata hai — har instance ka apna alag `requestCounts` hoga — share nahi hoga:

```
Instance 1 → requestCounts: { "192.x.x.x": { count: 8 } }
Instance 2 → requestCounts: { "192.x.x.x": { count: 6 } }
```

Effective limit 10 nahi rahi — har instance pe alag count. **Sahi production solution Redis** hai — shared database — sab instances ek jagah se padhte hain. Development ke liye abhi yeh theek hai.

## Summary

- Rate limiting — IP se count track karo — limit cross hone pe block karo
- Module level object — function ke bahar — bar bar fresh nahi hota
- `Date.now()` — milliseconds mein current time — compare karne ke liye number
- `count++` — increment — 1 add karo
- `||` — ya — koi bhi condition true ho toh
- `429` — "Too Many Requests" HTTP status code
- `new NextResponse(body, { status })` — custom response banana
- In-memory — development ke liye theek — production mein Redis chahiye
