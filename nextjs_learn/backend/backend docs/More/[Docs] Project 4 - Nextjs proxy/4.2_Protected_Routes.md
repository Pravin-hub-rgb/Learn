# Doc 4.2 — Protected Routes — login check karo proxy mein

## Abhi Tak Kya Hua

4.1 mein logging implement ki — har request ka record rakhna. Ab ek aur real scenario — **protected routes** — kuch pages sirf logged-in users ke liye hone chahiye.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Pehle problem practically dekhenge — phir proxy mein step by step login check + redirect implement karenge.

---

## Problem — koi bhi `/dashboard` khol sakta hai

Sabse pehle yeh try karo — cookies delete karo:

```
F12 → Application → Cookies → localhost:3000 → Right click → Clear
```

Ab visit karo:

```
http://localhost:3000/dashboard
```

Dashboard dikh gaya? — **Yeh problem hai.** Koi bhi URL bar mein type kare — page khul jaata hai — chahe login kiya ho ya nahi.

---

## Har page mein check karne ki problem

Ek approach — dashboard page ke andar hi check karo:

```tsx
// dashboard/page.tsx
import { cookies } from "next/headers"
import { redirect } from "next/navigation"

export default async function Dashboard() {
  const cookieStore = await cookies()
  const session = cookieStore.get("session")
  if (!session) redirect("/login")
  return <h1>Dashboard</h1>
}
```

Kaam karta hai — par agar 10 protected pages hain toh har jagah yeh likhna padega — `/dashboard`, `/profile`, `/settings`, `/orders`... Ek page mein bhool gaye — woh unprotected reh jaata hai.

**Proxy mein ek jagah likho — sab cover.**

---

## Step 1 — proxy mein cookie check karo

`request.cookies.has()` — 2.3 mein seekha tha — cookie exist karti hai ya nahi — `true`/`false`. Pehle sirf log karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const hasSession = request.cookies.has("session")
  console.log("Session hai?", hasSession)
  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Pehle cookie add karo DevTools mein:

```
F12 → Application → Cookies → + → Name: session, Value: abc123
```

Visit karo — terminal mein:

```
Session hai? true
```

Cookie delete karo — visit karo:

```
Session hai? false
```

Proxy cookie padh raha hai. ✅

---

## Step 2 — session nahi + dashboard — redirect karo

Ab condition add karo. Do cheezein check karni hain ek saath:

1. User `/dashboard` pe ja raha hai?
2. Session cookie nahi hai?

Dono true hain — `/login` pe redirect karo.

`&&` operator — "aur" — dono conditions true honi chahiye:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const hasSession = request.cookies.has("session")
  const pathname = request.nextUrl.pathname

  if (pathname === "/dashboard" && !hasSession) {
    return NextResponse.redirect(new URL("/login", request.url))
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`pathname === "/dashboard"` — exactly yahi path hai?

`!hasSession` — `!` matlab ulta — "session nahi hai?"

`&&` — dono true hain toh andar jaao.

`return NextResponse.redirect(...)` — `return` — yahan function khatam karo — yeh response bhejo. Agle `return NextResponse.next()` tak nahi pahunchega.

Test karo:

```
Cookie nahi + /dashboard → /login pe gaya ✅
Cookie hai  + /dashboard → dashboard dikh gaya ✅
```

## Step 3 — sirf dashboard nahi — saare protected routes

Abhi sirf `/dashboard` check hota hai. `/profile` aur `/settings` bhi protected chahiye.

Ek array banao — protected routes ka. Phir `.some()` se check karo:

`.some()` — array ka method hai — "kya array mein koi ek bhi element yeh condition satisfy karta hai?" — `true`/`false` return karta hai.

Simple example pehle:

```js
const fruits = ["apple", "banana", "mango"]
fruits.some((fruit) => fruit === "banana")  // → true
fruits.some((fruit) => fruit === "grapes")  // → false
```

Ab apply karo:

```ts
import { NextRequest, NextResponse } from "next/server"

const protectedRoutes = ["/dashboard", "/profile", "/settings"]

export function proxy(request: NextRequest) {
  const hasSession = request.cookies.has("session")
  const pathname = request.nextUrl.pathname

  const isProtected = protectedRoutes.some((route) =>
    pathname.startsWith(route)
  )

  if (isProtected && !hasSession) {
    return NextResponse.redirect(new URL("/login", request.url))
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`pathname.startsWith(route)` — `startsWith()` — string method hai — "kya yeh string us cheez se shuru hoti hai?" `/dashboard/settings` bhi cover hoga — kyunki woh `/dashboard` se shuru hota hai.

Test karo:

```
Cookie nahi + /dashboard          → /login ✅
Cookie nahi + /profile            → /login ✅
Cookie nahi + /dashboard/settings → /login ✅
Cookie nahi + /about              → /about dikh gaya ✅
Cookie hai  + /dashboard          → dashboard dikh gaya ✅
```

## Step 4 — callbackUrl — user kahan jaana chahta tha yaad rakho

Abhi user `/dashboard/settings` pe ja raha tha — login nahi tha — `/login` pe redirect hua — login kiya — ab seedha `/dashboard` pe jaata hai. Settings wali page bhool gayi.

**Better UX** — redirect karte waqt original URL URL mein daal do — login ke baad wahan bhejo.

`searchParams.set()` — URL object pe `.searchParams` hota hai — query parameters handle karne ke liye. `.set("key", "value")` naya parameter add karta hai.

```ts
const loginUrl = new URL("/login", request.url)
loginUrl.searchParams.set("callbackUrl", pathname)
// → /login?callbackUrl=/dashboard/settings
```

`proxy.ts` update karo:

```ts
import { NextRequest, NextResponse } from "next/server"

const protectedRoutes = ["/dashboard", "/profile", "/settings"]

export function proxy(request: NextRequest) {
  const hasSession = request.cookies.has("session")
  const pathname = request.nextUrl.pathname

  const isProtected = protectedRoutes.some((route) =>
    pathname.startsWith(route)
  )

  if (isProtected && !hasSession) {
    const loginUrl = new URL("/login", request.url)
    loginUrl.searchParams.set("callbackUrl", pathname)
    return NextResponse.redirect(loginUrl)
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Cookie delete karo — visit karo `/dashboard/settings` — URL bar mein dekho:

```
http://localhost:3000/login?callbackUrl=/dashboard/settings
```

`callbackUrl` dikh raha hai URL mein? ✅ — Login page isko padh ke baad wahan redirect kar sakta hai.

## Step 5 — logged-in user login page pe aa gaya

Ek aur case — user pehle se logged in hai — phir `/login` visit kiya URL bar mein type karke.

Woh `/login` pe reh jaata hai — confusing lagta hai already logged-in user ke liye.

Fix — session hai + `/login` pe aaya — `/dashboard` pe bhejo:

```ts
import { NextRequest, NextResponse } from "next/server"

const protectedRoutes = ["/dashboard", "/profile", "/settings"]
const publicOnlyRoutes = ["/login"]

export function proxy(request: NextRequest) {
  const hasSession = request.cookies.has("session")
  const pathname = request.nextUrl.pathname

  const isProtected = protectedRoutes.some((route) =>
    pathname.startsWith(route)
  )
  const isPublicOnly = publicOnlyRoutes.some((route) =>
    pathname.startsWith(route)
  )

  if (isProtected && !hasSession) {
    const loginUrl = new URL("/login", request.url)
    loginUrl.searchParams.set("callbackUrl", pathname)
    return NextResponse.redirect(loginUrl)
  }

  if (isPublicOnly && hasSession) {
    return NextResponse.redirect(new URL("/dashboard", request.url))
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`publicOnlyRoutes` — yeh routes sirf logged-OUT users ke liye hain.

Cookie add karo — visit karo `/login`:

```
http://localhost:3000/login
```

`/dashboard` pe redirect hua? ✅

---

## Summary

- Protected routes — ek array mein — proxy ek jagah sab check kare
- `.some()` — array mein koi ek element condition satisfy karta hai?
- `.startsWith()` — string us cheez se shuru hoti hai?
- `callbackUrl` — original URL mein save karo — login ke baad wahan bhejo
- `publicOnlyRoutes` — logged-in user yahan na aaye
- `&&` — dono conditions true honi chahiye
