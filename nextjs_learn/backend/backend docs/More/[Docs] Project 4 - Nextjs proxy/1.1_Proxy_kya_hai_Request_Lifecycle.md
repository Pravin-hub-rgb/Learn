## Problem pehle — kya issue hai?

Socho tera app hai — `/dashboard` page hai — sirf logged-in users ke liye.

Abhi kya hota hai:

```
User /dashboard visit karta hai
        ↓
Next.js seedha dashboard render karta hai
        ↓
Page dikh jaata hai — chahe logged in ho ya nahi
```

**Problem:** Koi authentication check nahi ho raha! Koi bhi user directly `/dashboard` visit kar sakta hai.

**Solution try karte hain:**

Option 1 — har page ke andar check karo:

```tsx
// dashboard/page.tsx
export default function Dashboard() {
  // yahan check karo...
  // profile/page.tsx mein bhi check karo...
  // settings/page.tsx mein bhi check karo...
  // har jagah likhna padega
}
```

**Error:** Bahut repetition — 10 protected pages hain toh 10 jagah likhna padega!

**Solution try karte hain:**

Option 2 — ek common component banao:

```tsx
// components/ProtectedRoute.tsx
export default function ProtectedRoute({ children }) {
  // check karo...
  return children
}
```

**Error:** Har page mein import karna padega — phir bhi 10 jagah likhna padega!

## Solution — Proxy ka idea

**Senior developer bol raha hai junior ko:**

"Sun, soch ek building ki tarah. Building mein security guard hota hai darwaze pe. Sab visitors wahan se guzarte hain — guard check karta hai — phir andar jaane deta hai.

Hum Next.js app ka bhi ek **security guard** bana sakte hain — jo har request pehle check kare!"

**Yehi hai Proxy** — Next.js app ka darwaza — har request pehle yahaan aati hai:

```
User ne /dashboard visit kiya
        ↓
Pehle Proxy chalta hai — "logged in hai?"
        ↓
Haan → /dashboard pe jaane do
Nahi → /login pe redirect karo
```

Ek jagah likho — saari requests cover ho jaati hain.

## Pehle "Middleware" tha — ab "Proxy" hai

Agar internet pe search karo — "Next.js middleware" dikhega har jagah. Yeh confusing lagta hai — kyunki ab Next.js mein yeh **"Proxy"** ho gaya hai.

**Kyu rename kiya?**

Next.js v16 se pehle — file ka naam tha `middleware.ts` — function ka naam tha `middleware`. Par yeh naam confusing tha — Express.js mein bhi "middleware" hota hai — dono alag cheezein hain.

Toh Next.js ne rename kar diya:

```
middleware.ts  →  proxy.ts
middleware()   →  proxy()
```

**Officially warning bhi aati hai ab agar purana naam use karo:**

```
⚠ The "middleware" file convention is deprecated.
  Please use "proxy" instead.
```

**Migrate karna ho purane project mein — yeh command run karo:**

```bash
npx @next/codemod@canary middleware-to-proxy .
```

Yeh automatically `middleware.ts` ko `proxy.ts` mein rename kar deta hai — aur function ka naam bhi change kar deta hai.

**Naya project mein** — seedha `proxy.ts` banao — `middleware.ts` nahi.

## Proxy ka flow — officially

```
User request karta hai koi bhi URL
        ↓
Proxy sabse pehle chalta hai
        ↓
Proxy decide karta hai:
  - Aage jaane do? (NextResponse.next())
  - Redirect karo? (NextResponse.redirect())
  - Rewrite karo? (NextResponse.rewrite())
        ↓
Phir actual page render hota hai
```

**Sabse important cheez** — Proxy **har** request pe chalta hai — page, API, image — sab. Isliye carefully likhna padta hai — warna slow ho jaata hai app.

## Real example — kaise kaam karta hai?

Socho proxy.ts mein likha:

```ts
export function proxy(request) {
  console.log("Koi request aayi:", request.nextUrl.pathname)
  return NextResponse.next()
}
```

Ab jab bhi koi URL visit karega:

```
http://localhost:3000/dashboard
```

**Flow hoga:**

1. User ne `/dashboard` visit kiya
2. Pehle proxy chala — `console.log("Koi request aayi: /dashboard")`
3. `NextResponse.next()` — "aage jaane do"
4. Phir actual dashboard page render hua

**Dekha?** Proxy beech mein chala — lekin kuch block nahi kiya — bas log kiya.

## Summary

- **Problem:** Har page mein authentication check karna padta hai — repetition
- **Solution:** Proxy — ek security guard jaise — har request pehle check kare
- **Proxy ka naam:** Pehle "middleware" tha — ab officially "proxy" hai
- **File:** `proxy.ts` — function: `proxy()`
- **Flow:** Request → Proxy → Page render
- **Important:** Proxy har request pe chalta hai — carefully likhna
