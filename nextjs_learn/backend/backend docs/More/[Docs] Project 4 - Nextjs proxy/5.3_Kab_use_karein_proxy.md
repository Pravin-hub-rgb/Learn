## Next.js khud kya kehta hai

Yeh interesting hai — Next.js ne jab `middleware` ko `proxy` rename kiya — toh officially likha:

> "We recommend users avoid relying on Middleware unless no other options exist."

Matlab — **proxy last resort hai** — pehle doosre options try karo.

Kyun? — Proxy har request pe chalta hai — agar heavy kaam karo — poori app slow ho jaati hai.

---

## Alternatives kya hain

**Case 1 — Route protect karna:**

```
Option A — Server Component mein auth() check karo
Option B — Proxy mein cookie check karo
```

Server Component mein directly check karo — specific page pe — sirf woh page check karega:

```tsx
// dashboard/page.tsx
import { auth } from "@/auth"
import { redirect } from "next/navigation"

export default async function Dashboard() {
  const session = await auth()
  if (!session) redirect("/login")
  return <div>Dashboard</div>
}
```

**Proxy tab use karo jab** — bahut saare protected routes hain — har page mein likhna mushkil ho — centralized check chahiye.

---

**Case 2 — Redirect karna:**

```
Option A — next.config.js mein redirects
Option B — Proxy mein redirect
```

Simple static redirects — `next.config.js` mein:

```js
// next.config.js
module.exports = {
  async redirects() {
    return [
      {
        source: "/old-page",
        destination: "/new-page",
        permanent: true,
      },
    ]
  },
}
```

**Proxy tab use karo jab** — redirect condition dynamic hai — cookie check karni hai, header padhna hai — static redirect se nahi hoga.

---

**Case 3 — Headers add karna:**

```
Option A — next.config.js mein headers
Option B — Proxy mein headers
```

Simple static headers — `next.config.js` mein:

```js
module.exports = {
  async headers() {
    return [
      {
        source: "/:path*",
        headers: [{ key: "X-Custom-Header", value: "hello" }],
      },
    ]
  },
}
```

**Proxy tab use karo jab** — header dynamic hai — request ke hisaab se value badalni ho.

---

## Decision Guide — simple

```
Kya karna hai?              Kahan karo?
─────────────────────────────────────────────────────
Static redirect             next.config.js → redirects
Static headers              next.config.js → headers
Page-specific auth check    Server Component mein
Centralized auth            Proxy ✅
Logging                     Proxy ✅
Rate limiting               Proxy ✅ (simple cases)
Geo blocking                Proxy ✅ (Vercel pe)
Heavy business logic        Server Component / API Route
Database queries            Server Component / API Route
```

---

## Proxy best practices — recap

**1. Matcher sahi set karo:**

```ts
export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Static files exclude karo — nahi toh har image pe bhi proxy chalega.

**2. Proxy mein sirf simple kaam karo:**

```ts
// ✅ Theek hai
const hasSession = request.cookies.has("session")
if (!hasSession) return NextResponse.redirect(...)

// ❌ Avoid — heavy kaam proxy mein mat karo
const user = await prisma.user.findMany(...)
```

**3. Edge compatible libraries use karo:**

```ts
// JWT ke liye
import { jwtVerify } from "jose"  // ✅

// bcrypt ke liye (agar zaroorat ho)
import bcryptjs from "bcryptjs"   // ✅
```

## Summary

- Next.js khud kehta hai — proxy last resort hai — pehle alternatives try karo
- Static redirects/headers → `next.config.js`
- Page-specific auth → Server Component
- Centralized auth, logging, rate limiting, geo blocking → Proxy ✅
- Proxy mein sirf simple kaam — heavy logic bahar
- Matcher sahi set karo — static files exclude karo
