# Doc 3.2 — Redirect karna

## Abhi Tak Kya Hua

3.1 mein `NextResponse` ke teen methods samjhe — `next()`, `redirect()`, `rewrite()`. Aaj `redirect()` properly implement karenge — real protected route example ke saath.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

`NextResponse.redirect()` use karenge — proxy mein login check karke — agar logged in nahi toh `/login` pe redirect karo.

---

## Problem — abhi redirect nahi ho raha

2.3 mein yeh tak pahunche the:

```ts
const hasSession = request.cookies.has("session")

if (!hasSession) {
  console.log("Session nahi hai — login pe bhejo")
} else {
  console.log("Session hai — aage jaane do")
}

return NextResponse.next()
```

`console.log` kiya — par redirect nahi kiya. Abhi bhi `NextResponse.next()` return ho raha hai — user `/dashboard` pe pahunch jaata hai chahe session ho ya nahi.

Yeh fix karte hain.

---

## `NextResponse.redirect()` ka syntax

```ts
NextResponse.redirect(new URL("/login", request.url))
```

Do cheezein hain yahan:

**`new URL("/login", request.url)`** — ek naya URL object bana rahe hain.

`URL` — JavaScript ka built-in class hai — URLs banane ke liye. `new URL(path, base)` — do arguments:
- `path` — kahan redirect karna hai — `"/login"`
- `base` — base URL kya hai — `request.url` — incoming request ka poora URL

Kyun `request.url` base mein? — Taaki Next.js sahi domain pe redirect kare. Development mein `localhost:3000/login` banega — production mein `myapp.com/login`.

`new` keyword — JavaScript mein kisi class ka naya instance banane ke liye likhte hain.

**`NextResponse.redirect()`** — yeh URL le kar response banata hai — browser ko bolta hai "is URL pe jaao."

---

## Step 1 — redirect implement karo

`proxy.ts` update karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const hasSession = request.cookies.has("session")

  if (!hasSession) {
    return NextResponse.redirect(new URL("/login", request.url))
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico|login).*)"],
}
```

**Kya badla?**

`return NextResponse.redirect(...)` — `console.log` ki jagah actual redirect. `return` — matlab "yahan se function khatam karo — yeh response bhejo."

Matcher mein `login` add kiya — `/((?!_next/static|_next/image|favicon.ico|login).*)` — kyunki `/login` pe proxy nahi chalna chahiye — warna infinite loop hoga:

```
/login visit kiya
        ↓
Proxy chala — session nahi hai
        ↓
/login pe redirect kiya
        ↓
Proxy chala — session nahi hai
        ↓
/login pe redirect kiya
        ↓
...infinite loop
```
Let me explain this with a simpler, real-world analogy that might make it clearer:

## Real-World Analogy: Security Guard at a Building

Imagine there's a security guard at the entrance of an office building:

### The Problem (Without Exclusion):

1. **You want to enter the office** (like visiting `/dashboard`)
2. **Security guard checks your ID** (like checking cookies)
3. **You don't have ID** → Guard says "Go get your ID at the reception"
4. **You go to reception** (like going to `/login`)
5. **Security guard is ALSO at reception** → Checks your ID again
6. **You still don't have ID** → Guard says "Go get your ID at the reception"
7. **You're stuck in a loop!** 🔄

### The Solution (With Exclusion):

1. **You want to enter the office** (like visiting `/dashboard`)
2. **Security guard checks your ID** (like checking cookies)
3. **You don't have ID** → Guard says "Go get your ID at the reception"
4. **You go to reception** (like going to `/login`)
5. **NO security guard at reception** → You can get your ID normally
6. **You get your ID** → Now you can enter the office

### In Code Terms:

**Without excluding `/login`:**
```typescript
// This matcher runs proxy on EVERY page including /login
matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"]
```

**With excluding `/login`:**
```typescript
// This matcher runs proxy on EVERY page EXCEPT /login
matcher: ["/((?!_next/static|_next/image|favicon.ico|login).*)"]
```

### The Key Point:

The `/login` page needs to be accessible WITHOUT going through the authentication check, otherwise users can never actually reach the login page to log in!

Think of it like this: **You can't check if someone is logged in when they're trying to log in.** 😊

Does this analogy help make it clearer?

## Step 3 — sirf protected routes pe redirect

Abhi har route pe redirect ho raha hai — `/about` pe bhi — jo public hona chahiye.

Protected routes define karo — sirf unhe check karo:

```ts
import { NextRequest, NextResponse } from "next/server"

const protectedRoutes = ["/dashboard", "/profile", "/settings"]

export function proxy(request: NextRequest) {
  const pathname = request.nextUrl.pathname
  const isProtected = protectedRoutes.some((route) =>
    pathname.startsWith(route)
  )

  if (isProtected && !request.cookies.has("session")) {
    return NextResponse.redirect(new URL("/login", request.url))
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico|login).*)"],
}
```

`protectedRoutes` — ek array hai — protected route paths ka.

`.some()` — array ka method hai — "kya array mein koi ek element yeh condition satisfy karta hai?" — `true` ya `false` return karta hai.

```js
["/dashboard", "/profile"].some((route) => "/dashboard/settings".startsWith(route))
// → true — kyunki "/dashboard/settings" startsWith "/dashboard"
```

`isProtected && !request.cookies.has("session")` — do conditions:
- `isProtected` — protected route hai?
- `!request.cookies.has("session")` — session nahi hai?

Dono true honi chahiye tabhi redirect hoga.

Test karo:

```
Cookie nahi → /dashboard → /login redirect ✅
Cookie nahi → /about     → /about dikh jaata hai ✅
Cookie hai  → /dashboard → dashboard dikh jaata hai ✅
```

---

## Summary

- `NextResponse.redirect(new URL("/login", request.url))` — redirect karta hai
- `new URL(path, base)` — naya URL object banata hai — sahi domain ke saath
- Matcher mein `/login` exclude karo — infinite loop rokne ke liye
- `.some()` — array mein koi element condition satisfy karta hai?
