## Problem — production mein kuch toot gaya — pata kaise chalega?

Tera app deploy hai. Ek user complaint karta hai — "kal raat dashboard open nahi ho raha tha."

Tu dekhne jaata hai — koi proof nahi. Kab hua? Kaunse page pe? Kitni baar? Kahan se request aayi?

**Koi record nahi toh kuch pata nahi chalta.** Yahi problem hai.

**Logs** — har request ka record — timestamp, URL, IP — sab note karo. Baad mein kuch bhi investigate kar sako.

---

## Proxy — logging ke liye best jagah

Agar har page mein alag alag log karo:

```tsx
// dashboard/page.tsx
console.log("dashboard visit hua")

// profile/page.tsx
console.log("profile visit hua")
```

Naya page bana — logging bhool gaye — woh page untracked. Aur saare pages mein repeat karna padega.

Proxy — **har request pehle wahan jaati hai** — ek jagah likho — sab cover.

---

## Step 1 — pehle sirf pathname log karo

`proxy.ts` mein abhi kya hai:

```ts
export function proxy(request: NextRequest) {
  return NextResponse.next()
}
```

Pehle bas yeh dekho — kaunsa URL aa raha hai. `request.nextUrl.pathname` — yeh 2.1 mein seekha tha — URL ka path. `console.log` karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  console.log(request.nextUrl.pathname)
  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Visit karo:

```
http://localhost:3000/dashboard
```

Terminal mein:

```
/dashboard
```

Chal raha hai. ✅

## Step 2 — request ka type bhi chahiye

Log mein sirf `/dashboard` hai — par yeh nahi pata ki request ka **type** kya tha.

HTTP mein requests alag alag types ki hoti hain — inhe **methods** kehte hain:

```
GET    → page visit karna — browser sirf data maang raha hai
POST   → form submit — data bhej raha hai server ko
PUT    → kuch update karna
DELETE → kuch delete karna
```

**Logging mein method kyun chahiye?**

Socho — `/api/user` pe kuch problem aayi. Agar sirf URL hai toh pata nahi — koi page visit kar raha tha ya form submit kar raha tha. Dono bahut alag cases hain.

`request.method` — yeh property request ka method deti hai — ek simple string — `"GET"`, `"POST"`, etc.

Abhi kya karna hai — `request.method` ek variable mein save karo — phir pathname ke saath log karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const method = request.method
  const pathname = request.nextUrl.pathname

  console.log(method, pathname)

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Terminal mein:

```
GET /dashboard
```

Dono aa gaye. ✅

## Step 3 — ek clean line mein format karo

`console.log(method, pathname)` — do alag arguments hain — Next.js unke beech space daal ke print karta hai. Kaam toh karta hai — par agar aur cheezein add karni hon toh messy ho jaayega.

**Template literal** — ek cleaner tarika hai strings banane ka. Normal string `"hello"` — template literal `` `hello` `` — backticks use karte hain. Andar `${}` mein koi bhi variable ya expression dal sakte hain:

```js
const naam = "Pravin"
console.log(`Hello ${naam}`)  // → Hello Pravin

const a = 5
const b = 3
console.log(`Sum: ${a + b}`)  // → Sum: 8
```

`${}` ke andar jo bhi likho — woh evaluate hota hai aur string mein fit ho jaata hai.

Ab `proxy.ts` mein template literal use karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const method = request.method
  const pathname = request.nextUrl.pathname

  console.log(`${method} ${pathname}`)

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Terminal mein:

```
GET /dashboard
```

Same output — par ab ek controlled string hai — aage aur cheezein add karna easy hoga. ✅

## Step 4 — timestamp add karo — kab hua?

Abhi pata nahi kab request aayi — subah aayi ya raat ko.

**`new Date()`** — JavaScript mein current date aur time ka object banata hai. `new` keyword — JavaScript mein kisi class ka object banane ke liye likhte hain — yeh `Date` class ka naya object hai.

**`.toISOString()`** — Date object ko ek standard readable string mein convert karta hai:

```
2024-01-15T10:30:45.123Z
```

Yeh format hai — `year-month-dayThour:minute:second.msZ`. `T` separator hai date aur time ke beech. `Z` matlab UTC timezone.

Pehle sirf yeh try karo separately — samjho kya milta hai:

```ts
console.log(new Date().toISOString())
// → 2024-01-15T10:30:45.123Z
```

Ab `proxy.ts` mein add karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const method = request.method
  const pathname = request.nextUrl.pathname
  const time = new Date().toISOString()

  console.log(`[${time}] ${method} ${pathname}`)

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Terminal mein:

```
[2024-01-15T10:30:45.123Z] GET /dashboard
```

Ab timestamp bhi hai. ✅
- Following is for better formating of the time:
```
const time = new Date().toLocaleString("en-IN", {
    timeZone: "Asia/Kolkata",
    year: "numeric",
    month: "short",
    day: "numeric",
    hour: "2-digit",
    minute: "2-digit",
    second: "2-digit",
    hour12: true,
  });
```

## Step 5 — IP bhi chahiye — kahan se aaya?

Last cheez — kaunsi IP se request aayi.

Yeh 2.2 mein seekha tha — `x-forwarded-for` header mein hota hai user ka IP. Locally `unknown` aayega — production pe actual IP.

Ab `proxy.ts` complete karo:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const method = request.method
  const pathname = request.nextUrl.pathname
  const time = new Date().toISOString()
  const ip = request.headers.get("x-forwarded-for") ?? "unknown"

  console.log(`[${time}] ${method} ${pathname} — ${ip}`)

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Alag alag pages visit karo — terminal mein har request ka record dikh raha hai:

```
[2024-01-15T10:30:45.123Z] GET /dashboard — unknown
[2024-01-15T10:30:46.456Z] GET /profile — unknown
[2024-01-15T10:30:47.789Z] GET /about — unknown
```

✅ — Production mein `unknown` ki jagah real IP aayega.

- `GET /` - This is an HTTP GET request to the root path
- `::ffff:127.0.0.1` - This is the IP address of the client making the request

## The IP Address `::ffff:127.0.0.1` means:
- `127.0.0.1` is the localhost IP address (your own computer)
- The `::ffff:` prefix indicates this is an IPv6-mapped IPv4 address
- This confirms the request is coming from your local machine

## So yes, you're getting requests locally! This means:
2. ✅ Your proxy middleware is active and logging requests
3. ✅ You're testing locally (which is great for development)

## Summary

- Logging — proxy mein ek jagah — sab routes cover
- `request.method` — request ka type — `"GET"`, `"POST"`, etc.
- Template literal `` `${variable}` `` — variables ko string mein mix karo cleanly
- `new Date().toISOString()` — current timestamp — readable format
- `request.headers.get("x-forwarded-for")` — user ka IP
