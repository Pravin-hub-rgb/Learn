# Doc 1.3 — Matcher — kaunse routes pe chale Proxy

## Abhi Tak Kya Hua

1.2 mein `proxy.ts` banaya — basic setup kiya — `console.log` se verify kiya ki proxy chal raha hai. `config.matcher` bhi lagaya — static files exclude kiye. Par woh regex pattern seedha copy kiya tha — samjha nahi tha. Aaj matcher properly samjhenge.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

`matcher` ke alag alag patterns samjhenge — specific routes include karna, exclude karna — aur practically test karenge.

---

## Problem pehle — kya issue hai?

Abhi tak humne `proxy.ts` banaya aur dekha ki woh har request pe chal raha hai. Lekin yeh problem hai:

**Problem:** Proxy har cheez pe chal raha hai — CSS files, JS files, images — sab pe! Yeh performance ka issue hai.

**Solution:** `matcher` use karna — taki proxy sirf unhi routes pe chale jahan humein check karna hai.

---

## Step 1 — Simple matcher try karo

Pehle ek simple case se shuru karte hain. `proxy.ts` mein matcher lagao:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  console.log("Proxy chal raha hai:", request.nextUrl.pathname)
  return NextResponse.next()
}

export const config = {
  matcher: "/dashboard",
}
```

**Test karo:** Browser mein visit karo:
```
http://localhost:3000/dashboard
```

**Expected:** Terminal mein "Proxy chal raha hai: /dashboard" aana chahiye
**Actual:** Kuch print nahi hua! ❌

**Error:** Kyu nahi chala? Kyunki Next.js server restart nahi hua!

**Fix:** Terminal mein `Ctrl+C` karo, phir dobara `npm run dev` chalao.

Ab dobara try karo — ab chalega ✅

---

## Step 2 — Multiple routes ke liye array

Ab multiple routes pe proxy chalana hai. Array use karo:

```ts
export const config = {
  matcher: ["/dashboard", "/profile", "/settings"],
}
```

**Test karo:**
```
http://localhost:3000/dashboard → proxy chala ✅
http://localhost:3000/profile   → proxy chala ✅  
http://localhost:3000/settings  → proxy chala ✅
http://localhost:3000/about     → kuch nahi ✅
```

**Server restart karna yaad rakhna!** Next.js config change hone pe restart karna padta hai.

---

## Step 3 — Children routes ke liye `:path*`

Socho `/dashboard` hai aur uske andar bhi pages hain:
- `/dashboard/settings`
- `/dashboard/analytics` 
- `/dashboard/users/123`

Sirf `/dashboard` likhne se sirf `/dashboard` pe chalega, children pe nahi.

`:path*` lagao:

```ts
export const config = {
  matcher: ["/dashboard/:path*"],
}
```

**Test karo:**
```
http://localhost:3000/dashboard              → proxy chala ✅
http://localhost:3000/dashboard/settings     → proxy chala ✅
http://localhost:3000/dashboard/users/123    → proxy chala ✅
```

**`*` ka matlab:** "zero ya zyada" — matlab `/dashboard` ke baad kuch bhi ho ya kuch na ho.

---

## Step 4 — Ab problem aayegi — static files

Abhi tak accha chal raha hai. Lekin ek problem hai:

**Problem:** Agar hum `/dashboard/:path*` use karenge, toh static files bhi block ho jayenge!

**Test karo:** Visit karo `http://localhost:3000/dashboard` — page load nahi hoga! ❌

**Kyun?** Kyunki Next.js static files load karta hai:
- `/_next/static/...` (CSS/JS files)
- `/_next/image/...` (optimized images)
- `/favicon.ico` (browser icon)

Yeh sab bhi `/dashboard` ke children ki tarah match ho jate hain!

---

## Step 5 — Negative matcher — solution

**Solution:** Negative matcher use karo — "sab pe chalo EXCEPT yeh specific cheezein":

```ts
export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico|.*\\.svg).*)"],
}
```

**Regex explain karte hain step-by-step:**

```ts
"/((?!_next/static|_next/image|favicon.ico).*)"
```

Break it down:
- `/` → path slash se shuru
- `(?!...)` → "negative lookahead" — matlab "agar yeh na ho"
- `_next/static|_next/image|favicon.ico` → in teeno ko chhod do
- `.*` → baaki kuch bhi

**Matlab:** "in teeno ko chhod ke baaki sab pe proxy chalao"

---

## Step 6 — Real scenario — login aur API routes

Real app mein aur bhi cheezein exclude karni padti hain:

```ts
export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico|login|api).*)"],
}
```

Ab test karo:

```
http://localhost:3000/              → proxy chala ✅
http://localhost:3000/dashboard     → proxy chala ✅
http://localhost:3000/login         → kuch nahi — public hai ✅
http://localhost:3000/api/anything  → kuch nahi — API route hai ✅
```

---

## Step 7 — Konsa pattern kab use karein?

**Rule of thumb:**

```
Agar kam protected routes hain (2-3) → specific list karo
  matcher: ["/dashboard/:path*", "/profile/:path*"]

Agar zyada protected routes hain → negative matcher use karo
  matcher: ["/((?!login|api|_next/static|_next/image|favicon.ico).*)"]
```

**Real apps mein:** Mostly negative matcher use hota hai kyunki protected routes zyada hote hain.

---

## Summary

- **Matcher kya hai:** Proxy kahan chale yeh control karta hai
- **Simple matcher:** Single string — sirf ek route
- **Multiple routes:** Array use karo
- **Children routes:** `:path*` lagao
- **Negative matcher:** Sab pe chalo EXCEPT specific cheezein
- **Server restart:** Config change pe restart karna yaad rakhna!
- **Real apps:** Mostly negative matcher use hota hai
