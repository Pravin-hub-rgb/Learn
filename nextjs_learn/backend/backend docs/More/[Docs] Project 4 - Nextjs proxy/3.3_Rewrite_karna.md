# Doc 3.3 — Rewrite karna — redirect se fark

## Abhi Tak Kya Hua

3.2 mein `redirect()` implement kiya — protected routes pe session check karke `/login` pe redirect kiya. Ab `rewrite()` seekhenge — jo bilkul alag kaam karta hai.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

`rewrite()` kya hai — `redirect()` se kaise alag hai — practical example ke saath implement karenge.

---

## Problem — redirect mein URL badal jaata hai

`redirect()` mein — user ko naye URL pe bheja jaata hai — URL bar mein URL badal jaata hai.

Par kabhi kabhi yeh nahi chahiye — URL same dikhao — content alag do.

---

## `rewrite()` — URL same, content alag

`rewrite()` kya karta hai:

```
User ne /home visit kiya
        ↓
Proxy ne rewrite kiya → /home-v2
        ↓
/home-v2 ka content aaya
        ↓
Par URL bar mein abhi bhi /home dikh raha hai
```

User ko pata hi nahi chala ki actually `/home-v2` ka page dikh raha hai.

---

## Practical use case — A/B testing

A/B testing kya hota hai? — Do versions banao page ke — kuch users ko version A dikhao — kuch ko version B — dekho kaun zyada kaam karta hai.

Example — `/home` pe do versions hain:

```
/home    → original version
/home-v2 → naya version — test kar rahe hain
```

Hum chahte hain — har doosra user `/home-v2` dekhe — par URL bar mein `/home` hi dikhe — user confuse na ho.

---

## Step 1 — pages banao

`app/home/page.tsx` banao:

```tsx
export default function Home() {
  return <h1>Home — Version A (Original)</h1>
}
```

`app/home-v2/page.tsx` banao:

```tsx
export default function HomeV2() {
  return <h1>Home — Version B (New)</h1>
}
```

Visit karo verify karne ke liye:

```
http://localhost:3000/home    → "Home — Version A" ✅
http://localhost:3000/home-v2 → "Home — Version B" ✅
```

---

## Step 2 — rewrite implement karo

`proxy.ts` update karo — abhi ke liye hardcode karenge — har `/home` visit pe rewrite karo `/home-v2` pe:

```ts
import { NextRequest, NextResponse } from "next/server"

export function proxy(request: NextRequest) {
  const pathname = request.nextUrl.pathname

  if (pathname === "/home") {
    return NextResponse.rewrite(new URL("/home-v2", request.url))
  }

  return NextResponse.next()
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`NextResponse.rewrite(new URL("/home-v2", request.url))` — same syntax hai `redirect()` jaisi — `new URL()` se URL banao — par `rewrite()` use kiya.

Visit karo:

```
http://localhost:3000/home
```

Kya dikh raha hai? — "Home — Version B (New)" — par URL bar mein `/home` dikh raha hai? ✅

Yahi hai `rewrite()` — content badla — URL nahi.

---

## Step 3 — redirect aur rewrite side by side dekhte hain

Dono try karo — fark clearly samajh aayega:

**Redirect:**

```ts
if (pathname === "/home") {
  return NextResponse.redirect(new URL("/home-v2", request.url))
}
```

`/home` visit karo → URL bar `/home-v2` ho jaata hai — content bhi `/home-v2` ka

**Rewrite:**

```ts
if (pathname === "/home") {
  return NextResponse.rewrite(new URL("/home-v2", request.url))
}
```

`/home` visit karo → URL bar `/home` hi rehta hai — content `/home-v2` ka

---

## Kab kaunsa use karein

```
Login required page    → redirect()
  User ko pata hona chahiye — "tu login pe gaya"

A/B testing           → rewrite()
  User ko pata nahi chalna chahiye — URL same dikhe

Old URL → new URL     → redirect()
  URL permanently change kar rahe hain

Maintenance page      → rewrite()
  /maintenance ka content dikhao — har URL pe — URL same rakho
```

---

## Summary

- `NextResponse.rewrite()` — URL same rakhta hai — content alag deta hai
- `NextResponse.redirect()` — URL bhi badalta hai — content bhi
- `new URL(path, request.url)` — dono mein same syntax
- A/B testing, maintenance page — rewrite ka common use case
