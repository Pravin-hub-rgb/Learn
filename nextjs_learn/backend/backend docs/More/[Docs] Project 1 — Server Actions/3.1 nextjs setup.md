# Doc 3.1 — Next.js Project Setup + Folder Structure Samjho

---

## Abhi Tak Kya Hua

Doc 1.1 mein samjha — Server Actions kyun.
Doc 2.1 mein samjha — Prisma kyun.

Ab theory khatam. **Actual kaam shuru karte hain.**

Sabse pehla kaam — ek Next.js project banana. Par sirf `npx create-next-app` chalana aur aage badh jaana nahi — **jo folders aur files banti hain, unhe samajhna bhi zaroori hai.** Kyunki aage ke saare docs mein hum inhi folders mein kaam karenge.

---

## Project Banao

Terminal mein yeh chalaao:

```bash
npx create-next-app@latest todo-app
```

Yeh command chalane ke baad kuch sawaal pooche jaayenge. **Exactly yeh choose karo:**

```
✔ Would you like to use TypeScript? → Yes
✔ Would you like to use ESLint? → Yes
✔ Would you like to use Tailwind CSS? → Yes
✔ Would you like your code inside a `src/` directory? → No
✔ Would you like to use App Router? → Yes
✔ Would you like to use Turbopack? → No
✔ Would you like to customize the import alias? → No
```

Kyun yeh choices? —

- **TypeScript** — types milenge, Prisma ke saath bahut kaam aata hai
- **Tailwind** — styling ke liye, alag CSS file nahi likhni padegi
- **App Router** — Server Actions sirf App Router mein kaam karte hain, Pages Router mein nahi
- **src/ directory — No** — ek extra folder kam, seedha `app/` root mein rahega

## Ab Folder Structure Dekho

Project ban gaya. Ab `todo-app` folder kholo. Kuch aisa dikhega:

```
todo-app/
├── app/
│   ├── favicon.ico
│   ├── globals.css
│   ├── layout.tsx
│   └── page.tsx
├── public/
├── .gitignore
├── next.config.ts
├── package.json
├── postcss.config.mjs
├── tailwind.config.ts
└── tsconfig.json
```

Bahut saari files hain. Par **tumhe abhi sirf kuch cheezein samajhni hain** — baaki apne aap clear hogi jab kaam karenge.

---

## Sirf Yeh Samjho Abhi — Kaunsa Folder Kisliye

### `app/` — Yahan Tumhara Poora App Rahega

Yeh **sabse important folder** hai. Tumhari saari pages, components, server actions — sab yahan aayenge.

Next.js mein ek special cheez hai — **`app/` folder ka structure hi tumhara URL structure hota hai.**

Matlab:
```
app/page.tsx              →   localhost:3000/
app/todos/page.tsx        →   localhost:3000/todos
app/todos/new/page.tsx    →   localhost:3000/todos/new
```

Folder banao — URL ban jaata hai. Koi alag routing config nahi likhni.

Abhi `app/` mein kya hai:

- **`page.tsx`** — yeh tumhara homepage hai — `localhost:3000/` pe jo dikhta hai
- **`layout.tsx`** — yeh ek wrapper hai — jo cheez **har page pe** dikhni chahiye woh yahan jaati hai, jaise navbar. Abhi ignore karo isko.
- **`globals.css`** — global CSS — Tailwind ki basic setup yahan hai

### `public/` — Static Files Yahan Jaati Hain

Images, icons, fonts — jo files directly browser ko deni hain bina kisi processing ke — woh yahan jaati hain.

Humara todo app mein koi image nahi — toh yeh folder abhi kaam nahi aayega.

### Baaki Files — Config Hain, Chhodo Abhi

`next.config.ts`, `tailwind.config.ts`, `tsconfig.json` — yeh sab configuration files hain. Next.js ne khud bana di hain. Inhe abhi touch nahi karna.

---

## Ek Aur Folder Banenge Hum — `lib/`

Yeh folder abhi nahi hai — **hum baad mein banayenge.** Par pehle se batata hoon kyun banayenge — taaki jab banayein toh out of blue na lage.

`lib/` folder ka convention yeh hai — **jo code kisi ek page ka nahi hai, jo reuse hoga multiple jagah** — woh `lib/` mein jaata hai.

Jaise Prisma ka database connection — woh ek jagah banana hai aur poore app mein reuse karna hai. Toh woh `lib/` mein jaayega.

```
lib/
└── prisma.ts    ← database connection yahan rahega
```

Jab woh step aayega, yaad rakhna — **`lib/` matlab reusable utility code.**

---

## Abhi Ka Project Structure — Itna Kaafi Hai

Abhi ke liye sirf yeh yaad rakho:

```
todo-app/
├── app/          ← tumhara poora app yahan — pages, components, actions
├── lib/          ← baad mein banayenge — reusable code ke liye
└── public/       ← static files — abhi kaam nahi
```

Baaki sab config hai — chhodo.

---

## Dev Server Chalao — Dekho Kaam Kar Raha Hai

```bash
cd todo-app
npm run dev
```

Browser mein kholo — `localhost:3000`

Next.js ka default page dikhega. Matlab project sahi se ban gaya.

## Summary

- `npx create-next-app@latest` se project bana — TypeScript, Tailwind, App Router choose kiya
- **`app/`** — sabse important folder — saara app yahan, folder structure = URL structure
- **`lib/`** — baad mein banayenge — reusable code ke liye, jaise database connection
- **`public/`** — static files ke liye — abhi kaam nahi
- Dev server chala ke check kiya — sab theek hai
