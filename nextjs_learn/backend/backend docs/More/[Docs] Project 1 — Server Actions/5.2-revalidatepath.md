# Doc 5.2 — Cache Fix Karo — `revalidatePath`

---

## Abhi Tak Kya Hua

Todos screen pe dikh rahe hain. Par naya todo add karo — screen pe nahi dikhta. Manually refresh karo — tab dikhta hai.

Problem — Next.js cache. Fix karna hai.

---

## `revalidatePath` — Next.js Ko Batao

`next/cache` se `revalidatePath` aata hai — yeh Next.js ko bolta hai:

> "Is path ka cache hatao — agli baar fresh data se render karo."

`TodoForm.tsx` mein `addTodo` function mein — `prisma.todo.create()` ke baad yeh add karo:

```tsx
import { prisma } from '@/lib/prisma'
import { revalidatePath } from 'next/cache'

export default function TodoForm() {

  async function addTodo(formData: FormData) {
    "use server"
    const title = formData.get('title') as string

    await prisma.todo.create({
      data: { title }
    })

    revalidatePath('/')    // ← yeh add hua
  }

  return (
    <form action={addTodo} className="flex gap-2">
      <input
        type="text"
        name="title"
        placeholder="Kya karna hai?"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        type="submit"
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Add Todo
      </button>
    </form>
  )
}
```

`revalidatePath('/')` — `'/'` matlab homepage. Is route ka cache hatao.

## Try Karo

"Buy groceries" likho — button dabao.

Naya todo **turant** screen pe dikh raha hai — refresh ki zaroorat nahi ✅

---

## Poora Flow Ab

```
User form submit karta hai
        ↓
addTodo server pe chala
        ↓
prisma.todo.create() — database mein save hua
        ↓
revalidatePath('/') — Next.js ko bataya "cache hatao"
        ↓
page.tsx dobara chala — prisma.todo.findMany() fresh data laya
        ↓
Screen pe naya todo dikh gaya ✅
```

---

## Summary

- `revalidatePath('/')` — server action ke baad cache clear kiya
- Ab naya todo add karne pe turant screen pe dikhta hai ✅

---

## Agla Step

Todo add ho raha hai, dikh raha hai — sab kaam kar raha hai.

Par `TodoForm.tsx` mein UI aur server action dono hain. Aage update, delete bhi aayenge — sab ek file mein rakhe toh messy hoga.

Convention yeh hai — **server actions alag file mein rakho.**

Yahi hoga **Doc 5.3 mein.**
