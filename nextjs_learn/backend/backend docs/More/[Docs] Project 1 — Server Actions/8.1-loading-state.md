# Doc 8.1 ‚Äî Loading State ‚Äî User Ko Pata Chale Kuch Ho Raha Hai

---

## Abhi Tak Kya Hua

CRUD poora ho gaya. Par ek cheez notice karo ‚Äî

Todo add karo ‚Äî button dabao ‚Äî kuch seconds tak kuch nahi dikhta. User sochega "hua ya nahi?" ‚Äî dobara click karega ‚Äî double submission ho sakti hai.

User ko feedback chahiye ‚Äî "haan bhai, kaam chal raha hai."

---

## Problem Practically Dekho

Abhi `TodoForm.tsx` mein yeh hai:

```tsx
<form action={addTodo}>
```

Button click karo ‚Äî button waise hi dikhta rehta hai ‚Äî koi indicator nahi. Loading state nahi hai.

Fix ‚Äî button ko "Adding..." kar dena hai jab submit ho raha ho ‚Äî aur disable karna hai.

---

## `useActionState` ‚Äî Introduce Karo

Abhi `form action={addTodo}` seedha function pass kiya hua hai.

`useActionState` ek React hook hai ‚Äî jo `addTodo` jaisa function leta hai aur **uske upar ek wrapper bana deta hai** ‚Äî jisme `isPending` bhi milta hai.

Aise use karte hain:

```tsx
const [state, formAction, isPending] = useActionState(addTodo, null)
```

Do cheezein pass ki hain ‚Äî samjho ek ek:

---

### Pehla ‚Äî `addTodo`

Yeh woh function hai jo actually run hoga jab form submit hoga. `useActionState` isko andar se call karta hai.

`useActionState` ek manager ki tarah hai ‚Äî tum manager ko bolte ho "yeh kaam karna hai" ‚Äî manager khud kaam karta hai aur tumhe update deta hai.

Toh:

```
form submit hua
      ‚Üì
formAction chala  ‚Üê yeh tumhara manager hai
      ‚Üì
andar se addTodo chala  ‚Üê yeh actual kaam hai
      ‚Üì
result state mein aaya
```

---

### Doosra ‚Äî `null`

Yeh **initial state** hai ‚Äî yaani `state` ki starting value kya ho.

`useState` yaad hai?

```tsx
const [count, setCount] = useState(0)
//                                  ‚Üë
//                            starting value ‚Äî shuru mein 0
```

Wahi yahan:

```tsx
const [state, formAction, isPending] = useActionState(addTodo, null)
//                                                             ‚Üë
//                                                   starting value ‚Äî shuru mein null
```

Shuru mein koi error nahi, koi message nahi ‚Äî blank slate ‚Äî toh `null` diya.

Baad mein jab `addTodo` kuch return karega ‚Äî woh `state` mein aa jaayega.

---

### Teen Cheezein Jo Milti Hain

```tsx
const [state, formAction, isPending] = useActionState(addTodo, null)
//     ‚Üë       ‚Üë           ‚Üë
//     |       |           |
//     |       |           action chal raha hai? true/false
//     |       |
//     |       form mein yeh denge ‚Äî action={formAction}
//     |
//     addTodo ne jo return kiya ‚Äî woh yahan milega
```

`state` ‚Äî abhi `null` ‚Äî baad mein error message aayega agar kuch galat hua

`formAction` ‚Äî form ke `action` mein yeh denge ‚Äî seedha `addTodo` nahi

`isPending` ‚Äî `true` jab form submit ho raha ho ‚Äî `false` jab ho gaya

---

## `TodoForm.tsx` Upgrade Karo

`useActionState` import karo aur hook use karo:

```tsx
import { useActionState } from 'react'
import { addTodo } from './actions'

export default function TodoForm() {
  const [state, formAction, isPending] = useActionState(addTodo, null)

  return (
    <form action={formAction} className="flex gap-2">
      <input
        type="text"
        name="title"
        placeholder="Kya karna hai?"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        type="submit"
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Add Todo
      </button>
    </form>
  )
}
```

Save karo ‚Äî **yeh error aayegi:**

```
You're importing a component that needs useActionState. 
This React Hook only works in a Client Component.
```

Kyun? `useActionState` ek React hook hai ‚Äî hooks sirf **Client Components** mein chalte hain. Abhi `TodoForm.tsx` Server Component hai.

Fix ‚Äî file ke top pe `'use client'` add karo:

```tsx
'use client'   // ‚Üê yeh add karo

import { useActionState } from 'react'
import { addTodo } from './actions'

export default function TodoForm() {
  const [state, formAction, isPending] = useActionState(addTodo, null)

  return (
    <form action={formAction} className="flex gap-2">
      ...
    </form>
  )
}
```

`action={addTodo}` ‚Üí `action={formAction}` ‚Äî manager ko diya, seedha worker ko nahi.

---

## Button Ko `isPending` Se Update Karo

```tsx
<button
  type="submit"
  disabled={isPending}
  className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600 disabled:opacity-50"
>
  {isPending ? 'Adding...' : 'Add Todo'}
</button>
```

`disabled={isPending}` ‚Äî jab chal raha ho ‚Äî button click nahi hoga.

`disabled:opacity-50` ‚Äî Tailwind class ‚Äî disabled hone pe button thoda faded dikhega.

`{isPending ? 'Adding...' : 'Add Todo'}` ‚Äî text bhi change hoga.

---

## `addTodo` Mein `prevState` Parameter Add Karo

`useActionState` internally action ko call karta hai ‚Äî aur ek extra parameter pass karta hai ‚Äî `prevState` ‚Äî yaani pehle ki state kya thi.

Toh `addTodo` ka signature thoda badlega ‚Äî `prevState` pehle aayega, `formData` baad mein:

```ts
export async function addTodo(prevState: unknown, formData: FormData) {
  const title = formData.get('title') as string
  await prisma.todo.create({ data: { title } })
  revalidatePath('/')
  return null
}
```

`prevState` ‚Äî abhi use nahi kar rahe ‚Äî `unknown` type de diya ‚Äî par parameter toh hona chahiye warna `useActionState` kaam nahi karega.

`return null` ‚Äî `useActionState` expect karta hai ki action kuch return kare ‚Äî abhi koi error nahi toh `null` return kiya.

## Summary

- `useActionState(addTodo, null)` ‚Äî `addTodo` ko wrap kiya, `null` initial state hai
- `formAction` ‚Äî `addTodo` ka wrapper ‚Äî form mein yeh diya
- `isPending` ‚Äî submit ho raha hai toh `true`
- `'use client'` ‚Äî hooks ke liye zaroori ‚Äî error aayi tab naturally add kiya
- Button `disabled` + text change ‚Äî user ko feedback milta hai
- `prevState` ‚Äî `useActionState` automatically pass karta hai ‚Äî signature mein add karna zaroori
- `return null` ‚Äî action kuch return kare ‚Äî `useActionState` ke liye
