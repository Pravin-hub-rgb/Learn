# Doc 3.3 — Prisma Install + Neon Se Connect

---

## Abhi Tak Kya Hua

Next.js project ban gaya. Neon pe database ban gaya. Connection string `.env.local` mein save hai.

Par abhi Next.js app ko **pata nahi** ki koi database bhi hai. Dono alag khade hain — connected nahi.

Ab karna yeh hai — **Prisma install karo aur use batao ki database kahan hai.**

---

## Prisma Install Karo

Terminal mein project folder mein jaao aur yeh chalaao:

```bash
npm install prisma --save-dev
npm install @prisma/client
```

Do alag cheezein install ho rahi hain — kyun?

**`prisma`** — yeh ek **tool** hai. Sirf development mein kaam aata hai — schema likhne ke liye, database migrate karne ke liye. Production mein nahi jaata. Isliye `--save-dev`.

**`@prisma/client`** — yeh woh cheez hai jo **actually database se baat karti hai** — tumhare app ke andar. Yeh production mein bhi chahiye. Isliye normal install.

Simple analogy — `prisma` ek blueprint banane ka tool hai, `@prisma/client` woh worker hai jo actually kaam karta hai.

---

## Prisma Initialize Karo

```bash
npx prisma init
```

Yeh command chalane ke baad do cheezein hoti hain:

**Pehli — `prisma/` folder ban jaata hai:**

```
todo-app/
├── app/
├── prisma/
│   └── schema.prisma    ← naya file — yahan database ka structure likhenge
├── lib/
└── ...
```

**Doosri — `.env` file ban jaati hai:**

```
DATABASE_URL="postgresql://johndoe:randompassword@localhost:5432/mydb?schema=public"
```

Yeh ek **placeholder** hai — tumhari actual connection string nahi. Isko abhi replace karna hai.

## `.env` Mein Apni Connection String Daalo

`.env` file kholo aur placeholder ki jagah apni Neon connection string paste karo:

```
DATABASE_URL="postgresql://username:password@ep-something.ap-south-1.aws.neon.tech/tododb?sslmode=require"
```

## Ek Zaroori Baat — `.env` vs `.env.local`

Yahan thoda confusing ho sakta hai — clear karte hain.

**`.env`** — Prisma ne yeh banayi. Prisma **sirf `.env`** padhta hai — `.env.local` nahi padhta.

**`.env.local`** — Next.js padhta hai isko.

Toh dono mein connection string honi chahiye. Tumne 3.2 mein `.env.local` mein daali thi — ab `.env` mein bhi daalo. Same string — dono jagah.

Aur `.gitignore` mein check karo — **dono listed hain ya nahi:**

```
# .gitignore
.env
.env.local
.env*.local
```

Agar `.env` nahi hai — **abhi add karo.** Warna connection string GitHub pe chali jaayegi.

---

## `schema.prisma` File — Dekho Andar Kya Hai

`prisma/schema.prisma` kholo. Kuch aisa dikhega:

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

**`generator client`** — Prisma ko JavaScript/TypeScript ke liye code generate karna hai. Hamesha yahi rahega — touch nahi karna.

**`datasource db`** — yeh **sabse important part hai abhi.**
- `provider = "postgresql"` — bata raha hai ki database PostgreSQL hai
- `url = env("DATABASE_URL")` — yeh bol raha hai "connection string `.env` file mein `DATABASE_URL` naam ke variable mein hai — wahan se le lena"

Matlab Prisma ne khud `.env` file mein dekha, `DATABASE_URL` nikala, aur Neon se connect ho jaayega.

## `lib/prisma.ts` — Ab Yahan Dhyan Do

Ab yaad hai 3.1 mein bataya tha — `lib/` folder reusable code ke liye hota hai?

Ab woh moment aa gaya. `lib/prisma.ts` file banao.

---

### Step 1 — Pehle Socho, Is File Ka Kaam Kya Hai?

Humara todo app mein kai jagah database se kaam hoga — todo add karna, todos laana, delete karna.

Har jagah database se baat karni hai — toh har jagah Prisma chahiye.

Toh sabse pehla kaam — **ek Prisma ka instance banana aur export karna** taaki poore app mein use ho sake.

`PrismaClient` ek class hai jo `@prisma/client` package mein hoti hai — iska instance banane se ek database connection milta hai.

Toh likho:

```ts
export const prisma = new PrismaClient()
```

`new PrismaClient()` — naya connection ban gaya. `export` — taaki doosri files import kar sakein.

---

### Step 2 — `PrismaClient` Kahan Se Aaya?

Abhi TypeScript error dega — `PrismaClient` defined nahi hai.

Kyunki `PrismaClient` `@prisma/client` package mein hai — import karna padega:

```ts
import { PrismaClient } from '@prisma/client'

export const prisma = new PrismaClient()
```

Ab file ready hai. Koi bhi file mein `import { prisma } from '@/lib/prisma'` likho — aur database ready.

---

### Step 3 — Practically Dekho, Kaam Kar Raha Hai?

`app/page.tsx` kholo. Wahan yeh likho temporarily:

```ts
import { prisma } from '@/lib/prisma'

export default async function Home() {
  console.log('Prisma connected:', prisma)
  return <div>Todo App</div>
}
```

Dev server chalao — `npm run dev`. Browser mein `localhost:3000` kholo.

**Par yahan ek error aayegi:**

```
@prisma/client did not initialize yet. 
Please run "prisma generate" and try to import it again.
```

Ruko — ghabrao mat. Yeh error **expected** hai. Samjhte hain kyun.

---

### Yeh Error Kyun Aayi — Samjho

Normal packages jaise `react`, `lodash` — install karo, seedha use karo. Done.

`@prisma/client` **alag hai.**

Yeh package install hone ke baad andar se **khali hoti hai** — koi code nahi hota. Kyunki yeh package tumhare **apne schema ke hisaab se** khud ko generate karti hai.

Matlab jab tak Prisma tumhara `schema.prisma` nahi padhega — `@prisma/client` ko pata hi nahi ki tumhare paas `Todo` model hai. Usme `prisma.todo` exist hi nahi karta abhi.

Isliye pehle ek baar **generate** karna padta hai — Prisma schema padhega aur `@prisma/client` ke andar code banayega.

---

### Step 4 — `prisma generate` Chalao

```bash
npx prisma generate
```

Yeh command kya karta hai?

`schema.prisma` padhta hai — `Todo` model dekh leta hai — aur `@prisma/client` ke andar TypeScript code generate karta hai. Ab `@prisma/client` ko pata hai:
- `prisma.todo` exist karta hai
- Usmein `id` hai, `title` hai, `done` hai
- `findMany()`, `create()`, `delete()` — yeh sab methods available hain

Output kuch aisa aayega:

```
✔ Generated Prisma Client (v5.x.x)
```

Ab dev server dobara chalao — `npm run dev`.

Terminal mein dekho:

```
Prisma connected: PrismaClient {
  _engineConfig: { ... },
  ...
}
```

**Yeh matlab — Prisma instance bana, import hua, kaam kar raha hai.** ✅

Console.log dekh liya? Ab yeh lines hata do `page.tsx` se — yeh sirf check ke liye tha.

---

### Step 5 — Par Ek Aur Problem Hai Development Mein

Abhi jo file hai:

```ts
import { PrismaClient } from '@prisma/client'

export const prisma = new PrismaClient()
```

Yeh production mein bilkul theek hai.

Par **development mein ek dikkat hai.**

Next.js development mein **hot reload** karta hai — matlab jab bhi koi file save karo, woh module dobara load hota hai.

Har reload pe `new PrismaClient()` dubara chalega — matlab **naya connection** ban jaayega. Phir se reload — phir naya. Phir se — phir naya.

Thodi der mein Neon bolega — "bhai, bahut zyada connections aa rahe hain" — aur error dega.

---

### Step 6 — Fix: Connection Global Mein Save Karo

Solution simple hai — **pehli baar connection banao, global mein rakh do. Agli baar reload pe — naaya mat banao, wahi purana use karo.**

JavaScript mein `globalThis` ek special object hai — yeh **poori application mein ek hi hota hai**, reload hone pe bhi naya nahi banta. Yahan connection store karenge.

Pehle check karo — kya global mein pehle se connection hai?

```ts
import { PrismaClient } from '@prisma/client'

const existing = (globalThis as any).prisma

export const prisma = existing ?? new PrismaClient()
```

`??` — nullish coalescing operator. Matlab:
- Left side kuch hai? — wahi lo
- Left side `undefined` ya `null` hai? — right side lo

---

### Step 7 — Aur Connection Global Mein Save Bhi Karo

Abhi check toh kar liya — par save kab karein?

Agar naya connection bana — toh use global mein save karo taaki **next reload pe existing mil jaaye:**

```ts
import { PrismaClient } from '@prisma/client'

const existing = (globalThis as any).prisma

export const prisma = existing ?? new PrismaClient()

if (process.env.NODE_ENV !== 'production') {
  (globalThis as any).prisma = prisma
}
```

Production mein yeh save nahi karte — production mein hot reload hoti nahi, toh problem hai hi nahi wahan.

---

### Step 8 — TypeScript Ko Sahi Batao

Abhi `(globalThis as any)` use kar rahe hain — `any` matlab TypeScript koi check nahi karta. Kaam karta hai par TypeScript ka fayda nahi milta.

Sahi tarika — TypeScript ko batao ki "globalThis mein ek `prisma` property hogi":

```ts
import { PrismaClient } from '@prisma/client'

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined
}

export const prisma =
  globalForPrisma.prisma ?? new PrismaClient()

if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma
}
```

`as unknown as { prisma: PrismaClient | undefined }` — TypeScript ko bata rahe hain ki is global object mein `prisma` property hogi — ya toh `PrismaClient` instance hogi, ya `undefined`. Ab TypeScript khush hai, type safety bhi hai.

## Poora Flow Ek Baar Dekho

```
.env  ←  DATABASE_URL yahan hai
  ↓
schema.prisma  ←  env("DATABASE_URL") se utha leta hai
  ↓
npx prisma generate  ←  @prisma/client ke andar code bana
  ↓
lib/prisma.ts  ←  PrismaClient banta hai — ek baar, global mein save, reuse hota hai
  ↓
Tumhara app  ←  jahan bhi chahiye, prisma import karo aur use karo
```

## Summary

- `prisma` (dev tool) aur `@prisma/client` (actual connector) install kiya
- `npx prisma init` se `schema.prisma` bani, `.env` mein connection string daali
- `lib/prisma.ts` mein **step by step** code badha — pehle simple instance, import, phir `prisma generate` kyun zaroori hai yeh samjha, phir hot reload problem, phir global fix, phir TypeScript
- `npx prisma generate` — yeh **ek baar zaroori hai** — tab `@prisma/client` kaam karta hai
- Console.log se practically verify kiya
