# Doc 3.3 — Prisma Install Karo

---

## Abhi Tak Kya Hua

Next.js project ban gaya. Neon pe database ban gaya. Connection string `.env.local` mein save hai.

Par abhi Next.js app ko **pata nahi** ki koi database bhi hai. Dono alag khade hain — connected nahi.

Ab karna yeh hai — **Prisma install karo.**

---

## Pehle Install Karo

```bash
npm install prisma --save-dev
npm install @prisma/client
npm install @prisma/adapter-pg pg
npm install -D @types/pg
```

Yeh sab ek saath install ho rahe hain — par actually teen alag cheezein hain. Todo app ke context mein samjhte hain —

**`prisma`** — yeh ek CLI tool hai. Jaise tum terminal mein `npx prisma generate` ya `npx prisma migrate` chalate ho — yeh woh tool hai. Sirf development mein kaam aata hai — jaise ek carpenter ka hammer. Hammer se ghar banta hai, par hammer ghar mein nahi rehta. Isliye `--save-dev`.

**`@prisma/client`** — yeh woh cheez hai jo tumhare todo app ke andar actually kaam karegi. Jab user "Add Todo" button dabayega — `@prisma/client` database se baat karega aur todo save karega. Yeh production mein bhi chahiye.

**`@prisma/adapter-pg` + `pg`** — Prisma v7 mein ek naya concept aaya — **adapter**. Socho aise — Prisma ek manager hai jo jaanta hai kya karna hai. Par actually database se baat karne ke liye usse ek local representative chahiye. PostgreSQL ke liye woh representative hai `pg`. Aur `@prisma/adapter-pg` dono ke beech translator ka kaam karta hai.

## `npx prisma init` Chalao

```bash
npx prisma init
```

Yeh command chalane ke baad do naye cheezein ban jaati hain project mein:

```
todo-app/
├── prisma/
│   └── schema.prisma    ← naya
├── prisma.config.ts     ← naya
└── ...
```

Dono ke baare mein jaante hain —

---

## `schema.prisma` — Yahan Todo Ka Structure Likhenge

Yeh file abhi kuch aisi dikhegi:

```prisma
generator client {
  provider = "prisma-client"
  output   = "../app/generated/prisma"
}

datasource db {
  provider = "postgresql"
}
```

Abhi sirf ek cheez change karni hai — `provider = "prisma-client"` ki jagah yeh likho:

```prisma
generator client {
  provider = "prisma-client-js"
  output   = "../app/generated/prisma"
}
```

Kyun? `"prisma-client"` Next.js ke Turbopack ke saath ek module error deta hai. `"prisma-client-js"` sahi kaam karta hai.

Baaki is file mein abhi kuch nahi karna — Todo model 3.5 mein likhenge.

## `prisma.config.ts` — Prisma V7 Mein Naya

Yeh file pehle nahi hoti thi — Prisma v7 mein aayi.

Isko kholo — kuch aisa dikhega:

```ts
import "dotenv/config";
import { defineConfig } from "prisma/config";

export default defineConfig({
  schema: "prisma/schema.prisma",
  migrations: {
    path: "prisma/migrations",
  },
  datasource: {
    url: process.env["DATABASE_URL"],
  },
});
```

Yeh file bol rahi hai — "bhai Prisma, teri saari config yahan hai — schema kahan hai, migrations kahan jaayenge, aur database ka address kahan se lena hai."

`process.env["DATABASE_URL"]` — yeh `.env` file mein se `DATABASE_URL` utha raha hai.

**Par abhi `.env` mein connection string nahi hai** — woh `.env.local` mein daali thi 3.2 mein. Prisma `prisma.config.ts` ke through sirf `.env` padhta hai. Toh `.env` mein bhi daalni padegi.

`.env` file kholo — yeh line add karo:

```
DATABASE_URL="postgresql://username:password@ep-something.ap-south-1.aws.neon.tech/tododb?sslmode=require"
```

Wahi connection string — jo 3.2 mein `.env.local` mein daali thi — same string yahan bhi.

Aur `.gitignore` mein `.env` add karo agar nahi hai:

```
.env
```

## Summary

- `prisma`, `@prisma/client`, adapter sab install kiya
- `npx prisma init` se `schema.prisma` aur `prisma.config.ts` bani
- `schema.prisma` mein `provider` fix kiya — Turbopack ke liye
- `prisma.config.ts` ne `DATABASE_URL` `.env` se leni hai — wahan string daali

Prisma install ho gaya. Ab `lib/prisma.ts` banani hai — jahan actual Prisma connection rahega — taaki poore app mein reuse ho sake.
