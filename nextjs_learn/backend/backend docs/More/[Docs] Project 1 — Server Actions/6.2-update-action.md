# Doc 6.2 — `TodoItem` Client Component + `updateTodo` Action

---

## Abhi Tak Kya Hua

`form action` checkbox pe kaam nahi karta — submit button chahiye hota hai. Toh `onChange` use karenge — aur uske liye Client Component banana padega.

---

## Step 1 — `TodoItem.tsx` Banao

`app/components/TodoItem.tsx` banao — abhi sirf UI:

```tsx
'use client'

export default function TodoItem({ todo }: { todo: { id: number; title: string; done: boolean } }) {
  return (
    <li className="flex items-center gap-2 p-3 border rounded">
      <input
        type="checkbox"
        defaultChecked={todo.done}
      />
      <span>{todo.title}</span>
    </li>
  )
}
```

`'use client'` — browser pe chalega — `onChange` use kar sakenge.

## Step 2 — `page.tsx` Mein `TodoItem` Use Karo

```tsx
import TodoItem from './components/TodoItem'

  <TodoItem key={todo.id} todo={todo} />

```

Screen pe same dikhega — par ab Client Component hai ✅

---

## Step 3 — `updateTodo` Action Banao

`actions.ts` mein `updateTodo` add karo — abhi sirf console.log:

```ts
export async function updateTodo(id: number, done: boolean) {
  console.log('updateTodo chala! id:', id, 'done:', done)
}
```

`formData` nahi — seedha `id` aur `done`. Kyun? Kyunki `onChange` se directly call karenge — `form action` nahi use karenge — toh directly values pass kar sakte hain.

---

## Step 4 — `onChange` Se Seedha Call Karo

`TodoItem.tsx` mein `onChange` lagao:

```tsx
'use client'

import { updateTodo } from '../actions'

export default function TodoItem({ todo }: { todo: { id: number; title: string; done: boolean } }) {
  return (
    <li className="flex items-center gap-2 p-3 border rounded">
      <input
        type="checkbox"
        defaultChecked={todo.done}
        onChange={(e) => {
          updateTodo(todo.id, e.target.checked)
        }}
      />
      <span>{todo.title}</span>
    </li>
  )
}
```

Checkbox click karo — **terminal** mein:

```
updateTodo chala! id: 1 done: true
```

✅ Kaam kar raha hai.

---

## Step 5 — Problem Notice Karo

Fast fast checkbox click karo — ek ke baad ek.

Terminal mein dekho — **multiple calls** ja rahe hain ek saath:

```
updateTodo chala! id: 1 done: true
updateTodo chala! id: 1 done: false
updateTodo chala! id: 1 done: true
```

Yeh problem hai — React ko pata nahi ki "ek action abhi chal raha hai — naya mat bhejo." Database mein race condition ho sakti hai — kaunsa update last mein aaya — woh save hoga.

Aur ek aur cheez — `revalidatePath` ke baad page re-render hoga — us waqt checkbox ki state "jump" karti hai — UX kharab lagta hai.

---

## `startTransition` Kyun?

React mein `startTransition` ek tarika hai — "yeh kaam background mein karo."

Jab `startTransition` mein action wrap karo — React samajh jaata hai ki "yeh transition chal rahi hai." Aur ek cheez milti hai — `isPending` — jo `true` hoti hai jab action chal raha ho.

`isPending` se checkbox `disabled` kar sakte hain — jab tak action complete na ho — user dobara click na kare.

---

## Step 6 — `useTransition` Import Karo

`useTransition` hook se `startTransition` aur `isPending` milte hain:

```tsx
import { useTransition } from 'react'

const [isPending, startTransition] = useTransition()
```

Abhi yeh add karo — bas import aur hook — kuch aur nahi:

```tsx
'use client'

import { useTransition } from 'react'
import { updateTodo } from '../actions'

export default function TodoItem({ todo }: { todo: { id: number; title: string; done: boolean } }) {
  const [isPending, startTransition] = useTransition()

  return (
    <li className="flex items-center gap-2 p-3 border rounded">
      <input
        type="checkbox"
        defaultChecked={todo.done}
        onChange={(e) => {
          updateTodo(todo.id, e.target.checked)
        }}
      />
      <span>{todo.title}</span>
    </li>
  )
}
```

---

## Step 7 — `startTransition` Mein Wrap Karo

`onChange` mein action ko `startTransition` ke andar rakho:

```tsx
onChange={(e) => {
  startTransition(() => {
    updateTodo(todo.id, e.target.checked)
  })
}}
```

Aur `isPending` se checkbox disable karo — jab tak action chal raha hai:

```tsx
<input
  type="checkbox"
  defaultChecked={todo.done}
  disabled={isPending}
  onChange={(e) => {
    startTransition(() => {
      updateTodo(todo.id, e.target.checked)
    })
  }}
/>
```

Ab fast click karo — checkbox disable ho jaata hai pehle click ke baad — dobara click nahi ho sakta — ek hi call jaayegi ✅

---

## Step 8 — Database Update Karo

`actions.ts` mein `updateTodo` complete karo:

```ts
export async function updateTodo(id: number, done: boolean) {
  await prisma.todo.update({
    where: { id },
    data: { done }
  })
  revalidatePath('/')
}
```

## Poora `TodoItem.tsx`

```tsx
'use client'

import { useTransition } from 'react'
import { updateTodo } from '../actions'

export default function TodoItem({ todo }: { todo: { id: number; title: string; done: boolean } }) {
  const [isPending, startTransition] = useTransition()

  return (
    <li className="flex items-center gap-2 p-3 border rounded">
      <input
        type="checkbox"
        defaultChecked={todo.done}
        disabled={isPending}
        onChange={(e) => {
          startTransition(() => {
            updateTodo(todo.id, e.target.checked)
          })
        }}
      />
      <span>{todo.title}</span>
    </li>
  )
}
```

## Poora `actions.ts`

```ts
"use server"

import { prisma } from '@/lib/prisma'
import { revalidatePath } from 'next/cache'

export async function addTodo(formData: FormData) {
  const title = formData.get('title') as string
  await prisma.todo.create({ data: { title } })
  revalidatePath('/')
}

export async function updateTodo(id: number, done: boolean) {
  await prisma.todo.update({ where: { id }, data: { done } })
  revalidatePath('/')
}
```

---

## Summary

- `TodoItem.tsx` — Client Component banaya — `onChange` ke liye
- `updateTodo` — seedha `id` aur `done` parameters — `formData` nahi
- `onChange` se seedha call kiya — kaam kiya — par fast click pe multiple calls
- `startTransition` — background mein action chalao
- `isPending` — action chal raha hai toh checkbox disable — dobara click nahi
- `prisma.todo.update()` — database mein `done` update kiya ✅
