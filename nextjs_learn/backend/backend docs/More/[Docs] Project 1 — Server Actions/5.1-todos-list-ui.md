# Doc 5.1 — Todos Screen Pe Dikhao

---

## Abhi Tak Kya Hua

Todo add ho raha hai — Prisma Studio mein dikh raha hai. Par screen pe kuch nahi dikh raha — sirf input aur button hai.

Coder soochega — "Todos screen pe dikhane hain. `page.tsx` mein fetch karta hoon."

---

## `page.tsx` Mein Fetch Karo

`prisma` import karo — `findMany()` se saare todos laao:

```tsx
import TodoForm from './TodoForm'
import { prisma } from '@/lib/prisma'

export default function Home() {
  const todos = await prisma.todo.findMany()

  return (
    <main className="max-w-xl mx-auto mt-10 p-4">
      <h1 className="text-2xl font-bold mb-6">My Todos</h1>
      <TodoForm />
    </main>
  )
}
```

Save karo — error aayegi:

```
'await' expressions are only allowed within async functions
```

---

## Kyun Error Aayi?

`await` ka matlab hai — "yeh kaam hone tak ruko." Database se response aane mein time lagta hai — isliye `await` chahiye.

Par `await` sirf `async` function ke andar use ho sakta hai.

Abhi `Home()` normal function hai — `async` nahi. Toh `await` nahi chala.

Fix — `async` lagao:

```tsx
export default async function Home() {
```

Next.js mein Server Components `async` ho sakte hain — yeh React ka normal feature nahi hai, Next.js ka feature hai. `page.tsx` ek Server Component hai — toh yeh kaam karega.

---

## Ab List Bhi Dikhao

Todos aa rahe hain — ab screen pe dikhao. `.map()` se har todo ke liye ek `<li>` banao:

```tsx
import TodoForm from './TodoForm'
import { prisma } from '@/lib/prisma'

export default async function Home() {
  const todos = await prisma.todo.findMany()

  return (
    <main className="max-w-xl mx-auto mt-10 p-4">
      <h1 className="text-2xl font-bold mb-6">My Todos</h1>
      <TodoForm />
      <ul className="mt-6 space-y-2">
        {todos.map((todo) => (
          <li key={todo.id} className="flex items-center gap-2 p-3 border rounded">
            <span>{todo.title}</span>
          </li>
        ))}
      </ul>
    </main>
  )
}
```

`todos.map()` — har todo ke liye ek `<li>` banao.

`key={todo.id}` — React ko har item ka unique identifier chahiye — `id` use karo.

Save karo — database mein jo todos hain woh screen pe dikh rahe hain ✅

---

## Naya Todo Add Karo — Par Dikh Nahi Raha

"Buy groceries" likho — button dabao.

Terminal mein log aata hai. Prisma Studio mein bhi save ho jaata hai.

Par **screen pe naya todo nahi dikh raha.** Manually refresh karo — tab dikh raha hai.

---

## Kyun Nahi Dikh Raha?

Next.js pages ko **cache** karta hai — ek baar render hua, store kar liya.

Jab Server Action se kuch save hota hai — Next.js ko khud pata nahi chalta ki "kuch badla, dobara render karo." Isliye purana cached version dikhata rehta hai.

Fix — Server Action ke baad Next.js ko batana padega — "is page ka cache hatao, fresh render karo."

Yahi hoga **Doc 5.2 mein.**

---

## Summary

- `prisma.todo.findMany()` — database se saare todos laaye
- Normal function mein `await` nahi chala — error aayi
- `async` component — naturally aaya — error fix hua
- `.map()` se har todo ke liye `<li>` banayi — screen pe dikh rahe hain ✅
- Naya todo add kiya — screen pe nahi dikha — Next.js cache ki wajah se
