## Pehle Socho — Ek Todo Mein Kya Kya Hoga?

Seedha code mein mat jaao. Pehle practically socho —

> "Mujhe todo app banana hai. Toh ek todo object mein kya kya information hogi?"

Sochte hain saath mein —

**Title toh hoga** — "Buy milk", "Call mom" — koi text hoga.

**Done hoga** — todo complete hua ya nahi — `true` ya `false`. `done: false` matlab abhi pending. `done: true` matlab complete.

**Id hogi** — socho agar do todos hain same title ke saath — "Buy milk" aur "Buy milk". Inhe alag kaise pehchanoge? Isliye har todo ka ek unique number hoga — `id`. Yeh automatically 1, 2, 3... generate hoga.

Bas. **Teen cheezein — `id`, `title`, `done`.** Itna kaafi hai abhi.

```
id     → 1
title  → "Buy milk"
done   → false
```

Yahi columns banenge database ki `todos` table mein.

## `schema.prisma` Mein Todo Model Likho

`prisma/schema.prisma` kholo. Abhi sirf yeh hai:

```prisma
generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
```

Ab socho — **Prisma ko kaise batayein ki ek `todos` table chahiye jisme `id`, `title`, `done` columns hों?**

Prisma mein iske liye `model` keyword hota hai. Toh likho:

```prisma
model Todo {

}
```

Khali model hai abhi. Iske andar teen cheezein add karni hain — ek ek karke.

---

### Pehle — `id`

Id ek number hai — integer. Aur yeh unique hona chahiye har todo ke liye. Prisma mein integer type hota hai `Int`:

```prisma
model Todo {
  id Int
}
```

Par abhi sirf type define kiya — do cheezein aur batani hain Prisma ko:

Pehli — yeh primary key hai, matlab **har row ka unique identifier**. Iske liye `@id` lagate hain:

```prisma
model Todo {
  id Int @id
}
```

Doosri — id manually nahi deni, automatically 1, 2, 3... badhti jaaye. Iske liye `@default(autoincrement())`:

```prisma
model Todo {
  id Int @id @default(autoincrement())
}
```

`@default()` matlab — "jab naya todo bane aur id na di jaaye, toh yeh default value use karo."
`autoincrement()` matlab — "khud se 1 se shuru karke badhte jao."

---

### Phir — `title`

Title ek text hai — Prisma mein text type hota hai `String`:

```prisma
model Todo {
  id    Int    @id @default(autoincrement())
  title String
}
```

Koi `@default` nahi — kyunki title toh user dega. Agar title nahi diya toh Prisma error dega — jo sahi hai, title ke bina todo ka koi matlab nahi.

---

### Phir — `done`

Done sirf `true` ya `false` hoga — Prisma mein yeh type hota hai `Boolean`:

```prisma
model Todo {
  id    Int     @id @default(autoincrement())
  title String
  done  Boolean
}
```

Par socho — naya todo banane pe `done` kya hoga by default?

Hamesha `false` — matlab incomplete. Toh `@default(false)` lagao:

```prisma
model Todo {
  id    Int     @id @default(autoincrement())
  title String
  done  Boolean @default(false)
}
```

Ab model complete hai.

## Abhi Schema Likha — Par Database Ko Pata Nahi

Yeh samajhna zaroori hai —

`schema.prisma` mein model likhne se **database mein table nahi banti.** Abhi sirf ek blueprint hai — kaagaz pe likha hai.

Neon ke database mein abhi bhi koi `todos` table nahi hai.

Blueprint ko actual table mein convert karne ke liye ek step hai — **migration.**

---

## Migration Kya Hoti Hai?

Simple concept —

> "Mera schema yeh hai — database ko bhi aise hi bana do."

Tum Prisma ko bolte ho — "bhai, `schema.prisma` dekh, aur Neon ke database mein waise hi tables bana do."

Prisma kya karta hai? Woh khud SQL likhta hai — `CREATE TABLE todos (...)` — aur Neon pe run karta hai.

Tumhe SQL nahi likhni. **Prisma likhta hai.**

---

## Migration Run Karo

```bash
npx prisma migrate dev --name init
```

Yeh command tod ke samjho:

- `prisma migrate dev` — development mein migration run karo
- `--name init` — is migration ko naam do "init" — yeh pehli migration hai toh "init" theek hai

Yeh command chalane ke baad kya hota hai:

1. Prisma `schema.prisma` padhta hai
2. Khud SQL generate karta hai — `CREATE TABLE "todos" (...)`
3. Woh SQL Neon pe run karta hai
4. Neon mein `todos` table ban jaati hai
5. `prisma/migrations/` folder banta hai — yahan migration history save hoti hai

Output kuch aisa dikhega:

```
✔ Generated Prisma Client
✔ Your database is now in sync with your schema.
```

Yeh message matlab — **connection sahi tha, table ban gayi.** Yahi tha actual connection test — migrate kaam kiya toh sab theek hai.

## Practically Verify Karo — Table Bani Ya Nahi?

`app/page.tsx` mein temporarily yeh likho:

```ts
import { prisma } from '@/lib/prisma'

export default async function Home() {
  const todos = await prisma.todo.findMany()
  console.log('Todos:', todos)
  return <div>Todo App</div>
}
```

Yahan kya ho raha hai —

`prisma.todo` — Prisma ko pata hai `Todo` model hai schema mein. Toh `prisma.todo` se us table pe operations kar sakte ho.

`findMany()` — yeh Prisma ka built-in method hai — "saare todos laao database se." Abhi table khali hai toh empty array aayega — `[]`.

Terminal mein dekho:

```
Todos: []
```

Empty array — matlab **table ban gayi, connection kaam kar raha hai, Prisma sahi se kaam kar raha hai.** Bas abhi koi data nahi hai.

Console.log dekh liya? Ab yeh lines hata do `page.tsx` se — yeh sirf verify karne ke liye tha.
