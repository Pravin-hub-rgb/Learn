# Doc 7.2 — `deleteTodo` Action Likho

---

## Abhi Tak Kya Hua

Delete button UI ban gayi — `form action={deleteTodo}` laga diya. Par `deleteTodo` abhi bani nahi — error aa rahi hogi.

Ab `actions.ts` mein `deleteTodo` banate hain.

---

## Step 1 — `deleteTodo` Banao — Pehle Console.log

`actions.ts` mein add karo:

```ts
export async function deleteTodo(formData: FormData) {
  console.log('deleteTodo chala!')
}
```

`deleteTodo` import karo `TodoItem.tsx` mein:

```tsx
import { updateTodo, deleteTodo } from '../actions'
```

Delete button click karo — **terminal** mein:

```
deleteTodo chala!
```

✅ Action connect ho gayi.

---

## Step 2 — `id` Nikalo

"Action chal rahi hai. Ab kaunsa todo delete karna hai — `id` chahiye."

Hidden input se `id` aa raha hai `formData` mein. Nikalo aur log karo:

```ts
export async function deleteTodo(formData: FormData) {
  const id = formData.get('id')
  console.log('delete karna hai id:', id)
  console.log('id ka type:', typeof id)
}
```

Delete button click karo — terminal mein:

```
delete karna hai id: 1
id ka type: string
```

Dekho — `id` ki value `1` hai par **type `string` hai** — number nahi.

---

## Kyun String Aaya?

`formData` browser ka feature hai — browser sab inputs ki values **string** mein store karta hai. Chahe tumne `value={todo.id}` mein number diya ho — HTML mein jaate jaate string ban jaata hai.

Toh `formData.get('id')` hamesha string deta hai — `"1"`, `"2"` — number nahi.

Par `prisma.todo.delete()` ko `id` **number** chahiye — string nahi chalegi.

Fix — `Number()` se convert karo:

```ts
const id = Number(formData.get('id'))
```

`Number("1")` → `1` — string se number ban gaya.

Dobara log karo:

```ts
export async function deleteTodo(formData: FormData) {
  const id = Number(formData.get('id'))
  console.log('delete karna hai id:', id)
  console.log('id ka type:', typeof id)
}
```

Terminal mein:

```
delete karna hai id: 1
id ka type: number
```

✅ Ab number hai.

---

## Step 3 — Database Se Delete Karo

`id` mil gayi — ab database se todo hatao.

`prisma.todo.delete()` — Prisma ka built-in method — row delete karta hai:

```ts
await prisma.todo.delete({
  where: { id }
})
```

`where` — kaunsa todo delete karna hai yeh batao. `{ id }` — yeh shorthand hai `{ id: id }` ka — variable ka naam aur key same hain toh ek baar hi likhte hain.

`await` — database operation hai — time lagta hai — wait karo.

Poora function:

```ts
export async function deleteTodo(formData: FormData) {
  const id = Number(formData.get('id'))

  await prisma.todo.delete({
    where: { id }
  })

  revalidatePath('/')
}
```

`revalidatePath('/')` — page fresh karo — deleted todo screen pe se gayab ho jaayega.

## Poora `actions.ts`

```ts
"use server"

import { prisma } from '@/lib/prisma'
import { revalidatePath } from 'next/cache'

export async function addTodo(formData: FormData) {
  const title = formData.get('title') as string
  await prisma.todo.create({ data: { title } })
  revalidatePath('/')
}

export async function updateTodo(id: number, done: boolean) {
  await prisma.todo.update({ where: { id }, data: { done } })
  revalidatePath('/')
}

export async function deleteTodo(formData: FormData) {
  const id = Number(formData.get('id'))
  await prisma.todo.delete({ where: { id } })
  revalidatePath('/')
}
```

## Poora `TodoItem.tsx`

```tsx
'use client'

import { useTransition } from 'react'
import { updateTodo, deleteTodo } from '../actions'

export default function TodoItem({ todo }: { todo: { id: number; title: string; done: boolean } }) {
  const [isPending, startTransition] = useTransition()

  return (
    <li className="flex items-center gap-2 p-3 border rounded">
      <input
        type="checkbox"
        defaultChecked={todo.done}
        disabled={isPending}
        onChange={(e) => {
          startTransition(() => {
            updateTodo(todo.id, e.target.checked)
          })
        }}
      />
      <span>{todo.title}</span>
      <form action={deleteTodo}>
        <input type="hidden" name="id" value={todo.id} />
        <button type="submit">Delete</button>
      </form>
    </li>
  )
}
```
