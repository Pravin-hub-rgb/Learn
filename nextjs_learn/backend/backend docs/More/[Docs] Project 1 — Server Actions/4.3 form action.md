# Doc 4.3 — `form action` + `FormData`

---

## Abhi Tak Kya Hua

4.2 mein yeh problem thi:

```
'use client' + useState + "use server"  ❌
```

`useState` ke liye `'use client'` chahiye — par `'use client'` ke saath `"use server"` nahi chalta.

Toh `useState` ke bina value kaise milegi server pe?

---

## Solution — `FormData`

Jab HTML `form` submit hota hai — browser automatically ek **`FormData` object** banata hai. Usme form ke saare inputs ki values hoti hain. Yeh automatically server action ko milta hai — parameter mein.

`useState` ki zaroorat hi nahi — browser khud value pakad ke server action ko de deta hai.

---

## `'use client'` Hatao, `useState` Hatao

Abhi jo code hai:

```tsx
'use client'
import { useState } from 'react'

export default function TodoForm() {
  const [title, setTitle] = useState('')

  async function addTodo() {
    "use server"
    console.log('Server pe chala!')
  }

  return (
    <div className="flex gap-2">
      <input
        type="text"
        value={title}
        onChange={(e) => setTitle(e.target.value)}
        placeholder="Kya karna hai?"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        onClick={addTodo}
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Add Todo
      </button>
    </div>
  )
}
```

`'use client'` hatao, `useState` hatao, `value` aur `onChange` hatao — clean component:

```tsx
export default function TodoForm() {

  async function addTodo() {
    "use server"
    console.log('Server pe chala!')
  }

  return (

```

## `div` Ko `form` Se Replace Karo

`FormData` milne ke liye `form` element chahiye — `div` nahi. Aur `onClick` ki jagah `action`:

```tsx
export default function TodoForm() {

  async function addTodo(formData: FormData) {
    "use server"
    console.log('Server pe chala!')
  }

  return (
    <form action={addTodo} className="flex gap-2">
      <input
        type="text"
        placeholder="Kya karna hai?"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        type="submit"
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Add Todo
      </button>
    </form>
  )
}
```

Teen cheezein badli —

**`div` → `form`** — ab proper HTML form hai.

**`onClick` → `action={addTodo}`** — form submit hone pe `addTodo` chalega — button click pe bhi, Enter key pe bhi.

**`type="submit"`** — button ko batao ki yeh form submit karega.

**`formData: FormData`** — parameter add kiya — browser yeh automatically pass karega.

---

## Value Nikalo `formData` Se

`formData` mein value dhundne ke liye — input ko `name` chahiye. `name="title"` add karo:

```tsx
<input
  type="text"
  name="title"
  placeholder="Kya karna hai?"
  className="border border-gray-300 rounded px-3 py-2 flex-1"
/>
```

Ab `formData.get('title')` se nikalo:

```tsx
async function addTodo(formData: FormData) {
  "use server"
  const title = formData.get('title')
  console.log('Title:', title)
}
```

`formData.get('title')` — "is object mein se `title` naam wali value nikalo." Isliye `name="title"` zaroori tha — naam se pehchaan hoti hai.

## Summary

- 4.2 ki problem — `'use client'` + `"use server"` ek saath nahi chalte
- Solution — `FormData` — browser khud value pakad ke server action ko deta hai
- `div` → `form`, `onClick` → `action={addTodo}`, `type="submit"` add kiya
- `name="title"` — input ko naam diya — `formData.get('title')` se value mili
- Value server pe aa gayi — bina `useState`, bina `'use client'` ✅
