# Doc 4.2 — Button Ko Kuch Karna Sikhao

---

## Abhi Tak Kya Hua

`TodoForm.tsx` mein input aur button dikh raha hai. Par button dabao — kuch nahi hota.

---

## Step 1 — Button Click Pe Kuch Karo

Coder ka pehla sawaal — "Button click pe kuch toh ho — pehle console.log karta hoon."

`onClick` lagao button pe:

```tsx
export default function TodoForm() {

  function handleClick() {
    console.log('Button clicked!')
  }

  return (
    <div className="flex gap-2">
      <input
        type="text"
        placeholder="Kya karna hai?"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        onClick={handleClick}
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Add Todo
      </button>
    </div>
  )
}
```

Save karo — error aayegi:

```
Event handlers cannot be passed to Client Component props.
```

Kyun? Next.js mein default mein har component **Server Component** hota hai. Server pe `onClick` jaisi cheezein nahi hoti — yeh browser ki cheezein hain.

Fix — file ke top pe `'use client'` lagao:

```tsx
'use client'

export default function TodoForm() {

  function handleClick() {
    console.log('Button clicked!')
  }

  return (
    <div className="flex gap-2">
      <input
        type="text"
        placeholder="Kya karna hai?"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        onClick={handleClick}
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Add Todo
      </button>
    </div>
  )
}
```

Button dabao — **browser console** (F12) mein dekho:

```
Button clicked!
```

✅ Kaam kar raha hai.

---

## Step 2 — Value Chahiye

"Button click pe log aa raha hai. Ab jo user ne type kiya woh bhi chahiye."

`useState` lagao — value track karne ke liye:

```tsx
'use client'
import { useState } from 'react'

export default function TodoForm() {
  const [title, setTitle] = useState('')

  function handleClick() {
    console.log('Title:', title)
  }

  return (
    <div className="flex gap-2">
      <input
        type="text"
        value={title}
        onChange={(e) => setTitle(e.target.value)}
        placeholder="Kya karna hai?"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        onClick={handleClick}
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Add Todo
      </button>
    </div>
  )
}
```

"Buy milk" likho, button dabao — **browser console** mein:

```
Title: Buy milk
```

✅ Value mil gayi.

---

## Step 3 — Ab Database Mein Save Karo

"Value mil gayi. Ab database mein save karte hain."

`handleClick` mein Prisma use karo:

```tsx
import { prisma } from '@/lib/prisma'

function handleClick() {
  prisma.todo.create({   // ❌
    data: { title }
  })
}
```

Yeh kaam nahi karega. Kyun?

`'use client'` laga hua hai — matlab yeh component **browser pe chal raha hai.** Browser ke paas Prisma nahi hai — database connection browser pe nahi hota.

Toh kya karein?

---

## Pehle Wala Tarika — API Route

Ek option hai — API route banao:

```tsx
fetch('/api/todos', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ title })
})
```

Kaam karta hai — par alag file, `fetch`, `JSON.stringify`, `headers` — bahut boilerplate sirf ek todo add karne ke liye. Yahi toh Doc 1.1 mein problem thi.

---

## Naya Tarika — Server Action

Ek aur option hai — **Server Action.**

Idea yeh hai — ek function banao jo **server pe chale** — browser pe nahi. Toh Prisma seedha use kar sako.

Yeh karne ke liye function ke andar `"use server"` likhte hain — Next.js samajh jaata hai ki yeh function server pe chalana hai:

`'use client'` hatao. `useState` hatao. Naya function banao `addTodo`:

```tsx
export default function TodoForm() {

  function addTodo() {
    "use server"
    console.log('Server pe chala!')
  }

  return (
    <div className="flex gap-2">
      ...
    </div>
  )
}
```

Save karo — error aayegi:

```
Server Actions must be async functions
```

Kyun `async`? Yeh function server pe jaayega — network ke through. Network pe jaane mein time lagta hai — JavaScript mein jo bhi time leta hai woh `async` hona chahiye.

```tsx
async function addTodo() {
  "use server"
  console.log('Server pe chala!')
}
```

Error gayi ✅

---

## `onClick` Se Connect Karo

```tsx
export default function TodoForm() {

  async function addTodo() {
    "use server"
    console.log('Server pe chala!')
  }

  return (
    <div className="flex gap-2">
      <input
        type="text"
        placeholder="Kya karna hai?"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        onClick={addTodo}
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Add Todo
      </button>
    </div>
  )
}
```

Button dabao — **terminal** mein dekho:

```
Server pe chala!
```

✅ Terminal mein aaya — matlab **server pe chala.**

## Ab Value Chahiye Server Pe

"Server pe chal raha hai — ab `title` bhi chahiye wahan."

`useState` wapas lagao — `title` lene ke liye:

```tsx
'use client'
import { useState } from 'react'

export default function TodoForm() {
  const [title, setTitle] = useState('')

  async function addTodo() {
    "use server"
    console.log('Title:', title)
  }

  return (
    <div className="flex gap-2">
      <input
        type="text"
        value={title}
        onChange={(e) => setTitle(e.target.value)}
        placeholder="Kya karna hai?"
        className="border border-gray-300 rounded px-3 py-2 flex-1"
      />
      <button
        onClick={addTodo}
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Add Todo
      </button>
    </div>
  )
}
```

Save karo — **yeh error aayegi:**

```
It is not allowed to define inline "use server" annotated Server Actions 
in Client Components.
```

---

## Kyun Yeh Error Aayi?

`useState` ke liye `'use client'` lagana pada — matlab component **browser pe** hai.

Par `"use server"` function **server pe** chalta hai.

**Ek hi file mein dono nahi ho sakte.** Next.js ne rok diya.

Toh yeh approach kaam nahi karegi:

```
'use client' + useState + "use server"  ❌
```

Yahan 4.2 khatam — yeh problem hai. Solution **Doc 4.3 mein.**

---

## Summary

- `onClick` lagaya → error → `'use client'` lagaya → browser console mein log ✅
- `useState` se value li → browser console mein title ✅
- Prisma use karna tha → browser pe nahi chalta ❌
- API route option → bahut boilerplate
- **Server Action** introduce kiya — `"use server"` function ke andar — server pe chala ✅
- `useState` wapas lagaya value ke liye → `'use client'` + `"use server"` → **error** ❌

---

## Agla Step

Problem clear hai — `'use client'` aur `"use server"` ek saath nahi chalte.

Toh `useState` ke bina value kaise milegi server pe?

Yahi solve hoga **Doc 4.3 mein.**
