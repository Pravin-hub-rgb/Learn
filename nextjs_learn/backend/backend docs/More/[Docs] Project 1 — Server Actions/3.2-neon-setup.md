# Doc 3.2 — Neon Pe PostgreSQL Setup

---

## Abhi Tak Kya Hua

Next.js project ban gaya — `app/` folder samjha, dev server chal raha hai.

Ab ek sawaal —

> **"Data store kahan hoga?"**

Humara todo app mein jo bhi todos add honge — woh kahin save hone chahiye. Warna page refresh karo — sab gone.

Iske liye chahiye — **database.**

---

## Database Kahan Chalega?

Do options hain —

**Option 1 — Local machine pe PostgreSQL install karo**
Apne computer pe PostgreSQL install karo, wahan database banao.
Dikkat — har developer ke machine pe alag setup, version issues, configuration alag alag. Beginner ke liye painful.

**Option 2 — Cloud pe database — Neon**
Browser mein jaao, account banao, ek button click karo — database ready. Koi installation nahi. Koi configuration nahi.

Hum **Neon** use karenge. Kyun Neon specifically?

- Free tier hai — koi credit card nahi chahiye
- PostgreSQL ka hi hai — koi alag syntax nahi seekhni
- Next.js ke saath best fit hai — Vercel ne khud recommend kiya hai
- Ek **connection string** milti hai — woh copy karo, app mein paste karo, done

---

## Neon Pe Database Banao

**Step 1 — Account banao**

[neon.tech](https://neon.tech) pe jaao → Sign up karo — GitHub se kar sakte ho, easy hai.

---

**Step 2 — Naya Project banao**

Login ke baad dashboard dikhega. **"New Project"** button dikhega — click karo.

Ek form aayega:

```
Project name:  todo-app         ← kuch bhi rakho
Database name: tododb           ← yeh rakho
Region:        AWS ap-south-1   ← India ke paas wala choose karo
```

**"Create Project"** click karo.

---

**Step 3 — Connection String Copy Karo**

Project banne ke baad ek popup aayega — yahan tumhe ek **connection string** milegi. Kuch aisi dikhegi:

```
postgresql://username:password@ep-something.ap-south-1.aws.neon.tech/tododb?sslmode=require
```

Yeh string **tumhara database ka address + password** hai — ek saath.

Isko samjho thoda —

```
postgresql://    ← yeh PostgreSQL hai, yeh batata hai
username         ← tumhara Neon username
password         ← tumhara password
@ep-something... ← database server ka address — kahan hai woh
/tododb          ← database ka naam
?sslmode=require ← secure connection use karo
```

**Abhi copy karo yeh string — dialog band mat karo.** Agले step mein kaam aayegi.

---

## `.env` File — Connection String Yahan Jaayegi

Ab Next.js project mein wapas aao.

Root mein ek file hogi — `.env.local` — agar nahi hai toh banao. Yahan environment variables jaate hain — matlab **sensitive information jo code mein directly nahi likhni.**

Kyun code mein directly nahi likhte? Kyunki code GitHub pe jaata hai — aur connection string GitHub pe gayi toh koi bhi tumhara database access kar sakta hai.

`.env.local` mein yeh likho:

```
DATABASE_URL="postgresql://username:password@ep-something.ap-south-1.aws.neon.tech/tododb?sslmode=require"
```

Woh string jo copy ki thi — woh yahan paste karo quotes ke andar.

---

## Check Karo — `.gitignore` Mein Hai?

Project root mein `.gitignore` file hai. Isko kholo aur dekho — `.env.local` usmein listed hai ya nahi.

```
# .gitignore
.env.local    ← yeh hona chahiye
.env*.local
```

`create-next-app` ne yeh automatically add kar diya hoga. Matlab `.env.local` kabhi GitHub pe nahi jaayegi — tumhara password safe hai.

---

## Summary

- Data store karne ke liye database chahiye — **Neon** use kiya, cloud pe PostgreSQL
- Neon pe project banaya, connection string mili
- Connection string `.env.local` mein rakhi — code mein directly nahi — security ke liye
- `.gitignore` mein `.env.local` hai — GitHub pe nahi jaayegi
