## Gender Radio Button Problem

Jab hum radio buttons use karte hain aur koi bhi radio select nahi hota — browser `null` bhejta hai, string nahi.

`z.string()` expects ek string — `null` receive karta hai toh validation fail ho jati hai with error:
```
Invalid input: expected string, received null
```

Par hum chahte hain ki:
- Agar koi radio select hai — toh validate ho
- Agar koi radio select nahi hai — toh bhi validation run ho aur custom error message dikhe

## Solution: `.nullable()` + `.refine()`

Zod mein `.nullable()` ka matlab hai — "yeh field `null` bhi ho sakta hai."

```tsx
gender: z.string().nullable()
```

Ab `null` valid hai. Par humein chahiye ki `null` invalid ho — kyunki gender select karna zaroori hai.

Toh `.refine()` use karte hain — jo `null` ko bhi check kare:

```tsx
gender: z.string().nullable().refine(val => val !== null, {
  message: 'Gender select karo'
})
```

**`.nullable()`** — Zod ko bolta hai "null bhi accept kar, error mat de type mismatch ka."

**`.refine(val => val !== null)`** — custom validation — agar `null` hai toh fail karo.

Schema update karo:

```tsx
const formSchema = z.object({
  naam: z.string().min(1, 'Naam zaroori hai'),
  email: z.email('Email sahi format mein nahi hai'),
  city: z.string().min(1, 'City select karo'),
  terms: z.boolean().refine(val => val === true, {
    message: 'Terms accept karne zaroori hain'
  }),
  gender: z.string().nullable().refine(val => val !== null, {
    message: 'Gender select karo'
  })
})
```

Gender radio buttons ko uncomment karo jo pehle comment out kiye the:

```tsx
<div>
  <label className="block text-sm font-medium mb-1">Gender</label>
  <div className="flex gap-4">
    <label className="flex items-center gap-1">
      <input
        type="radio"
        value="male"
        {...register('gender')}
      />
      Male
    </label>
    <label className="flex items-center gap-1">
      <input
        type="radio"
        value="female"
        {...register('gender')}
      />
      Female
    </label>
  </div>
</div>
```

Gender ke neeche error add karo:

```tsx
{errors.gender && (
  <p className="text-red-500 text-sm">{errors.gender.message}</p>
)}
```

Gender select kiye bina submit karo — "Gender select karo" ✅

Gender select karo — submit karo — error nahi ✅

## `z.infer` — Schema Se TypeScript Type Banao

Abhi tak hum manually `FormFields` type bana rahe the:

```tsx
type FormFields = {
  naam: string
  email: string
  city: string
  terms: boolean
  gender: string
}
```

Par yeh **duplicate** hai — schema mein bhi wahi define kiya hai. Agar schema change hua — type manually update karna padega.

**Doubly maintenance** — ek jagah change kiya, doosri jagah bhool gaye — bug aa gaya.

## Solution: `z.infer`

Zod ka `z.infer` schema se automatically TypeScript type nikal deta hai:

```tsx
import { z } from 'zod'

const formSchema = z.object({
  naam: z.string().min(1, 'Naam zaroori hai'),
  email: z.email('Email sahi format mein nahi hai'),
  city: z.string().min(1, 'City select karo'),
  terms: z.boolean().refine(val => val === true, {
    message: 'Terms accept karne zaroori hain'
  }),
  gender: z.string().nullable().refine(val => val !== null, {
    message: 'Gender select karo'
  })
})

// Schema se type automatically nikalo
type FormFields = z.infer<typeof formSchema>
```

**`z.infer<typeof formSchema>`** — iska matlab "formSchema ki type nikalo."

Zod khud samajh jata hai:
- `z.string()` → `string`
- `z.boolean()` → `boolean`
- `.nullable()` → `| null`
- `.min(1)` → still `string` (validation rule, type nahi badalta)

Toh `FormFields` automatically ban jayega:

```ts
type FormFields = {
  naam: string
  email: string
  city: string
  terms: boolean
  gender: string | null
}
```

Dekha? `gender: string | null` — kyunki `.nullable()` use kiya hai.

Ab `useForm` mein yeh type use karo:

```tsx
const { register, handleSubmit, formState: { errors } } = useForm<FormFields>({
  resolver: zodResolver(formSchema)
})
```

Ab agar schema mein kuch change kiya — type automatically update ho jayega. No more doubly maintenance.

## Poora `ContactForm.tsx` — Final Version

```tsx
'use client'

import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'

// Step 1: Schema banao
const formSchema = z.object({
  naam: z.string().min(1, 'Naam zaroori hai'),
  email: z.email('Email sahi format mein nahi hai'),
  city: z.string().min(1, 'City select karo'),
  terms: z.boolean().refine(val => val === true, {
    message: 'Terms accept karne zaroori hain'
  }),
  gender: z.string().nullable().refine(val => val !== null, {
    message: 'Gender select karo'
  })
})

// Step 2: Schema se type nikalo
type FormFields = z.infer<typeof formSchema>

export default function ContactForm() {
  // Step 3: useForm mein resolver aur type do
  const { register, handleSubmit, formState: { errors } } = useForm<FormFields>({
    resolver: zodResolver(formSchema)
  })

  return (
    <form
      onSubmit={handleSubmit((data) => {
        console.log(data)
      })}
      className="flex flex-col gap-4 max-w-md"
    >
      {/* Naam */}
      <div>
        <label className="block text-sm font-medium mb-1">Naam</label>
        <input
          type="text"
          placeholder="Apna naam likho"
          className="border border-gray-300 rounded px-3 py-2 w-full"
          {...register('naam')}
        />
        {errors.naam && (
          <p className="text-red-500 text-sm mt-1">{errors.naam.message}</p>
        )}
      </div>

      {/* Email */}
      <div>
        <label className="block text-sm font-medium mb-1">Email</label>
        <input
          type="email"
          placeholder="Apni email likho"
          className="border border-gray-300 rounded px-3 py-2 w-full"
          {...register('email')}
        />
        {errors.email && (
          <p className="text-red-500 text-sm mt-1">{errors.email.message}</p>
        )}
      </div>

      {/* City */}
      <div>
        <label className="block text-sm font-medium mb-1">City</label>
        <select
          {...register('city')}
          className="border border-gray-300 rounded px-3 py-2 w-full"
        >
          <option value="">-- Select karo --</option>
          <option value="mumbai">Mumbai</option>
          <option value="delhi">Delhi</option>
          <option value="bangalore">Bangalore</option>
        </select>
        {errors.city && (
          <p className="text-red-500 text-sm mt-1">{errors.city.message}</p>
        )}
      </div>

      {/* Terms */}
      <div className="flex items-center gap-2">
        <input
          type="checkbox"
          {...register('terms')}
          className="w-4 h-4"
        />
        <label className="text-sm">Terms aur Conditions accept karta hoon</label>
      </div>
      {errors.terms && (
        <p className="text-red-500 text-sm">{errors.terms.message}</p>
      )}

      {/* Gender */}
      <div>
        <label className="block text-sm font-medium mb-1">Gender</label>
        <div className="flex gap-4">
          <label className="flex items-center gap-1">
            <input
              type="radio"
              value="male"
              {...register('gender')}
            />
            Male
          </label>
          <label className="flex items-center gap-1">
            <input
              type="radio"
              value="female"
              {...register('gender')}
            />
            Female
          </label>
        </div>
        {errors.gender && (
          <p className="text-red-500 text-sm">{errors.gender.message}</p>
        )}
      </div>

      <button
        type="submit"
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Submit
      </button>
    </form>
  )
}
```
