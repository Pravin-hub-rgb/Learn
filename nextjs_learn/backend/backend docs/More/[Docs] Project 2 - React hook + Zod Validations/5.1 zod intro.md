# Doc 1.5.1 — Zod Kya Hai — Schema Banao — RHF Se Connect Karo

---

## Abhi Tak Kya Hua

1.4.2 mein React Hook Form ka built-in validation seekha — `register` mein `required`, `pattern` likhe.

Kaam karta tha — par email format check ke liye yeh likhna pada:

```tsx
pattern: {
  value: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,
  message: 'Email sahi format mein nahi hai'
}
```

Yeh regex padhna mushkil hai. Complex rules ke liye aur mushkil hoga.

**Kya hoga agar validation rules seedhi padhne wali language mein likh sakein?**

Yahi karta hai **Zod.**

---

## Is Doc Mein Kya Karenge — Broader Goal

Humara kaam hai — Zod ko form ke saath connect karna taaki validation Zod handle kare.

Iske liye do cheezein karni hain:
1. **Zod ka schema banana** — "form mein kaunse fields hain, unki type kya hai" define karna
2. **Schema ko RHF se jodna** — taaki submit pe Zod validate kare

Pehle samjhte hain schema kya hota hai — phir connect karenge.

---

## Schema Kya Hota Hai

Schema matlab — **"mera data kaisa dikhna chahiye" — ek blueprint.**

Zod mein yeh aise likhte hain:

```ts
z.object({
  naam: z.string(),
  email: z.email()
})
```

Koi regex nahi. Seedha padhne wala. `z.email()` — Zod pehle se jaanta hai email format kya hota hai.

## Install Karo

Do packages chahiye:

```bash
npm install zod @hookform/resolvers
```

**`zod`** — schema likhne ki library

**`@hookform/resolvers`** — yeh ek bridge hai. Zod aur RHF do alag libraries hain — yeh package dono ko connect karta hai.

---

## Schema Banana Shuru Karo

`ContactForm.tsx` mein `z` import karo — component ke bahar schema banana shuru karo.

Pehle sirf `z` import karo aur ek simple schema dekho:

```tsx
import { z } from 'zod'

const formSchema = z.object({
  naam: z.string()
})

console.log(formSchema)
```

Browser console mein ek Zod object dikhega. Console.log hata do.

Ab baaki sab fields add karo:

```tsx
const formSchema = z.object({
  naam: z.string(),
  email: z.string(),
  city: z.string(),
  terms: z.boolean(),
  gender: z.string()
})
```

`z.boolean()` — true ya false — terms ke liye.

Abhi sirf types hain — rules baad mein.

---

## RHF Se Connect Karo — `zodResolver`

`zodResolver` import karo — `useForm` mein pass karo:

```tsx
import { zodResolver } from '@hookform/resolvers/zod'

const { register, handleSubmit, formState: { errors } } = useForm({
  resolver: zodResolver(formSchema)
})
```

`resolver: zodResolver(formSchema)` — RHF ko bol raha hai — "validation ke liye Zod ka schema use karo."

**Ek zaroori baat** — `register` mein se pehle wale rules hatao:

```tsx
// Pehle aisa tha — hatao
{...register('naam', { required: 'Naam zaroori hai' })}

// Ab sirf yeh
{...register('naam')}
```

RHF ke built-in validators aur `zodResolver` ek saath kaam nahi karte.

## Submit Karo — Verify Karo

Submit karo — temporarily doosra function lagao:

```tsx
onSubmit={handleSubmit(
  (data) => console.log('valid:', data),
  (errors) => console.log('invalid:', errors)
)}
```

Sab fields fill karke submit karo — "valid" aaya ✅

Par gender select mat karo — submit karo — yeh error aayegi:

```js
invalid: {
  gender: {
    message: 'Invalid input: expected string, received null'
  }
}
```

Kyun? Radio buttons jab koi select nahi hota — browser `null` bhejta hai. `z.string()` ko string chahiye — `null` nahi.

Gender ek special case hai — abhi schema mein comment out karo:

```tsx
const formSchema = z.object({
  naam: z.string(),
  email: z.string(),
  city: z.string(),
  terms: z.boolean(),
  // gender: z.string()  ← baad mein sahi tarike se handle karenge
})
```

Gender radio buttons bhi UI mein comment out karo.

Ab submit karo — "valid" aata hai ✅

---

## Doosra Function Hata Do

Yeh verify wala doosra function sirf debugging ke liye tha. Real app mein errors `formState.errors` mein automatically aate hain — `handleSubmit` ke doosre function mein nahi.

Hata do — sirf pehla rakho:

```tsx
onSubmit={handleSubmit((data) => {
  console.log(data)
})}
```

✅ Zod aur RHF connected hain — kaam kar rahe hain.

## Summary

- **Zod** — schema likhte hain — "data kaisa hona chahiye"
- `z.object({})` — object schema — fields define karo
- `z.string()`, `z.boolean()` — types
- `@hookform/resolvers` — bridge hai Zod aur RHF ke beech
- `resolver: zodResolver(formSchema)` — `useForm` mein pass kiya
- `register` mein se built-in rules hatao — Zod handle karega
- Gender radio buttons — `null` aata hai unselected pe — alag handle karna padta hai — baad mein
