# Doc 1.4.2 — React Hook Form Ka Built-in Validation

---

## Abhi Tak Kya Hua

1.4 mein manually validation ki — `useState`, `if` checks — bahut zyada code.

Par React Hook Form khud bhi validation deta hai — `register` ke andar hi rules likh sakte hain.

Pehle jo manual code tha — woh clean karte hain — phir RHF ka validation add karenge.

---

## Step 1 — Manual Code Clean Karo

Abhi code mein yeh sab hai jo RHF ke saath nahi chahiye:

```tsx
const [naamError, setNaamError] = useState("")
const [emailError, setEmailError] = useState("")
```

Aur `handleSubmit` mein manual checks:

```tsx
setNaamError("")
setEmailError("")
if (!data.naam) {
  setNaamError("Naam Zaroori hai")
}
// ...
```

Yeh sab hata do. `useState` import bhi hata do.

`handleSubmit` sirf itna rakho — clean:

```tsx
onSubmit={handleSubmit((data) => {
  console.log(data)
})}
```

Ab form bilkul clean hai — koi manual validation nahi.

---

## Step 2 — `formState` Aur `errors` Nikalo

`useForm()` se sirf `register` aur `handleSubmit` nikaale the pehle. Ab `formState` bhi chahiye — jisme errors rahenge.

Abhi:

```tsx
const { register, handleSubmit } = useForm<FormFields>()
```

Update karo:

```tsx
const { register, handleSubmit, formState: { errors } } = useForm<FormFields>()
```

`formState: { errors }` — yeh kya hai?

`formState` ek bada object hai jisme form ke baare mein bahut saari information hoti hai — `errors`, `isDirty`, `isSubmitting` — sab kuch.

Abhi sirf `errors` chahiye — toh directly `formState` ke andar se `errors` nikaal liya.

`errors` ek object hai — jisme har field ke errors hote hain:

```js
{
  naam: { type: 'required', message: '...' },
  email: { type: 'pattern', message: '...' }
}
```

Abhi `errors` khali object hai `{}` — koi error nahi.

---

## Step 3 — `register` Mein Validation Rules Do

Yaad hai `register('naam')` — sirf ek string dete the?

Ab doosra argument bhi de sakte hain — **rules ka object**:

```tsx
{...register('naam', {
  required: 'Naam zaroori hai'
})}
```

Pehla argument — `'naam'` — field ka naam

Doosra argument — `{ required: 'Naam zaroori hai' }` — rules

`required` rule kehta hai — "yeh field khali nahi hona chahiye." Aur jo string di — `'Naam zaroori hai'` — woh error message hai.

---

## Step 4 — Connection Samjho — `register` Se `errors` Tak

Yeh sabse important cheez hai — samjho.

Jab `register` mein yeh likha:

```tsx
{...register('naam', {
  required: 'Naam zaroori hai'
})}
```

Toh React Hook Form ne internally note kar liya — "naam field pe `required` rule hai — aur agar fail hua toh message `'Naam zaroori hai'` hoga."

Jab user khali submit karta hai — React Hook Form check karta hai — naam khali hai — rule fail hua — toh `errors` object mein daalta hai:

```js
errors = {
  naam: {
    type: 'required',          // ← kaunsa rule fail hua
    message: 'Naam zaroori hai'  // ← wahi message jo register mein diya tha
  }
}
```

Toh `errors.naam.message` mein wahi string milti hai jo tumne `register` mein `required` ke saath di thi.

---

## Step 5 — Error UI Mein Dikhao

`errors.naam` mein error hai — `errors.naam.message` mein message — UI mein dikhao:

```tsx
<div>
  <label className="block text-sm font-medium mb-1">Naam</label>
  <input
    type="text"
    placeholder="Apna naam likho"
    className="border border-gray-300 rounded px-3 py-2 w-full"
    {...register('naam', {
      required: 'Naam zaroori hai'
    })}
  />
  {errors.naam && (
    <p className="text-red-500 text-sm mt-1">{errors.naam.message}</p>
  )}
</div>
```

`{errors.naam && <p>...</p>}` — agar `errors.naam` exist karta hai toh `<p>` dikhao — nahi hai toh kuch nahi.

`{errors.naam.message}` — woh message jo `register` mein diya tha — `'Naam zaroori hai'`

Naam khali chhod ke submit karo — "Naam zaroori hai" screen pe dikh raha hai ✅

Naam likho — phir submit karo — error khud hat jaati hai ✅

**React Hook Form khud errors clear karta hai — tumhe manually `setNaamError('')` nahi karna.**

---

## Step 6 — Email Pe Bhi Validation Add Karo

Email ke liye do rules chahiye:

**`required`** — khali nahi hona chahiye

**`pattern`** — email format sahi hona chahiye

```tsx
<div>
  <label className="block text-sm font-medium mb-1">Email</label>
  <input
    type="email"
    placeholder="Apni email likho"
    className="border border-gray-300 rounded px-3 py-2 w-full"
    {...register('email', {
      required: 'Email zaroori hai',
      pattern: {
        value: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,
        message: 'Email sahi format mein nahi hai'
      }
    })}
  />
  {errors.email && (
    <p className="text-red-500 text-sm mt-1">{errors.email.message}</p>
  )}
</div>
```

`pattern` ek object hai:
- `value` — regex — email format check karta hai
- `message` — agar format galat ho toh yeh message aayega

Wahi connection — `pattern.message` mein jo diya — woh `errors.email.message` mein aayega.

"abc" likho — submit karo — "Email sahi format mein nahi hai" ✅

"abc@gmail.com" likho — submit karo — error nahi ✅

## Poora `ContactForm.tsx` Ab — Clean

```tsx
'use client'

import { useForm } from 'react-hook-form'

type FormFields = {
  naam: string
  email: string
  city: string
  terms: boolean
  gender: string
}

export default function ContactForm() {
  const { register, handleSubmit, formState: { errors } } = useForm<FormFields>()

  return (
    <form
      onSubmit={handleSubmit((data) => {
        console.log(data)
      })}
      className="flex flex-col gap-4"
    >
      <div>
        <label className="block text-sm font-medium mb-1">Naam</label>
        <input
          type="text"
          placeholder="Apna naam likho"
          className="border border-gray-300 rounded px-3 py-2 w-full"
          {...register('naam', {
            required: 'Naam zaroori hai'
          })}
        />
        {errors.naam && (
          <p className="text-red-500 text-sm mt-1">{errors.naam.message}</p>
        )}
      </div>

      <div>
        <label className="block text-sm font-medium mb-1">Email</label>
        <input
          type="email"
          placeholder="Apni email likho"
          className="border border-gray-300 rounded px-3 py-2 w-full"
          {...register('email', {
            required: 'Email zaroori hai',
            pattern: {
              value: /^[^\s@]+@[^\s@]+\.[^\s@]+$/,
              message: 'Email sahi format mein nahi hai'
            }
          })}
        />
        {errors.email && (
          <p className="text-red-500 text-sm mt-1">{errors.email.message}</p>
        )}
      </div>

      <div>
        <label className="block text-sm font-medium mb-1">City</label>
        <select
          className="border border-gray-300 rounded px-3 py-2 w-full"
          {...register('city')}
        >
          <option value="">-- Select karo --</option>
          <option value="mumbai">Mumbai</option>
          <option value="delhi">Delhi</option>
          <option value="bangalore">Bangalore</option>
        </select>
      </div>

      <div className="flex items-center gap-2">
        <input
          type="checkbox"
          {...register('terms')}
          className="w-4 h-4"
          id="terms"
        />
        <label className="text-sm" htmlFor="terms">
          Terms aur Conditions accept karta hoon
        </label>
      </div>

      <div>
        <label className="block text-sm font-medium mb-1">Gender</label>
        <div className="flex gap-4">
          <label className="flex items-center gap-1">
            <input type="radio" value="male" {...register('gender')} />
            Male
          </label>
          <label className="flex items-center gap-1">
            <input type="radio" value="female" {...register('gender')} />
            Female
          </label>
        </div>
      </div>

      <button
        type="submit"
        className="bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600"
      >
        Submit
      </button>
    </form>
  )
}
```

## Fark Dekho — Manual vs RHF

**Manual — 1.4 mein:**
- Do alag `useState` — `naamError`, `emailError`
- `handleSubmit` mein `if` checks manually
- Manually `setNaamError('')` karna — errors clear karne ke liye

**RHF Built-in:**
- Koi `useState` nahi
- `handleSubmit` clean — sirf `console.log(data)`
- `register` mein rules likho — bas
- `errors` mein automatically aate hain — automatically clear bhi hote hain

---

## Summary

- `register('naam', { required: 'message' })` — doosra argument rules ka object
- `required: 'Naam zaroori hai'` — rule aur message ek saath
- React Hook Form internally store karta hai — fail hua toh `errors` mein daalega
- `formState: { errors }` — `useForm` se nikalo
- `errors.naam.message` — wahi message jo `register` mein `required` ke saath diya tha
- `pattern` — regex format check — `value` aur `message` dono
- Errors automatically clear hote hain — manually nahi karna

---

## Agla Step

RHF ka built-in validation simple cases ke liye theek hai — `required`, `pattern`.

Par complex rules ke liye — jaise "password mein ek number hona chahiye, ek capital letter hona chahiye" — regex likhna mushkil ho jaata hai.

Iske liye **Zod** aata hai — rules seedhi readable language mein likhte hain.

Yahi hoga **Doc 1.5 mein.**
