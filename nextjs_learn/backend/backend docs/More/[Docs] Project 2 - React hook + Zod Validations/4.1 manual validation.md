# Doc 1.4 — Validation Manually Karo — `if` Checks — Problem Dikhao

---

## Abhi Tak Kya Hua

Form complete hai — naam, email, city, terms, gender — saari values aa rahi hain.

Par koi validation nahi hai. Try karo —

Naam khali chhod ke submit karo — submit ho gaya. Email mein "abc" likho — submit ho gaya.

User galat data submit kar sakta hai — aur humein pata bhi nahi chalega.

Ab validation add karenge — manually — `if` checks se.

---

## Pehle Samjho — JavaScript Mein Empty String `false` Kyun Hoti Hai

JavaScript mein kuch values hoti hain jo **"falsy"** hain — matlab `if` condition mein yeh `false` ki tarah behave karti hain:

```js
''        // khali string — falsy
0         // zero — falsy
null      // falsy
undefined // falsy
false     // falsy
```

Toh jab user kuch type nahi karta — `data.naam` ki value `''` hoti hai — khali string.

Aur `!` ka matlab hai "ulta" — `!true` = `false`, `!false` = `true`.

Toh:

```js
!''        // → true  — khali string thi, ulta kiya, true mila
!'Pravin'  // → false — kuch tha, ulta kiya, false mila
```

Isliye `if (!data.naam)` ka matlab hai — "agar naam khali hai toh."

---

## Step 1 — Pehla Check Lagao — Console Mein

`handleSubmit` mein naam check karo — pehle sirf console mein:

```tsx
onSubmit={handleSubmit((data) => {
  if (!data.naam) {
    console.log('Naam khali hai!')
    return
  }
  console.log(data)
})}
```

`return` kyun? — Function ko wahan rokna hai — aage nahi jaana:

```
naam khali hai?
    ↓ haan
    console.log('Naam khali hai!')
    return  ← yahan ruk gaya

naam khali hai?
    ↓ nahi
    if block skip hua
    console.log(data)  ← yahan aaya
```

Naam khali chhod ke submit karo — console mein:

```
Naam khali hai!
```

✅ Check kaam kar raha hai — par sirf console mein. User ko nahi dikha.

---

## Step 2 — Error Store Karo

"User ko dikhana hai — toh error kahin store karni padegi."

`useState` banao naam error ke liye:

```tsx
const [naamError, setNaamError] = useState('')
```

Shuru mein `''` — khali string — koi error nahi.

`handleSubmit` mein `console.log` ki jagah `setNaamError` karo:

```tsx
onSubmit={handleSubmit((data) => {
  if (!data.naam) {
    setNaamError('Naam zaroori hai')
    return
  }
  console.log(data)
})}
```

Naam khali chhod ke submit karo — `naamError` mein `'Naam zaroori hai'` aa gaya.

Par screen pe abhi bhi kuch nahi dikha — UI mein dikhana hai.

---

## Step 3 — Error UI Mein Dikhao

Input ke neeche `<p>` add karo:

```tsx
<div>
  <label className="block text-sm font-medium mb-1">Naam</label>
  <input
    type="text"
    placeholder="Apna naam likho"
    className="border border-gray-300 rounded px-3 py-2 w-full"
    {...register('naam')}
  />
  {naamError && (
    <p className="text-red-500 text-sm mt-1">{naamError}</p>
  )}
</div>
```

`{naamError && <p>...</p>}` — yeh kya hai?

`&&` React mein conditional rendering ke liye use hota hai:

```
naamError = ''          → falsy → kuch nahi dikhega
naamError = 'Naam...'   → truthy → <p> dikhega
```

Naam khali chhod ke submit karo — "Naam zaroori hai" screen pe dikh raha hai ✅

---

## Step 4 — Error Kab Hati Hai?

"Error aa gayi — ab naam likhte hain — phir submit karte hain."

Naam likhke submit karo — error hati hai ya nahi?

Nahi hati — kyunki `naamError` mein abhi bhi `'Naam zaroori hai'` hai — clear nahi kiya.

Fix — jab naam sahi ho — `setNaamError('')` karo — purani error clear karo:

```tsx
onSubmit={handleSubmit((data) => {
  if (!data.naam) {
    setNaamError('Naam zaroori hai')
    return
  }
  setNaamError('')   // ← naam sahi hai — error clear karo

  console.log(data)
})}
```

Ab flow:

```
Submit 1 — naam khali   → setNaamError('Naam zaroori hai') → error dikh rahi
Submit 2 — naam likha   → if skip hua → setNaamError('') → error hat gayi
```

✅ Naam likhke submit karo — error hat gayi.

---

## Step 5 — Email Ke Liye Bhi Wohi Karo

Exactly wahi pattern — email ke liye:

```tsx
const [emailError, setEmailError] = useState('')
```

`handleSubmit` mein naam ke baad email check add karo:

```tsx
onSubmit={handleSubmit((data) => {
  if (!data.naam) {
    setNaamError('Naam zaroori hai')
    return
  }
  setNaamError('')

  if (!data.email) {
    setEmailError('Email zaroori hai')
    return
  }
  setEmailError('')

  console.log(data)
})}
```

Email input ke neeche error dikhao:

```tsx
<div>
  <label className="block text-sm font-medium mb-1">Email</label>
  <input
    type="email"
    placeholder="Apni email likho"
    className="border border-gray-300 rounded px-3 py-2 w-full"
    {...register('email')}
  />
  {emailError && (
    <p className="text-red-500 text-sm mt-1">{emailError}</p>
  )}
</div>
```

Try karo — email khali chhod ke submit karo — error aa rahi hai ✅

---

## Problem Dikhti Hai — Abhi Tak Sirf Do Fields

Dekho abhi tak kitna code ho gaya — sirf naam aur email ke liye:

- Do alag `useState`
- `handleSubmit` mein do `if` blocks + dono ko clear karna
- UI mein do alag error paragraphs

Aur baaki teen fields — city, terms, gender — abhi bhi validate nahi kiye. Teeno ke liye bhi yahi karo toh:

- Teen aur `useState`
- Teen aur `if` blocks
- Teen aur error paragraphs

5 fields ke liye yeh sab — bahut zyada kaam.
