## Chaining — Rules Kaise Add Karte Hain

Rules add karne ke liye Zod mein **chaining** hoti hai — `.` se jodo:

```ts
z.string().min(1); // string + minimum 1 character
z.string().min(2).max(50); // string + min 2 + max 50
```

"Aur yeh bhi hona chahiye" — dot se agle rule ko jodo.

---

## Naam Pe Rule Add Karo

Naam khali nahi hona chahiye:

```tsx
naam: z.string().min(1, "Naam zaroori hai");
```

`.min(1, 'message')` — kam se kam 1 character. 0 = khali = fail.

Schema update karo:

```tsx
const formSchema = z.object({
  naam: z.string().min(1, "Naam zaroori hai"),
  email: z.email(),
  city: z.string(),
  terms: z.boolean(),
  // gender comment out hai
});
```

Naam khali submit karo — temporarily verify ke liye:

```tsx
onSubmit={handleSubmit(
  (data) => console.log('valid:', data),
  (errors) => console.log('errors:', errors)
)}
```

Console mein:

```js
errors: {
  naam: {
    message: "Naam zaroori hai";
  }
}
```

Connection dekho — `.min(1, 'Naam zaroori hai')` mein jo message diya — wahi `errors.naam.message` mein aaya.

Yeh verify wala doosra function hata do — real errors `formState.errors` mein aate hain:

```tsx
onSubmit={handleSubmit((data) => {
  console.log(data)
})}
```

## Errors Kahan Se Aate Hain — `formState`

Abhi tak humne sirf `register` aur `handleSubmit` nikala tha `useForm` se.

Par `useForm` aur bhi cheezein deta hai — unmein se ek hai **`formState`**.

`formState` ek object hai jisme form ki current state hoti hai — errors, isSubmitting, isValid, etc.

```tsx
const {
  register,
  handleSubmit,
  formState: { errors },
} = useForm({
  resolver: zodResolver(formSchema),
});
```

**`formState: { errors }`** — iska matlab `formState` object mein se sirf `errors` property nikalo.

Yeh `errors` object automatically populate hota hai jab validation fail hoti hai. Har field ke liye:

- `errors.naam` — naam field ka error
- `errors.email` — email field ka error
- `errors.naam.message` — error ka message

---

## Error UI Mein Dikhao

Ab `errors` object hai — usko use karke UI mein error dikha sakte hain:

```tsx
<div>
  <label className="block text-sm font-medium mb-1">Naam</label>
  <input
    type="text"
    placeholder="Apna naam likho"
    className="border border-gray-300 rounded px-3 py-2 w-full"
    {...register("naam")}
  />
  {errors.naam && (
    <p className="text-red-500 text-sm mt-1">{errors.naam.message}</p>
  )}
</div>
```

**`{errors.naam && (...)}`** — agar `errors.naam` hai (yani error hai), tabhi yeh `<p>` render hoga.

**`{errors.naam.message}`** — woh message jo schema mein diya tha — "Naam zaroori hai".

Naam khali submit karo — "Naam zaroori hai" screen pe ✅

Naam likho — submit karo — error hat gayi ✅

## Email Ka Rule — Zod V4 Mein Naya Tarika

Pehle `z.string().email()` hota tha — Zod v3 mein. Zod v4 mein yeh deprecated ho gaya.

Ab **top-level** `z.email()` use karte hain:

```tsx
email: z.email("Email sahi format mein nahi hai");
```

`z.email('message')` — Zod khud email format check karta hai. Koi regex nahi.

Schema update karo:

```tsx
const formSchema = z.object({
  naam: z.string().min(1, "Naam zaroori hai"),
  email: z.email("Email sahi format mein nahi hai"),
  city: z.string(),
  terms: z.boolean(),
  // gender comment out hai
});
```

Email input ke neeche error add karo:

```tsx
{
  errors.email && (
    <p className="text-red-500 text-sm mt-1">{errors.email.message}</p>
  );
}
```

"abc" likho — submit karo — "Email sahi format mein nahi hai" ✅

"abc@gmail.com" — error nahi ✅

## Terms Ka Rule

Terms checkbox — unchecked matlab `false` — woh invalid hona chahiye.

Pehla idea — `z.boolean()` try karo:

```tsx
terms: z.boolean();
```

Submit karo bina terms check kiye — koi error nahi aaya.

Kyun? `z.boolean()` ka matlab sirf "true ya false hona chahiye" — `false` bhi valid boolean hai.

Toh specifically `true` check karna hai — `.refine()` use karte hain:

```tsx
terms: z.boolean().refine((val) => val === true, {
  message: "Terms accept karne zaroori hain",
});
```

`.refine()` ek function leta hai — `val` form se aayi value. `true` return karo — valid. `false` return karo — invalid.

`val => val === true` — exactly `true` hona chahiye.

Schema update karo:

```tsx
const formSchema = z.object({
  naam: z.string().min(1, "Naam zaroori hai"),
  email: z.email("Email sahi format mein nahi hai"),
  city: z.string().min(1, "City select karo"),
  terms: z.boolean().refine((val) => val === true, {
    message: "Terms accept karne zaroori hain",
  }),
  // gender comment out hai
});
```

Terms ke neeche error add karo:

```tsx
{
  errors.terms && (
    <p className="text-red-500 text-sm">{errors.terms.message}</p>
  );
}
```

Terms uncheck submit karo — "Terms accept karne zaroori hain" ✅

---

## Summary

- `.min(1, 'message')` — minimum 1 character — khali nahi
- **`z.email()`** — Zod v4 ka naya tarika — `z.string().email()` deprecated hua
- `z.boolean()` akela — `false` bhi valid — kaam nahi kiya
- `.refine(val => val === true)` — custom rule — exactly true chahiye
- `formState: { errors }` — `useForm` se nikalo — validation errors yahan aate hain
- `{errors.field && <p>{errors.field.message}</p>}` — UI mein error dikhane ka tarika
- Schema ka message → `errors.field.message` mein aata hai
