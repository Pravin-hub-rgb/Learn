# Doc 2.3 — useSession() kya hai + kyun chahiye

## Abhi Tak Kya Hua
2.1 mein session kya hoti hai samjha — cookie story. 2.2 mein `auth()` se session read kiya — page pe naam dikha — `signOut()` se logout kiya. Abhi tak sab kuch `page.tsx` mein tha — Server Component.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

`auth()` toh kaam karta hai — par sirf **Server Components** mein. Agar Client Component mein session chahiye toh kya karein? Yeh samjhenge aaj.

---

## Server Component vs Client Component — pehle yeh clear karo

Tu sahi soch raha tha — `page.tsx` mein `return` mein UI hai — toh kya woh client wala part hai?

**Nahi** — poora `page.tsx` Server Component hai.

By default Next.js mein **har component Server Component hota hai** — jab tak upar `"use client"` na likha ho.

Server Component ka kaam:
```
Request aati hai
      ↓
Server pe code chalta hai — auth(), data fetch, etc.
      ↓
Server HTML banata hai — return wala UI
      ↓
Woh HTML browser ko bhejta hai
      ↓
Browser sirf HTML dikhata hai — React code nahi aata browser tak
```

Toh `page.tsx` mein jo `return` hai — woh HTML server pe banta hai — browser ko sirf final HTML milta hai.

---

**Client Component kab chahiye?**

Jab browser mein kuch **interactive** karna ho — server se HTML aane ke baad:

**`useState` kyun Client chahiye?**

Socho Navbar mein dropdown hai — user click kare toh khule:

```
User click karta hai
      ↓
State change — isOpen: true
      ↓
Component re-render — dropdown dikha
```

Yeh sab browser mein real time mein hota hai — server se koi request nahi gayi. Server toh pehle hi HTML bhej chuka — ab browser khud handle kar raha hai.

Server pe `useState` ka koi matlab nahi — server ek baar HTML banata hai — khatam. Woh yaad nahi rakh sakta ki dropdown khula hai ya band.

**`useEffect` kyun Client chahiye?**

`useEffect` tab chalta hai jab component browser mein load ho jaata hai:

```tsx
useEffect(() => {
  console.log("page browser mein load hua")
}, [])
```

Server pe "browser mein load hona" ka concept nahi — server sirf HTML banata hai — load koi nahi hota wahan.

**`onClick`, `onChange` kyun Client chahiye?**

Yeh browser ke events hain — user ne click kiya, kuch type kiya. Server pe koi click nahi hota — server toh sirf request aane pe response deta hai.

## Problem — `auth()` Client Component mein kaam nahi karta

Abhi `page.tsx` mein `auth()` use kar rahe hain — aur woh Server Component hai.

Par socho — **Navbar** banana hai — jisme user ka naam dikhega. Navbar mein `useState` use hoga dropdown ke liye — aur `useState` sirf Client Component mein hota hai — toh Navbar `"use client"` hoga.

Toh Navbar mein `auth()` use kar sakte hain?

Try karo — `app/components/Navbar.tsx` banao:

```tsx
"use client"

import { auth } from "@/auth"

export default function Navbar() {
  const session = auth() // ← yeh kaam nahi karega
  return <nav>Navbar</nav>
}
```

Ab `app/layout.tsx` mein import karke add karo:

```tsx
import Navbar from "./components/Navbar"

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        <Navbar />
        {children}
      </body>
    </html>
  )
}
```

Error aayega — kuch aisa:
```
Error: `headers` was called outside a request scope
```

> **Note:** Error message exact alag ho sakta hai Next.js version ke hisaab se — par matlab ek hi hai — `auth()` Client Component mein nahi chalta kyunki server ki cookie browser mein directly nahi padh sakte.

**Kyun?** — `auth()` server pe cookie padhta hai — browser mein cookie directly nahi padh sakta. Client Component browser mein run hota hai — isliye `auth()` wahan kaam nahi karta.

---

## Solution — `useSession()` hook

Client Components ke liye Next-Auth ne alag cheez banayi hai — `useSession()` hook.

`useSession()` kaise kaam karta hai?

```
Client Component render hota hai browser mein
        ↓
useSession() ek API call karta hai server pe
        ↓
Server cookie padhta hai — session return karta hai
        ↓
useSession() woh data deta hai component ko
```

Matlab — `useSession()` bhi server se hi session leta hai — par automatically — ek API call ke through.

---

## `useSession()` use karo

Ab `Navbar.tsx` fix karo — `auth()` hata do — `useSession()` lagao:

```tsx
"use client"

import { useSession } from "next-auth/react"

export default function Navbar() {
  const { data: session, status } = useSession()
  console.log(session)
  console.log(status)

  return <nav>Navbar</nav>
}
```

`useSession()` ek object return karta hai — hum dono cheezein nikal rahe hain:
- `data` — session ka data — humne `session` naam diya isse
- `status` — teen values ho sakti hain: `"loading"`, `"authenticated"`, `"unauthenticated"`

`"use client"` — upar likha kyunki `useSession` hook hai — hooks sirf Client Components mein kaam karte hain.

Browser mein jao — yeh error aayegi:
```
[next-auth]: `useSession` must be wrapped in a <SessionProvider />
```

**Kyun?** — `useSession()` akela kaam nahi karta — isko ek `SessionProvider` ke andar wrap karna padta hai. Woh provider session ka data saare components ko share karta hai.

Yeh `SessionProvider` kya hai aur kahan lagaate hain — **2.4 mein karenge.**

---

## Summary

- By default Next.js mein har component Server Component hota hai
- Server Component — server pe HTML banata hai — browser ko bhejta hai
- Client Component — `"use client"` likhte hain — browser mein interactivity ke liye
- `auth()` sirf Server Components mein — server pe cookie padhta hai
- `useSession()` Client Components ke liye — par akela kaam nahi karta
- `SessionProvider` chahiye — 2.4 mein lagayenge

**Doc 2.4** mein `SessionProvider` samjhenge — kya hota hai, kyun chahiye — aur lagane ke baad `useSession()` kaam karega.
