# Doc 5.5 — Production Security Checklist

## Abhi Tak Kya Hua

5.4 mein `AUTH_SECRET` properly generate kiya — `.env` samjha — `process.env` se code mein padhna seekha — `auth.ts` complete ho gaya. Ab ek kaam bacha — production deploy karne se pehle kya kya check karna hai.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

App locally toh chal raha hai — par production pe deploy karne se pehle kuch cheezein verify karni hain. Ek ek karke dekhenge — kya missing hai, kya fix karna hai.

---

## Check 1 — `AUTH_SECRET` strong hai?

`.env` kholo — `AUTH_SECRET` ki value dekho.

`npx auth secret` se generate ki thi? ✅ — Strong hai.

Manually likhi thi — jaise `"hello123"`? ❌ — Abhi regenerate karo:

```bash
npx auth secret
```

---

## Check 2 — `.env` GitHub pe toh nahi gayi?

`.gitignore` file kholo — yeh line honi chahiye:

```
.env
.env.local
```

Hai? ✅ — Good.

Nahi hai? — Abhi add karo — aur ek important kaam karo — **GitHub pe already push ho gayi toh:**

```bash
git rm --cached .env
git commit -m "remove .env from tracking"
git push
```

Yeh command `.env` ko git tracking se hata deta hai — file local machine pe rahegi — par git isko ignore karega aage se.

---

## Check 3 — Production mein secrets kahan jaayenge?

Locally `.env` file se secrets milte hain. Par jab Vercel ya kisi aur platform pe deploy karoge — `.env` file wahan nahi jaayegi — kyunki `.gitignore` mein hai.

Toh production mein secrets kaise milenge?

**Environment Variables** — deployment platform pe manually enter karne padte hain.

Vercel pe:

```
Project → Settings → Environment Variables

AUTH_SECRET        → teri generated value
AUTH_GITHUB_ID     → tera GitHub Client ID
AUTH_GITHUB_SECRET → tera GitHub Client Secret
```

Yeh Vercel apne secure storage mein rakhta hai — aur `process.env` se accessible hote hain — bilkul `.env` ki tarah.

---

## Check 4 — GitHub OAuth app mein production URL add kiya?

Abhi GitHub OAuth app mein `localhost:3000` set hai. Jab deploy karoge — production URL add karna padega — warna login kaam nahi karega.

GitHub pe jao:

```
Settings → Developer Settings → OAuth Apps → tera app
```

Yeh add karo:

```
Homepage URL:  https://teri-production-url.com
Callback URL:  https://teri-production-url.com/api/auth/callback/github
```

`localhost:3000` wali entries reh sakti hain — development ke liye — sirf production wali add karo.

---

## Check 5 — Final `auth.ts` verify karo

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [GitHub],
  session: {
    strategy: "jwt",
    maxAge: 30 * 24 * 60 * 60, // 30 din
  },
  secret: process.env.AUTH_SECRET,
  pages: {
    signIn: "/login",
  },
})
```

Yeh sab hai? ✅

---

## Poora checklist ek jagah

```
✅ AUTH_SECRET — npx auth secret se generate kiya — weak nahi
✅ .env — .gitignore mein listed — GitHub pe nahi gayi
✅ Vercel pe environment variables set kiye — AUTH_SECRET, AUTH_GITHUB_ID, AUTH_GITHUB_SECRET
✅ GitHub OAuth app mein production URL add kiya
✅ auth.ts — session.strategy, maxAge, secret, pages — sab set hai
```

---

## Summary

- Production mein `.env` nahi jaati — platform pe manually environment variables set karo
- GitHub OAuth app mein production URL add karna zaroori hai — warna login fail
- `AUTH_SECRET` strong honi chahiye — `npx auth secret` se generate karo
- `.gitignore` mein `.env` hona chahiye — already push ho gayi toh `git rm --cached .env`
