## Session kya hoti hai — story se samjho

Tu Instagram use karta hai — kal login kiya tha — aaj phone uthaya — abhi bhi logged in hai.

**Yeh kaise hota hai?**

Jab tu pehli baar login karta hai — Instagram server ek **token** banata hai — ek random string:
```
"x7fK92mNpQ3rT8..."
```

Yeh token tere **browser mein save ho jaata hai** — ek jagah hoti hai jise **cookie** kehte hain.

```
Pehli baar login kiya
        ↓
Server ne token banaya
        ↓
Token cookie mein save hua — browser mein
```

Ab jab bhi tu Instagram kholega:
```
Browser request bhejta hai
        ↓
Cookie automatically saath jaati hai — token bhi
        ↓
Server check karta hai — "yeh token valid hai — yeh Pravin hai"
        ↓
Logged in dikhata hai
```

**Yahi session hai** — woh saved information jo batati hai "yeh user logged in hai."

## Hamare app mein session kahan hai?

Tu 1.3 mein login kar chuka hai. Browser mein **DevTools** kholo:

```
F12 → Application → Cookies → localhost:3000
```

Ek cookie dikhegi — kuch aisa naam hoga:
```
authjs.session-token
```

**Yahi hai teri session** — Next-Auth ne yeh cookie set ki jab tu login hua. Yeh encrypted hai — `AUTH_SECRET` se lock hai — toh seedha read nahi kar sakte.

## `auth()` — session read karne ka function

Ab page pe dikhana hai ki kaun logged in hai. Iske liye Next-Auth ka `auth()` function hai.

`auth()` kya karta hai? — Woh cookie padta hai — decrypt karta hai — aur batata hai ki kaun logged in hai.

Pehle `auth.ts` mein export karo — abhi zaroorat pad rahi hai:

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers, signIn, auth } = NextAuth({
  providers: [GitHub],
})
```

Ab `page.tsx` mein use karo — pehle sirf `console.log` karke dekho kya aata hai:

```tsx
import { signIn, auth } from "@/auth"

export default async function Home() {
  const session = await auth()
  console.log(session)

  return (
    <main>
      <h1>Home Page</h1>
      <form
        action={async () => {
          "use server"
          await signIn("github")
        }}
      >
        <button type="submit">Sign In with GitHub</button>
      </form>
    </main>
  )
}
```

`async` function banana pada — kyunki `await auth()` use kiya. `auth()` server pe cookie check karta hai — thoda time leta hai — isliye `await`.

**Terminal mein dekho** — kya print hua?

Agar logged in ho:
```js
{
  user: {
    name: "Pravin",
    email: "pravin@gmail.com",
    image: "https://avatars.githubusercontent.com/..."
  },
  expires: "2024-..."
}
```

Agar logged in nahi:
```js
null
```

## Ab page pe dikhao

`session` mein user ka data hai — ab page pe dikhate hain. `console.log` hata do — aur yeh add karo:

```tsx
import { signIn, auth } from "@/auth"

export default async function Home() {
  const session = await auth()

  return (
    <main>
      <h1>Home Page</h1>
      <p>Session: {JSON.stringify(session)}</p>
      <form
        action={async () => {
          "use server"
          await signIn("github")
        }}
      >
        <button type="submit">Sign In with GitHub</button>
      </form>
    </main>
  )
}
```

`JSON.stringify(session)` — session object ko string mein convert kiya — taaki page pe raw format mein dikh sake.

Browser mein dekho — agar logged in ho toh kuch aisa dikhega:
```
Session: {"user":{"name":"Pravin","email":"pravin@gmail.com",...}}
```

Dikh raha hai? ✅

## `signOut` — logout kaise karein?

Abhi page pe sirf Sign In button hai — par agar logged in hain toh Sign Out button hona chahiye.

`signOut` bhi Next-Auth ka function hai — yeh cookie delete karta hai — session khatam ho jaati hai.

Pehle `auth.ts` mein export karo:

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [GitHub],
})
```

Ab `page.tsx` mein — humko decide karna hai:
- Agar `session` hai → naam dikhao + Sign Out button
- Agar `session` nahi → Sign In button

Yeh `if/else` jaisi cheez hai — React mein isko **ternary operator** se likhte hain:

```
session ? (logged in wala UI) : (logged out wala UI)
```

`?` matlab "agar session hai toh" — `:` matlab "warna."

Pehle sirf text se try karo — bina buttons ke:

```tsx
import { signIn, signOut, auth } from "@/auth"

export default async function Home() {
  const session = await auth()

  return (
    <main>
      <h1>Home Page</h1>
      {session ? (
        <p>Logged in hai tu — {session.user?.name}</p>
      ) : (
        <p>Logged in nahi hai</p>
      )}
    </main>
  )
}
```

`session.user?.name` — `?.` matlab "agar `user` exist karta hai tabhi `name` lo — warna error mat do."

Browser mein dekho:
- Logged in ho toh → "Logged in hai tu — Pravin" ✅
- Logged in nahi → "Logged in nahi hai" ✅

---

Ab buttons add karo:

```tsx
import { signIn, signOut, auth } from "@/auth"

export default async function Home() {
  const session = await auth()

  return (
    <main>
      <h1>Home Page</h1>
      {session ? (
        <>
          <p>Welcome, {session.user?.name}!</p>
          <form
            action={async () => {
              "use server"
              await signOut()
            }}
          >
            <button type="submit">Sign Out</button>
          </form>
        </>
      ) : (
        <form
          action={async () => {
            "use server"
            await signIn("github")
          }}
        >
          <button type="submit">Sign In with GitHub</button>
        </form>
      )}
    </main>
  )
}
```

`<>...</>` — yeh **Fragment** hai. React mein ek se zyada elements return karne ke liye ek wrapper chahiye. `<div>` laga sakte the — par extra HTML ban jaata — `<>` empty wrapper hai, HTML mein kuch nahi aata.

## Summary

- Session = cookie mein saved token — batata hai "kaun logged in hai"
- `auth()` — cookie read karta hai — session return karta hai

- Page pe login/logout UI banaya — ternary operator se
