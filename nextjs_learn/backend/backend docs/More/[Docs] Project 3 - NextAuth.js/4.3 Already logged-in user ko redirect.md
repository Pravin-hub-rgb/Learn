# Doc 4.3 — Already Logged-In User Ko Redirect

## Abhi Tak Kya Hua

4.1 mein custom `/login` page banaya — `auth.ts` mein `pages.signIn: "/login"` add kiya — Navbar ka `signIn()` fix kiya — middleware bhi `/login` pe redirect karne laga. 4.2 mein error handling add kiya — URL se error padh kar page pe dikhaya.

Abhi login page pe ek problem hai — agar user pehle se logged in hai aur `/login` visit kare toh bhi login form dikhta hai. User ko seedha `/dashboard` pe jaana chahiye.

## Broad Goal — Yeh Doc Mein Kya Karenge

Ek cheez fix karenge — agar user pehle se logged in hai aur `/login` page visit kare toh usko seedha `/dashboard` pe redirect karenge. Yeh user experience improve karega.

## Problem — Logged-in User Ko Bhi Login Form Dikhta Hai

Abhi agar aap logged in hoke `/login` visit karte hain:

```
http://localhost:3000/login
```

Toh login form dikhta hai — "Sign In with GitHub" button dikhta hai. User ko confuse hota hai — "Maine toh pehle hi login kiya hai, phir bhi login page dikha raha hai?"

**Solution:** Login page mein check karenge ki user logged in hai ya nahi — agar logged in hai toh seedha `/dashboard` pe redirect karenge.

## Step 1 — `auth()` se session padho

Login page mein session check karna padega. `auth()` function use karenge jo `auth.ts` se export hota hai.

Pehle `auth` import karo:

```tsx
import { signIn } from "@/auth"
import { auth } from "@/auth"
```

Ab `auth()` call karo — yeh ek async function hai:

```tsx
export default async function LoginPage({
  searchParams,
}: {
  searchParams: { error?: string }
}) {
  const session = await auth()
  
  return (
    <main>
      <h1>Sign In</h1>
      {/* ... login form ... */}
    </main>
  )
}
```

`const session = await auth()` — session object milta hai agar user logged in hai, warna `null` milta hai.

## Step 2 — Session check karo aur redirect karo

Ab session check karo — agar logged in hai toh redirect karo:

```tsx
export default async function LoginPage({
  searchParams,
}: {
  searchParams: { error?: string }
}) {
  const session = await auth()
  
  if (session) {
    // User logged in hai - redirect karo
    return <div>Redirecting...</div>
  }
  
  // User logged out hai - login form dikhao
  return (
    <main>
      <h1>Sign In</h1>
      {/* ... login form ... */}
    </main>
  )
}
```

Abhi redirect ke liye `<div>Redirecting...</div>` dikhaya hai — test karne ke liye.

Browser mein test karo:

1. **Logged out state:** `/login` visit karo → "Sign In" heading dikhegi ✅
2. **Logged in state:** Login karo → phir `/login` visit karo → "Redirecting..." dikhega ✅

## Step 3 — `redirect()` function use karo

Ab redirect ke liye proper function use karenge — `redirect()` jo Next.js se aata hai:

```tsx
import { signIn } from "@/auth"
import { auth } from "@/auth"
import { redirect } from "next/navigation"
```

Ab `redirect()` use karo:

```tsx
export default async function LoginPage({
  searchParams,
}: {
  searchParams: { error?: string }
}) {
  const session = await auth()
  
  if (session) {
    // User logged in hai - seedha redirect karo
    redirect("/dashboard")
  }
  
  // User logged out hai - login form dikhao
  // (Error handling already exists from 4.2)
  
  return (
    <main>
      <h1>Sign In</h1>
      {/* Existing error handling and form from 4.2 */}
    </main>
  )
}
```

`redirect("/dashboard")` — yeh ek server-side redirect hai jo immediate redirect karta hai.

## Step 4 — Final complete code

Ab poora login page ka final code:

```tsx
import { signIn } from "@/auth"
import { auth } from "@/auth"
import { redirect } from "next/navigation"

const errorMessages = {
  OAuthAccountNotLinked: "Yeh email pehle se kisi aur account se linked hai",
  OAuthSignin: "GitHub se connect karne mein dikkat aayi — dobara try karo",
  OAuthCallback: "Login process mein kuch gadbad hui — dobara try karo",
  Default: "Kuch galat hua — dobara try karo",
}

export default async function LoginPage({
  searchParams,
}: {
  searchParams: { error?: string }
}) {
  const session = await auth()
  
  if (session) {
    redirect("/dashboard")
  }
  
  const error = (await searchParams).error
  const errorMessage = error
    ? (errorMessages[error as keyof typeof errorMessages] ?? errorMessages["Default"])
    : null

  return (
    <main>
      <h1>Sign In</h1>
      
      {errorMessage && (
        <p style={{ color: "red" }}>
          {errorMessage}
        </p>
      )}

      <form
        action={async () => {
          "use server"
          await signIn("github")
        }}
      >
        <button type="submit">Sign In with GitHub</button>
      </form>
    </main>
  )
}
```

## Step 5 — Test karo

Ab poora flow test karo:

1. **Logged out state:**
   ```
   http://localhost:3000/login
   ```
   → "Sign In" heading dikhegi ✅

2. **Logged in state:**
   - Login karo
   - Phir `/login` visit karo
   → Seedha `/dashboard` pe redirect hoga ✅

3. **Error handling:**
   ```
   http://localhost:3000/login?error=OAuthAccountNotLinked
   ```
   → Error message dikhega ✅

4. **Logged in + error:**
   - Login karo
   - Phir `/login?error=OAuthAccountNotLinked` visit karo
   → Seedha `/dashboard` pe redirect hoga (error nahi dikhega) ✅

## Summary

- `auth()` se session check kiya
- Agar session hai toh `redirect("/dashboard")` use kiya
- Error handling ke saath combine kiya
- Logged-in user ko `/login` pe nahi rukna padta
