# Doc 3.3 — Server Component Protection (auth() helper)

## Abhi Tak Kya Hua
3.1 mein `/dashboard` ko client-side protection di — `useSession()` + `useEffect` + `useRouter`. Kaam karta hai — par flash aata hai aur server ko pata nahi hota. 3.2 mein samjha — Server Component mein server pe hi check ho sakta hai — page browser tak pahuncha hi nahi.

Aaj wahi karenge.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

`/dashboard` page ko Server Component mein convert karenge — `auth()` se server pe session check karenge — agar logged in nahi toh server pe hi redirect karenge. Koi flash nahi — koi `useEffect` nahi — koi `useSession` nahi.

---

## Abhi ka `/dashboard` dekho

Abhi `app/dashboard/page.tsx` mein yeh hai:

```tsx
"use client"

import { useSession } from "next-auth/react"
import { useRouter } from "next/navigation"
import { useEffect } from "react"

export default function Dashboard() {
  const { status } = useSession()
  const router = useRouter()

  useEffect(() => {
    if (status === "unauthenticated") {
      router.push("/")
    }
  }, [status, router])

  if (status === "loading") return <p className="p-8">Loading...</p>
  if (status === "unauthenticated") return null

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p>Yeh sirf logged-in users ke liye hai.</p>
    </main>
  )
}
```

Bahut code hai — sirf ek kaam ke liye — "logged in hai ya nahi check karo."

Server Component mein yeh kitna simple ho jaayega — dekhte hain.

---

## Step 1 — `"use client"` hatao

Pehle sirf `"use client"` hatao — baaki sab same rakho:

```tsx
import { useSession } from "next-auth/react"
import { useRouter } from "next/navigation"
import { useEffect } from "react"

export default function Dashboard() {
  const { status } = useSession()
  const router = useRouter()

  useEffect(() => {
    if (status === "unauthenticated") {
      router.push("/")
    }
  }, [status, router])

  if (status === "loading") return <p className="p-8">Loading...</p>
  if (status === "unauthenticated") return null

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p>Yeh sirf logged-in users ke liye hai.</p>
    </main>
  )
}
```

Save karo — error aayegi:

```
You're importing a component that needs `useSession`. 
This React Hook only works in a Client Component.
```

**Kyun?** — `"use client"` hataya toh component Server Component ban gaya. Server Component mein hooks — `useSession`, `useEffect`, `useRouter` — kaam nahi karte. Hooks browser ke liye hain.

Toh yeh sab hatana padega.

## Step 3 — `auth()` se session check karo

Server Component mein session check karne ke liye `auth()` use karte hain — jo `auth.ts` mein banaya tha.

`auth` import karo — zaroorat ab pad rahi hai:

```tsx
import { auth } from "@/auth"

export default async function Dashboard() {
  const session = await auth()
  console.log(session)

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p>Yeh sirf logged-in users ke liye hai.</p>
    </main>
  )
}
```

`async` — `await auth()` use kiya — server pe cookie check karta hai — thoda time leta hai — isliye `async/await`.

**Terminal mein dekho** — kya print hua?

Logged in ho toh:
```js
{ user: { name: "Pravin", ... }, expires: "..." }
```

Logged out ho toh:
```js
null
```

---

## Step 4 — agar `null` hai toh redirect karo

`session` null hai matlab logged in nahi — redirect karna hai.

Pehle socho — `router.push()` use kar sakte hain? Nahi — `useRouter` hook hai — Server Component mein kaam nahi karta.

Server Component mein redirect karne ke liye Next.js ka `redirect()` function hai — `next/navigation` se aata hai.

`redirect` import karo:

```tsx
import { auth } from "@/auth"
import { redirect } from "next/navigation"

export default async function Dashboard() {
  const session = await auth()

  if (!session) {
    redirect("/")
  }

  console.log(session)

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p>Yeh sirf logged-in users ke liye hai.</p>
    </main>
  )
}
```

`!session` — `!` matlab "nahi" — agar session nahi hai — redirect karo.

`redirect("/")` — yeh server pe hi redirect kar deta hai — browser tak page pahunchta hi nahi.

`console.log` bhi ab `redirect` ke baad hai — agar redirect hua toh console tak pahunchega hi nahi.

Test karo — logout karke `/dashboard` visit karo:
- Flash aaya? — **Nahi** ✅
- Redirect hua? — **Haan** ✅
- Terminal mein kya dikha? — Kuch nahi — server ne redirect kar diya page render hone se pehle ✅

---

## Step 5 — `console.log` hata do — session use karo

```tsx
import { auth } from "@/auth"
import { redirect } from "next/navigation"

export default async function Dashboard() {
  const session = await auth()

  if (!session) {
    redirect("/")
  }

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p className="text-gray-600">
        Welcome, {session.user?.name}!
      </p>
    </main>
  )
}
```

`session.user?.name` — yahan TypeScript jaanta hai ki `session` null nahi hai — kyunki upar `if (!session) redirect()` hai — toh yahan tak sirf logged in users pahunchenge.

## Pehle vs Abhi — comparison

**Client-side (3.1):**
```
Browser pe page load → JS chala → useSession fetch kiya
→ unauthenticated → useEffect → redirect
```
- Flash possible
- Bahut code — useSession, useEffect, useRouter
- Browser mein hota hai

**Server Component (3.3):**
```
Server pe request aayi → auth() → null → redirect
```
- Koi flash nahi
- Sirf 2 lines — `auth()` aur `if (!session) redirect()`
- Server pe hota hai — browser tak page pahunchta hi nahi

---

## Summary

- `"use client"` hataya — Server Component ban gaya — hooks kaam nahi karte
- `auth()` — server pe session check karta hai
- `redirect()` — `next/navigation` se — server pe redirect karta hai
- `if (!session) redirect("/")` — 2 lines — poora protection
- Koi flash nahi — server ne page bhejne se pehle hi rok diya
