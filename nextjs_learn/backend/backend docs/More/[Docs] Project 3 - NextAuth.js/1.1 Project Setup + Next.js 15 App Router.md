## Next.js 15 kya hai — aur App Router kyun?

Bhai, pehle broadly samjho — hum kya banana chahte hain aur Next.js kyun use kar rahe hain.

Humara goal hai ek **full-stack web app** banana jisme:
- User login kar sake (GitHub se)
- Kuch pages sirf logged-in users ke liye hon
- Navbar mein user ka naam dikhe

Ab socho — yeh sab karne ke liye sirf React kaafi nahi hai. React sirf UI banata hai — **server pe kya hoga, routes kaise handle honge, API kaise banti hai** — yeh sab React nahi karta.

Isliye **Next.js** — yeh React ke upar ek framework hai jo yeh sab handle karta hai.

---

### Pages Router vs App Router — kya difference hai?

Next.js ke 2 versions the:

**Pehle wala — Pages Router (`pages/` folder)**
```
pages/
  index.js        ← home page
  about.js        ← about page
  api/
    hello.js      ← API route
```
Yahan har file ek page thi — simple tha, lekin limitations thi.

**Naya wala — App Router (`app/` folder) — Next.js 13+ se**
```
app/
  page.tsx        ← home page
  layout.tsx      ← shared layout
  about/
    page.tsx      ← about page
```

App Router mein **React Server Components (RSC)** aate hain — yeh concept important hai, aage explain karenge. Abhi bas itna samjho ki App Router naya aur better hai — hum yahi use karenge.

## Project banao

Humko pehle Next.js 15 ka project setup karna hai. Iske liye ek command hai:

```bash
bun create next-app@latest nextauth-project
```

Yeh command chalate hi kuch options aayenge — **yeh select karo:**

```
✔ TypeScript → Yes
✔ ESLint → Yes
✔ Tailwind CSS → Yes
✔ src/ directory → No
✔ App Router → Yes
✔ Turbopack → Yes
✔ Import alias → No
```

Ek ek option kyun:
- **TypeScript** → type safety milegi — galtiyan pehle hi pakad jaayengi
- **App Router** → naya system — yahi use karenge
- **Turbopack** → faster dev server
- **src/ directory → No** → `app/` folder seedha root mein rahega — simpler
- **Import alias → No** → abhi ke liye zaroorat nahi

## `app/layout.tsx` aur `app/page.tsx` kya hote hain?

### `layout.tsx` — poori app ka wrapper

```tsx
// app/layout.tsx — abhi jo default hai
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        {children}
      </body>
    </html>
  )
}
```

Yeh `{children}` kya hai? — Jab bhi koi page open hoga, woh page ka content `children` mein aayega aur `<body>` ke andar render hoga.

Matlab — **Navbar yahan lagaenge** — taaki har page pe dikhE.

### `page.tsx` — actual page content

```tsx
// app/page.tsx — abhi jo default hai
export default function Home() {
  return (
    <main>
      <h1>Hello World</h1>
    </main>
  )
}
```

`/` route pe yahi dikhega.

## React Server Components (RSC) — briefly

Yeh concept 1.2 aur 1.3 mein zyada kaam aayega — abhi sirf itna samjho:

**Pehle React mein** — saara code browser mein run hota tha (client-side)

**App Router mein** — by default **server pe run hota hai** — "React Server Component"

```tsx
// Yeh SERVER pe run hoga — browser tak sirf HTML jaayega
export default async function Page() {
  const data = await fetch('...') // ← directly fetch kar sakte ho server pe
  return <div>{data}</div>
}
```

Agar client pe run karna ho (useState, useEffect, browser events) — toh file ke upar likhna padta hai:

```tsx
"use client"  // ← yeh likha toh client component ban jaata hai

import { useState } from 'react'
// ...
```

**Hum yeh kab use karenge:**
- `auth()` call karni ho → Server Component (1.3 mein)
- `useSession()` hook use karna ho → Client Component (`"use client"`)

Abhi bas yeh concept yaad rakho — aage naturally samajh aayega.
