# Doc 3.6 — proxy.ts mein Auth Check + Redirect

## Abhi Tak Kya Hua
3.5 mein `proxy.ts` banaya — verify kiya ki har request pe chal raha hai — static files exempt kiye. Abhi proxy mein sirf `console.log` hai — koi auth check nahi.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Proxy ko session ka pata lagana hai — phir `/dashboard` pe logged-out user ko redirect karna hai.

---

## Proxy ko session kaise milegi?

Yaad hai — page mein session aise check karte the:

```ts
const session = await auth()
```

Proxy mein yeh directly nahi kar sakte — proxy har request pe chalta hai — thoda alag convention hai.

Auth.js ne proxy ke liye ek special tarika diya hai — `auth` ko ek function pass karo — andar `req.auth` mein session automatically milegi:

```ts
// Page mein
const session = await auth()
session  // ← yahan session hai

// Proxy mein
auth((req) => {
  req.auth  // ← yahan session hai — Auth.js ne inject ki
})
```

Dono mein `auth` same hai — sirf use karne ka tarika alag.

---

## Step 1 — `auth` import karo aur verify karo

`auth` import karo — zaroorat ab pad rahi hai:

```ts
import { auth } from "@/auth"

export default auth((req) => {
  console.log("req.auth:", req.auth)
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Browser mein `/dashboard` visit karo — **logged in hoke** — terminal mein dekho:

```
req.auth: { user: { name: "Pravin", email: "...", ... } }
```

Ab **logout karke** `/dashboard` visit karo:

```
req.auth: null
```

Proxy ko session ka pata chal raha hai ✅

---

## Step 2 — `/dashboard` protect karo

Ab decision karna hai — agar `/dashboard` pe request aayi aur `req.auth` null hai — redirect karo.

Pehle socho — kaise check karein ki current request `/dashboard` ke liye hai?

`req.nextUrl.pathname` — current URL ka path hai — jaise `/dashboard` ya `/` ya `/about`.

Pehle sirf yeh log karo:

```ts
import { auth } from "@/auth"

export default auth((req) => {
  console.log("path:", req.nextUrl.pathname)
  console.log("session:", req.auth)
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`/dashboard` visit karo — terminal mein confirm karo ki path sahi aa raha hai ✅

---

## Step 3 — condition lagao

Abhi `req.auth` ka pata chal raha hai — ab decision karna hai.

Pehle browser console mein try karo — `!!` kya karta hai practically:

```js
// Browser console mein yeh try karo
const user = { name: "Pravin" }  // ← object hai — truthy
console.log(!!user)               // true

const noUser = null               // ← null hai — falsy
console.log(!!noUser)             // false
```

`!!` sirf ek kaam karta hai — **koi bhi value ko `true` ya `false` mein convert karo.**

`req.auth` mein ya toh object hoga ya `null` — hum sirf `true`/`false` chahiye — isliye `!!` use karenge.

---

Ab proxy mein pehle sirf `isLoggedIn` banao — verify karo:

```ts
import { auth } from "@/auth"

export default auth((req) => {
  let isLoggedIn

  if (req.auth) {
    isLoggedIn = true
  } else {
    isLoggedIn = false
  }

  console.log("isLoggedIn:", isLoggedIn)
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Terminal mein dekho — logged in aur logged out dono baar:
```
isLoggedIn: true   ← logged in
isLoggedIn: false  ← logged out
```

Yeh if/else lamba hai — `!!` se same kaam ek line mein:

```ts
const isLoggedIn = !!req.auth
// ↑ yeh wahi hai jo upar if/else mein kiya
```

Ab `isProtectedRoute` banao — pehle verify karo:

```ts
import { auth } from "@/auth"

export default auth((req) => {
  const isLoggedIn = !!req.auth
  console.log("isLoggedIn:", isLoggedIn)
  console.log("path:", req.nextUrl.pathname)
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`/dashboard` aur `/` dono visit karo — path sahi aa raha hai? ✅

Ab `isProtectedRoute` add karo:

```ts
import { auth } from "@/auth"

export default auth((req) => {
  const isLoggedIn = !!req.auth
  const isProtectedRoute = req.nextUrl.pathname.startsWith("/dashboard")

  console.log("isLoggedIn:", isLoggedIn)
  console.log("isProtectedRoute:", isProtectedRoute)
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`/dashboard` visit karo — `isProtectedRoute: true` dikh raha hai? ✅
`/` visit karo — `isProtectedRoute: false` dikh raha hai? ✅

Ab `NextResponse` import karo — redirect lagane ki zaroorat ab pad rahi hai:

```ts
import { auth } from "@/auth"
import { NextResponse } from "next/server"

export default auth((req) => {
  const isLoggedIn = !!req.auth
  const isProtectedRoute = req.nextUrl.pathname.startsWith("/dashboard")

  if (isProtectedRoute && !isLoggedIn) {
    return NextResponse.redirect(new URL("/", req.nextUrl))
  }
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`isProtectedRoute && !isLoggedIn` — dono true hain tabhi redirect karo:
- `isProtectedRoute` — `/dashboard` pe hai
- `!isLoggedIn` — logged in nahi hai

`NextResponse.redirect(new URL("/", req.nextUrl))` — home pe redirect karo. `new URL("/", req.nextUrl)` full URL banata hai — `req.nextUrl` se base URL leta hai — `"/"` path lagata hai.

Logout karke `/dashboard` visit karo — redirect hua? ✅

---

## Step 4 — `/api/auth/*` ka kya?

Sign In try karo — GitHub pe redirect ho raha hai? ✅

**Kyun kaam kar raha hai bina extra code ke?**

Auth.js v5 ka `auth()` wrapper smart hai — jab request `/api/auth/*` pe jaati hai — Auth.js internally handle kar leta hai — koi conflict nahi hota.

Toh abhi kuch add karne ki zaroorat nahi.

> **Future mein** agar kabhi sign in break ho — redirect loop aaye — tab yeh add karna:
> ```ts
> if (req.nextUrl.pathname.startsWith("/api/auth")) {
>   return NextResponse.next()
> }
> ```
> Abhi nahi — sirf tab jab problem aaye.

---

## Step 5 — `/dashboard` page clean karo

Ab proxy protect kar raha hai — `/dashboard/page.tsx` mein 3.3 mein likha hua `auth()` + `redirect()` redundant ho gaya — hata do:

```tsx
export default function Dashboard() {
  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p className="text-gray-600">Yeh sirf logged-in users ke liye hai.</p>
    </main>
  )
}
```

Logout karke `/dashboard` visit karo — redirect hua? ✅

**Proxy ne handle kar liya — page mein kuch nahi likha.**

---

## Summary

- `auth((req) => {...})` — proxy convention — `req.auth` mein session milti hai
- `req.nextUrl.pathname` — current URL path
- `!!req.auth` — boolean convert — logged in hai ya nahi
- `startsWith("/dashboard")` — protected route check
- `NextResponse.redirect()` — redirect karo
- `NextResponse.next()` — allow karo
- `/api/auth/*` explicitly allow kiya
