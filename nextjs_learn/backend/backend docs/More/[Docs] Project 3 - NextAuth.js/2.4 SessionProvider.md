# Doc 2.4 ‚Äî SessionProvider kya hai + kahan lagao

## Abhi Tak Kya Hua
2.3 mein `useSession()` introduce kiya ‚Äî Navbar mein lagaya ‚Äî par error aayi:
```
[next-auth]: `useSession` must be wrapped in a <SessionProvider />
```
Yeh error fix karni hai aaj.

---

## Broad Goal ‚Äî Yeh Doc Mein Kya Karenge

`SessionProvider` kya hai samjhenge ‚Äî kahan lagaate hain ‚Äî aur lagane ke baad `useSession()` kaam karega aur console mein session data dikhega.

---

## SessionProvider kya hai ‚Äî story se samjho

Socho ek office hai ‚Äî manager ke paas ek important file hai ‚Äî "kaun logged in hai" ki information.

Ab office mein saare employees ko yeh file chahiye ‚Äî par har baar manager ke paas jaana inefficient hai.

**Solution** ‚Äî manager ek **notice board** lagata hai ‚Äî file wahan pin kar deta hai ‚Äî ab sabko direct milti hai information.

Yeh **notice board** hai `SessionProvider` ‚Äî session ki information ek jagah rakhta hai ‚Äî saare components usse le sakte hain bina baar baar server se maange.

```
SessionProvider ‚Äî notice board
        ‚Üì
useSession() ‚Äî koi bhi component notice board se padhta hai
```

---

## Kahan lagaate hain SessionProvider?

Humko `SessionProvider` wahan lagana hai jahan se **saare components cover ho jaayein** ‚Äî taaki Navbar ho ya koi aur page ‚Äî sab `useSession()` use kar sakein.

Woh jagah hai ‚Äî `app/layout.tsx` ‚Äî kyunki yeh poori app ka wrapper hai ‚Äî har component iske andar hai.

`app/layout.tsx` open karo ‚Äî abhi kuch aisa hoga:

```tsx
import Navbar from "./components/Navbar"

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        <Navbar />
        {children}
      </body>
    </html>
  )
}
```

Ab `SessionProvider` import karo ‚Äî zaroorat ab pad rahi hai:

```tsx
import { SessionProvider } from "next-auth/react"
import Navbar from "./components/Navbar"
```

`next-auth/react` ‚Äî yeh Next-Auth ka client-side package hai ‚Äî `SessionProvider` yahan se aata hai.

Ab `Navbar` aur `{children}` ko `SessionProvider` se wrap karo:

```tsx
import { SessionProvider } from "next-auth/react"
import Navbar from "./components/Navbar"

export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>
        <SessionProvider>
          <Navbar />
          {children}
        </SessionProvider>
      </body>
    </html>
  )
}
```

**Kyun `Navbar` aur `{children}` dono wrap kiye?**

Kyunki dono ko session chahiye hogi ‚Äî Navbar mein user naam dikhega ‚Äî aur aage kisi bhi page mein `useSession()` use karna pad sakta hai. `SessionProvider` ke bahar koi bhi component `useSession()` use nahi kar sakta.

## Verify karo

Browser mein jao ‚Äî error gayi? ‚úÖ

Ab **browser ka console** kholo ‚Äî F12 ‚Üí Console:

Pehle yeh dikhega:
```js
null
"loading"
```

Thodi der baad:
```js
{ user: { name: "Pravin", email: "...", image: "..." } }
"authenticated"
```

**Dekha?** ‚Äî `status` pehle `"loading"` tha ‚Äî phir `"authenticated"` ho gaya.

Yeh isliye ‚Äî `useSession()` server se session fetch karta hai ‚Äî thoda time lagta hai ‚Äî tab tak `"loading"` ‚Äî phir data aa gaya toh `"authenticated"`.

Agar logged in nahi ho toh:
```js
null
"unauthenticated"
```

---

## `status` kyun important hai?

Teen states hoti hain:

```
"loading"         ‚Üí session abhi fetch ho rahi hai ‚Äî wait karo
"authenticated"   ‚Üí user logged in hai ‚Äî session available hai
"unauthenticated" ‚Üí user logged in nahi hai
```

Agar `status` check nahi kiya aur seedha `session.user.name` access kiya ‚Äî toh `"loading"` state mein `session` null hoga ‚Äî error aayega:
```
Cannot read properties of null (reading 'user')
```

Isliye hamesha `status` pehle check karo ‚Äî phir session use karo. Yeh 2.5 mein karenge.

## Summary

- `SessionProvider` session ki information saare components ko share karta hai
- `app/layout.tsx` mein lagao ‚Äî taaki poori app cover ho
- `SessionProvider` ke bahar `useSession()` kaam nahi karta
- `status` ‚Äî `"loading"`, `"authenticated"`, `"unauthenticated"`
