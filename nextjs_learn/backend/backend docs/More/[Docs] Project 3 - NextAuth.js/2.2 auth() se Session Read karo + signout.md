# Doc 2.2 — auth() se Session Read karo + signOut

## Abhi Tak Kya Hua
2.1 mein samjha — session ek cookie hai browser mein — encrypted token hai — Next-Auth set karta hai login pe, delete karta hai logout pe. Par abhi page pe kuch nahi dikhta — session read karna implement nahi kiya.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Cookie toh hai browser mein — par **page pe kaise dikhayein ki kaun logged in hai?** Iske liye `auth()` function hai — woh cookie padhta hai aur user ka data deta hai. Aur `signOut()` se proper logout button banayenge.

---

## `auth()` — cookie padhne wala function

`auth()` Next-Auth ka function hai — yeh kya karta hai:

```
auth() call kiya
        ↓
Server pe jaata hai — cookie padhta hai
        ↓
Cookie decrypt karta hai — AUTH_SECRET se
        ↓
User ka data return karta hai
```

Pehle `auth.ts` mein export karo — abhi zaroorat pad rahi hai:

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers, signIn, auth } = NextAuth({
  providers: [GitHub],
})
```

Ab `page.tsx` mein use karo — pehle sirf `console.log` karo — dekho kya aata hai:

```tsx
import { signIn, auth } from "@/auth"

export default async function Home() {
  const session = await auth()
  console.log(session)

  return (
    <main>
      <h1>Home Page</h1>
      <form
        action={async () => {
          "use server"
          await signIn("github")
        }}
      >
        <button type="submit">Sign In with GitHub</button>
      </form>
    </main>
  )
}
```

`async` — function ke aage lagaya kyunki `await auth()` use kiya. `auth()` server pe cookie check karta hai — thoda time leta hai — isliye `await` — matlab "pehle yeh complete ho phir aage badho."

**Terminal mein dekho** — kya print hua?

Agar logged in ho:
```js
{
  user: {
    name: "Pravin",
    email: "pravin@gmail.com",
    image: "https://avatars.githubusercontent.com/..."
  },
  expires: "2024-..."
}
```

Agar logged in nahi:
```js
null
```

`null` matlab koi session nahi — koi logged in nahi.

## Page pe dikhao

`console.log` se confirm ho gaya — `session` mein data hai. Ab page pe dikhate hain — `console.log` hata do:

```tsx
import { signIn, auth } from "@/auth"

export default async function Home() {
  const session = await auth()

  return (
    <main>
      <h1>Home Page</h1>
      <p>Session: {JSON.stringify(session)}</p>
      <form
        action={async () => {
          "use server"
          await signIn("github")
        }}
      >
        <button type="submit">Sign In with GitHub</button>
      </form>
    </main>
  )
}
```

`JSON.stringify(session)` — session object ko string mein convert kiya — taaki page pe raw format mein dikh sake.

Browser mein dekho — agar logged in ho toh kuch aisa dikhega:
```
Session: {"user":{"name":"Pravin","email":"pravin@gmail.com",...}}
```

Dikh raha hai? ✅

## `signOut` — proper logout button

Abhi page pe sirf Sign In button hai. Logged in hone ke baad Sign Out button bhi hona chahiye.

`signOut` Next-Auth ka function hai — yeh cookie delete karta hai — session khatam.

Pehle `auth.ts` mein export karo:

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [GitHub],
})
```

Ab `page.tsx` mein — humko decide karna hai:
- Agar `session` hai → naam dikhao + Sign Out button
- Agar `session` nahi → Sign In button

Yeh `if/else` jaisi cheez hai. React mein isko **ternary operator** se likhte hain:

```
session ? (logged in wala UI) : (logged out wala UI)
```

`?` matlab "agar session hai toh" — `:` matlab "warna."

Pehle sirf text se try karo — bina buttons ke — sirf dekho ternary kaam kar rahi hai:

```tsx
import { signIn, signOut, auth } from "@/auth"

export default async function Home() {
  const session = await auth()

  return (
    <main>
      <h1>Home Page</h1>
      {session ? (
        <p>Logged in hai — {session.user?.name}</p>
      ) : (
        <p>Logged in nahi hai</p>
      )}
    </main>
  )
}
```

`session.user?.name` — `?.` matlab "agar `user` exist karta hai tabhi `name` lo — warna error mat do."

Browser mein dekho:
- Logged in ho → "Logged in hai — Pravin" ✅
- Logged in nahi → "Logged in nahi hai" ✅

---

Ab buttons add karo:

```tsx
import { signIn, signOut, auth } from "@/auth"

export default async function Home() {
  const session = await auth()

  return (
    <main>
      <h1>Home Page</h1>
      {session ? (
        <>
          <p>Welcome, {session.user?.name}!</p>
          <form
            action={async () => {
              "use server"
              await signOut()
            }}
          >
            <button type="submit">Sign Out</button>
          </form>
        </>
      ) : (
        <form
          action={async () => {
            "use server"
            await signIn("github")
          }}
        >
          <button type="submit">Sign In with GitHub</button>
        </form>
      )}
    </main>
  )
}
```

`<>...</>` — yeh **Fragment** hai. React mein ek se zyada elements return karne ke liye ek wrapper chahiye — `<div>` ki jagah `<>` use karte hain taaki extra HTML na bane.
