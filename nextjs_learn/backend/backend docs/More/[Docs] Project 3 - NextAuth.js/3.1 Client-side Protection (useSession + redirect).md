# Doc 3.1 — Client-side Protection (useSession + redirect)

## Abhi Tak Kya Hua
Phase 2 complete — Session samjha, `auth()` aur `useSession()` use kiya, Navbar banaya with login/logout. Abhi app mein sirf ek page hai — `page.tsx` home page.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Ek `/dashboard` page banayenge — jo sirf logged-in users dekh sakein. Agar koi logged-in nahi hai aur directly `/dashboard` visit kare — toh home page pe redirect ho jaaye.

---

## Pehle `/dashboard` page banao

`app/dashboard/page.tsx` banao:

```tsx
export default function Dashboard() {
  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p>Yeh sirf logged-in users ke liye hai.</p>
    </main>
  )
}
```

Browser mein jao:
```
http://localhost:3000/dashboard
```

Page dikh raha hai? — Par abhi koi bhi dekh sakta hai — logged in ho ya na ho. Yeh problem hai.

---

## Protection lagao — pehla attempt

`useSession()` se `status` check karenge — agar `"unauthenticated"` hai toh redirect karenge.

Redirect ke liye `useRouter` chahiye — Next.js ka hook hai — programmatically kisi bhi page pe bhejta hai.

> **`next/router` vs `next/navigation`** — VS Code mein `useRouter` type karo toh do options aayenge. `next/router` purana hai — Pages Router ke liye tha. Hum App Router use kar rahe hain — toh hamesha `next/navigation` wala lo.

```tsx
"use client"

import { useSession } from "next-auth/react"
import { useRouter } from "next/navigation"

export default function Dashboard() {
  const { status } = useSession()
  const router = useRouter()

  if (status === "unauthenticated") {
    router.push("/")
  }

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p>Yeh sirf logged-in users ke liye hai.</p>
    </main>
  )
}
```

Browser mein jao dashboard pe aur yaha se signout pe click karo - yaha se tum redirect toh hogaye "/" par — yeh error aayegi console mein:
```
Cannot update a component (`Router`) while rendering a different component (`Dashboard`)
```

---

## Yeh error kya keh raha hai?

Error ka matlab samjho — do parts hain:

```
Cannot update a component (Router)        ← Router ko update karna chahte ho
while rendering a different component     ← par Dashboard abhi render ho raha hai
(Dashboard)
```

Matlab — **"render hote waqt Router mat badlo."**

Kya ho raha hai actually:

```
Dashboard render ho raha hai
      ↓
status === "unauthenticated" — TRUE
      ↓
router.push("/") call hua — render ke BEECH mein
      ↓
React bola — "bhai pehle render complete hone do"
      ↓
ERROR
```

Yeh error sirf tab aata hai jab `unauthenticated` ho — kyunki tab hi `router.push()` call hota hai. Jab logged in ho toh condition false hai — code chalta hi nahi — koi error nahi.

---

## Worst solution — `setTimeout`

Tu soch sakta hai — "1 second baad redirect karo — tab tak render complete ho jaayega":

```tsx
if (status === "unauthenticated") {
  setTimeout(() => router.push("/"), 1000)
}
```

Yeh kaam karega — error nahi aayegi — par ek bada problem hai:

**1 second tak dashboard content dikh jaayega** — agar yeh secret page hota — koi screenshot le sakta hai.

Toh `setTimeout` sahi solution nahi hai.

---

## Sahi solution — render ke BAAD redirect karo

Problem yeh hai ki hum render ke **beech** mein redirect kar rahe hain.

Solution — render **complete hone ke baad** redirect karo.

Iske liye `useEffect` hai — yeh sirf itna hai:

```
useEffect = "render hone ke BAAD yeh kaam karo"
```

```tsx
// Render ke BEECH — GALAT
if (status === "unauthenticated") router.push("/")

// Render ke BAAD — SAHI
useEffect(() => {
  if (status === "unauthenticated") router.push("/")
}, [status])
```

Bas itna fark hai — same logic — sirf **timing** alag.

`[status]` — yeh "watch list" hai — "jab bhi `status` change ho — yeh code dobara chalao." Yeh isliye important hai — agar tu dashboard pe logged-in hai aur Navbar se logout kare — `status` change hoga — `useEffect` dobara chalega — redirect ho jaayega bina refresh kiye.

Ab update karo — pehle sirf `[status]` daalo watch list mein:

```tsx
"use client"

import { useSession } from "next-auth/react"
import { useRouter } from "next/navigation"
import { useEffect } from "react"

export default function Dashboard() {
  const { status } = useSession()
  const router = useRouter()

  useEffect(() => {
    if (status === "unauthenticated") {
      router.push("/")
    }
  }, [status]) // ← abhi sirf status

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p>Yeh sirf logged-in users ke liye hai.</p>
    </main>
  )
}
```

Kaam karta hai — par vs code editor mein ek **warning** aayegi:

```
React Hook useEffect has a missing dependency: 'router'
```

**Kyun?** — `useEffect` ke andar `router` use kar rahe hain — par watch list mein nahi daala. React bol raha hai — "bhai jo cheez andar use kar rahe ho — usse watch list mein daalo."

Fix — `router` bhi add karo:

```tsx
}, [status, router]) // ← dono daalo
```

Warning gayi? ✅ Error gayi? ✅

## Abhi bhi ek problem — flash

Logout karke directly `/dashboard` visit karo — **dhyan se dekho** — ek split second ke liye dashboard content dikh jaata hai — phir redirect hota hai.

Kyun yeh hota hai? — `status` ka flow socho jab page load hota hai:

```
/dashboard visit kiya
      ↓
status = "loading"   ← useSession fetch kar raha hai
      ↓
status = "unauthenticated"  ← ab pata chala
      ↓
useEffect chala — redirect hua
```

`"loading"` state mein — humne kuch nahi kiya — toh dashboard content dikh gaya.

**Fix** — `"loading"` state mein dashboard content mat dikhao — sirf "Loading..." dikhao. Aur `"unauthenticated"` mein bhi kuch mat dikhao — redirect ho raha hai toh khaali screen better hai `null` se.

`null` return karna matlab — "kuch bhi render mat karo" — React mein yeh valid hai.

```tsx
"use client"

import { useSession } from "next-auth/react"
import { useRouter } from "next/navigation"
import { useEffect } from "react"

export default function Dashboard() {
  const { status } = useSession()
  const router = useRouter()

  useEffect(() => {
    if (status === "unauthenticated") {
      router.push("/")
    }
  }, [status, router])

  if (status === "loading") return <p className="p-8">Loading...</p>
  // ↑ loading hai — dashboard mat dikhao

  if (status === "unauthenticated") return null
  // ↑ redirect ho raha hai — kuch bhi mat dikhao

  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Dashboard</h1>
      <p>Yeh sirf logged-in users ke liye hai.</p>
    </main>
  )
  // ↑ sirf "authenticated" state mein yahan tak pahunchega
}
```

Flash gaya? ✅

## Client-side protection ka bada limitation

Flash toh kam ho gaya — par yeh approach fundamentally client-side hai:

```
Browser pe page load hota hai
      ↓
JavaScript run hota hai
      ↓
Check hota hai — redirect hota hai
```

**Server ko pata hi nahi tha ki user logged in nahi tha — usne page bhej diya — phir browser ne redirect kiya.**

Isse better tarika hai — **server pe hi check karo** — page bhejne se pehle. Yeh 3.3 mein karenge.
