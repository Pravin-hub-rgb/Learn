# Doc 5.4 — AUTH_SECRET kya hai + .env properly set karo

## Abhi Tak Kya Hua

5.3 mein JWT explicitly configure kiya — `session.strategy` aur `maxAge` likh diya. Par ek cheez abhi bhi hai — `AUTH_SECRET` — 2.1 mein briefly mention hua tha "cookie encrypted hai AUTH_SECRET se" — ab isko properly samjhenge.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

- `AUTH_SECRET` kya hai — kyun zaroori hai
- Sahi se generate kaise karein
- `.env` file kya hoti hai — kahan hoti hai — kyun use karte hain
- `process.env` se code mein kaise padhte hain
- `auth.ts` mein explicitly add karenge

---

## `AUTH_SECRET` kya hai?

5.1 mein JWT ka flow dekha tha:

```
User ka data — naam, email, image
        ↓
Encrypt kiya — lock kar diya
        ↓
Encrypted token cookie mein save kiya
```

**Encrypt karne ke liye ek key chahiye** — jaise ghar ki chabi. Woh key hai `AUTH_SECRET`.

```
Data + AUTH_SECRET → encrypted token → cookie mein
Cookie + AUTH_SECRET → decrypt → data wapas
```

Agar `AUTH_SECRET` weak ho — jaise `"hello123"` — toh koi guess kar sakta hai — fake tokens bana sakta hai — "main Pravin hun" bol ke — bina login kiye access mil jaayega.

Isliye yeh **strong, random, long string** honi chahiye — jo koi guess na kar sake.

---

## `.env` file — secrets kahan rakhte hain

Secrets — jaise `AUTH_SECRET`, GitHub Client Secret — **code mein directly nahi likhte**.

Kyun? — Socho `auth.ts` mein seedha likha:

```ts
secret: "mera-secret-123"
```

Ab yeh code GitHub pe push kiya — `auth.ts` public ho gayi — **duniya ne tera secret dekh liya**. Koi bhi fake tokens bana sakta hai.

Isliye secrets ek alag file mein rakhte hain — **`.env`** — aur yeh file GitHub pe nahi jaati.

`.env` file project root mein hoti hai — kholo — kuch aisa dikhega:

```
AUTH_SECRET="kuch_string_yahan"
AUTH_GITHUB_ID="tera_github_client_id"
AUTH_GITHUB_SECRET="tera_github_client_secret"
```

Aur `.gitignore` file mein dekho — `.env` listed hogi — matlab git isko ignore karta hai — GitHub pe nahi jaati.

---

## `AUTH_SECRET` sahi se generate karo

`.env` mein abhi jo `AUTH_SECRET` hai — woh strong hai? Shayad nahi.

Terminal mein yeh command run karo:

```bash
npx auth secret
```

Yeh Next-Auth ka official tool hai — **cryptographically secure** random secret generate karta hai — aur automatically `.env` mein daal deta hai.

Terminal mein kuch aisa dikhega:

```
AUTH_SECRET=AbC123XyZ789... (ek lamba random string)
```

`.env` file dobara kholo — `AUTH_SECRET` update hua? ✅

**Kyun `npx auth secret`?** — Khud banana risky hai — is tool se guaranteed secure value milti hai.

---

## `process.env` — code mein `.env` kaise padhte hain

`.env` mein likha — par `auth.ts` mein kaise use karein?

`process.env` — Node.js ka ek object hai — isme saari environment variables hoti hain — `.env` wali bhi.

```ts
process.env.AUTH_SECRET  // → .env mein jo AUTH_SECRET likha woh milega
```

Pehle verify karo — `auth.ts` mein temporarily add karo:

```ts
console.log(process.env.AUTH_SECRET)
```

Terminal mein dekho — teri `.env` wali value print hogi? ✅

`console.log` hata do — sirf verify karne ke liye tha.

---

## `auth.ts` mein `AUTH_SECRET` explicitly add karo

Abhi Next-Auth khud `.env` se `AUTH_SECRET` padh leta hai — automatically. Par explicitly likhna better practice hai:

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [GitHub],
  session: {
    strategy: "jwt",
    maxAge: 30 * 24 * 60 * 60,
  },
  secret: process.env.AUTH_SECRET,
  pages: {
    signIn: "/login",
  },
})
```

`secret: process.env.AUTH_SECRET` — Next-Auth ko explicitly bola — "yeh secret use karo JWT encrypt karne ke liye."

App test karo:

```
http://localhost:3000
```

Login karo — sab kaam kar raha hai? ✅

DevTools mein cookie dekho:

```
F12 → Application → Cookies → localhost:3000
```

`authjs.session-token` abhi bhi hai? ✅

## Summary

- `AUTH_SECRET` — JWT encrypt karne ki secret key — weak hogi toh security risk
- `npx auth secret` — properly secure secret generate karta hai
- `.env` — secrets yahan rakhte hain — code mein directly nahi
- `.gitignore` mein `.env` listed — GitHub pe nahi jaati
- `process.env.AUTH_SECRET` — code mein `.env` ki value padhne ka tarika
