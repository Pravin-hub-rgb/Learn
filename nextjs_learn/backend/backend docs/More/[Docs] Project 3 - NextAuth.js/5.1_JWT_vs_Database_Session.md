## Sawaal — data kahan jaata hai?

Socho — tune GitHub se login kiya. Next-Auth ne tera naam, email, image le liya GitHub se.

Ab yeh data **kahan gaya?**

Do jagah ho sakta hai:

```
Option 1 — Cookie mein hi rakh do — JWT
Option 2 — Database mein save karo — Database Session
```

Dono samjhte hain.

---

## Option 1 — JWT Session

JWT — **JSON Web Token** — ek special token hota hai.

Jab tu login karta hai:

```
GitHub se naam, email, image aaya
        ↓
Next-Auth ne in sab cheezein ko ek token mein pack kiya
        ↓
Token ko encrypt kiya — lock kar diya — AUTH_SECRET se
        ↓
Woh encrypted token cookie mein save kar diya — browser mein
```

Ab jab bhi tu koi page visit karta hai:

```
Browser cookie bhejta hai — token ke saath
        ↓
Server token ko decrypt karta hai — unlock karta hai
        ↓
Token ke andar se naam, email nikal leta hai
        ↓
"Yeh Pravin hai — logged in hai"
```

**Khaas baat** — server ne **kuch bhi save nahi kiya apne paas**. Saara data cookie mein hai — browser mein. Server sirf token padhta hai — khatam.

---

## Option 2 — Database Session

Jab tu login karta hai:

```
GitHub se naam, email, image aaya
        ↓
Next-Auth ne database mein user save kiya — users table mein
        ↓
Phir ek random ID banaya — sessions table mein save kiya
        ↓
Woh random ID cookie mein save kar di — browser mein
```

Ab jab bhi tu koi page visit karta hai:

```
Browser cookie bhejta hai — random ID ke saath
        ↓
Server database mein jaata hai — "yeh ID kiske liye hai?"
        ↓
Database se user ka data aata hai
        ↓
"Yeh Pravin hai — logged in hai"
```

**Khaas baat** — cookie mein sirf ek **random ID** hai — koi data nahi. Saara actual data database mein hai.

---

## Dono ka comparison — kab kaunsa use karein

```
                JWT                     Database Session
────────────────────────────────────────────────────────
Data kahan?   Cookie mein              Database mein
Database?     Nahi chahiye             Chahiye
Speed         Fast                     Thoda slow
Logout        Tricky                   Easy
Use karo jab  Simple apps,             User roles, permissions,
              database nahi hai        instant logout chahiye
```

**Logout kyun tricky JWT mein?**

JWT cookie mein hai — Sign Out kiya — cookie delete ho gayi browser se. Par agar kisine woh token pehle copy kar liya? — Woh token technically abhi bhi valid hai — `maxAge` expire hone tak. Server ke paas koi record nahi — woh sirf token padhta hai.

Database session mein — logout pe sirf database ki row delete karo — woh ID invalid — done.

## Database Session ka syntax — briefly

Agar kabhi database add karo — `auth.ts` mein kuch aisa hoga:

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"
import { PrismaAdapter } from "@auth/prisma-adapter"
import { prisma } from "@/lib/prisma"

export const { handlers, signIn, signOut, auth } = NextAuth({
  adapter: PrismaAdapter(prisma),  // ← database adapter
  providers: [GitHub],
  session: {
    strategy: "database",          // ← explicitly database
  },
})
```

`adapter` — Next-Auth ko batata hai "database se baat kaise karni hai." Alag databases ke liye alag adapters — Prisma, MongoDB, Drizzle, etc.

Abhi implement nahi kar rahe — sirf dikhaya ki syntax kaisa hota hai jab zaroorat pade.

## Hamare app mein kya ho raha hai?

Hamare `auth.ts` mein abhi sirf yeh hai:

```ts
export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [GitHub],
})
```

Koi `adapter` nahi — koi `session.strategy` nahi.

Next-Auth kya karta hai? — **Default use karta hai** — aur default hai **JWT**.

Matlab hamare app mein already JWT chal raha hai — tune kuch nahi kiya — Next-Auth ne khud decide kiya.

---

## Summary

- Session store karne ke do tarike — JWT aur Database
- JWT — data cookie mein — no database needed — fast
- Database Session — data server pe — database chahiye — logout easy
- Hamare app mein JWT already chal raha hai — Next-Auth ka default
