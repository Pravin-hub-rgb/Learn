# Doc 3.5 — proxy.ts banao + Verify karo

## Abhi Tak Kya Hua
3.4 mein Middleware ka concept samjha — ek jagah sab handle — har request pe page render hone se pehle chalta hai. Ab implement karte hain.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Pehle `proxy.ts` banayenge aur verify karenge ki middleware actually kaam kar raha hai — har request pe chal raha hai. Auth check 3.6 mein karenge.

---

## Step 1 — file banao

Pehle documentation mein `middleware.ts` milega — woh banao root mein:

```ts
export default function middleware() {
  console.log("middleware chala!")
}
```

Terminal mein warning aayegi:

```
⚠ The "middleware" file convention is deprecated.
Please use "proxy" instead.
```

`middleware.ts` delete karo — `proxy.ts` banao:

```ts
export default function proxy() {
  console.log("proxy chala!")
}
```

---

## `export` kyun karna padta hai?

`proxy.ts` mein kai cheezein ho sakti hain — functions, variables, constants:

```ts
function helperFunction() { ... }  // koi helper kaam
function proxy() { ... }           // middleware
const config = { ... }             // settings
```

Next.js confused ho jaayega — **"kaun sa function run karun har request pe?"**

`export default` se batao — "yeh wala":

```ts
// Next.js nahi chalayega — export nahi kiya
function helperFunction() { ... }

// Next.js chalayega har request pe — export kiya
export default function proxy() {
  console.log("proxy chala!")
}
```

---

## Step 2 — verify karo

Browser mein `/dashboard` visit karo — terminal mein dekho:

```
proxy chala!
proxy chala!
proxy chala!
... bahut saari baar
```

**Itni baar kyun?**

Ek page visit pe browser bahut saari cheezein load karta hai — CSS, JavaScript, fonts — **har ek ke liye alag request** — proxy **har request pe** chalta hai.

CSS aur fonts pe auth check ki zaroorat nahi — woh public files hain. `config` export karo — Next.js ko batao kahan chalana hai:

```ts
export default function proxy() {
  console.log("proxy chala!")
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`config` bhi export kiya — kyunki Next.js ko matcher bhi is file se lena hai. Bina export ke Next.js ko pata nahi chalega.

Ab `/dashboard` visit karo — logs bahut kam aaye? ✅

---

## Summary

- `middleware.ts` → warning → `proxy.ts` rename kiya
- `export default` — Next.js ko batata hai kaun sa function run karna hai
- `config` export — matcher batata hai kahan chalana hai
- Static files exempt kiye — logs kam ho gaye
