## Broad Goal — Yeh Doc Mein Kya Karenge

Aaj hum Auth.js install karenge aur do cheezein banayenge:
1. `auth.ts` — Auth.js ka config file
2. API route — GitHub callback handle karne ke liye

Dono kyun chahiye — yeh naturally samajh aayega neeche.

---

## Auth.js kya hai — kyun chahiye?

Agar tu khud login system banana chaahe bina kisi library ke — toh yeh sab karna padega:
- Session banana — "kaun logged in hai" track karna
- Cookies set karna browser mein
- GitHub ke saath poora OAuth flow handle karna
- Security — token expiry, CSRF attacks, etc.

Yeh bahut kaam hai — aur bahut jagah galti ho sakti hai.

**Auth.js** yeh sab pehle se kar chuka hai — tera kaam sirf configure karna hai.

## Install karo

```bash
bun add next-auth@beta
```

> **`@beta` kyun?** — Auth.js v5 abhi beta mein hai — lekin stable enough hai. v4 se syntax kaafi alag hai — hum directly v5 seekh rahe hain.

---

## `AUTH_SECRET` — kyun chahiye?

Jab user login karta hai — Auth.js ek token banata hai jo browser ki cookie mein store hota hai. Woh token seedha readable nahi hona chahiye — koi bhi cookie dekh ke samajh jaaye ki "yeh user kaun hai" — yeh safe nahi.

Isliye woh token **encrypt** hota hai — lock ho jaata hai — aur lock kholne ki key hai `AUTH_SECRET`.

```
User login karta hai
      ↓
Auth.js token banata hai: "user_id=123, name=Pravin"
      ↓  AUTH_SECRET se lock karta hai
Cookie mein jaata hai: "x7fK92mNpQ..." ← gibberish — koi read nahi kar sakta
```

Agar `AUTH_SECRET` set nahi kiya — Auth.js error dega — kuch kaam nahi karega.

**⚠️ Yeh commands mat chalao:**
```bash
npx auth secret   ← GALAT
bunx auth secret  ← GALAT
```

Yeh dono `BETTER_AUTH_SECRET` generate karte hain — jo ek alag library ke liye hai — Auth.js ke liye nahi.

**Sahi tarika — `.env.local` manually banao root mein:**

```
nextauth-project/
├── app/
├── .env.local     ← yahan banao
├── package.json
```

`.env.local` andar:
```env
AUTH_SECRET=koi_bhi_lambi_random_string_yahan_likho
```

> **`.env.local` kyun?** — Yeh file Git mein commit nahi hoti — secrets safe rehte hain. GitHub pe nahi jaayenge kabhi.

Server restart karo taaki `.env.local` load ho:
```bash
# Ctrl+C phir:
bun dev
```

## GitHub Callback — ek problem hai

Tu jaanta hai ki "Login with GitHub" ka flow kya hota hai:

```
User click karta hai
      ↓
GitHub pe jaata hai — "Authorize karo?"
      ↓
User haa karta hai
      ↓
GitHub wapas redirect karta hai hamare app pe
      ↓
User logged in hai
```

Ab socho — jab GitHub wapas redirect karta hai hamare app pe — **kahan redirect karta hai exactly?**

Ek specific URL pe:
```
http://localhost:3000/api/auth/callback/github
```

Yeh URL GitHub ko tu khud batayega — 1.3 mein jab GitHub pe OAuth App setup karenge.

**Par problem yeh hai** — abhi hamare app mein yeh URL exist nahi karta. Koi nahi sun raha iss URL pe. Toh GitHub redirect karega — aur `404 Not Found` aayega.

Isliye humein ek **API route** banana hai jo iss URL pe sun'e — aur Auth.js ko handle karne de.

---

## API Route banao

Pehle ek cheez samjho — Next-Auth ne **pehle se fix** kar rakha hai ki callback URLs kaisi hongi:

```
GitHub se login  → /api/auth/callback/github
Google se login  → /api/auth/callback/google
Facebook se login → /api/auth/callback/facebook
```

Tu kuch nahi likhta — Next-Auth khud jaanta hai inhe handle karna hai. Yeh inka standard hai.

Par in URLs ko "sun'ne" ke liye hamare app mein ek route chahiye. Agar route nahi hoga toh GitHub redirect karega aur `404 Not Found` aayega.

Ab socho — agar hum manually banayein toh:
```
app/api/auth/callback/github/route.ts
app/api/auth/callback/google/route.ts
app/api/auth/callback/facebook/route.ts
... bahut saare
```

Yeh bakwaas kaam hai. Isliye ek shortcut hai — **`[...]` catch-all** — ek hi folder sab pakad leta hai:

```
app/
└── api/
    └── auth/
        └── [...nextauth]/
            └── route.ts    ← /api/auth/* sab kuch yahi aayega
```

`route.ts` abhi khali rakho — pehle `auth.ts` banate hain.

---

## `auth.ts` banao

Root mein banao — `app/` ke bahar:

```
nextauth-project/
├── app/
├── auth.ts        ← yahan
├── .env.local
├── package.json
```

`NextAuth` — yeh Auth.js ka main function hai. Isme config pass karte hain — providers, settings, etc. Abhi providers empty rakhte hain — 1.3 mein GitHub add karenge.

```ts
import NextAuth from "next-auth"

export const { handlers } = NextAuth({
  providers: [],
})
```

**`handlers` kya hai?**

`NextAuth()` ek object return karta hai. Usme ek cheez hai `handlers` — jisme `GET` aur `POST` functions hain. Yeh functions woh kaam karte hain jo `/api/auth/*` URLs pe hona chahiye — jaise GitHub callback handle karna.

Humne `handlers` export kiya — taaki `route.ts` isme se `GET` aur `POST` le sake.

---

## `route.ts` complete karo

Ab `app/api/auth/[...nextauth]/route.ts` mein:

```ts
import { handlers } from "@/auth"

export const { GET, POST } = handlers
```

**Kya ho raha hai yahan?**

- `handlers` import kiya `auth.ts` se — jisme Next-Auth ke `GET` aur `POST` functions hain
- Export kar diye — matlab Next-Auth ko bol diya "tu handle kar — main gate khol raha hun"

Ab jab GitHub `/api/auth/callback/github` pe redirect karega — yeh route sun raha hai — Next-Auth handle kar lega.

---

## Verify karo

Browser mein jao:
```
http://localhost:3000/api/auth/providers
```

Yeh aana chahiye:
```json
{}
```

Empty object — kyunki abhi `providers: []` hai — koi login method nahi. Par **error nahi aaya** — matlab Auth.js connected hai ✅

Agar error aa raha hai:
- `AUTH_SECRET` `.env.local` mein hai?
- Server restart kiya?
- `app/api/auth/[...nextauth]/route.ts` sahi jagah bani?
