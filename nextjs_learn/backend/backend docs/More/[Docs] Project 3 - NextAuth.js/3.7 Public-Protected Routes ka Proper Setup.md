# Doc 3.7 — Public/Protected Routes ka Proper Setup

## Abhi Tak Kya Hua
3.6 mein proxy mein auth check lagaya — `/dashboard` protect ho gaya. Abhi `proxy.ts` mein yeh hai:

```ts
import { auth } from "@/auth"
import { NextResponse } from "next/server"

export default auth((req) => {
  const isLoggedIn = !!req.auth
  const isProtectedRoute = req.nextUrl.pathname.startsWith("/dashboard")

  if (isProtectedRoute && !isLoggedIn) {
    return NextResponse.redirect(new URL("/", req.nextUrl))
  }
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Abhi sirf `/dashboard` protect hai. Real app mein bahut saare protected pages hote hain. Ek scalable pattern banayenge.

---

## Problem — naya page add karo

Ek naya page banao — `app/profile/page.tsx`:

```tsx
export default function Profile() {
  return (
    <main className="p-8">
      <h1 className="text-2xl font-bold">Profile</h1>
      <p>Yeh bhi sirf logged-in users ke liye hai.</p>
    </main>
  )
}
```

Logout karke `/profile` visit karo — redirect hua?

**Nahi** — kyunki proxy mein sirf `/dashboard` check kiya tha.

Ab `/profile` bhi protect karna hai — kya karoge?

Pehla khayal — ek aur `startsWith` add karo:

```ts
const isProtectedRoute =
  req.nextUrl.pathname.startsWith("/dashboard") ||
  req.nextUrl.pathname.startsWith("/profile")
```

Kaam karega — par socho — kal `/settings` bhi add karna hai, `/orders` bhi, `/account` bhi. Har baar yeh line aur lambi hoti jaayegi:

```ts
const isProtectedRoute =
  req.nextUrl.pathname.startsWith("/dashboard") ||
  req.nextUrl.pathname.startsWith("/profile") ||
  req.nextUrl.pathname.startsWith("/settings") ||
  req.nextUrl.pathname.startsWith("/orders") ||
  req.nextUrl.pathname.startsWith("/account")
```

Bahut messy — aur bhoolne ka chance bhi hai.

---

## Solution — array use karo

Protected routes ki ek list banao — array mein:

```ts
const protectedRoutes = ["/dashboard", "/profile"]
```

Ab check karna hai ki current path is list mein hai ya nahi.

Pehle manually karo — familiar `for` loop se:

```ts
const protectedRoutes = ["/dashboard", "/profile"]

let isProtectedRoute = false

for (let i = 0; i < protectedRoutes.length; i++) {
  if (req.nextUrl.pathname.startsWith(protectedRoutes[i])) {
    isProtectedRoute = true
  }
}
```

`protectedRoutes.length` — array mein kitne elements hain — `2`.
`protectedRoutes[i]` — index se element nikala — `protectedRoutes[0]` = `"/dashboard"`, `protectedRoutes[1]` = `"/profile"`.

Yeh kaam karta hai — par `forEach` se aur clean kar sakte hain:

```ts
let isProtectedRoute = false

protectedRoutes.forEach((route) => {
  if (req.nextUrl.pathname.startsWith(route)) {
    isProtectedRoute = true
  }
})
```

`forEach` — har element pe ek function chalaata hai — index likhne ki zaroorat nahi — seedha `route` milta hai.

Par ek problem — `forEach` **rukta nahi** — `/dashboard` mil gaya, `true` set ho gaya — par loop baaki elements pe bhi chalta rehega — wasteful.

`for` loop mein `break` laga sakte the — rukwa sakte the:

```ts
for (let i = 0; i < protectedRoutes.length; i++) {
  if (req.nextUrl.pathname.startsWith(protectedRoutes[i])) {
    isProtectedRoute = true
    break // ← mil gaya — ab loop band
  }
}
```

Yahi kaam `some()` ek line mein karta hai — aur pehla `true` milte hi ruk jaata hai — internally `break` jaisa:

```ts
const isProtectedRoute = protectedRoutes.some(
  (route) => req.nextUrl.pathname.startsWith(route)
)
```

`some()` andar ek function leta hai — har element pe chalaata hai — agar kisi bhi ek ke liye `true` aaye toh `some()` `true` return karta hai aur ruk jaata hai — warna `false`.

## Update karo

Pehle sirf array banao aur log karo — verify karo:

```ts
import { auth } from "@/auth"
import { NextResponse } from "next/server"

const protectedRoutes = ["/dashboard", "/profile"]

export default auth((req) => {
  const isLoggedIn = !!req.auth
  const isProtectedRoute = protectedRoutes.some(
    (route) => req.nextUrl.pathname.startsWith(route)
  )

  console.log("path:", req.nextUrl.pathname)
  console.log("isProtectedRoute:", isProtectedRoute)
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

`/dashboard` visit karo — `isProtectedRoute: true` ✅
`/profile` visit karo — `isProtectedRoute: true` ✅
`/` visit karo — `isProtectedRoute: false` ✅

Ab redirect add karo — `console.log` hata do:

```ts
import { auth } from "@/auth"
import { NextResponse } from "next/server"

const protectedRoutes = ["/dashboard", "/profile"]

export default auth((req) => {
  const isLoggedIn = !!req.auth
  const isProtectedRoute = protectedRoutes.some(
    (route) => req.nextUrl.pathname.startsWith(route)
  )

  if (isProtectedRoute && !isLoggedIn) {
    return NextResponse.redirect(new URL("/", req.nextUrl))
  }
})

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
}
```

Test karo — logout karke:
- `/dashboard` → redirect ✅
- `/profile` → redirect ✅
- `/` → kuch nahi — home page dikhega ✅

Aage koi naya protected page add karna ho — sirf array mein add karo:

```ts
const protectedRoutes = ["/dashboard", "/profile", "/settings"]
```

Bas — proxy mein kuch aur nahi likhna. ✅

---

## Summary

- Ek `startsWith` — scale nahi karta
- `protectedRoutes` array — ek jagah sab routes
- `some()` — check karta hai ki koi bhi route match hota hai
- Naya page add karna — sirf array mein ek line
