# Doc 5.2 — JWT mein data kahan hai actually — "Hawa mein" hai kya?

## Abhi Tak Kya Hua

5.1 mein samjha — JWT vs Database session — hamare app mein JWT chal raha hai by default — data cookie mein encrypted hai. Par ek sawaal naturally uthta hai —

---

## Sawaal — yeh data real hai kya?

JWT mein data cookie mein hai — database mein kuch save nahi hota. Toh kya yeh **hawa mein** hai? Agar server band ho jaaye — agar browser band ho jaaye — data kahan hai?

Yeh confusion bilkul valid hai. Seedha DevTools se dekho — reality samajh aayegi.

---

## DevTools mein khud dekho

Browser mein jao — login karo — phir:

```
F12 → Application → Cookies → localhost:3000
```

`authjs.session-token` cookie dikhegi — uski value copy karo — kuch aisi hogi:

```
eyJhbGciOiJkaXIiLCJlbmMiOiJBMjU2Q0JDLUhTNTEyIiwia2lkIjoidGVzdCJ9...
```

Yeh **encrypted JWT token** hai — tera saara data iske andar hai — naam, email, image — sab. Encrypt hone ki wajah se seedha nahi padh sakte.

---

## Toh kya real hai?

Haan — **bilkul real hai**.

```
JWT data kahan save hota hai?
        ↓
Browser ki cookie mein
        ↓
Cookie browser ke disk pe store hoti hai
        ↓
Tab band karo — browser band karo — data wahan hi rahega
        ↓
Wapas kholo — cookie wahan hai — login abhi bhi hai
```

Fark sirf itna hai ki **kahan save hai**:

```
Database Session → data server ki disk pe — database mein
JWT Session      → data teri machine pe — browser mein cookie mein
```

Dono real hain — dono disk pe hain — sirf jagah alag hai.

---

## Practically verify karo

**Test 1 — Tab band karo**

Login karo — tab band karo — wapas kholo:

```
http://localhost:3000
```

Abhi bhi logged in hai? ✅ — Data cookie mein save tha — tab band karne se cookie nahi gayi.

**Test 2 — Browser band karo**

Login karo — poora browser band karo — wapas kholo:

```
http://localhost:3000
```

Abhi bhi logged in hai? ✅ — Cookie browser close hone pe bhi nahi gayi — disk pe thi.

**Test 3 — Cookie manually delete karo**

DevTools mein jao:

```
F12 → Application → Cookies → localhost:3000
```

`authjs.session-token` pe right click → Delete.

Page refresh karo — ab logged out hai? ✅

**Yahi JWT ka sach hai** — data cookie mein hai — cookie delete karo — data gaya — session khatam.

---

## JWT ka ek limitation — data stale ho sakta hai

Database session mein — user ka data server pe hai — koi bhi change instantly reflect hota hai.

JWT mein — data token mein packed hai — token expire hone tak wahi data rahega.

Example:

```
User ne login kiya — JWT token bana — naam "Pravin" cookie mein pack hua
        ↓
Server pe database mein naam change kiya — "Pravin Sharma"
        ↓
Par cookie mein abhi bhi "Pravin" hai — token expire nahi hua
        ↓
User ko purana naam dikhega — jab tak session expire na ho
```

Yeh JWT ka known limitation hai — simple apps mein problem nahi — complex apps mein sochna padta hai.

---

## Summary

- JWT data hawa mein nahi — browser ki cookie mein disk pe save hai
- Tab band karo, browser band karo — cookie wahan hi rehti hai
- Cookie delete karo — session khatam — logout
- JWT limitation — data stale ho sakta hai — server pe change reflect nahi hota turant
