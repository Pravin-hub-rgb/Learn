# Doc 4.1 — Custom Sign-In Page banao

## Abhi Tak Kya Hua

Phase 3 complete hai — middleware ban gaya, `/dashboard` aur `/profile` protected hain — unauthenticated user ko redirect ho raha hai. Navbar mein Sign In button hai — par abhi woh seedha GitHub pe bhej raha hai.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Humko ek **custom Sign-In page** banana hai — `/login` pe — jahan saare providers dikhein. Iske liye teen kaam honge:

- Navbar ka `signIn("github")` fix karna — taaki seedha GitHub na jaaye
- `/login/page.tsx` banana — Sign In button ke saath
- `auth.ts` mein Next-Auth ko batana — "mera custom page `/login` pe hai"

Shuru karte hain problem se.

---

## Problem — Navbar ka Sign In seedha GitHub pe jaata hai

Abhi `Navbar.tsx` mein yeh hai:

```tsx
onClick={() => signIn("github")}
```

`"github"` specify kiya — toh directly GitHub pe chala jaata hai — beech mein koi page nahi.

Par socho — agar kal Google bhi add karo, Email/Password bhi — toh user ko choose karna hoga. Kahan choose karega?

Usi jagah pe — **custom Sign-In page** — jahan saare options dikhenge.

Toh pehla kaam — Navbar fix karo.

---

## Step 1 — Navbar fix karo

`Navbar.tsx` kholo — `signIn("github")` wali line dhundo:

```tsx
onClick={() => signIn("github")}
```

Isko yeh karo:

```tsx
onClick={() => signIn()}
```

Bas `"github"` hata diya. Ab `signIn()` bina provider ke call hoga.

`signIn()` bina provider ke kya karta hai? — Next-Auth ka **default Sign-In page** dikhata hai.

Browser mein dekho — Sign In click karo:

```
http://localhost:3000/api/auth/signin
```

Ek page aayega — kuch aisa:

```
Sign in with GitHub
```

Yeh Next-Auth ka **default page** hai — automatically bana hua. GitHub provider add kiya tha `auth.ts` mein — isliye woh button dikh raha hai.

Theek hai — kaam toh kar raha hai. Par yeh ugly hai — customize nahi kar sakte — apna branding nahi daal sakte.

Isliye **custom page** banana padega.

---

## Step 2 — `/login` folder aur page banao

Next.js mein pages `app/` folder ke andar hote hain. `/login` route ke liye:

```
app/
  login/
    page.tsx    ← yeh banana hai
```

`app/login/` folder banao — andar `page.tsx` banao.

**Kyun yeh naming?** — Next.js ka convention hai — folder ka naam route ban jaata hai — aur `page.tsx` woh route ka UI hota hai. Toh `app/login/page.tsx` → `localhost:3000/login`.

Pehle sirf heading dikhao — kuch bhi nahi — bas dekho route kaam karta hai:

```tsx
export default function LoginPage() {
  return (
    <main>
      <h1>Sign In</h1>
    </main>
  )
}
```

Browser mein jao:
```
http://localhost:3000/login
```

"Sign In" heading dikh raha hai? ✅ — Route kaam kar raha hai.

---

## Step 3 — Sign In button add karo

Ab button chahiye — jab click karo toh GitHub se login ho.

`signIn` import karna padega — par **kaunsa wala?**

Yeh Client Component hai ya Server Component?

Abhi `page.tsx` mein `"use client"` nahi likha — toh **Server Component** hai by default.

Server Component mein `onClick` nahi hota — toh `signIn` ka server wala version use karenge — `@/auth` se — `form action` ke andar — jaise `home/page.tsx` mein kiya tha.

`signIn` import karo — zaroorat ab pad rahi hai:

```tsx
import { signIn } from "@/auth"

export default function LoginPage() {
  return (
    <main>
      <h1>Sign In</h1>
      <form
        action={async () => {
          "use server"
          await signIn("github")
        }}
      >
        <button type="submit">Sign In with GitHub</button>
      </form>
    </main>
  )
}
```

`"use server"` — form action ke andar likha — matlab yeh function server pe chalega jab form submit hoga.

`await signIn("github")` — yahan provider specify kiya — kyunki yeh button specifically GitHub ke liye hai.

Browser mein dekho:
```
http://localhost:3000/login
```

Button click karo — GitHub pe jaata hai? ✅

---
Bilkul valid confusion hai yaar — acha pakda!

Dekho yeh thoda tricky hai —

**Poora `page.tsx`** — Server Component hai — `"use client"` nahi hai upar.

Par **`form` ka `action`** — woh ek alag cheez hai. Woh ek **inline function** hai jo form submit hone pe chalta hai.

Problem yeh hai ki — form `action` mein jo function likhte hain — woh normally **client pe** run hoga — browser pe. Par `signIn` server pe hi kaam karta hai.

Toh `"use server"` us **specific function** ke liye likha — poore page ke liye nahi. Matlab —

```tsx
action={async () => {
  "use server"        // ← sirf yeh chhota function server pe chalega
  await signIn("github")
}}
```

Poora page Server Component hai — ✅
Us function ko explicitly bola — "tu bhi server pe hi chalna" — ✅

Isko **Server Action** kehte hain — ek function jo client se trigger hota hai — par server pe execute hota hai.

Confusing kyun lagta hai? Kyunki normally socho — "Server Component mein sab server pe hi chalega" — par `form action` ka default behavior browser wala hota hai — isliye explicitly `"use server"` likhna padta hai usme.

Samajh aaya? Ya aur discuss karein?

## Step 4 — Next-Auth ko batao custom page kahan hai

Abhi ek problem hai — Navbar mein `signIn()` click karo — kahan jaata hai?

```
http://localhost:3000/api/auth/signin
```

Abhi bhi default page pe — `/login` pe nahi.

Kyun? — Next-Auth ko pata nahi hai ki humne custom page banaya hai. Usko batana padega.

`auth.ts` kholo — abhi kuch aisa hai:

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [GitHub],
})
```

`pages` config add karo — zaroorat ab pad rahi hai:

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [GitHub],
  pages: {
    signIn: "/login",
  },
})
```

`pages.signIn: "/login"` — Next-Auth ko bol diya — "Sign In ke liye mera custom page `/login` pe hai."

Ab Navbar mein Sign In click karo:

```
http://localhost:3000/login
```

Custom page pe aa gaya? ✅

---

## Step 5 — Middleware update karo

Abhi `proxy.ts` mein unauthenticated user ko kahan redirect ho raha hai?

```ts
return NextResponse.redirect(new URL("/", req.nextUrl))
```

`"/"` pe — home page pe. Par ab custom login page hai — `/login` pe bhejna chahiye.

`proxy.ts` kholo — redirect wali line update karo:

```ts
return NextResponse.redirect(new URL("/login", req.nextUrl))
```

Ab test karo — logged out ho jao — seedha `/dashboard` visit karo:

```
http://localhost:3000/dashboard
```

`/login` pe redirect hua? ✅

## Summary

- `signIn()` bina provider — custom page pe jaata hai
- `app/login/page.tsx` — custom Sign-In page banaya
- `auth.ts` mein `pages.signIn: "/login"` — Next-Auth ko bataya
- `proxy.ts` mein redirect `/login` pe kiya
