# Doc 2.5 — Navbar mein user naam + dropdown

## Abhi Tak Kya Hua
2.4 mein `SessionProvider` lagaya `layout.tsx` mein — `useSession()` kaam karne laga — console mein session data aa raha hai. Abhi `Navbar.tsx` mein sirf `<nav>Navbar</nav>` hai — koi UI nahi.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Navbar ko proper banayenge — teen states handle karenge:
- `"loading"` → kuch mat dikhao — ya loading indicator
- `"unauthenticated"` → Sign In button dikhao
- `"authenticated"` → user ka naam + dropdown mein Sign Out

---

## Step 1 — pehle `status` se teen states handle karo

Abhi Navbar mein sirf `<nav>Navbar</nav>` hai. `console.log` hata do — aur pehle sirf text dikhao `status` ke hisaab se:

```tsx
"use client"

import { useSession } from "next-auth/react"

export default function Navbar() {
  const { data: session, status } = useSession()

  return (
    <nav>
      <span>MyApp</span>
      {status === "loading" && <span>Loading...</span>}
      {status === "unauthenticated" && <span>Logged out hai</span>}
      {status === "authenticated" && <span>Logged in hai — {session?.user?.name}</span>}
    </nav>
  )
}
```

**Kya ho raha hai yahan?**

`&&` — yeh "aur" operator hai — par yahan ek alag kaam karta hai:

```
status === "loading" && <span>Loading...</span>
```

Matlab — "agar `status` `"loading"` hai — tabhi `<span>` dikhao — warna kuch nahi."

Toh teeno lines teen alag conditions hain — sirf ek waqt pe ek hi dikhega.

Browser mein dekho:
- Logged in ho → "Logged in hai — Pravin" ✅
- Logged in nahi → "Logged out hai" ✅
- Thodi der ke liye "Loading..." bhi dikhega page load pe ✅

## Step 2 — Sign In button lagao

`"unauthenticated"` state mein sirf text hai — button chahiye.

`signIn` import karna hai — par **ek important baat pehle.**

### `signIn` ke do versions hain

Next-Auth ne do jagah se `signIn` export kiya hai — dono alag hain:

**1. Server-side** — `@/auth` se:
```ts
import { signIn } from "@/auth"
```
Yeh Server Components aur Server Actions mein use hota hai — jaise `page.tsx` mein `form action` ke andar kiya tha.

**2. Client-side** — `next-auth/react` se:
```ts
import { signIn } from "next-auth/react"
```
Yeh Client Components mein use hota hai — `"use client"` wale components mein — `onClick` ke saath.

Next-Auth ne yeh isliye alag rakha — kyunki server aur browser pe kaam karne ka tarika alag hota hai. Server pe cookie directly padh sakte hain — browser mein API call karni padti hai.

**Navbar Client Component hai** — toh yahan `next-auth/react` wala import karenge:

```tsx
"use client"

import { useSession, signIn } from "next-auth/react"

export default function Navbar() {
  const { data: session, status } = useSession()

  return (
    <nav className="flex justify-between items-center px-6 py-3 bg-gray-900 text-white">
      <span className="font-bold text-lg">MyApp</span>
      <div>
        {status === "loading" && <span className="text-gray-400">Loading...</span>}
        {status === "unauthenticated" && (
          <button
            onClick={() => signIn("github")}
            className="bg-white text-gray-900 px-4 py-1.5 rounded-lg text-sm font-medium hover:bg-gray-200 cursor-pointer"
          >
            Sign In
          </button>
        )}
        {status === "authenticated" && <span>{session?.user?.name}</span>}
      </div>
    </nav>
  )
}
```

`onClick` use kiya — `signIn` ke liye — kyunki Client Component hai — yahan `onClick` kaam karta hai.

Browser mein Sign In button click karo — GitHub pe jaata hai? ✅

## Step 3 — dropdown banao

`"authenticated"` state mein naam ke saath dropdown chahiye — click karo toh Sign Out option aaye.

Dropdown ke liye `useState` chahiye — open/close track karne ke liye. `signOut` bhi import karna hai — `next-auth/react` se — same package — client-side wala:

```tsx
"use client"

import { useSession, signIn, signOut } from "next-auth/react"
import { useState } from "react"

export default function Navbar() {
  const { data: session, status } = useSession()
  const [dropdownOpen, setDropdownOpen] = useState(false)

  return (
    <nav className="flex justify-between items-center px-6 py-3 bg-gray-900 text-white">
      <span className="font-bold text-lg">MyApp</span>

      <div>
        {status === "loading" && (
          <span className="text-gray-400">Loading...</span>
        )}

        {status === "unauthenticated" && (
          <button
            onClick={() => signIn("github")}
            className="bg-white text-gray-900 px-4 py-1.5 rounded-lg text-sm font-medium hover:bg-gray-200 cursor-pointer"
          >
            Sign In
          </button>
        )}

        {status === "authenticated" && (
          <div className="relative">
            <button
              onClick={() => setDropdownOpen(!dropdownOpen)}
              className="flex items-center gap-2 cursor-pointer hover:opacity-80"
            >
              {session?.user?.name} ▾
            </button>

            {dropdownOpen && (
              <div className="absolute right-0 top-full mt-2 bg-gray-800 rounded-lg shadow-lg min-w-32 p-1">
                <button
                  onClick={() => signOut()}
                  className="w-full text-left px-4 py-2 text-sm text-white hover:bg-gray-700 rounded-md cursor-pointer"
                >
                  Sign Out
                </button>
              </div>
            )}
          </div>
        )}
      </div>
    </nav>
  )
}
```

**Naye cheezein:**

`useState(false)` — `dropdownOpen` ek state variable hai — shuru mein `false` — matlab band hai dropdown.

`setDropdownOpen(!dropdownOpen)` — `!` matlab "ulta karo" — agar `true` tha toh `false`, agar `false` tha toh `true`. Har click pe toggle hoga.

`{dropdownOpen && (...)}` — sirf tab dikhao jab `dropdownOpen` true ho.

`relative` parent pe — `absolute` dropdown pe — taaki dropdown parent ke relative position pe aaye — baaki content push nahi hoga.
