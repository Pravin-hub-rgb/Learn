# Doc 1.3 — GitHub OAuth Setup + Pehli Baar Login

## Abhi Tak Kya Hua
1.1 mein Next.js project banaya. 1.2 mein Auth.js install kiya — `auth.ts` banai — `handlers` export kiye — API route banai jo `/api/auth/*` sun rahi hai. Par abhi `providers: []` empty hai — koi login method nahi.

---

## Broad Goal — Yeh Doc Mein Kya Karenge

Do kaam:
1. GitHub pe OAuth App register karenge — Client ID aur Secret lenge
2. `auth.ts` mein GitHub provider add karenge — pehli baar actual login karenge

---

## Step 1 — GitHub pe OAuth App banao

GitHub ko pehle batana padega — "ek app hai jo mere users se login karna chahta hai." Yeh registration GitHub ke Developer Settings mein hoti hai.

```
GitHub → Settings → Developer Settings → OAuth Apps → New OAuth App
```

Yeh form fill karo:

```
Application name:           nextauth-project
Homepage URL:               http://localhost:3000
Authorization callback URL: http://localhost:3000/api/auth/callback/github
```

**`Authorization callback URL` kya hai?**

Yaad hai flow? Jab user GitHub pe "Authorize" click karta hai — GitHub usse wapas hamare app pe redirect karta hai. Kahan redirect kare — yeh URL hai.

`/api/auth/callback/github` — yeh Next-Auth ka standard URL hai — 1.2 mein `[...nextauth]` route banaya tha — wohi yeh handle karega.

**Register App** click karo.

---

## Client ID aur Client Secret lo

Register karne ke baad GitHub ek page dikhayega:

**Client ID** — dikhega wahan — copy karo.

**Client Secret** — "Generate a new client secret" button click karo — **abhi copy karo** — ek baar hi dikhta hai. Agar miss kiya toh dobara generate karna padega.

---

## `.env.local` mein daalo

`.env.local` open karo — jo 1.2 mein banaya tha:

```env
AUTH_SECRET=jo_pehle_se_likha_hai_woh_rehne_do

AUTH_GITHUB_ID=yahan_client_id_paste_karo
AUTH_GITHUB_SECRET=yahan_client_secret_paste_karo
```

**Kyun `.env.local` mein?**

`AUTH_GITHUB_SECRET` ek password jaisa hai — agar GitHub pe commit ho gaya toh koi bhi dekh sakta hai — account abuse ho sakta hai. `.env.local` kabhi commit nahi hoti — safe rehta hai.

Server restart karo — `.env.local` changes ke liye zaroori hai:
```bash
# Ctrl+C phir:
bun dev
```

## Step 2 — `auth.ts` mein GitHub provider add karo

Abhi `auth.ts` mein `providers: []` empty hai. GitHub provider add karo.

`next-auth/providers/github` — yeh Next-Auth ka GitHub provider hai — GitHub ke saath poora OAuth flow handle karta hai — Client ID, Secret, callback — sab internally.

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers } = NextAuth({
  providers: [GitHub],
})
```

`[GitHub]` — array mein daala — matlab "GitHub se login allow karo." Kal Google bhi add karna ho toh `[GitHub, Google]` — bas itna.

Verify karo:
```
http://localhost:3000/api/auth/providers
```

Yeh aana chahiye:
```json
{
  "github": {
    "id": "github",
    "name": "GitHub",
    "type": "oauth"
  }
}
```

GitHub provider dikh raha hai ✅ — ab login button banate hain.

## Step 3 — Login button banao

`app/page.tsx` kholo — default content hata do — sirf yeh likho:

```tsx
export default function Home() {
  return (
    <main>
      <h1>Home Page</h1>
      <button>Sign In with GitHub</button>
    </main>
  )
}
```

Browser mein dekho — button dikh raha hai. Click karo — kuch nahi hoga — kyunki button se koi function connect nahi kiya abhi.

---

## `signIn` — button se kaise connect karein?

`signIn` Next-Auth ka function hai — yeh GitHub pe redirect karta hai.

Ab socho — button pe `onClick` laga sakte hain:
```tsx
<button onClick={() => signIn("github")}>Sign In</button>
```

Ya form pe `onSubmit`:
```tsx
<form onSubmit={() => signIn("github")}>
```

**Par yeh dono kaam nahi karenge.**

Kyun? — `page.tsx` by default **Server Component** hai — server pe run hota hai, browser mein nahi. `onClick` aur `onSubmit` browser ke events hain — server pe nahi hote.

Toh kya karein? — **Server Action** use karte hain. Yeh ek function hota hai jo server pe run hota hai par button click se trigger hota hai. `form` ke `action` prop mein dete hain — andar `"use server"` likhte hain:

Pehle bina `signIn` ke — sirf verify karo ki Server Action kaam kar rahi hai:

```tsx
export default function Home() {
  return (
    <main>
      <h1>Home Page</h1>
      <form
        action={async () => {
          "use server"
          console.log("button click hua!")
        }}
      >
        <button type="submit">Sign In with GitHub</button>
      </form>
    </main>
  )
}
```

Button click karo — **terminal mein** dekho — "button click hua!" print hona chahiye.

Terminal mein print hua? ✅ Server Action kaam kar rahi hai.

---

Ab `signIn` add karo. Pehle `auth.ts` mein export karo — abhi zaroorat pad rahi hai:

```ts
import NextAuth from "next-auth"
import GitHub from "next-auth/providers/github"

export const { handlers, signIn } = NextAuth({
  providers: [GitHub],
})
```

Ab `page.tsx` mein import karke use karo:

```tsx
import { signIn } from "@/auth"

export default function Home() {
  return (
    <main>
      <h1>Home Page</h1>
      <form
        action={async () => {
          "use server"
          await signIn("github")
        }}
      >
        <button type="submit">Sign In with GitHub</button>
      </form>
    </main>
  )
}
```

`signIn("github")` — `"github"` pass kiya matlab "GitHub wala login flow shuru karo."

`await` — `signIn` GitHub pe redirect karta hai, thoda time leta hai — `await` matlab "pehle yeh complete ho phir aage badho."

---

## Pehli baar login karo

```
http://localhost:3000
```

Button click karo — kya hota hai:

```
Button click
      ↓
Server Action chali — signIn("github") call hua
      ↓
GitHub pe redirect — "Authorize nextauth-project?"
      ↓
"Authorize" click karo
      ↓
GitHub wapas bhejta hai → /api/auth/callback/github
      ↓
Next-Auth handle karta hai
      ↓
Wapas localhost:3000 pe
```

Terminal mein yeh logs aane chahiye:
```
POST / 303
GET /api/auth/callback/github?code=... 302
GET / 200
```

`303` aur `302` — yeh redirect codes hain — matlab sab sahi chal raha hai ✅

Login ho gaya — par page pe abhi bhi kuch nahi dikhega — kyunki session read karna implement nahi kiya abhi.

**Phase 2 — Doc 2.1 mein karenge** — pehle samjhenge ki session actually kya hoti hai, kahan store hoti hai — phir read karenge.
