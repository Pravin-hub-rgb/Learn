# Doc 4.1.1 — Login Route: `(auth)/login/page.tsx` (Basic)

- **Phase 2 complete** — Registration system ban gaya!

## Aaj Ka Goal (4.1.1)

**Bahut simple:** Sirf `/login` route banana hai aur ek heading dikhani hai.

```
User /login par jaye → "Login" heading dikhe → Bas!
```

Kuch nahi — na form, na inputs, na validation. Sirf route aur heading.

## Step-by-Step Implementation

### Step 1: File Structure Samjho

Humare paas already `(auth)` route group hai:

```
app/
  (auth)/
    register/
      page.tsx    ← Already exists
    verify/
      page.tsx    ← Already exists
```

Ab humein `(auth)` ke andar `login` folder banana hai:

```
app/
  (auth)/
    login/
      page.tsx    ← Naya banana hai
```

**Note:** `(auth)` route group hai — URL mein nahi aata. Isliye:
- `app/(auth)/login/page.tsx` ka URL hoga: `/login`
- `app/(auth)/register/page.tsx` ka URL hoga: `/register`

### Step 2: `app/(auth)/login/page.tsx` Banao

Naya file create karo aur yeh code likho:

```tsx
// app/(auth)/login/page.tsx

export default function LoginPage() {
  return (
    <div>
      <h1>Login</h1>
    </div>
  );
}
```

**Explanation:**
- `export default function LoginPage()` — Next.js page component
- `<h1>Login</h1>` — Simple heading
- Kuch nahi aur — na form, na inputs

### Step 3: Test Karo

1. Development server chal raha hona chahiye: `npm run dev`
2. Browser mein jao: `http://localhost:3000/login`
3. Screen par dikha chahiye: **"Login"** (badi heading)

**Expected Output:**

```
┌─────────────────────────────┐
│                             │
│         Login               │  ← Sirf yeh heading
│                             │
└─────────────────────────────┘
```

Bas! Kuch aur nahi.

## Common Issues

### Issue 1: Page not found (404)

**Check:**
- File ka naam `page.tsx` hai na? (`login.tsx` nahi hona chahiye)
- File sahi folder mein hai? `app/(auth)/login/page.tsx`

### Issue 2: Heading nahi dikhti

**Check:**
- Development server chal raha hai?
- Browser cache clear karo (Ctrl+Shift+R)
