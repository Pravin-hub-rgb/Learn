## Step-by-Step Implementation (Natural Coder Flow)

### Step 1: Problem Samjho — "Mujhe Password Hash Karna Hai..."

Soch rahe ho: *"Mujhe user ka password secure tarike se store karna hai. Plain text mein save karna dangerous hai. Kya karun?"*

Pehle manually try karte hain — bina kisi library ke:

```typescript
// Pehla attempt — manually hash (bahut basic)
function simpleHash(str: string) {
  let hash = 0;
  for (let i = 0; i < str.length; i++) {
    hash = str.charCodeAt(i) + ((hash << 5) - hash);
  }
  return hash.toString();
}

const hashed = simpleHash("myPassword123");
console.log(hashed); // Output: "123456789" (kuch bhi number)
```

*"Hmm... yeh toh kaam ka nahi hai. Koi bhi isko easily reverse kar sakta hai. Aur same input hamesha same output dega — yeh secure nahi hai."*

### Step 2: Solution Socho — "Koi Secure Library Use Karni Padegi"

*"Mujhe ek aisi library chahiye jo:*

1. *One-way hashing kare*
2. *Salt use kare (random data add kare)*
3. *Slow ho (taaki brute force mushkil ho)*
4. *Battle-tested ho (logon ne pehle use kiya ho)*

*Yeh sab karta hai — `bcrypt`!*

### Step 3: Package Install Karo

Terminal mein jao aur run karo:


**Kyun `bcryptjs` aur `bcrypt` nahi?**

- `bcrypt` native C++ bindings use karta hai — Node.js ke saath compile karna padta hai
- Next.js ke saath kabhi-kabhi compatibility issues aate hain
- `bcryptjs` pure JavaScript implementation hai — Next.js ke saath koi issue nahi
- Performance thoda kam hai, lekin humare use case ke liye perfectly fine hai

### Step 5: `hashPassword()` Function Banao

```typescript
// lib/password.ts
import bcrypt from "bcryptjs";

// Pehla version — basic
export async function hashPassword(password: string): Promise<string> {
  const saltRounds = 10;
  const hashedPassword = await bcrypt.hash(password, saltRounds);
  return hashedPassword;
}

// Test karo
const hash = await hashPassword("mySecretPassword123");
console.log(hash);
// Output: "$2a$10$EixZaYVK1fsbw1ZfbX3OXePaWxn96p36WQoeG6Lruj3vjPGga31lW"
```

**Kya ho raha hai:**

1. `bcrypt.hash()` function le raha hai:
   - `password` — plain text password
   - `saltRounds` — kitni baar salt apply karna hai (10 = 2^10 = 1024 iterations)
2. Yeh automatically:
   - Random salt generate karta hai
   - Salt ko password mein mix karta hai
   - Multiple rounds mein hash karta hai
3. Return karta hai: `$2a$10$...` format mein
   - `$2a$` — bcrypt version
   - `10$` — salt rounds
   - Baaki — actual hash + salt

### Issue 2: Performance Issue — Hashing Bahut Slow Hai

**Problem:**
```
Hashing le raha hai 2-3 seconds... yeh toh bahut slow hai!
```

**Reason:** `saltRounds` zyada high hai

**Solution:**
```typescript
// saltRounds kam karo (default 10 hai)
export async function hashPassword(password: string): Promise<string> {
  const saltRounds = 8; // 10 se 8 karo — faster hoga
  const hashedPassword = await bcrypt.hash(password, saltRounds);
  return hashedPassword;
}
```

**Trade-off:**
- `saltRounds = 10` → 2^10 = 1024 iterations → ~300ms
- `saltRounds = 8` → 2^8 = 256 iterations → ~75ms
- `saltRounds = 12` → 2^12 = 4096 iterations → ~1.2s

**Recommendation:** 10 default acha hai — security aur performance ka balance

### Issue 3: Hash Length Fixed Kyun Hai?

**Question:**
```
"Maine 5 character ka password hash kiya, aur 100 character ka password hash kiya.
Dono ka hash 60 characters ka hai. Kaise?"
```

**Answer:**
bcrypt ka output format fixed hai:
- `$2a$` (4 chars) — version identifier
- `10$` (3 chars) — salt rounds
- Salt (22 chars) — base64 encoded
- Hash (31 chars) — base64 encoded
- **Total: 60 characters**

Input password ki length output pe asar nahi daalti.

### Issue 4: Same Password, Different Hashes — Kaise Handle Karein?

**Problem:**
```typescript
const hash1 = await hashPassword("password123");
const hash2 = await hashPassword("password123");
console.log(hash1 === hash2); // false!
```

*"Agar same password ke different hashes hain, toh login ke time compare kaise hoga?"*

**Solution:**
Login ke time, hum database se hash uthate hain aur input password ko us hash ke saath compare karte hain:

```typescript
// Login flow
const user = await prisma.user.findUnique({
  where: { email: "user@example.com" },
});

if (user) {
  // user.password mein stored hash hai
  const isMatch = await comparePassword(inputPassword, user.password);
  if (isMatch) {
    // Login successful
  } else {
    // Wrong password
  }
}
```

bcrypt internally salt extract kar leta hai hash se aur usi salt ko use karke compare karta hai.

## Best Practices

### 1. Never Log Hashed Passwords

```typescript
// ❌ GALAT
console.log("Hashed password:", hashedPassword);

// ✅ SAHI
console.log("Password hashed successfully");
```

### 2. Always Use Async Versions

```typescript
// ❌ GALAT — Synchronous (blocks the event loop)
const hash = bcrypt.hashSync(password, 10);

// ✅ SAHI — Asynchronous (non-blocking)
const hash = await bcrypt.hash(password, 10);
```

### 3. Validate Password Before Hashing

```typescript
export async function hashPassword(password: string): Promise<string> {
  // Validation
  if (!password || password.length === 0) {
    throw new Error("Password is required");
  }

  if (password.length < 8) {
    throw new Error("Password must be at least 8 characters long");
  }

  if (password.length > 100) {
    throw new Error("Password must be less than 100 characters");
  }

  const saltRounds = 10;
  const hashedPassword = await bcrypt.hash(password, saltRounds);
  return hashedPassword;
}
```

### 4. Handle Errors Gracefully

```typescript
export async function hashPassword(password: string): Promise<string> {
  try {
    if (!password || password.length === 0) {
      throw new Error("Password is required");
    }

    const saltRounds = 10;
    const hashedPassword = await bcrypt.hash(password, saltRounds);
    return hashedPassword;
  } catch (error) {
    console.error("Error hashing password:", error);
    throw new Error("Failed to hash password");
  }
}
```

## Final Code

```typescript
// lib/password.ts
import bcrypt from "bcryptjs";

/**
 * Hash a plain text password using bcrypt
 * @param password - The plain text password to hash
 * @returns The hashed password (60 characters)
 */
export async function hashPassword(password: string): Promise<string> {
  // Validation
  if (!password || password.length === 0) {
    throw new Error("Password is required");
  }

  if (password.length < 8) {
    throw new Error("Password must be at least 8 characters long");
  }

  if (password.length > 100) {
    throw new Error("Password must be less than 100 characters");
  }

  try {
    const saltRounds = 10;
    const hashedPassword = await bcrypt.hash(password, saltRounds);
    return hashedPassword;
  } catch (error) {
    console.error("Error hashing password:", error);
    throw new Error("Failed to hash password");
  }
}

/**
 * Compare a plain text password with a hashed password
 * @param plainPassword - The plain text password to verify
 * @param hashedPassword - The hashed password to compare against
 * @returns true if passwords match, false otherwise
 */
export async function comparePassword(
  plainPassword: string,
  hashedPassword: string
): Promise<boolean> {
  try {
    if (!plainPassword || !hashedPassword) {
      return false;
    }

    const isMatch = await bcrypt.compare(plainPassword, hashedPassword);
    return isMatch;
  } catch (error) {
    console.error("Error comparing passwords:", error);
    return false;
  }
}
```
