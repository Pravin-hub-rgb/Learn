# Doc 1.1 — Packages Install

## Abhi Tak Kya Hua
Abhi kuch nahi hua — yeh humara pehla step hai. Hum ek naya project shuru kar rahe hain jahan hum email-based authentication system banayenge **bina kisi auth library ke** — seedha Prisma v7 + PostgreSQL + Next.js ke saath.

## Broader Goal
**Hum kya banana chahte hain:**
- Email se registration/login system
- Database sessions (cookies mein session ID store hoga, DB mein session data)
- Email verification
- Password reset
- Baad mein OAuth bhi integrate karenge

**Iske liye kya chahiye:**
- **Prisma v7** — database ORM (database se baat karne ke liye)
- **PostgreSQL driver** — database se connect karne ke liye (baad mein install hoga)
- **bcryptjs** — password hash karne ke liye (baad mein install hoga)
- **Resend** — emails bhejne ke liye (baad mein install hoga)

**Aaj kya karenge:**
Sirf **core Prisma v7 packages** install karenge — jo abhi ke abhi chahiye.

**Package Installation Strategy:**
- **Just-in-time installation** — packages tab install honge jab zaroorat pade
- Koi bhi package "out of blue" nahi aayega
- Baaki packages baad mein install honge:
  - `pg`, `@types/pg` → **Phase 2.3** mein (jab `lib/prisma.ts` banayenge)
  - `bcryptjs` → **Phase 2.6** mein (jab password hash karna ho)
  - `resend` → **Phase 5.1** mein (jab emails bhejni ho)

## Step-by-Step Implementation

### Step 1: Terminal kholo aur project directory mein jao

```bash
cd nextjs_learn/backend
```

Ya jahan bhi tumhara Next.js project hai.

### Step 2: Core Prisma packages install karo

```bash
npm install @prisma/client @prisma/adapter-pg
```

**Kya install ho raha hai aur kyun:**

| Package | Kyun chahiye | Example |
|---------|-------------|---------|
| `@prisma/client` | Prisma ka client — database queries chalane ke liye | Jab hum User model banayenge, toh `prisma.user.create()` se naya user add karenge. Yeh `create()`, `findUnique()`, `update()` jaise methods `@prisma/client` se aate hain. |
| `@prisma/adapter-pg` | Prisma v7 ka PostgreSQL adapter — driver adapter mandatory hai v7 mein | Hum PostgreSQL use kar rahe hain, isliye `@prisma/adapter-pg` chahiye. Agar MongoDB hota, toh `@prisma/adapter-mongodb` use karte. Yeh adapter Prisma ko database se connect karta hai. |

### Step 3: Dev dependency install karo

```bash
npm install -D prisma
```

**Kya install ho raha hai aur kyun:**

| Package | Kyun chahiye | Example |
|---------|-------------|---------|
| `prisma` | Prisma CLI — migration, schema manage karne ke liye | `npx prisma init` se project setup hota hai, `npx prisma migrate dev` se database migration chalta hai. Yeh CLI tool hai jo schema manage karta hai. |

**Note:** Prisma v7 automatically TypeScript config handle kar leta hai — alag se `tsx` install karne ki zaroorat nahi hai.

## Summary
- **Core packages:** `@prisma/client`, `@prisma/adapter-pg`
- **Dev package:** `prisma`
- `@prisma/client` — database queries chalane ke liye (create, find, update)
- `@prisma/adapter-pg` — PostgreSQL ke liye adapter (MongoDB hota toh alag adapter)
- `prisma` — CLI tool for migrations aur schema management
- Prisma v7 automatically TypeScript config handle kar leta hai — `tsx` ki zaroorat nahi
- Baaki packages (`pg`, `bcryptjs`, `resend`) baad mein install honge jab zaroorat pade
