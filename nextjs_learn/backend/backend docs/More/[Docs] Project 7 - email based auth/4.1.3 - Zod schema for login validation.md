---

## Step 1: Zod Schema Define Karna

**Soch:** "Form validation ke liye Zod use karna hai. Zod schema define karta hai ki form data kis format mein hona chahiye. Schema ko `lib/validations/auth.ts` mein rakhunga kyunki validation logic ko alag file mein rakhna better practice hai."

**Zod kya karta hai:**
- Schema define karta hai ki data kis format mein hona chahiye
- `.parse()` ya `.safeParse()` se validate kar sakte hain
- Invalid data par error messages return karta hai

```typescript
// lib/validations/auth.ts

import { z } from "zod";

export const loginSchema = z.object({
  email: z
    .string()
    .min(1, "Email is required")
    .email("Please enter a valid email address"),

  password: z
    .string()
    .min(1, "Password is required")
    .min(8, "Password must be at least 8 characters"),
});

export type LoginInput = z.infer<typeof loginSchema>;
```

**Explanation:**

- `z.object({ ... })` — Object schema define karta hai
- `z.string()` — Field string type ka hona chahiye
- `.min(1, "message")` — Minimum 1 character (required field)
- `.email("message")` — Valid email format check karta hai
- `.min(8, "message")` — Minimum 8 characters (password ke liye)
- `z.infer<typeof loginSchema>` — Zod schema se TypeScript type automatically infer karta hai. Yeh type hoga: `{ email: string; password: string; }`

## Step 2: Schema Test Karna (Optional)

**Soch:** "Schema ban gaya, lekin kaam kar raha hai ya nahi verify karna hai."

```typescript
import { loginSchema } from "@/lib/validations/auth";

// Valid data test
const validData = { email: "test@example.com", password: "password123" };
const result1 = loginSchema.safeParse(validData);
console.log(result1.success);  // true

// Invalid email test
const invalidEmail = { email: "galat", password: "password123" };
const result2 = loginSchema.safeParse(invalidEmail);
console.log(result2.success);  // false
console.log(result2.error.flatten().fieldErrors);
// { email: ['Please enter a valid email address'] }
```

**Explanation:**

- `safeParse(data)` — Data validate karta hai, error nahi throw karta
- `result.success` — Boolean, validation pass hua ya nahi
- `result.error.flatten().fieldErrors` — Field-wise error messages

## Complete Code

```typescript
// lib/validations/auth.ts

import { z } from "zod";

export const registrationSchema = z.object({
  email: z.string().email("Invalid email address"),
  password: z.string().min(8, "Password must be at least 8 characters"),
});

export const loginSchema = z.object({
  email: z
    .string()
    .min(1, "Email is required")
    .email("Please enter a valid email address"),

  password: z
    .string()
    .min(1, "Password is required")
    .min(8, "Password must be at least 8 characters"),
});

export type LoginInput = z.infer<typeof loginSchema>;
export type RegistrationInput = z.infer<typeof registrationSchema>;
```

## Summary

**Aaj humne kya seekha:**

1. **Zod schema:**
   - `z.object({ field: z.string().rules() })`
   - Har field par multiple validation rules laga sakte hain

2. **Validation rules:**
   - `.min(1, "message")` — Required field
   - `.email("message")` — Valid email format
   - `.min(8, "message")` — Minimum length

3. **Type inference:**
   - `z.infer<typeof schema>` se TypeScript type milta hai

**Abhi kya nahi kiya:**
- RHF ke saath integrate nahi kiya (4.1.4 mein karenge)
- Error messages UI par nahi dikhaye
