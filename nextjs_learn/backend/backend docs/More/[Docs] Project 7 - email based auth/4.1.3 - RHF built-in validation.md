# Doc 4.1.3 — RHF Built-in Validation: The Messy Way

## Abhi Tak Kya Hua (4.1.2)

- React Hook Form se basic form ban gaya
- `register` se inputs connect kiye
- `handleSubmit` se form submit handle kiya
- Form submit par console.log hota hai
- **Problem:** Koi validation nahi hai!

---

## Step 1: Problem Samjho — Validation Kyun Zaroori Hai?

**Soch:** "Abhi form bina validation ke submit ho raha hai. Mujhe validation add karni hai:"

**Validation rules jo chahiye:**
1. **Email required hai** — khali nahi hona chahiye
2. **Email format valid hona chahiye** — `something@domain.com` pattern
3. **Password required hai** — khali nahi hona chahiye
4. **Password minimum 8 characters** — chhota password nahi chalega

**Options:**
1. **RHF built-in validation** — `register()` ke second argument mein rules
2. **Zod validation** — Alag schema, cleaner approach (baad mein karenge)

**Aaj:** RHF built-in validation use karenge — yeh messy hai, lekin samajhne se Zod ki value pata chalegi.

---

## Step 2: Form Type Define Karna (TypeScript Best Practice)

**Soch:** "TypeScript mein form fields ka type define karna best practice hai. Isse type safety milti hai aur `errors` object ka sahi type hoga."

**Type kyun chahiye:**
- `useForm` ko batane ke liye ki form fields kaunse hain
- `errors` object ka type sahi se define ho
- Auto-completion mile IDE mein
- Compile-time errors pakad jaye

**Type definition:**
```tsx
type LoginFormValues = {
  email: string;
  password: string;
};
```

**Explanation:**
- `LoginFormValues` — Form fields ka type
- `email: string` — Email field string hai
- `password: string` — Password field string hai

**Type ko `useForm` ke saath use karna:**
```tsx
const { register, handleSubmit, formState: { errors } } = useForm<LoginFormValues>();
```

**Kya hua:**
- `useForm<LoginFormValues>()` — Generic type pass kiya
- Ab `errors.email.message` ka type `string | undefined` hoga (sirf)
- TypeScript error nahi aayega

## Step 3: Email Validation Rules Add Karna

**Soch:** "Email ke liye validation rules add karna hai. `register()` ke second argument mein rules pass karunga."

**Email validation rules:**
- `required: "Email is required"` — Khali nahi hona chahiye
- `pattern: { value: /regex/, message: "..." }` — Valid email format

**Email regex pattern:**
```
/^[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}$/i
```
- `^[A-Z0-9._%+-]+` — Start with alphanumeric, dots, etc.
- `@` — @ symbol
- `[A-Z0-9.-]+` — Domain name
- `\.` — Dot
- `[A-Z]{2,}$` — TLD (.com, .org, etc.)

**Code:**
```tsx
<input
  type="email"
  {...register("email", {
    required: "Email is required",
    pattern: {
      value: /^[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}$/i,
      message: "Please enter a valid email address"
    }
  })}
  placeholder="Enter your email"
/>
```

**Test karo:**
1. Khali email submit karo → Error aayega (lekin abhi dikha nahi rahe)
2. Galat email format → Error aayega (lekin abhi dikha nahi rahe)
3. Valid email → Koi error nahi

**Problem:** Error toh aa rahe hain, lekin dikha nahi rahe! Kyunki `errors` object extract nahi kiya aur error display nahi kiya.

## Step 4: Password Validation Rules Add Karna

**Soch:** "Password ke liye bhi validation rules add karna hai:"
- `required: "Password is required"` — Khali nahi hona chahiye
- `minLength: { value: 8, message: "..." }` — Minimum 8 characters

**Code:**
```tsx
<input
  type="password"
  {...register("password", {
    required: "Password is required",

      value: 8,

    }

/>
```

**Test karo:**
1. Khali password submit karo → Error aayega (lekin abhi dikha nahi rahe)
2. Chhota password → Error aayega (lekin abhi dikha nahi rahe)
3. Valid password → Koi error nahi

## Step 5: `formState: { errors }` Introduce Karna

**Soch:** "Validation errors ko capture karne ke liye mujhe `formState` se `errors` object extract karna padega. Tabhi error messages dikha paunga."

**`formState` kya hai:**
- `useForm` hook ka return value mein ek object
- Isme form ka saara state hota hai — errors, isSubmitting, isDirty, etc.
- `errors` object mein validation errors store hote hain
- `errors.email` — Email field ka error
- `errors.email.message` — Error message

**Code:**
```tsx
const { register, handleSubmit, formState: { errors } } = useForm();
```

**Explanation:**
- `formState: { errors }` — `formState` object mein se sirf `errors` extract kiya
- Ab `errors` object mein saare validation errors aayenge

---

## Step 6: Error Display Logic Banana

**Soch:** "Ab errors ko UI par dikhana hai. Conditional rendering use karunga — agar error hai toh dikha, nahi toh kuch mat dikha."

**Syntax:**
```tsx
{errors.email && <p style={{ color: "red" }}>{errors.email.message}</p>}
```

**Explanation:**
- `{errors.email && ...}` — Agar `errors.email` truthy hai (error hai), tabhi render karo
- `errors.email.message` — Error message jo validation rule mein define kiya tha
- `style={{ color: "red" }}` — Red color for error

---

## Step 7: Error Display Add Karna

**Soch:** "Ab dono fields ke liye error display add karta hoon."

**Code:**
```tsx
<input
  type="email"
  {...register("email", {
    required: "Email is required",
    pattern: {
      value: /^[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}$/i,
      message: "Please enter a valid email address"
    }
  })}
  placeholder="Enter your email"
/>
{errors.email && <p style={{ color: "red" }}>{errors.email.message}</p>}

<input
  type="password"
  {...register("password", {
    required: "Password is required",
    minLength: {
      value: 8,
      message: "Password must be at least 8 characters"
    }
  })}
  placeholder="Enter your password"
/>
{errors.password && <p style={{ color: "red" }}>{errors.password.message}</p>}
```

**Test karo:**
1. Khali email submit karo → "Email is required" error dikhega
2. Galat email format → "Please enter a valid email address" error dikhega
3. Khali password submit karo → "Password is required" error dikhega
4. Chhota password → "Password must be at least 8 characters" error dikhega
5. Valid data → Koi error nahi, console.log chalega

## Step 8: Complete Code

```tsx
// app/(auth)/login/page.tsx

"use client";

import { useForm } from "react-hook-form";

export default function LoginPage() {
  const { register, handleSubmit, formState: { errors } } = useForm();

  const onSubmit = (data: any) => {
    console.log("Form submitted:", data);
  };

  return (
    <div>
      <h1>Login</h1>
      <form onSubmit={handleSubmit(onSubmit)}>
        <div>
          <label htmlFor="email">Email:</label>
          <input
            id="email"
            type="email"
            {...register("email", {
              required: "Email is required",
              pattern: {
                value: /^[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}$/i,
                message: "Please enter a valid email address"
              }
            })}
            placeholder="Enter your email"
          />
          {errors.email && (
            <p style={{ color: "red" }}>{errors.email.message}</p>
          )}
        </div>

        <div>
          <label htmlFor="password">Password:</label>
          <input
            id="password"
            type="password"
            {...register("password", {
              required: "Password is required",
              minLength: {
                value: 8,
                message: "Password must be at least 8 characters"
              }
            })}
            placeholder="Enter your password"
          />
          {errors.password && (
            <p style={{ color: "red" }}>{errors.password.message}</p>
          )}
        </div>

        <button type="submit">Login</button>
      </form>
    </div>
  );
}
```

## Problem: Yeh Approach Messy Kyun Hai?

**Dekho code kitna verbose ho gaya:**

```tsx
register("email", {
  required: "Email is required",
  pattern: {
    value: /^[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,}$/i,
    message: "Please enter a valid email address"
  }
})
```

**Problems:**
1. **Regex manually likhna padta hai** — complex, error-prone
2. **Verbose code** — har field ke liye itna code
3. **Type safety nahi hai** — `data: any` use karna padta hai
4. **Reusable nahi hai** — har form mein yeh rules repeat karne padenge
5. **Maintainability** — Regex change karna mushkil

**Solution:** Zod use karna — cleaner, simpler, type-safe.

## Summary

**Aaj humne kya seekha:**

1. **RHF built-in validation:**
   - `register("field", { rules })` — second argument mein validation rules
   - `required`, `pattern`, `minLength`, `maxLength`

2. **Email validation:**
   - `required: "Email is required"`
   - `pattern: { value: /regex/, message: "..." }`

3. **Password validation:**
   - `required: "Password is required"`
   - `minLength: { value: 8, message: "..." }`

4. **`formState: { errors }`:**
   - Validation errors capture karne ke liye
   - `errors.fieldName.message` se error text milta hai

5. **Error display:**
   - `{errors.field && <p>{errors.field.message}</p>}` — conditional rendering

**Abhi kya nahi kiya:**
- Zod use nahi kiya (4.1.4 mein karenge)
- Type inference nahi kiya

---

## Agla Step (4.1.4)

**4.1.4 — Zod Schema: The Clean Way:** Ab hum Zod use karenge — yeh sab rules kitne simple ho jate hain:
- `z.string().email()` — No regex!
- `z.string().min(8)` — Clean syntax!
- Type inference — `z.infer<typeof schema>` — automatic types!
