## Step 1: Model Kyun Banaya Ja Raha Hai?

🧠 **SOCH:**
Ab hum token generate karenge. Ab in tokens ko kaha store karenge? Humko in tokens ko database mein save karna padega. Isliye hum ek naya model banayenge.

Ye token:
- Unique hoga
- Expiry hogi
- Kisi user se associated hoga
- Ek baar use ho jayega

---

## Step 2: Model Banaye

🧠 **SOCH:**
Ab `prisma/schema.prisma` file open karo. Uske end mein yeh naya model add karo.

📝 **CODE:**
```prisma
model PasswordResetToken {
  id        String   @id @default(cuid())
  userId    String   @unique
  token     String   @unique
  expiresAt DateTime
  createdAt DateTime @default(now())

  user User @relation(fields: [userId], references: [id], onDelete: Cascade)
}
```

📚 **EXPLANATION (Har field ke liye):**
1.  `id String @id @default(cuid())`:
    - Har token ka unique id
    - Prisma automatically generate karega

2.  `userId String @unique`:
    - Konsa user hai is token ka
    - `@unique` isliye hai kyuki ek user ke liye ek hi time pe ek hi token hoga
    - Agar naya token generate karoge toh purana automatically overwrite ho jayega

3.  `token String @unique`:
    - Actual random token jo hum email mein bhejenge
    - Unique hoga kyuki 2 user ka same token nahi ho sakta

4.  `expiresAt DateTime`:
    - Kab ye token expire hoga
    - Hum isme 15 minute baad ka time set karenge

5.  `createdAt DateTime @default(now())`:
    - Kab token generate hua tha

6.  `user User @relation(...)`:
    - User model se relation banaya hai
    - Agar user delete hoga toh uske saare tokens bhi automatically delete ho jayenge

---

## Step 2.1: Prisma Format Ke Baad Kya Hua?

🧠 **SOCH:**
Ab tum `npx prisma format` chalaoge toh tum User model mein automatically yeh line add ho jayegi:

```prisma
passwordResetToken PasswordResetToken?
```

📚 **EXPLANATION:**
| Field | Type | Kyun? |
|---|---|---|
| `verificationTokens VerificationToken[]` | Array `[]` | Ek user ke multiple verification tokens ho sakte hai. Baar baar register karega naya token banega isliye array |
| `sessions Session[]` | Array `[]` | Ek user multiple devices se login kar sakta hai. Har device ka alag session hoga isliye array |
| `passwordResetToken PasswordResetToken?` | Single `?` | ✅ Humne hi `@unique` lagaya tha `userId` par PasswordResetToken model mein. Isliye ek user ke liye **ek hi time pe ek hi password reset token hoga**. Agar naya banayege toh purana automatically overwrite ho jayega. `?` optional hai kyuki har user ka kabhi bhi password reset token nahi bhi ho sakta hai. |

✅ Yeh 100% expected hai. Prisma automatically reverse relation add karta hai. Koi problem nahi hai.
