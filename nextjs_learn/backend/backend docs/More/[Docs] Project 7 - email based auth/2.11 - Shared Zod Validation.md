# Doc 2.11 — Shared Zod Validation: Frontend + Backend

## Abhi Tak Kya Hua

- **Doc 2.10 tak** humne RHF + Zod se client-side validation kiya
- **Problem notice kiya:** Server-side pe abhi bhi koi bhi data save ho raha hai!
- **Example:** Postman se 2 character ka password bhejo — woh bhi save ho jata hai!

## Broader Goal

**Aaj kya karenge:**

- Zod schema ko `lib/validations/auth.ts` mein shift karenge
- Same schema frontend aur backend dono mein use karenge
- Server Action mein `safeParse` se validate karenge
- Better error messages dikhayenge

**Problem jo hum solve karenge:**

```typescript
// Abhi kya ho raha hai (bina server-side validation):
POST /register { email: "abc", password: "12" }
// → Save ho jata hai! ❌

// Hum chahte hain:
POST /register { email: "abc", password: "12" }
// → Error: "Invalid email" + "Password must be at least 6 characters" ✅
```

---

## Step-by-Step Implementation

### Step 1: Problem Samjho — "Server-Side Validation Kyun?"

Soch rahe ho: _"Mera client-side validation toh hai (RHF + Zod). Phir bhi server-side kyun?"_

**Reasons:**

1. **Client-side validation bypass ho sakta hai:**
   - Postman se direct API call
   - Browser console se form submit
   - JavaScript disabled
   - Malicious user

2. **Security:**
   - Server hamesha validate kare
   - Client-side sirf UX ke liye hai
   - Server-side security ke liye hai

3. **Single source of truth:**
   - Ek hi schema frontend + backend mein
   - Dono jagah same validation rules
   - Code duplication kam

### Step 2: Validation Schema Shift Karo

Nayi file banao: `lib/validations/auth.ts`

```typescript
// lib/validations/auth.ts

import { z } from "zod";

// Registration form ka validation schema
export const registrationSchema = z.object({
  email: z.string().min(1, "Email is required").email("Invalid email format"),

  password: z
    .string()
    .min(1, "Password is required")
    .min(6, "Password must be at least 6 characters"),
});

// TypeScript type infer karo schema se
export type RegistrationInput = z.infer<typeof registrationSchema>;
```

**Kya hua:**

- Schema ab `lib/validations/auth.ts` mein hai
- `RegistrationInput` type bhi export kiya
- Frontend aur backend dono import kar sakte hain

### Step 3: Server Action Mein Validation Add Karo

Update `app/actions/auth.ts`:

```typescript
// app/actions/auth.ts

"use server";

import prisma from "@/lib/prisma";
import { hashPassword } from "@/lib/password";
import { registrationSchema } from "@/lib/validations/auth";

export type RegistrationResult = {
  error?: string;
  success?: boolean;
};

export async function registerUser(
  state: RegistrationResult,
  formData: FormData
): Promise<RegistrationResult> {
  // FormData se raw data nikalo
  const rawData = {
    email: formData.get("email") as string,
    password: formData.get("password") as string,
  };

  // 👇 ZOD SE VALIDATE KARO
  const result = registrationSchema.safeParse(rawData);

  // `safeParse` kya return karta hai?
  // Yeh ek object return karta hai jo do tarah ka ho sakta hai:
  //
  // 1. Agar validation SUCCESS:
  //    { success: true, data: { email: "...", password: "..." } }
  //
  // 2. Agar validation FAILED:
  //    { success: false, error: ZodError }
  //
  // ZodError mein `issues` array hota hai:
  //    { issues: [{ message: "Invalid email format" }, ...] }

  // Agar validation failed
  if (!result.success) {
    // result.error.issues[0].message se pehla error nikalo
    return {
      error: result.error.issues[0].message,
    };
  }

  // Agar validation passed — data extract karo
  // result.data ab safe hai — validated and typed
  const { email, password } = result.data;

  try {
    const hashedPassword = await hashPassword(password);

    const newUser = await prisma.user.create({
      data: {
        email,
        password: hashedPassword,
        isVerified: false,
      },
    });

    console.log("User created successfully:", newUser.id);
    return { success: true };
  } catch (error) {
    if (
      typeof error === "object" &&
      error !== null &&
      "code" in error &&
      (error as { code: string }).code === "P2002"
    ) {
      return { error: "Email already exists" };
    }

    console.log("Error occurred:", error);
    return { error: "Something went wrong. Please try again." };
  }
}
```

**Key changes:**

1. `registrationSchema` import kiya
2. `safeParse` se validate kiya
3. Agar validation failed → error return
4. Agar validation passed → `result.data` se clean data liya

### Step 4: Frontend Mein Bhi Same Schema Use Karo

Update `app/(auth)/register/page.tsx`:

```tsx
// app/(auth)/register/page.tsx

"use client";

import { useActionState } from "react";
import { registerUser } from "@/app/actions/auth";
import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import { registrationSchema, RegistrationInput } from "@/lib/validations/auth"; // 👈 Import same schema

export default function RegisterPage() {
  const [state, formAction, isPending] = useActionState(registerUser, {
    error: "",
    success: false,
  });

  const {
    register,
    handleSubmit,
    formState: { errors },
  } = useForm<RegistrationInput>({
    resolver: zodResolver(registrationSchema), // 👈 Same schema use
  });

  const onSubmit = async (data: RegistrationInput) => {
    const formData = new FormData();
    formData.append("email", data.email);
    formData.append("password", data.password);

    startTransition(() => {
      formAction(formData);
    });
  };

  return (
    <div>
      <h1>Register</h1>

      {state.error && (
        <div style={{ color: "red", marginBottom: "10px" }}>{state.error}</div>
      )}

      {state.success && (
        <div style={{ color: "green", marginBottom: "10px" }}>
          Registration successful! Please login.
        </div>
      )}

      <form onSubmit={handleSubmit(onSubmit)}>
        <div>
          <label htmlFor="email">Email:</label>
          <input type="email" id="email" {...register("email")} />
          {errors.email && (
            <span style={{ color: "red" }}>{errors.email.message}</span>
          )}
        </div>

        <div>
          <label htmlFor="password">Password:</label>
          <input type="password" id="password" {...register("password")} />
          {errors.password && (
            <span style={{ color: "red" }}>{errors.password.message}</span>
          )}
        </div>

        <button type="submit" disabled={isPending}>
          {isPending ? "Registering..." : "Register"}
        </button>
      </form>
    </div>
  );
}
```

**Key changes:**

1. `registrationSchema` import kiya from `lib/validations/auth`
2. `RegistrationInput` type use kiya
3. Same schema `zodResolver` mein use kiya

## Key Benefits

| Before (Doc 2.10)                        | After (Doc 2.11)                          |
| ---------------------------------------- | ----------------------------------------- |
| Schema component ke andar tha            | Schema `lib/validations/auth.ts` mein hai |
| Frontend aur backend alag-alag schemas   | Same schema dono jagah                    |
| Server-side validation nahi tha          | Server-side bhi validate hota hai         |
| Code duplication                         | Single source of truth                    |
| Postman se invalid data save ho jata tha | Ab validate hota hai                      |

## Summary

**Aaj humne kya seekha:**

1. **Shared validation schema:**
   - `lib/validations/auth.ts` mein schema define kiya
   - Frontend aur backend dono mein import kiya
   - Single source of truth banaya

2. **Server-side validation:**
   - `safeParse` se validate kiya
   - Invalid data ko reject kiya
   - Better error messages diye

3. **Security improve hui:**
   - Ab Postman se bhi invalid data nahi ja sakta
   - Server hamesha validate karta hai
   - Client-side sirf UX ke liye hai

**Ab registration fully secure hai!** Client-side validation (UX) + Server-side validation (Security) dono kaam kar rahe hain.
