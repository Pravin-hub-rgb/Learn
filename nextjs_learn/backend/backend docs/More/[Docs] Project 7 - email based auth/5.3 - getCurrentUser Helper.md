## Step 1: Solution Kya Hai?

Hum ek common helper function banayenge `lib/session.ts` mein. Usme saare checks dalenge ek baar. Phir har page par bas is function ko call karo.

- Ek jagah code
- Har jagah use
- Kal agar change karna hai toh ek jagah edit karo
- Sab jagah automatically update ho jayega

## Step 2: Helper Function Banaye Step by Step

Hum function ko step by step banayenge har cheez explain karte hue.

---

### Step 2.1 Pehle getSession() call karo

```typescript
export async function getCurrentUser() {
  const session = await getSession();
}
```

> 🧠 Kya ho raha hai:
> Hum pehle `getSession()` ko call kar rahe hai.
> Yeh woh hi function hai jo humne phase 4.9 mein banaya tha.
> Woh function already:
> - Cookie se sessionId nikal leta hai
> - Database mein session check karta hai
> - Expiry check karta hai
> - Agar sab sahi hai toh session return karta hai
> - Agar kuch galat hai toh null return karta hai

---

### Step 2.2 Session null check karo

```typescript
export async function getCurrentUser() {
  const session = await getSession();

  if (!session) {
    return null;
  }
}
```

> 🧠 Kya ho raha hai:
> Agar session null hai matlab user login nahi hai.
> Isliye hum bhi direct null return kar dete hai.
> Koi aur check karne ki zaroorat nahi hai.

---

### Step 2.3 Ab actual User fetch karo database se

```typescript
export async function getCurrentUser() {
  const session = await getSession();

  if (!session) {
    return null;
  }

  const user = await prisma.user.findUnique({
    where: { id: session.id },
    select: {
      id: true,
      email: true,
      isVerified: true,
      createdAt: true
    }
  });
}
```

> 🧠 **SABSE IMPORTANT QUESTION: Hum user ko kyun nikal rahe hai jab session mein already user hai???**
>
> ✅ Jawab:
> `getSession()` return karta hai sirf session object. Usme sirf userId hai.
> Hamein actual user ka data chahiye: email, name, etc.
> Isliye hum session ke userId se actual user ko database se nikal rahe hai.
>
> Lekin yaha 1 aur cheez hai:
> Agar user ke account ko delete kar diya gaya hai lekin session abhi bhi exist karta hai.
> Isliye hum user ko manually fetch karte hai. Agar user nahi milta toh null return karo.

---

### ✅ `select` kya hai? Kyun use karte hai?

> 🧠 Bahut important concept hai:
>
> Jab tum `prisma.user.findUnique()` call karte ho toh by default saare fields return karta hai.
> Usme `password` field bhi hota hai!
>
> Agar tum `select` nahi lagao toh tumhara user object mein hashed password bhi aayega.
> Hum usko kabhi bhi nahi chahiye. Kabhi bhi nahi.
>
> Isliye `select` ka use karo:
> - Sirf wahi fields le lo jo tumhe chahiye
> - Password field kabhi bhi mat lo
> - Required fields hi select karo hamesha
>
> ✅ `select: { id: true, email: true }`
> Matlab sirf id aur email hi aayega, password nahi.
>
> ❌ Agar tum select nahi lagao:
> User object mein password bhi aa jayega. Security risk hai.
>
> ✅ Isliye hamesha `select` use karo.

---

### Step 2.4 User return karo

```typescript
export async function getCurrentUser() {
  const session = await getSession();

  if (!session) {
    return null;
  }

  const user = await prisma.user.findUnique({
    where: { id: session.id },
    select: {
      id: true,
      email: true,
      isVerified: true,
      createdAt: true
    }
  });

  return user;
}
```

> 🧠 Kya ho raha hai:
> Ab hum user ko return kar dete hai.
> Ab jisne bhi is function ko call kiya hai usko seedha user mil jayega.
> Usko session ya cookie ke baare mein kuch nahi sochna hai.

## Step 3: Ab Dashboard Page Mein Use Karo

✅ Ab `app/(protected)/dashboard/page.tsx` ko update karo:

```tsx
import { getCurrentUser } from "@/lib/session";
import { redirect } from "next/navigation";

export default async function DashboardPage() {
  const user = await getCurrentUser();

  if (!user) {
    redirect("/login");
  }

  return (
    <div>
      <h1>Dashboard</h1>
      <p>Yeh sirf logged in user hi dekh sakta hai!</p>
      <p>Welcome: {user.email}</p>
    </div>
  );
}
```

✅ Ab har naye protected page par bas yeh 3 line likho:
```tsx
const user = await getCurrentUser();
if (!user) redirect("/login");
```
