# Doc 2.8 — Duplicate Email Check + Error Handling (Error-First Approach)

## Abhi Tak Kya Hua

- **Doc 2.1 mein** humne `User` model define kiya — `schema.prisma` mein
- **Doc 2.2 mein** humne migration run ki — `users` table database mein ban gayi
- **Doc 2.3.1 se 2.3.4 tak** humne `lib/prisma.ts` complete setup kiya
- **Doc 2.4 mein** humne `lib/password.ts` banaya — password hashing ke liye
- **Doc 2.5 mein** humne `app/actions/auth.ts` ka basic structure banaya — sirf `console.log` tak
- **Doc 2.6 mein** humne `(auth)/register/page.tsx` banaya — simple HTML form
- **Doc 2.7 mein** humne password hash kiya aur user database mein save kiya
- **Problem notice kiya:** Same email se multiple users create ho rahe hain!

## Broader Goal

**Aaj kya karenge:**

- Duplicate email check karenge — `prisma.user.findUnique` use karke
- **Error-First Approach:** Prisma ka `P2002` error naturally aane denge — phir handle karenge
- Proper error messages dikhayenge — `{ error: "Email already exists" }`
- Success case bhi handle karenge — `{ success: true }`

**Error-First Approach kya hai?**

- Pehle error aane do — naturally
- Phir dekho error kyun aayi
- Phir usko handle karo
- Fake errors mat banao — jo actually aati hai woh dikhao

## Step-by-Step Implementation (Natural Coder Flow)

### Step 1: Pichla Code Dekho (Doc 2.7)

Abhi humara `registerUser()` function aisa hai:

```typescript
// app/actions/auth.ts

"use server";

import prisma from "@/lib/prisma";
import { hashPassword } from "@/lib/password";

export async function registerUser(formData: FormData) {
  console.log("Registration attempt");
  
  // FormData se values nikalo
  const email = formData.get("email") as string;
  const password = formData.get("password") as string;
  
  console.log("Email:", email);
  console.log("Password:", password);
  
  // Password hash karo
  const hashedPassword = await hashPassword(password);
  console.log("Password hashed successfully");
  
  // User database mein save karo
  const newUser = await prisma.user.create({
    data: {
      email,
      password: hashedPassword,
      isVerified: false,
    },
  });
  
  console.log("User created successfully:", newUser.id);
}
```

*"Yeh code kaam kar raha hai — user create ho rahe hain. Lekin ek problem hai..."*

### Step 3: Error-First Approach — Pehle Error Aane Do

Chalo dekhte hain ki kya hota hai jab hum duplicate email se register karte hain:

1. Browser mein jao: `http://localhost:3000/register`
2. Wahi email daalo jo pehle use ki thi: `test@example.com`
3. Password daalo: `password123`
4. Register button click karo
5. Console check karo

**Expected error in console:**
```
Registration attempt
Email: test@example.com
Password: password123
Password hashed successfully
Error: 
Invalid `prisma.user.create()` invocation in...

Unique constraint failed on the fields: (`User`.`email`)
```

*"Perfect! Prisma ne error phenk di — `Unique constraint failed on the fields: (User.email)`"*

### Step 4: Error Ko Samjho

**Error kyun aayi?**

- Humne `schema.prisma` mein `email` field ko `@unique` mark kiya tha (Doc 2.1 mein)
- Database mein `users` table pe `UNIQUE` constraint hai
- Jab hum same email se doosra user create karte hain, database error deta hai
- Prisma woh error hum tak pahunchata hai

**Error message:**
```
Unique constraint failed on the fields: (`User`.`email`)
```

*"Yeh error expected hai — humein isko handle karna chahiye."*

### Step 6: Error Type Check Karo (TypeScript Safe Way)

Prisma errors ko identify karne ke liye, humein error code check karna padega. Lekin TypeScript mein catch block ka error `unknown` type ka hota hai, isliye humein proper type guard use karna padta hai:

```typescript
// app/actions/auth.ts

"use server";

import prisma from "@/lib/prisma";
import { hashPassword } from "@/lib/password";

export async function registerUser(formData: FormData): Promise<void> {
  console.log("Registration attempt");
  
  // FormData se values nikalo
  const email = formData.get("email") as string;
  const password = formData.get("password") as string;
  
  console.log("Email:", email);
  console.log("Password:", password);
  
  try {
    // Password hash karo
    const hashedPassword = await hashPassword(password);
    console.log("Password hashed successfully");
    
    // User database mein save karo
    const newUser = await prisma.user.create({
      data: {
        email,
        password: hashedPassword,
        isVerified: false,
      },
    });
    
    console.log("User created successfully:", newUser.id);
    
  } catch (error) {
    // TypeScript safe type guard — error unknown type se object type mein check karo
    if (
      typeof error === "object" &&
      error !== null &&
      "code" in error &&
      (error as { code: string }).code === "P2002"
    ) {
      // P2002 = Unique constraint failed (duplicate email)
      console.log("Error: Email already exists");
      return; // void return - form action expects void
    }
    
    console.log("Error occurred:", error);
    return; // void return
  }
}
```

*"Ab hum error type check kar rahe hain — TypeScript safe tarike se. `P2002` code ka matlab hai — unique constraint failed."*

**Important Note:** Hum abhi `void` return kar rahe hain kyunki form `action` directly use kar rahe hain. Form action expects `Promise<void>`, isliye hum object return nahi kar sakte. Error messages abhi console mein dikh rahe hain — UI mein dikhane ke liye Doc 2.9 mein `useActionState` hook use karenge.

**Type Guard Explanation:**

```typescript
if (
  typeof error === "object" &&    // Check karo ki error object hai
  error !== null &&               // Null nahi hai
  "code" in error &&              // 'code' property exist karti hai
  (error as { code: string }).code === "P2002"  // Code 'P2002' hai
) {
  // Yeh Prisma P2002 error hai
  console.log("Error: Email already exists");
  return; // void return
}
```

Yeh approach type-safe hai — TypeScript ko pata hai ki hum `error.code` access kar rahe hain.

## Common Issues & Solutions

### Issue 1: `error.code` Undefined Hai

**Error:**
```
Property 'code' does not exist on type 'Error'
```

**Reason:** TypeScript ko nahi pata ki yeh Prisma error hai

**Solution:**
```typescript
// Type assertion use karo
if ((error as any).code === 'P2002') {
  // ...
}

// Ya Prisma error type import karo
import { Prisma } from '@prisma/client';

if (error instanceof Prisma.PrismaClientKnownRequestError && error.code === 'P2002') {
  // ...
}
```

### Issue 2: TypeScript Form Action Error

**Error:**
```
Type '(formData: FormData) => Promise<{ error: string; } | { success: true; userId: string; }>' is not assignable to type '(formData: FormData) => void | Promise<void>'.
```

**Reason:** Form action expects `void` return type

**Solution:**
```typescript
// Return type `void` rakho
export async function registerUser(formData: FormData): Promise<void> {
  // ...
  return; // void return
}
```

### Issue 3: Error Message English Mein Hai

**Error:**
```
Unique constraint failed on the fields: (`User`.`email`)
```

**Reason:** Prisma default English error deta hai

**Solution:** Hum khud user-friendly message return karte hain:
```typescript
if (error.code === 'P2002') {
  return { error: "Email already exists" }; // Hindi/English mix
}
```
