# Doc 2.9 — UI mein error/success messages: `useActionState` Hook

## Abhi Tak Kya Hua

- **Doc 2.1-2.6** tak humne registration ka basic flow banaya
- **Doc 2.7** mein humne password hash kiya aur user save kiya
- **Doc 2.8** mein humne duplicate email error ko handle kiya — `try-catch` aur P2002 error check
- **Problem notice kiya:** Error messages sirf console mein dikh rahe hain, UI mein nahi!

## Broader Goal

**Aaj kya karenge:**

- `useActionState` hook seekhenge — basics se
- Server Action ke response ko UI mein kaise dikhayein
- Error aur success messages form ke neeche dikhayenge
- User-friendly feedback denge

**Problem jo hum solve karenge:**
- Abhi form submit karne ke baad user ko pata nahi chalta ki registration successful hua ya failed
- Sirf console mein logs dikhte hain
- Hum chahte hain ki user ko turant pata chal jaye

*Abhi toh bas console mein logs dikhte hain. User toh browser mein form submit karta hai aur wait karta hai — kuch hua ya nahi?"*

### Step 3: Solution Socho — "Response Kaise Handle Karein?"

*"Humein ek aisa tarika chahiye jo:*

1. *Form submit hone pe Server Action call kare*
2. *Action ka response capture kare*
3. *Response ko UI mein dikha sake*
4. *Error ya success message show kare*

*Yeh kaam karta hai — `useActionState` hook!*

### Step 4: `useActionState` Hook Kya Hai?

**Simple bhasha mein:**
- `useActionState` ek React hook hai
- Jo Server Action ke result ko state mein store karta hai
- Aur uss state ko hum UI mein dikha sakte hain

**Syntax:**
```tsx
const [state, formAction, isPending] = useActionState(action, initialState);
```

**Matlab:**
- `state` — Server Action ka response (jaise `{ error: "Email exists" }`)
- `formAction` — Form mein use karne ke liye action
- `isPending` — Boolean, batata hai ki action abhi chal raha hai ya nahi

---

### Step 4.1: `useActionState` — Wrapper Kaise Kaam Karta Hai?

**Pehle (bina `useActionState`):**

```
Form Submit → registerUser(formData) → void (kuch nahi return)
                                      ↓
                              Bas kaam karta tha
                              Result kisi ko nahi pata
```

**Ab (`useActionState` ke saath):**

```
Form Submit → useActionState (wrapper) → registerUser(state, formData)
                                              ↓
                                         Kaam karta hai
                                              ↓
                                         Result return karta hai
                                              ↓
                                       useActionState state update karta hai
                                              ↓
                                         UI automatically re-render hota hai
```

**`useActionState` bas ek wrapper hai jo:**
1. Form submit hone pe `registerUser` ko call karta hai
2. `registerUser` ka result leta hai
3. State update karta hai
4. UI ko batata hai ki re-render kare

---

### Step 4.2: `state` Kya Hai? (Form Ki Current Situation)

**`state` hai form ka current situation:**

```typescript
// Shuru mein (initial state)
{
  error: "",      // Koi error nahi
  success: false  // Success nahi hua
}

// Agar email already exists error aaya
{
  error: "Email already exists",  // Error aa gaya
  success: false                  // Success nahi hua
}

// Agar registration successful hua
{
  error: "",       // Koi error nahi
  success: true    // Success ho gaya!
}
```

---

### Step 4.3: `isPending` Kya Hai? (Loading State)

**`isPending` batata hai ki form submit ho raha hai ya nahi:**

```typescript
isPending = false  // Form submit nahi ho raha (idle)
isPending = true   // Form submit ho raha hai (loading)
```

---

### Step 4.4: Complete Timeline

```
1. Page load hua
   → state: { error: "", success: false }
   → isPending: false
   → UI: Form dikha, button "Register"

2. User ne form submit kiya
   → isPending: true (loading shuru)
   → UI: Button "Registering..." ho gaya

3. registerUser function chala (database mein save hua)
   → isPending: false (loading khatam)
   
   Agar error aaya:
   → state: { error: "Email exists", success: false }
   → UI: Error message dikha
   
   Agar success hua:
   → state: { error: "", success: true }
   → UI: Success message dikha
```

**Visual timeline:**

```
Time →

[Page Load]
  state: { error: "", success: false }
  isPending: false
  UI: Form + "Register" button

[User clicks Submit]
  isPending: true
  UI: Form + "Registering..." button (disabled)

[registerUser complete]
  isPending: false
  
  Case 1: Error
    state: { error: "Email exists", success: false }
    UI: Error message + Form + "Register" button
  
  Case 2: Success
    state: { error: "", success: true }
    UI: Success message + Form + "Register" button
```

### Step 6: Hook Use Karo

```tsx
// app/(auth)/register/page.tsx

"use client";

import { useActionState } from "react";
import { registerUser } from "@/app/actions/auth";

export default function RegisterPage() {
  // useActionState use karo
  const [state, formAction, isPending] = useActionState(registerUser, {
    error: "",
    success: false,
  });

  return (
    <div>
      <h1>Register</h1>
      <form action={formAction}>
        <input type="email" name="email" placeholder="Email" required />
        <input type="password" name="password" placeholder="Password" required />
        <button type="submit" disabled={isPending}>
          {isPending ? "Registering..." : "Register"}
        </button>
      </form>
    </div>
  );
}
```

*"Ab humare paas `state` hai jo Server Action ka response store karega!"*

### Step 7: Server Action Ko Response Return Karne Do (IMPORTANT CHANGE!)

**Yeh step dhyan se padhna — yahan ek important change ho raha hai!**

#### Pehle Kya Tha (Doc 2.8 tak)?

Doc 2.8 tak humara `registerUser` function aisa tha:

```typescript
// Pehle (Doc 2.8)
export async function registerUser(formData: FormData): Promise<void> {
  // ... logic ...
  return; // void return - kuch return nahi karta tha
}
```

**Kyun?** Kyunki hum directly form `action={registerUser}` use kar rahe the, aur form action expects `void` return.

#### Ab Kya Ho Raha Hai (Doc 2.9)?

Ab hum `useActionState` hook use kar rahe hain. Yeh hook action function se **do cheezein expect karta hai**:

Haan bhai, bilkul sahi! **"action function" = `registerUser`** jo humne banaya hai.

**Simple:**

```typescript
// Yeh hai "action function"
export async function registerUser(state, formData) {
  // ... kaam ...
  return { success: true }; // ya { error: "..." }
}

// Aur useActionState usko wrap karta hai
const [state, formAction, isPending] = useActionState(registerUser, initialState);
```

**Matlab:**
- `registerUser` = humara Server Action function
- `useActionState` = wrapper jo `registerUser` ko call karta hai
- `useActionState` expect karta hai ki `registerUser` do parameters le:
  1. `state` (current state)
  2. `formData` (form data)

**"action function"** bas ek fancy naam hai Server Action function ke liye jo `useActionState` ke saath use hota hai.

Samajh aa gaya? "action function" = `registerUser` hi hai!

1. **Pehla parameter:** `state` — current state (RegistrationResult)
2. **Doosra parameter:** `formData` — form data (FormData)
3. **Return:** Naya state (RegistrationResult)



**Kyun change hua?**

`useActionState` hook ko pata hona chahiye:
- Current state kya hai (taaki update kar sake)
- Form data kya hai (taaki process kar sake)
- Result kya hua (taaki UI update kar sake)

**Simple analogy:**

```
Pehle (bina useActionState):
  Form Submit → registerUser(formData) → void (kuch nahi return)
  Result: Kuch nahi hota, bas console mein log

Ab (useActionState ke saath):
  Form Submit → useActionState → registerUser(currentState, formData) → nayaState
  Result: State update hota hai, UI re-render hota hai
```

#### New Code:

```typescript
// app/actions/auth.ts

"use server";

import prisma from "@/lib/prisma";
import { hashPassword } from "@/lib/password";

export type RegistrationResult = {
  error?: string;
  success?: boolean;
};

// AB: Do parameters - state aur formData
export async function registerUser(
  state: RegistrationResult,    // Current state (hum ignore karenge)
  formData: FormData            // Form data
): Promise<RegistrationResult> {
  console.log("Registration attempt");
  
  const email = formData.get("email") as string;
  const password = formData.get("password") as string;
  
  try {
    const hashedPassword = await hashPassword(password);
    
    const newUser = await prisma.user.create({
      data: {
        email,
        password: hashedPassword,
        isVerified: false,
      },
    });
    
    console.log("User created successfully:", newUser.id);
    return { success: true }; // Naya state return karo
    
  } catch (error) {
    if (
      typeof error === "object" &&
      error !== null &&
      "code" in error &&
      (error as { code: string }).code === "P2002"
    ) {
      return { error: "Email already exists" }; // Naya state return karo
    }
    
    console.log("Error occurred:", error);
    return { error: "Something went wrong. Please try again." }; // Naya state return karo
  }
}
```

**Key Points:**

1. **Function signature change:** `(state, formData) => result`
2. **`state` parameter:** Hum isko use nahi kar rahe, lekin `useActionState` ke liye zaroori hai
3. **Return type:** Ab `RegistrationResult` return karna padega, `void` nahi

*"Ab action proper response return kar raha hai — `useActionState` usko capture kar lega!"*

### Step 8: UI Mein Error/Success Message Dikhao

```tsx
// app/(auth)/register/page.tsx

"use client";

import { useActionState } from "react";
import { registerUser } from "@/app/actions/auth";

export default function RegisterPage() {
  const [state, formAction, isPending] = useActionState(registerUser, {
    error: "",
    success: false,
  });

  return (
    <div>
      <h1>Register</h1>
      
      {/* Error message */}
      {state.error && (
        <div style={{ color: "red", marginBottom: "10px" }}>
          {state.error}
        </div>
      )}
      
      {/* Success message */}
      {state.success && (
        <div style={{ color: "green", marginBottom: "10px" }}>
          Registration successful! Please login.
        </div>
      )}
      
      <form action={formAction}>
        <div>
          <label htmlFor="email">Email:</label>
          <input type="email" id="email" name="email" required />
        </div>
        <div>
          <label htmlFor="password">Password:</label>
          <input type="password" id="password" name="password" required />
        </div>
        <button type="submit" disabled={isPending}>
          {isPending ? "Registering..." : "Register"}
        </button>
      </form>
    </div>
  );
}
```

*"Ab user ko turant pata chal jayega ki registration successful hua ya error aaya!"*

## Common Issues & Solutions

### Issue 1: `"use client"` Missing

**Error:**
```
Error: useActionState is not defined
```

**Reason:** `useActionState` ek React hook hai — client component mein hona chahiye

**Solution:**
```tsx
// File ke top pe
"use client";

import { useActionState } from "react";
```

### Issue 2: Server Action Returns `void`

**Error:**
```
Type 'void' is not assignable to type 'RegistrationResult'
```

**Reason:** Server Action abhi bhi `void` return kar raha hai

**Solution:** Action ko proper object return karne do:
```typescript
export async function registerUser(formData: FormData): Promise<RegistrationResult> {
  // ...
  return { success: true }; // ya { error: "..." }
}
```

### Issue 3: State Initial Value Galat Hai

**Error:**
```
Type '{ error: string; success: boolean; }' is not assignable to type 'never'
```

**Reason:** Initial state ka type match nahi kar raha

**Solution:**
```tsx
const [state, formAction, isPending] = useActionState(registerUser, {
  error: "",
  success: false,
} as RegistrationResult);
```
