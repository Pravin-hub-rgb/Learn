# Doc 2.1 — User Model

## Abhi Tak Kya Hua
- **Phase 1 mein** humne Prisma v7 setup kiya — packages install kiye, `prisma init` chalaya, database connect kiya
- `npx prisma db pull` chalaya — error aaya "P4001 introspected database was empty" — yeh expected tha
- Database connection verify ho gaya — config sahi kaam kar rahi hai
- Ab humare paas empty database hai — koi tables nahi hain

## Broader Goal
**Aaj kya karenge:**
- Pehla model banayenge — `User` model
- `User` model database mein `users` table banayega
- Har user ka data is table mein store hoga

**Kyun yeh fields chune:**
- `id` — har user ka unique identifier (har auth system mein hota hai)
- `email` — login ke liye use hoga (username ki jagah email use karenge)
- `password` — authentication ke liye (hash karke store karenge)
- `createdAt` — kab user register hua track karne ke liye
- `isVerified` — email verification ke liye (Phase 6 mein use hoga)

**`isVerified` kyun zaroori hai?**
- Socho: 100 log signup karte hain → sab database mein add ho jate hain
- Lekin kuch log fake email use kar rahe hain — woh actually exist hi nahi karte
- Agar hum un sab ko allow karenge, toh database mein faltu ke entries store ho jayengi
- **Solution:** Email verification — user ko verification email bhejo, woh click kare, tabhi `isVerified = true` ho
- Sirf verified users ko login allow karenge — fake accounts filter ho jayenge
- Isliye `isVerified` field zaroori hai — security aur data quality ke liye

## Prisma Schema File Structure

`schema.prisma` file ke **3 main parts** hote hain:

### 1. Generator Block
- Prisma client kaise generate hoga define karta hai
- Example:
```prisma
generator client {
  provider = "prisma-client"
}
```

### 2. Datasource Block
- Database connection define karta hai
- Example:
```prisma
datasource db {
  provider = "postgresql"
}
```

### 3. Model Block
- Database tables define karte hain
- **Keyword:** `model` use karte hain
- Example:
```prisma
model User {
  id    String @id @default(cuid())
  email String @unique
}
```

## Step-by-Step Implementation

### Step 1: `schema.prisma` kholo

`prisma/schema.prisma` file kholo — yeh file `prisma/` folder mein honi chahiye:

```prisma
generator client {
  provider = "prisma-client"
}

datasource db {
  provider = "postgresql"
}
```

**Abhi ke liye yeh file mein sirf generator aur datasource hai — koi model nahi hai.**

### Step 2: User model define karo

Model define karne ke liye **`model` keyword** use karte hain:

```prisma
model User {
  id        String   @id @default(cuid())
  email     String   @unique
  password  String
  isVerified Boolean @default(false)
  createdAt DateTime @default(now())
}
```

**Yeh model kya karta hai:**
- Database mein `users` table banayega
- Har row ek user ko represent karega

**Har field kyun chahiye:**
- `id` — har user ka unique identifier (primary key)
- `email` — login ke liye use hoga (unique hona chahiye)
- `password` — authentication ke liye (hash karke store karenge)
- `isVerified` — email verification ke liye (Phase 6 mein use hoga) — fake accounts filter karne ke liye
- `createdAt` — registration timestamp track karne ke liye

### Step 3: Poori `schema.prisma` aisi dikhni chahiye:

```prisma
// 1. Generator Block
generator client {
  provider = "prisma-client"
}

// 2. Datasource Block
datasource db {
  provider = "postgresql"
}

// 3. Model Block
model User {
  id        String   @id @default(cuid())
  email     String   @unique
  password  String
  isVerified Boolean @default(false)
  createdAt DateTime @default(now())
}
```

### Step 4: File save karo

`Ctrl+S` ya `Cmd+S` se file save karo.

## Model Fields Explained

| Field | Type | Attributes | Example Value |
|-------|------|------------|---------------|
| `id` | `String` | `@id @default(cuid())` | `"clx123abc"` (CUID format) |
| `email` | `String` | `@unique` | `"user@example.com"` |
| `password` | `String` | — | `"$2a$10$..."` (hashed) |
| `isVerified` | `Boolean` | `@default(false)` | `false` ya `true` |
| `createdAt` | `DateTime` | `@default(now())` | `"2024-01-01T12:00:00Z"` |

**CUID kya hai?**
- **CUID** = Collision-resistant Unique Identifier
- Example: `"clx123abc"`, `"cly456def"`
- Automatically generate hota hai
- Unique hota hai — duplicate nahi ho sakta

## Common Issues & Solutions

### Issue 1: `@unique` attribute nahi hai
**Reason:** Bhool gaye add karna
**Solution:** 
```prisma
email String @unique  // ← @unique add karo
```

### Issue 2: `@id @default(cuid())` syntax galat hai
**Reason:** Spaces ya brackets galat hain
**Solution:** 
```prisma
id String @id @default(cuid())  // ← Exact yeh syntax
```

### Issue 3: `model` keyword nahi likha
**Reason:** Model define karne ke liye `model` keyword zaroori hai
**Solution:** 
```prisma
model User {  // ← "model" keyword use karo
  id String @id @default(cuid())
}
```
