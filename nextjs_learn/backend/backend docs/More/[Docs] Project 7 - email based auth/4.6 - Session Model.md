## Step 2: Session Model Add Karo

```prisma
model Session {
  id        String   @id @default(cuid())
  userId    String
  user      User     @relation(fields: [userId], references: [id], onDelete: Cascade)
  expiresAt DateTime
  createdAt DateTime @default(now())
}
```

> 🧠 Har field ka kya kaam hai aur hume kya fayda hoga isse:
>
> ✅ `id` → Random unique 25 character long CUID. 
> Yeh woh ID hai jo hum browser ko cookie mein bhejenge. 
> Isse kya hoga: 
> - Kabhi koi guess nahi kar sakta
> - Koi isko edit karke dusre user ke session nahi le sakta
> - Har login par nayi alag ID milegi
>
> ✅ `userId` → User ka original ID jiska yeh session hai
> Hum database mein is ID se user find karenge har request par
> Is field ko kabhi bhi browser nahi bhejenge, sirf database mein store hai
>
> ✅ `user` → Prisma relation User table se
> Isse hum easily `session.user` karke poora user nikal sakte hai
>
> ✅ `onDelete: Cascade` → Bahut important!
> Agar future mein tum user ko delete karoge toh uske saare login sessions bhi apne aap delete ho jayenge
> Koi bakwaas stale session nahi bachenge database mein
>
> ✅ `expiresAt` → Session kab mar jayega
> Hum 7 din baad expire karenge session
> Agar user 7 din se zyada nahi aaya toh usko wapas login karna padega
>
> ✅ `createdAt` → Kab login kiya tha is user ne
> Tum isse dekh sakte ho ki user kab aaya tha akhiri baar

✅ **Important Note:**
Hum `expiresAt` field isliye banate hai kyunki har session ka ek fixed time hota hai. Uske baad woh session automatically invalid ho jata hai.

## Step 3: `prisma format` Chalao (Important!)

*"Migration se pehle, schema ko format karna zaroori hai."*

**Kyun?** Prisma ko relation validate karne ke liye dono sides par field chahiye. Humne Session mein User field add kiya, lekin User model mein opposite relation field nahi tha.

```bash
npx prisma format
```

✅ **Kya hoga is command se:**
- Prisma ne automatically User model mein `sessions Session[]` field add kar diya
- Ab relation dono sides se complete hai
- User model ab is tarah dikhega:
```prisma
model User {
  id                 String              @id @default(cuid())
  email              String              @unique
  password           String
  isVerified         Boolean             @default(false)
  createdAt          DateTime            @default(now())
  verificationTokens VerificationToken[]
  sessions           Session[]  // ← Automatically added!
}
```

> Yeh bilkul same tareeke se hua tha jab tumne VerificationToken banaya tha. Us time bhi Prisma ne apne aap `verificationTokens` add kiya tha User model mein.

## Step 4: Migration Generate Karo

✅ Ab terminal mein ye command chalao:
```bash
npx prisma migrate dev --name add-session-model
```

✅ Kya hoga is command se:
1.  Prisma database mein Session table banayega
2.  Migration file create hoga `prisma/migrations` folder mein

## Step 4.1: Prisma Client Generate Karo

✅ Ab ye command jarur chalao:
```bash
npx prisma generate
```

> 🧠 Yeh command kyun zaroori hai:
> Jab bhi tum naya model add karte ho, Prisma Client ke types update nahi hote automatically.
> `prisma generate` command naye models ke TypeScript types regenerate karta hai.
> Iske baad tum `prisma.session` use kar sakte ho, TypeScript error nahi aayega.

✅ Ab database mein ek nayi table `Session` ban chuki hai.
