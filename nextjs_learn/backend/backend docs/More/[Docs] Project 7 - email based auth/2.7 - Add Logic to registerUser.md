### Step 5: User Database Mein Save Karo

```typescript
// app/actions/auth.ts

"use server";

import prisma from "@/lib/prisma";
import { hashPassword } from "@/lib/password";

export async function registerUser(formData: FormData) {
  console.log("Registration attempt");
  
  // FormData se values nikalo
  const email = formData.get("email") as string;
  const password = formData.get("password") as string;
  
  console.log("Email:", email);
  console.log("Password:", password);
  
  // Password hash karo
  const hashedPassword = await hashPassword(password);
  console.log("Password hashed successfully");
  
  // User database mein save karo
  const newUser = await prisma.user.create({
    data: {
      email,
      password: hashedPassword,
      isVerified: false, // Abhi verify nahi hua
    },
  });
  
  console.log("User created successfully:", newUser.id);
}
```

*"Excellent! User database mein save ho gaya. Ab test karte hain — form submit karo aur database check karo."*

## Common Issues & Solutions

### Issue 1: `prisma.user.findUnique` Error

**Error:**
```
TypeError: Cannot read properties of undefined (reading 'findUnique')
```

**Reason:** Prisma client properly generate nahi hua

**Solution:**
```bash
# Prisma client regenerate karo
npx prisma generate
```

### Issue 2: Email Already Exists Error (Expected)

**Error:**
```
Unique constraint failed on the fields: (`User`.`email`)
```

**Reason:** Same email se pehle hi koi register ho chuka hai

**Solution:** Yeh expected behavior hai — hum isko handle karenge (Doc 2.8/2.9 mein).

### Issue 3: Database Connection Error

**Error:**
```
Error: connect ECONNREFUSED 127.0.0.1:5432
```

**Reason:** PostgreSQL server nahi chal raha

**Solution:**
```bash
# PostgreSQL start karo
pg_ctl start

# Ya Windows Services mein "PostgreSQL" start karo
```
