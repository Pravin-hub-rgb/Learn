# Doc 1.2 — Prisma Init

## Abhi Tak Kya Hua
- **Doc 1.1 mein** humne core Prisma packages install kiye: `@prisma/client`, `@prisma/adapter-pg`, `prisma`
- Ab humare paas basic Prisma setup ready hai.

## Broader Goal
**Aaj kya karenge:**
- `npx prisma init` chalayenge — jisse Prisma ka setup ho jayega
- Isse do files banengi:
  1. `prisma/schema.prisma` — database models yahan define honge
  2. `prisma.config.ts` — Prisma v7 ka configuration file

**Note:** Prisma v7 automatically TypeScript config handle kar leta hai — alag se `tsx` install karne ki zaroorat nahi hai.

---

## Step-by-Step Implementation

### Step 1: Prisma init command chalao

Terminal mein yeh command chalao:

```bash
npx prisma init
```

**Kya hoga:**
- Prisma initialize hoga
- `prisma/schema.prisma` banegi
- `prisma.config.ts` banegi
- `.env` file banegi (agar nahi hai toh)

**Output:**
```
Initialized Prisma in your project

  prisma/
    schema.prisma
  prisma.config.ts
  .env

warn You already have a .gitignore file. Don't forget to add .env in it to not commit any private information.
```

### Step 2: Files check karo

Command run karne ke baad, yeh files banengi:

```
project-root/
├── prisma/
│   ├── schema.prisma      ← Naya bana
│   └── .env               ← Naya bana (DATABASE_URL placeholder ke saath)
├── prisma.config.ts       ← Naya bana (Prisma v7 specific)
├── .env                   ← Already hoga
└── ...
```

### Step 3: `prisma.config.ts` dekho

Naya bana file kholo — `prisma.config.ts`:

```typescript
import "dotenv/config";
import { defineConfig } from "prisma/config";

export default defineConfig({
  schema: "prisma/schema.prisma",
  migrations: {
    path: "prisma/migrations",
  },
  datasource: {
    url: process.env["DATABASE_URL"],
  },
});
```

**Dhyan se dekho:**
- `import "dotenv/config"` — already included hai! Prisma ne automatically add kar diya
- `schema: "prisma/schema.prisma"` — Schema file ka path
- `datasource.url: process.env["DATABASE_URL"]` — Database URL environment variable se aata hai

### Step 4: `schema.prisma` dekho

`prisma/schema.prisma` file kholo:

```prisma
generator client {
  provider = "prisma-client"
}

datasource db {
  provider = "postgresql"
}
```

**Dhyan se dekho:**
- `datasource` mein sirf `provider` hai — `url` nahi hai!
- Prisma v7 mein `url` `prisma.config.ts` mein hota hai — `schema.prisma` mein nahi
- `prisma.config.ts` mein already `datasource.url: process.env["DATABASE_URL"]` set hai

## Common Issues & Solutions

### Issue 1: `prisma.config.ts` nahi bana
**Reason:** `prisma init` sahi se nahi chala
**Solution:** 
- Dobara `npx prisma init` chalao
- Ya manually file bana lo (upar code hai)

### Issue 2: `dotenv/config` import nahi hai
**Reason:** Purana Prisma version hai
**Solution:** 
- Prisma update karo: `npm install -D prisma@latest`
- Ya manually add karo: `import "dotenv/config";`
