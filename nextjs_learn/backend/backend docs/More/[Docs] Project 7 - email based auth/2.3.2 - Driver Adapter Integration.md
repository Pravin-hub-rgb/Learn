# Doc 2.3.2 — Driver Adapter Integration: Database Connection

## Abhi Tak Kya Hua
- **Doc 2.3.1 mein** humne `lib/prisma.ts` file banayi — PrismaClient import kiya aur instance banaya
- VS Code mein error aa raha hai: "Expected 1 arguments, but got 0"
- Prisma v7 mein architecture change ho gaya hai — bina adapter ke PrismaClient kaam nahi karta

## Broader Goal
**Aaj kya karenge:**
- `pg` package install karenge — PostgreSQL driver
- `@prisma/adapter-pg` install karenge — Prisma adapter
- `lib/prisma.ts` update karenge — pool, adapter, aur PrismaClient ke saath
- Tab jaake humara code kaam karega!

---

## Step-by-Step Implementation

### Step 1: Problem Samjho — Adapter Kyun Chahiye?

Humare paas abhi yeh code hai:

```typescript
// lib/prisma.ts
import { PrismaClient } from "../app/generated/prisma/client";

const prisma = new PrismaClient();

export default prisma;
```

VS Code error de raha hai: "Expected 1 arguments, but got 0"

**Kyun?**
- Prisma v7 ne architecture change kar diya
- Ab `PrismaClient` constructor mein ek argument mandatory hai — adapter
- Adapter ke bina PrismaClient database se connect nahi kar sakta
- Adapter database aur Prisma client ke beech mein bridge ka kaam karta hai

### Step 2: Pehle Packages Install Karo

Terminal mein jao aur yeh packages install karo:

```bash
npm install pg @types/pg @prisma/adapter-pg
```

**Kya install hua:**
- `pg` — PostgreSQL driver (JavaScript mein database se connect karne ke liye)
- `@types/pg` — TypeScript types for `pg`
- `@prisma/adapter-pg` — Prisma v7 ka PostgreSQL adapter

### Step 3: Ab Code Likho — Natural Flow Mein

Ab `lib/prisma.ts` file kholo aur code update karo. Main step-by-step batata hoon:

**Soch:** Mujhe database se connect karna hai. Database URL `.env` file mein `DATABASE_URL` mein stored hai. Mujhe usse use karna hai.

**Pehle:** Connection manage karne ke liye pool chahiye. `pg` package mein `Pool` class hai jo connection pool banata hai. Toh pehle `Pool` import karta hoon:

```typescript
import { Pool } from "pg";
```

**Ab:** Pool ka instance banata hoon. Pool ko connection string chahiye — woh `DATABASE_URL` hai:

```typescript
const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
});
```

**Soch:** Ab mujhe Prisma ko database se connect karna hai. Prisma v7 mein adapter use karna padta hai. PostgreSQL ke liye `PrismaPg` adapter hai. Toh `PrismaPg` import karta hoon:

```typescript
import { PrismaPg } from "@prisma/adapter-pg";
```

**Ab:** Adapter banata hoon. Adapter ko pool chahiye:

```typescript
const adapter = new PrismaPg(pool);
```

**Soch:** Ab mujhe PrismaClient banana hai. PrismaClient constructor mein adapter pass karna padta hai:

```typescript
const prisma = new PrismaClient({ adapter });
```

**Last:** Export kar do taaki doosre files use kar sakein:

```typescript
export default prisma;
```

**Poori file aisi dikhegi:**

```typescript
// lib/prisma.ts
import { Pool } from "pg";
import { PrismaClient } from "../app/generated/prisma/client";
import { PrismaPg } from "@prisma/adapter-pg";

// Connection pool banao — DATABASE_URL se
const pool = new Pool({
  connectionString: process.env.DATABASE_URL,
});

// Prisma adapter banao — pool se
const adapter = new PrismaPg(pool);

// PrismaClient banao — adapter ke saath
const prisma = new PrismaClient({ adapter });

// Export kar do
export default prisma;
```


### Step 4: Connection Pool Kya Hai?

**Simple example:**
- Socho tumhare paas ek library hai jahan 100 students hain
- Har student ko padhne ke liye ek table chahiye
- Agar har student ke liye alag table banayenge, toh 100 tables chahiye — bahut expensive!
- Iske bajaye, 10 tables rakho — jab student ko chahiye, use karo, kaam khatam hone ke baad wapas pool mein daal do

**Database Connection Pool bhi wahi karta hai:**
- Database connections limited aur expensive hote hain
- Pool ek set of connections maintain karta hai
- Jab query execute karni hai, pool se connection leta hai
- Query complete hone ke baad, connection wapas pool mein daal deta hai
- Doosri query use kar sakti hai

**Benefits:**
- ✅ Performance improve hoti hai (connection reuse)
- ✅ Resource management efficient hota hai
- ✅ Connection limit exceed nahi hota
- ✅ Scalability improve hoti hai

### Step 6: DATABASE_URL Kahan Se Aata Hai?

`process.env.DATABASE_URL` `.env` file se aata hai jo humne Doc 1.3 mein set kiya tha:

```env
DATABASE_URL="postgresql://postgres:mysecretpassword@localhost:5432/email_auth_db?schema=public"
```

**Connection string breakdown:**
- `postgresql://` — Protocol (PostgreSQL)
- `postgres:mysecretpassword` — Username:Password
- `localhost:5432` — Host:Port
- `email_auth_db` — Database ka naam
- `?schema=public` — Schema name

**Yeh string pool ko batata hai:**
- Kaunse database se connect karna hai
- Kaunse credentials use karne hain
- Kaunse server pe connect karna hai

### Step 7: Test Karo — API Route Banao

Ab code kaam kar raha hai — chalo test karte hain! Ek simple API route banao jo database se connect karke users fetch kare:

**Kya code likhna hai:**
1. `app/api/test-db/route.ts` file banao
2. `NextResponse` import karo
3. `prisma` ko `lib/prisma` se import karo
4. GET handler mein `prisma.user.findMany()` call karo
5. Response return karo

```typescript
// app/api/test-db/route.ts
import { NextResponse } from "next/server";
import prisma from "@/lib/prisma";

export async function GET() {
  try {
    // Database se users fetch karo
    const users = await prisma.user.findMany();
    
    // Success response
    return NextResponse.json({
      success: true,
      message: "Database connection successful",
      users: users,
    });
  } catch (error) {
    // Error response
    return NextResponse.json({
      success: false,
      message: "Database connection failed",
      error: error,
    }, { status: 500 });
  }
}
```

**Ab test karo:**
1. Server start karo — `npm run dev`
2. Browser mein jao — `http://localhost:3000/api/test-db`
3. Response dekho

**Expected response:**
```json
{
  "success": true,
  "message": "Database connection successful",
  "users": []
}
```

**Kya ho raha hai:**
- `prisma.user.findMany()` — `users` table se saare users fetch kar raha hai
- Abhi table empty hai, toh `users: []` aayega
- Agar error aata hai, toh connection mein problem hai

### Step 8: Error Aayega — Hot Reload Issue!

Ab code kaam kar raha hai, lekin development mode mein ek problem aayegi:

**Problem:** Jab tum code change karoge, Next.js hot reload karega. Har hot reload ke time `lib/prisma.ts` dobara execute hoga. Har baar naya `PrismaClient` instance ban jayega.

**Result:**
- 5 hot reloads = 5 different `PrismaClient` instances
- Har instance ka apna connection pool hai
- PostgreSQL ka default connection limit hota hai (usually 100)
- 50 hot reloads = 50 connection pools = 50+ connections
- Error aayega: `Error: too many clients already`

**Solution:** Singleton pattern — woh hum next doc (2.3.3) mein implement karenge.

## Common Issues & Solutions

### Issue 1: `Cannot find module 'pg'`
**Error:**
```
Cannot find module 'pg' or its corresponding type declarations.
```

**Reason:** `pg` package install nahi hua

**Solution:**
```bash
npm install pg @types/pg
```

### Issue 2: `Cannot find module '@prisma/adapter-pg'`
**Error:**
```
Cannot find module '@prisma/adapter-pg'
```

**Reason:** `@prisma/adapter-pg` package install nahi hua

**Solution:**
```bash
npm install @prisma/adapter-pg
```

### Issue 3: `DATABASE_URL is undefined`
**Error:**
```
Error: Environment variable DATABASE_URL is not defined
```

**Reason:** `.env` file load nahi ho rahi ya `DATABASE_URL` set nahi hai

**Solution:**
- `.env` file check karo — `DATABASE_URL` hona chahiye
- `prisma.config.ts` mein `import "dotenv/config"` top pe hai verify karo
- `.env` file root directory mein hai confirm karo

### Issue 4: `ECONNREFUSED` error
**Error:**
```
Error: connect ECONNREFUSED 127.0.0.1:5432
```

**Reason:** PostgreSQL server nahi chal raha

**Solution:**
```bash
# PostgreSQL start karo
pg_ctl start

# Ya Windows Services mein "PostgreSQL" start karo
```

### Issue 5: `28P01` (invalid password)
**Error:**
```
error: 28P01: invalid password
```

**Reason:** `DATABASE_URL` mein password galat hai

**Solution:**
- `.env` file mein `DATABASE_URL` check karo
- PostgreSQL credentials sahi hain verify karo
- Agar password change kiya hai, toh `.env` update karo

### Issue 6: `3D000` (database does not exist)
**Error:**
```
error: 3D000: database "email_auth_db" does not exist
```

**Reason:** Database nahi bana

**Solution:**
```bash
# psql mein login karo
psql -U postgres

# Database banao
CREATE DATABASE email_auth_db;

# Exit
\q
```

### Issue 7: `too many clients already`
**Error:**
```
Error: too many clients already
```

**Reason:** Hot reload ke time multiple instances ban rahe hain

**Solution:** Next doc (2.3.3) mein singleton pattern implement karenge

## Summary
- **Problem:** Prisma v7 mein adapter mandatory hai — bina adapter ke PrismaClient kaam nahi karta
- **Solution:** `pg`, `@types/pg`, aur `@prisma/adapter-pg` install kiye
- **Implementation:**
  - `Pool` banaya — `DATABASE_URL` se connection manage karne ke liye
  - `PrismaPg` adapter banaya — pool ko Prisma se connect karne ke liye
  - `PrismaClient` banaya — adapter ke saath
- **Connection pool benefits:**
  - Performance improve hoti hai (connection reuse)
  - Resource management efficient hota hai
  - Connection limit exceed nahi hota
  - Scalability improve hoti hai
- **Abhi bhi ek problem:** Hot reload ke time multiple instances ban sakte hain
- **Solution:** Singleton pattern — woh hum next doc (2.3.3) mein implement karenge
