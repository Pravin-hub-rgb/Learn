## Step-by-Step Implementation (Natural Coder Flow)

### Step 1: Problem Samjho — "Mujhe User Register Karna Hai"

Soch rahe ho: *"Mere paas database ready hai, password hashing ready hai. Ab mujhe ek aisa function chahiye jo user se email aur password le aur database mein save kare. Yeh function hoga — `registerUser()`*"

### Step 2: Folder Structure Banao

Pehle check karo — `app/actions/` folder hai ya nahi:

```bash
# Check karo
ls app/

# Agar actions folder nahi hai, toh banao
mkdir app/actions
```

### Step 3: Pehla Attempt — Sirf Structure (WITH "use server")

**Important:** Server Action banane ke liye `"use server"` directive sabse pehle zaroori hai. Bina iske, form action kaam nahi karega.

File banao: `app/actions/auth.ts`

```typescript
// app/actions/auth.ts

"use server"; // Sabse pehle yeh directive - bina iske form action kaam nahi karega!

// Pehle sirf structure — kuch nahi, bas function ka shell
export async function registerUser(formData: FormData) {
  console.log("Registration attempt");
  console.log("Form data:", formData);
}
```

*"Bas itna hi — ek empty function jo FormData le raha hai. Ab test karte hain ki yeh kaam kar raha hai ya nahi."*

### Step 4: Test Karne Ka Tarika Socho

*"Abhi ke liye, main manually test kar sakta hoon — ek temporary page bana ke:"*

```typescript

*"Ab browser mein `/test-register` pe jao, form bharo, submit karo, aur console check karo."*

**Expected console output:**
```

*"Great! Form data aa raha hai. Ab isme logic add karenge — lekin woh agle docs mein."*

## Common Issues & Solutions

### Issue 1: `"use server"` Directive Missing

**Error:**
```
Functions cannot be passed directly to Client Components unless you explicitly expose it by marking it with "use server".
```

**Reason:** `"use server"` directive nahi hai

**Solution:**
```typescript
// File ke top pe yeh directive zaroori hai
"use server";

export async function registerUser(formData: FormData) {
  // ...
}
```

### Issue 2: Import Path Error

**Error:**
```
Cannot find module '../actions/auth'
```

**Reason:** Path galat hai ya folder nahi hai

**Solution:**
```bash
# Check karo ki actions folder hai ya nahi
ls app/

# Agar nahi hai, toh banao
mkdir app/actions
```
