# Doc 4.4 — Password Compare

## Abhi Tak Kya Hua (4.3)

- User database mein mil gaya hai
- Zod validation ho chuki hai
- `isVerified` check bhi ho gaya hai
- ✅ **Ab bas ek cheez baaki hai:** Password sahi hai ya nahi

## Aaj Ka Goal (4.4)

**Aaj hum karenge:**
✅ Password compare karenge bcrypt se
✅ Agar password galat hai toh error return karenge

> 🎯 is phase mein itna hi hai. Frontend error display next step mein karenge.

## Step 1: Problem Samjho

**Soch:** "Ab user verified hai. Ab hume dekhna hai ki jo password user ne diya hai woh database mein stored hashed password se match karta hai ya nahi."

**Kyun direct compare nahi kar sakte:**
- Database mein password plain nahi hota
- Humne Phase 2 mein password ko bcrypt se hash kiya tha store karte time
- Hash ko wapas original password mein convert nahi kar sakte
- Isliye hum `comparePassword` helper function use karenge

## Step 3: Password Compare Karo

✅ **Ab `isVerified` check ke neeche yeh add karo:**
```typescript
// ✅ Ab password compare karo
const passwordMatch = await comparePassword(password, user.password);

if (!passwordMatch) {
  return { error: "Invalid credentials", success: false };
}

console.log("✅ Password match ho gaya:", user.email);

return { error: "", success: true };
```

> 🧠 Bahut important cheezein:
> 1.  `comparePassword` hamesha `await` karo, async function hai
> 2.  Hum yahan par bhi generic error hi dete hain: `Invalid credentials`
> 3.  Kabhi bhi "Wrong password" nahi bolte, security risk hai
> 4.  Attacker ko pata nahi chalta ki email sahi hai ya password

✅ **Ab poora `loginUser` function ab yeh dikhta hai:**
```typescript
"use server";

import prisma from "@/lib/prisma";
import { loginSchema } from "@/lib/validations/auth";
import { comparePassword } from "@/lib/password";

export async function loginUser(prevState, formData) {
  const email = formData.get("email") as string;
  const password = formData.get("password") as string;

  const result = loginSchema.safeParse({ email, password });
  if (!result.success) {
    return { error: result.error.issues[0].message, success: false };
  }

  const user = await prisma.user.findUnique({
    where: { email },
  });

  if (!user) {
    return { error: "Invalid credentials", success: false };
  }

  if (!user.isVerified) {
    return { error: "Please verify your email first", success: false };
  }

  const passwordMatch = await comparePassword(password, user.password);

  if (!passwordMatch) {
    return { error: "Invalid credentials", success: false };
  }

  console.log("✅ Login 100% successful:", user.email);

  return { error: "", success: true };
}
```

## Step 4: Ab Test Karo

1.  Sahi verified email daalo
2.  Sahi password daalo
3.  Submit karo

1.  Unverified user ka email daalo
2.  Sahi password daalo
3.  Submit karo

## Summary

**Aaj humne kya seekha:**

1.  **Bcrypt compare:**
    - Hashed password ko compare karna
    - Kabhi bhi plain text mein nahi store karte
    - Kabhi bhi hash ko reverse nahi karte

2.  **Security Practice:**
    - Generic error messages for invalid credentials
    - Attacker ko koi hint nahi dena
    - Dono case mein same error
