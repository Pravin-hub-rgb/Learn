## Problem: Registration Ke Baad Email Kaise Bhejein?

Soch rahe ho: *"Mera registration system toh ban gaya. User register kar sakta hai. Lekin abhi tak toh sirf database mein save ho raha hai. Verification email kaise bhejun?"*

**Current state (Phase 2):**
```typescript
// app/actions/auth.ts (Phase 2)

export async function registerUser(formData: FormData) {
  const email = formData.get("email") as string;
  const password = formData.get("password") as string;
  
  // Password hash karo
  const hashedPassword = await hashPassword(password);
  
  // User database mein save karo
  await prisma.user.create({
    data: {
      email,
      password: hashedPassword,
      isVerified: false,
    },
  });
  
  return { success: true };
}
```

**Problem:** User register ho gaya, lekin verification email nahi gaya!

**Solution:** Registration ke baad token generate karo aur email bhejo.

## Step-by-Step Implementation

### Step 1: Problem Samjho — "Token Return Kahan Se Aayega?"

**Phase 3.4 mein** humne `generateVerificationToken()` banaya tha, lekin token return nahi kiya tha kyunki abhi zaroorat nahi thi.

**Ab Phase 3.5 mein:**
- Humein token chahiye email link banane ke liye
- Toh `generateVerificationToken()` ko modify karenge — token return karega
- Phir token ko email link mein use karenge

### Step 2: `generateVerificationToken()` Ko Modify Karo

*"Sabse pehle, `lib/token.ts` mein `generateVerificationToken()` function ko token return karne ke liye modify karte hain."*

```typescript
// lib/token.ts

import { randomBytes } from "crypto";
import { prisma } from "./prisma";

/**
 * Verification token generate karta hai aur database mein save karta hai
 * @param userId - User jiske liye token generate karna hai
 * @returns Generated token (ab return kar rahe hain!)
 */
export async function generateVerificationToken(userId: string) {
  try {
    // 32 bytes ka random token (64 characters hex string)
    const token = randomBytes(32).toString("hex");
    
    // 24 hours ki expiry
    const expiresAt = new Date(Date.now() + 24 * 60 * 60 * 1000);
    
    // Purane tokens delete karo (agar koi hai)
    await prisma.verificationToken.deleteMany({
      where: {
        userId,
      },
    });
    
    // Naya token database mein save karo
    await prisma.verificationToken.create({
      data: {
        userId,
        token,
        expiresAt,
      },
    });
    
    // Ab token return kar rahe hain — email link banane ke liye!
    return token;
  } catch (error) {
    console.error("Error generating verification token:", error);
    throw new Error("Failed to generate verification token");
  }
}
```

**Kya change hua?**
- Pehle: Token return nahi ho raha tha
- Ab: `return token;` add kiya
- Kyun? Kyunki ab email link banane ke liye token chahiye!

**Important: `http://` kyun? `https://` kyun nahi?**

Local development mein Next.js dev server **HTTP** par chalta hai (port 3000 par), HTTPS par nahi. Agar aap `https://localhost:3000` use karoge toh browser SSL error dega:

```
SSL_ERROR_RX_RECORD_TOO_LONG
```

Kyunki browser HTTPS expect kar raha hai, lekin server HTTP respond kar raha hai.

**Rule:**
- **Development:** `http://localhost:3000` ✅
- **Production:** `https://yourdomain.com` ✅ (production mein HTTPS hi use hona chahiye)

Agar production ke liye environment variable use karna ho:
```typescript
const baseUrl = process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000';
const verificationLink = `${baseUrl}/verify?token=${token}`;
```

### Production Setup — Domain Ke Saath

**Scenario:** Maan lo tumhare paas domain hai `myapp.com`. Production mein kaise set karoge?

#### Step 1: Resend Dashboard Par Domain Verify Karo

Free plan mein sirf apne verified email ko hi emails bhej sakte ho. Production mein dusron ko email bhejne ke liye:

1. **Resend Dashboard** par jao → **Domains** section
2. **Add Domain** click karo → `myapp.com` enter karo
3. Resend tumhe **DNS records** dega (usually 3-4 records):
   ```
   Type: MX    Name: @          Value: feedback-smtp.us-east-1.amazonses.com
   Type: TXT   Name: @          Value: v=spf1 include:resend.com ~all
   Type: TXT   Name: resend._domainkey  Value: k=rsa; p=MIGfMA0GCS...
   Type: CNAME Name: resend._domainkey  Value: resend._domainkey.resend.com
   ```
4. Yeh records apne domain ke DNS settings mein add karo (GoDaddy, Namecheap, Cloudflare, etc.)
5. DNS propagate hone ka wait karo (5-30 minutes)
6. Resend dashboard par **Verify** click karo

### `app/actions/auth.ts` (Updated — registerUser)

```typescript
// app/actions/auth.ts

"use server";

import { prisma } from "@/lib/prisma";
import { hashPassword } from "@/lib/password";
import { registerSchema } from "@/lib/validations/auth";
import { generateVerificationToken } from "@/lib/token";
import { sendEmail } from "@/lib/email";

export async function registerUser(formData: FormData) {
  try {
    // Step 1: Form data extract karo
    const email = formData.get("email") as string;
    const password = formData.get("password") as string;
    
    // Step 2: Validate karo
    const validationResult = registerSchema.safeParse({ email, password });
    if (!validationResult.success) {
      return {
        error: validationResult.error.errors[0].message,
      };
    }
    
    // Step 3: Password hash karo
    const hashedPassword = await hashPassword(password);
    
    // Step 4: User database mein save karo
    const user = await prisma.user.create({
      data: {
        email,
        password: hashedPassword,
        isVerified: false,
      },
    });
    
    // Step 5: Verification token generate karo
    const token = await generateVerificationToken(user.id);
    
    // Step 6: Verification email bhejo
    const verificationLink = `http://localhost:3000/verify?token=${token}`;
    
    await sendEmail({
      to: user.email,
      subject: "Verify your email address",
      html: `
        <h1>Welcome!</h1>
        <p>Thanks for registering. Please verify your email by clicking the link below:</p>
        <a href="${verificationLink}">Verify Email</a>
        <p>Yeh link 24 hours tak valid hai.</p>
      `,
    });
    
    // Step 7: Success message return karo
    return {
      success: true,
      message: "Registration successful! Please check your email to verify your account.",
    };
  } catch (error) {
    console.error("Registration error:", error);
    return {
      error: "Something went wrong. Please try again.",
    };
  }
}
```
