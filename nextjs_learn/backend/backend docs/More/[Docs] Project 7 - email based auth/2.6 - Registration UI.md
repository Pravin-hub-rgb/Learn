## Broader Goal

**Aaj kya karenge:**

- `(auth)/register/page.tsx` file banayenge — yeh hoga permanent registration page
- Simple HTML form — koi component library nahi
- `<form action={registerUser}>` — directly action call hogi
- Email + password input fields
- Submit button
- Browser se test karenge — console verify karenge

**Route Groups ka use:**
- `(auth)` folder ka matlab hai ki yeh auth-related pages hain
- URL mein `(auth)` appear nahi hoga — sirf `/register` dikhega
- Future mein `(auth)` folder mein login, forgot-password jaise aur pages bhi add honge

## Step-by-Step Implementation (Natural Coder Flow)

### Step 1: Folder Structure Banao

Pehle check karo — `(auth)/register/` folder structure hai ya nahi:

```bash
# Check karo
ls app/

# Agar (auth) folder nahi hai, toh banao
mkdir app/\(auth\)
mkdir app/\(auth\)/register
```

**Note:** `(auth)` folder mein parentheses hain — yeh Next.js ka route group feature hai.

### Step 2:  `app/(auth)/register/page.tsx`

```typescript
// app/(auth)/register/page.tsx

import { registerUser } from "@/app/actions/auth";

export default function RegisterPage() {
  return (
    <div>
      <h1>Register</h1>
      <form action={registerUser}>
        <div>
          <label htmlFor="email">Email:</label>
          <input type="email" id="email" name="email" required />
        </div>
        <div>
          <label htmlFor="password">Password:</label>
          <input type="password" id="password" name="password" required />
        </div>
        <button type="submit">Register</button>
      </form>
    </div>
  );
}
```

*"Bas itna hi — simple HTML form jo `registerUser` action ko call karta hai."*

### Step 3: Test Karo

Browser mein jao: `http://localhost:3000/register`

**Kya dikha:**
- "Register" heading
- Email input field
- Password input field
- Register button

### Step 4: Form Submit Karke Dekho

1. Email daalo: `test@example.com`
2. Password daalo: `password123`
3. Register button click karo
4. Console check karo

**Expected console output:**
```
Registration attempt
Form data: FormData { email: 'test@example.com', password: 'password123' }
```

*"Great! Form submit ho raha hai aur action call ho raha hai!"*
