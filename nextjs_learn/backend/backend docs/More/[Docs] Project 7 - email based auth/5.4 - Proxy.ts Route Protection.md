### ✅ Step 3.1 Sabse basic proxy function

🧠 **SOCH:**
Pehle sirf empty function banate hai. Next.js ko batao ki yeh proxy hai.

📝 **CODE:**
```typescript
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export default async function proxy(request: NextRequest) {
  console.log("Proxy chala hai!");

  return NextResponse.next();
}
```

📚 **EXPLANATION:**
1.  `import { NextResponse } from "next/server"`:
    - Next.js se hi inbuilt hai yeh class
    - Isi se hum response bhejte hai proxy se
    - Proxy mein normal response nahi bhej sakte sirf yahi use kar sakte

2.  `import type { NextRequest }`:
    - Request object ka type hai
    - Typescript ke liye hai, isse type safety milti hai

3.  `export default async function proxy(request: NextRequest)`:
    - `export default` isliye lagaya hai taaki Next.js ko pata chale ki yeh woh function hai jo har request pe chalaega
    - `request` object har request ke saath automatically aata hai
    - Isme poora information hota hai ki user kaha ja raha hai, kya bhej raha hai

4.  `console.log("Proxy chala hai!")`:
    - Bas test ke liye hai, baad mein hata denge

5.  `return NextResponse.next()`:
    - Matlab sab thik hai aage badho
    - Page ko normally load hone do

### ✅ Step 3.3 Ab sirf dashboard par hi check lagao

🧠 **SOCH:**
Ab sirf dashboard par hi check lagate hai. Baaki pages par nahi. Ab hum dekh rahe hai ki user kaha ja raha hai.

📝 **CODE:**
```typescript
import { getSession } from "@/lib/session";
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export default async function proxy(request: NextRequest) {
  const session = await getSession();

  if (request.nextUrl.pathname === "/dashboard" && !session) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  return NextResponse.next();
}
```

📚 **EXPLANATION:**
1.  `request.nextUrl.pathname`:
    - Yeh `request` object ke andar hota hai
    - Next.js har request ke saath yeh automatically deta hai
    - Isse hum pata kar sakte hai ki user abhi kaun sa page open kar raha hai
    - Hum isse `/dashboard`, `/login`, `/profile` sab check kar sakte hai

2.  `=== "/dashboard"`:
    - Hum exact match kar rahe hai ki woh exact dashboard par hai ya nahi

3.  `&& !session`:
    - Dono condition true honi chahiye: dashboard par hai + session nahi hai
    - Tab hi hum redirect karenge

4.  `NextResponse.redirect()`:
    - Yahan pe normal `redirect()` function nahi use kar sakte
    - Proxy.ts / Middleware mein sirf NextResponse se hi redirect kar sakte hai

5.  `new URL("/login", request.url)`:

    🧠 Yeh bahut important hai. Yeh 2 parameter leta hai: `new URL(path, base)`

    Yeh `base` parameter hi woh second argument hai. Isme hum `request.url` dete hai. Yeh automatically current domain le lega.

    ✅ Real Example Table:
    | Jab tum kaha ho | Base `request.url` | Path | Final URL banega |
    |---|---|---|---|
    | Localhost par | `http://localhost:3000/dashboard` | `/login` | `http://localhost:3000/login` |
    | Production par | `https://yourapp.com/dashboard` | `/login` | `https://yourapp.com/login` |
    | Staging par | `https://staging.yourapp.com/dashboard` | `/login` | `https://staging.yourapp.com/login` |

    ✅ Kyun yeh best practice hai:
    - Tumhe kabhi bhi apna domain hardcode nahi karna padta
    - Har jagah same code same tareeke se chalaega
    - Kabhi bhi production mein redirect tootega nahi

    ❌ Agar tum yeh nahi karoge toh production mein tumhara redirect poora hi toot jayega.

> ✅ Ab agar tum logout karke `/dashboard` par jaoge toh login par chala jaoge. Ab sirf dashboard hi protected hai.

### ✅ Step 3.4 Ab aur routes add karo

🧠 **SOCH:**
Ab aage jaake profile aur settings pages banayenge. Unko bhi abhi se protected rakh lo.

📝 **CODE:**
```typescript
import { getSession } from "@/lib/session";
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export default async function proxy(request: NextRequest) {
  const session = await getSession();

  const protectedRoutes = ["/dashboard", "/profile", "/settings"];
  const isProtected = protectedRoutes.some(route => 
    request.nextUrl.pathname.startsWith(route)
  );

  if (isProtected && !session) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  return NextResponse.next();
}
```

📚 **EXPLANATION:**
1.  `const protectedRoutes = [...]`:
    - Array banaya hai jisme saare protected routes daal diye
    - Ab tum bas is array mein add karo naye pages. Woh automatically protected ho jayenge

2.  `protectedRoutes.some()`:
    - Javascript ka inbuilt array function hai
    - Har route ko check karega
    - Agar ek bhi route match ho jaye toh true return karega

3.  `startsWith(route)`:
    - Exact match nahi hai balki start se match karta hai
    - `/dashboard/orders`, `/dashboard/payments` sab automatically protected honge

### ✅ Step 3.5 Ab config add karo

🧠 **SOCH:**
Ab ek problem hai. Har static file par bhi proxy chalta hai. CSS, JS, fonts par bhi. Hum unko rokna hai.

📝 **CODE:**
```typescript
import { getSession } from "@/lib/session";
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export default async function proxy(request: NextRequest) {
  const session = await getSession();

  const protectedRoutes = ["/dashboard", "/profile", "/settings"];
  const isProtected = protectedRoutes.some(route => 
    request.nextUrl.pathname.startsWith(route)
  );

  if (isProtected && !session) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  return NextResponse.next();
}

export const config = {
  matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"],
};
```

📚 **EXPLANATION:**
1.  `export const config`:
    - Next.js ko settings dete hai isse
    - Isme hum batate hai ki kahan proxy chalao kahan nahi

2.  `matcher`:
    - Regex pattern hai
    - Yeh neeche wali cheezein exclude karta hai:
      - `_next/static` → CSS JS files
      - `_next/image` → images
      - `favicon.ico` → favicon
    - Ab in par proxy nahi chalaega. Sirf actual pages par chalaega.

✅ ✅ Ab isme koi warning nahi aati!

---

### 🧠 Kyun isme warning nahi aati?

Proxy.ts by default Node.js Runtime mein chalta hai. Edge Runtime nahi. Isliye poora Node.js ka saara code isme chalta hai. `getSession()`, `prisma`, `process.env` sab chalta hai koi problem nahi.

✅ Tum ab samajh rahe ho na?
Middleware.ts = Edge Runtime = warning
Proxy.ts = Node.js Runtime = no warning = sab kuch chalta hai

Yeh hi latest official tareeka hai ab.
