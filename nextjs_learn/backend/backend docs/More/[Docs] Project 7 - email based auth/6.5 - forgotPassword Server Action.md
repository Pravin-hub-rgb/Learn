### ✅ Step 2.2.1 Ab validation add karo

🧠 **SOCH:**
Ab hume email validation karni padegi. Koi bhi random string nahi chahiye. Hum already `lib/validations/auth.ts` mein Zod schema use kar rahe hai wahi use karenge.

📝 **CODE:**
```typescript
"use server";

import prisma from "@/lib/prisma";
import { ForgotPasswordSchema } from "@/lib/validations/auth";

export async function forgotPassword(prevState: unknown, formData: FormData) {
  const email = formData.get("email") as string;

  const validated = ForgotPasswordSchema.safeParse({ email });
  if (!validated.success) {
    return {
      error: validated.error.issues[0].message
    };
  }

  const user = await prisma.user.findUnique({
    where: { email: validated.data.email }
  });

  console.log("User mila:", user);

  return {};
}
```

### ✅ Step 2.2.2 Ab isVerified check add karo

🧠 **SOCH:**
Ab agar user ne apna email verify nahi kiya hai toh hum password reset nahi karne denge. Kyuki agar woh apna email hi nahi verify kiya hai toh usko password reset nahi chahiye.

```typescript
"use server";

import prisma from "@/lib/prisma";
import { ForgotPasswordSchema } from "@/lib/validations/auth";

export async function forgotPassword(prevState: unknown, formData: FormData) {
  const email = formData.get("email") as string;

  const validated = ForgotPasswordSchema.safeParse({ email });
  if (!validated.success) {
    return {
      error: validated.error.issues[0].message
    };
  }

  const user = await prisma.user.findUnique({
    where: { email: validated.data.email }
  });

  if (user && !user.isVerified) {
    return {
      error: "Please verify your email first"
    };
  }

  console.log("User mila:", user);

  return {};
}
```

✅ Ab agar unverified email daaloge toh error dikhega ✅

### ✅ Step 2.3 Ab token generate karo

🧠 **SOCH:**
Ab agar user exist karta hai aur verified bhi hai toh hume ek unique token generate karna hai. Is token ko hum database mein save karenge 15 minute ke expiry ke saath. Aur is token ko email mein bhejenge user ko.

📝 **CODE:**
```typescript
"use server";

import prisma from "@/lib/prisma";
import { ForgotPasswordSchema } from "@/lib/validations/auth";
import { generatePasswordResetToken } from "@/lib/token";

export async function forgotPassword(prevState: unknown, formData: FormData) {
  const email = formData.get("email") as string;

  const validated = ForgotPasswordSchema.safeParse({ email });
  if (!validated.success) {
    return {
      error: validated.error.issues[0].message
    };
  }

  const user = await prisma.user.findUnique({
    where: { email: validated.data.email }
  });

  if (user && !user.isVerified) {
    return {
      error: "Please verify your email first"
    };
  }

  if (!user) {
    return {
      success: true,
      message: "Check your email for password reset link"
    };
  }

  const passwordResetToken = await generatePasswordResetToken(user.id);
  console.log("Token generate ho gaya:", passwordResetToken);

  return {
    success: true,
    message: "Check your email for password reset link"
  };
}
```

> 🧠 Important:
> Chahe user exist kare ya nahi hum hamesha same message return karenge. Kyunki hum nahi batayenge ki yeh email exist karta hai ya nahi. Security risk hai.

### ✅ Step 2.5 Ab email bhejo

🧠 **SOCH:**
Ab hume user ko email bhejna hai uske email par. Us email mein reset password ka link hoga jisme token hoga. Jab user us link par click karega toh woh reset password page par jayega.

📝 **CODE:**
```typescript
"use server";

import prisma from "@/lib/prisma";
import { ForgotPasswordSchema } from "@/lib/validations/auth";
import { generatePasswordResetToken } from "@/lib/token";
import { sendPasswordResetEmail } from "@/lib/email";

export async function forgotPassword(prevState: unknown, formData: FormData) {
  const email = formData.get("email") as string;

  const validated = ForgotPasswordSchema.safeParse({ email });
  if (!validated.success) {
    return {
      error: validated.error.issues[0].message
    };
  }

  const user = await prisma.user.findUnique({
    where: { email: validated.data.email }
  });

  if (user && !user.isVerified) {
    return {
      error: "Please verify your email first"
    };
  }

  if (!user) {
    return {
      success: true,
      message: "Check your email for password reset link"
    };
  }

  const passwordResetToken = await generatePasswordResetToken(user.id);
  
  await sendPasswordResetEmail(user.email, passwordResetToken.token);

  return {
    success: true,
    message: "Check your email for password reset link"
  };
}
```

- Email inbox mein check karo
- Reset password link aa gaya hai? ✅

---

> 🧠 🧠 🧠 **Bilkul Clear Karta Hu Ab return values ke baare mein:**
> 
> **✅ Hum `error:` field mein return kar rahe hai kyunki:**
> Ye humara apna rule hai.
> 
> Jab bhi kuch galat hai validation mein, ya user nahi mila, ya verified nahi hai - woh sab **problem** hai. Isliye hum `error:` mein return karte hai.
> 
> Jab bhi sab kuch theek hai - woh **success** hai. Isliye hum `success: true` mein return karte hai.
> 
> **Yeh bas humara apna format hai:**
> ❌ Problem hua: `{ error: "problem ka message" }`
> ✅ Sab theek hua: `{ success: true, message: "good message" }`
> 
> Ye koi inbuilt rule nahi hai. Humne ye apne aap banaya hai taki hum frontend par aasani se check kar sake ki kya hua hai.
> 
> Frontend mein hum bas aise check karte hai:
> ```tsx
> {state?.error && <p className="text-red-500">{state.error}</p>}
> {state?.success && <p className="text-green-500">{state.message}</p>}
> ```
> 
> Bas itna hi hai. Koi magic nahi hai. Humne ye apne aap hi banaya hai format taki frontend par messages dikhaane mein aasan ho.
> 
> 
> **Ab ye lo poora logic:**
> ✅ Agar email galat format hai: return `error: "Invalid email address"`
> ✅ Agar user nahi mila: return `error: "No account found with this email"`
> ✅ Agar user mila lekin verified nahi: return `error: "Please verify your email first"`
> ✅ Agar sab theek hai: return `success: true, message: "Check your email for reset link"`
> 
> Sab alag alag message hai. Koi hidden nahi hai. Kyuki user apna hi email daal raha hai woh apna account hi bhool gaya hai. Usko sahi reason batana hi chahiye.
