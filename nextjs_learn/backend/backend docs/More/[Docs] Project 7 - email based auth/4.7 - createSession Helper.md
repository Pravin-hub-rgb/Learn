## Step 5: Expiry Time Calculate Karo

```typescript
// ✅ Step 1: 7 din tak session expire hoga
const expiresAt = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000);
```

> 🧠 Deep explanation:
> - `Date.now()` → Abhi ka time milliseconds mein
> - `7 * 24 * 60 * 60 * 1000` → 7 din ka total milliseconds
> - Inko add kar ke naya date banaya hai jo 7 din baad ka hai
> Is din ke baad yeh session automatically invalid ho jayega

## Step 7: Cookie Set Karo

✅ **Next.js 15 Important Update:**
Ab Next.js 15 mein `cookies()` function Promise return karta hai. Isliye tumhe isko `await` karna padega.

```typescript
// ✅ Step 3: Cookie set karo
const cookieStore = await cookies();

cookieStore.set("sessionId", session.id, {
  httpOnly: true,
  secure: process.env.NODE_ENV === "production",
  sameSite: "lax",
  expires: expiresAt,
  path: "/",
});
```

> 🧠 Yeh error kyun aa raha tha:
> Next.js ke naye version mein `cookies()` async banaya gaya hai.
> Pehle woh directly value deta tha, ab Promise deta hai.
> Isliye `await cookies()` likhna zaroori hai ab.
> Isliye tumhe `set` nahi mil raha tha, kyunki uspe Promise tha.

> 🧠 Is line ko har part ko deep mein samjho:
>
> ✅ `cookies()`
> Yeh Next.js ka function hai jo humne upar `next/headers` se import kiya tha.
> Yeh function hame cookie ko set karne ya read karne ke liye access deta hai Server Actions mein.
>
> ✅ `.set()`
> Yeh method cookie set karne ke liye hota hai. Isme 3 parameters hote hain:
> 1.  **Cookie ka Name**
> 2.  **Cookie ka Value**
> 3.  **Cookie ke Options** (flags)
>
> ✅ `"sessionId"`
> Yeh cookie ka naam hai. Hum apne cookie ko is naam se store kar rahe hain.
> Tum isko kuch bhi rakh sakte ho lekin `sessionId` standard naam hai.
>
> ✅ `session.id`
> Yeh woh random CUID hai jo Prisma ne automatically generate kiya tha jab humne `prisma.session.create()` call kiya tha.
> Jab tum `prisma.session.create()` call karte ho toh return mein jo object aata hai usme `id` field already hoti hai jo database mein generate hui thi.
> Wahi id hum cookie mein daal rahe hai.
>
> ✅ `{ options }`
> Yeh third parameter hai jisme hum saare security flags lagate hai.

> 🧠 Har option ko deep mein samjho:
>
> ✅ `httpOnly: true`
> - Sabse important flag hai
> - Cookie ko JavaScript se access nahi kar sakte
> - XSS attack se 100% bachata hai
> - Agar yeh nahi lagao toh koi bhi JS chura lega cookie aur dusre ke account mein ghuss jayega
>
> ✅ `secure: process.env.NODE_ENV === "production"`
> - Production mein sirf HTTPS par hi cookie bheji jayegi
> - Localhost par HTTP mein bhi kaam karega
> - Local development mein problem nahi aayegi
>
> ✅ `sameSite: "lax"`
> - Dusre website se request aaye toh cookie nahi bhejega
> - CSRF attack se bachata hai
> - Yeh bhi security ke liye zaroori hai
>
> ✅ `expires: expiresAt`
> - Cookie bhi same date par expire hogi jitna session expire hoga
> - Browser apne aap is date ke baad cookie delete kar dega
>
> ✅ `path: "/"`
> - Poori website par cookie available hogi
> - Har route par cookie bheji jayegi

## Step 8: Function Return Karo

```typescript
return session;
```

> 🧠 Hum isko kyun return kar rahe hain?
> - Abhi hum sirf function ban rahe hain
> - Jab hum is function ko `loginUser` action mein call karenge tab humein session ka data chahiye hoga
> - Koi future mein agar hume session ke baare mein aur information chahiye toh yeh return se mil jayegi
> - Abhi is function mein hume kya karna hai usko poora kar liya hai, ab jo isko call karega usko result return kar dete hain
> - Tum agar abhi kuch nahi return karoge bhi toh kaam chalega lekin good practice hai ki function apna result return kare

✅ **Ab poora function yeh dikhta hai:**
```typescript
import prisma from "./prisma";
import { cookies } from "next/headers";

export async function createSession(userId: string) {
  const expiresAt = new Date(Date.now() + 7 * 24 * 60 * 60 * 1000);

  const session = await prisma.session.create({
    data: {
      userId,
      expiresAt,
    },
  });

  const cookieStore = await cookies();

  cookieStore.set("sessionId", session.id, {
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
    sameSite: "lax",
    expires: expiresAt,
    path: "/",
  });

  return session;
}
```
