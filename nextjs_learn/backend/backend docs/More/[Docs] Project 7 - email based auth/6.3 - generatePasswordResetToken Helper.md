## Step 1: Helper Kyun Banaya Ja Raha Hai?

🧠 **SOCH (BIG PICTURE):**
Jab user forgot password page par apna email daal kar submit karega:
1.  Hum backend mein email se user ko dhundenge
2.  Agar user exist karta hai toh hum yeh `generatePasswordResetToken()` function call karenge
3.  Yeh function unique token generate karega
4.  Database mein save karega 15 minute expiry ke saath
5.  Phir hum us token ko lekar email bhejenge user ke email par
6.  User email mein link par click karega toh usko naya password daalne ka page milega

Isliye hum isko alag helper banaye hai. Ye apna kaam sirf token generate karke save karna hai. Baaki sab alag alag jagah handle hoga.

Ye function sirf ek kaam karta hai, aur acchi tareeke se karta hai.

## Step 2: Helper Function Banaye

🧠 **SOCH:**
Ab `lib/token.ts` file open karo. Uske end mein yeh naya function add karo.

📝 **CODE:**
```typescript
import crypto from "crypto";
import prisma from "./prisma";

export async function generatePasswordResetToken(userId: string) {
  // Generate 32 character random token
  const token = crypto.randomBytes(32).toString("hex");

  // 15 minute expiry
  const expiresAt = new Date(Date.now() + 15 * 60 * 1000);

  // Save token to database
  const passwordResetToken = await prisma.passwordResetToken.upsert({
    where: { userId },
    update: {
      token,
      expiresAt,
    },
    create: {
      userId,
      token,
      expiresAt,
    },
  });

  return passwordResetToken;
}
```

📚 **EXPLANATION (Har line ke liye):**
1.  `import crypto from "crypto"`:
    - Node.js ka inbuilt module hai
    - Isse secure random token generate karte hai
    - Koi third party package nahi chahiye

2.  `crypto.randomBytes(32).toString("hex")`:
    - 32 bytes ka random data generate karta hai
    - Hexadecimal string mein convert karta hai
    - 64 characters ka unique token ban jata hai
    - Yeh 100% secure aur unique hota hai

3.  `new Date(Date.now() + 15 * 60 * 1000)`:
    - Abhi ke time mein 15 minute add kar rahe hai
    - 15 * 60 = 900 seconds
    - 900 * 1000 = 900000 milliseconds
    - Token 15 minute baad expire ho jayega

4.  `prisma.passwordResetToken.upsert()`:

    🧠 Yeh Prisma ka most useful function hai. Iska matlab hai **UPDATE OR INSERT**.

    ✅ Real Life Example:
    Agar user ne pehle 5 minute pehle hi forgot password click kiya tha aur phir se dobara click karta hai:
    - ❌ Agar hum sirf `create` use karte toh **error aata** "Unique constraint failed on userId"
    - ❌ Agar hum pehle delete kar ke phir create karte toh 2 database query karni padti
    - ✅ `upsert()` ek hi query mein dono ka kaam kar deta hai

    📚 Isme 3 cheezein hoti hai:
    1.  `where: { userId }`: Pehle check karo is userId ke liye already token hai ya nahi
    2.  `update: { ... }`: Agar **pehle se token hai toh yeh fields update karo** (naya token, naya expiry time)
    3.  `create: { ... }`: Agar **pehle se token nahi hai toh naya banao** inn fields ke saath

    ✅ Isliye ek user ke liye ek hi time pe ek hi token hoga. Chahe kitni baar bhi forgot password click kare har baar naya token generate hoga purana overwrite ho jayega.

    ✅ Humne isliye `userId` par `@unique` lagaya tha model mein. Isliye yeh kaam karta hai.

## Common Issue: Typescript Error

🧠 **Problem:**
Agar tumhe yeh error aaye:
```

```

✅ **Fix:**
1.  Terminal mein:
    ```bash
    npx prisma generate
    ```
2.  VS Code mein `Ctrl + Shift + P` press karo
3.  Search karo: `TypeScript: Restart TS Server`
4.  Select karo

✅ 10 second mein error chala jayega. Yeh Prisma ka common issue hai jab naya model add karte hai.
