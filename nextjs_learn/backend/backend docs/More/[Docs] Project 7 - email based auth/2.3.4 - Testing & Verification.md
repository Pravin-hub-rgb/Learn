### Test 2: Database Connection Test

Ek simple test file banao — `test-prisma.ts`:

```typescript
// test-prisma.ts
import prisma from "./lib/prisma";

async function test() {
  try {
    console.log("🔌 Connecting to database...");
    
    // Simple query — check karo ki connection kaam kar raha hai
    const result = await prisma.$queryRaw`SELECT 1 as num`;
    console.log("✅ Database connection successful:", result);
    
    // Check karo ki users table exist karti hai
    const users = await prisma.user.findMany();
    console.log("✅ Users table accessible. Current users:", users.length);
    
    // Disconnect
    await prisma.$disconnect();
    console.log("✅ Disconnected from database");
    
    console.log("\n🎉 All tests passed!");
  } catch (error) {
    console.error("❌ Error:", error);
    process.exit(1);
  }
}

test();
```

Run karo:
```bash
npx tsx test-prisma.ts
```

**Expected output:**
```
🔌 Connecting to database...
✅ Database connection successful: [ { num: 1 } ]
✅ Users table accessible. Current users: 0
✅ Disconnected from database

🎉 All tests passed!
```

**Har check ka matlab:**
- `SELECT 1` — Basic database connection verify karta hai
- `prisma.user.findMany()` — `users` table exist karti hai verify karta hai
- `$disconnect()` — Connection properly close hota hai verify karta hai

### Test 5: Using Prisma in API Routes

Ab test karte hain ki `lib/prisma.ts` ko doosre files mein kaise use karna hai.

**Example: `app/api/auth/register/route.ts`**

```typescript
// app/api/auth/register/route.ts
import { NextRequest, NextResponse } from "next/server";
import prisma from "@/lib/prisma";

export async function POST(request: NextRequest) {
  try {
    // Request body parse karo
    const { email, password } = await request.json();
    
    console.log("📧 Registration attempt:", email);
    
    // Check karo ki user already exist toh nahi karta
    const existingUser = await prisma.user.findUnique({
      where: { email },
    });
    
    if (existingUser) {
      return NextResponse.json(
        { error: "User already exists with this email" },
        { status: 409 }
      );
    }
    
    // Naya user create karo (abhi ke liye sirf test)
    const user = await prisma.user.create({
      data: {
        email,
        password, // Abhi ke liye plain text — baad mein hash karenge
        isVerified: false,
      },
    });
    
    console.log("✅ User created:", user.id);
    
    return NextResponse.json(
      { message: "User registered successfully", userId: user.id },
      { status: 201 }
    );
  } catch (error) {
    console.error("❌ Registration error:", error);
    return NextResponse.json(
      { error: "Internal server error" },
      { status: 500 }
    );
  }
}
```

**Test karo:**

## Common Issues & Solutions

### Issue 1: `Cannot find module '@/lib/prisma'`
**Error:**
```
Cannot find module '@/lib/prisma' or its corresponding type declarations.
```

**Reason:** Path alias `@` configure nahi hai ya galat hai

**Solution:**
- `tsconfig.json` check karo — `@/*` path alias hona chahiye
- Ya direct relative path use karo: `import prisma from "../../lib/prisma"`

### Issue 2: `prisma.user is not defined`
**Error:**
```
TypeError: Cannot read property 'user' of undefined
```

**Reason:** Prisma client properly generate nahi hua

**Solution:**
```bash
# Prisma client regenerate karo
npx prisma generate
```

### Issue 3: `ECONNREFUSED` during tests
**Error:**
```
Error: connect ECONNREFUSED 127.0.0.1:5432
```

**Reason:** PostgreSQL server nahi chal raha

**Solution:**
```bash
# PostgreSQL start karo
pg_ctl start

# Ya Windows Services mein "PostgreSQL" start karo
```

### Issue 4: `DATABASE_URL is not defined`
**Error:**
```
Environment variable DATABASE_URL is not defined
```

**Reason:** `.env` file load nahi ho rahi

**Solution:**
- `.env` file root mein hai verify karo
- `prisma.config.ts` mein `import "dotenv/config"` top pe hai check karo
- `.env` file mein `DATABASE_URL` properly set hai confirm karo

### Issue 5: `relation "users" does not exist`
**Error:**
```
error: relation "users" does not exist
```

**Reason:** Migration run nahi hui ya table nahi bani

**Solution:**
```bash
# Migration run karo
npx prisma migrate dev

# Ya manually table banao
psql -U postgres -d email_auth_db
CREATE TABLE users (...);
```

### Issue 6: `PrismaClient requires a driver adapter`
**Error:**
```
PrismaClientInitializationError: PrismaClient requires a driver adapter.
```

**Reason:** Adapter properly configure nahi hua

**Solution:**
- `lib/prisma.ts` check karo — adapter pass kiya hai na?
- `npm install @prisma/adapter-pg` installed hai verify karo
- `import { PrismaPg } from "@prisma/adapter-pg"` sahi hai check karo

### Issue 7: Memory leak — connections badhte ja rahe hain
**Error:**
```
Error: too many clients already
```

**Reason:** Kahin aur se direct `new PrismaClient()` call ho raha hai

**Solution:**
- Search karo — kahin bhi `new PrismaClient()` toh nahi likha?
- Sirf `lib/prisma.ts` mein hi `new PrismaClient()` hona chahiye
- Doosre files mein sirf `import prisma from "@/lib/prisma"` use karo

## Best Practices

### 1. Always Import from `lib/prisma.ts`
```typescript
// ✅ Sahi
import prisma from "@/lib/prisma";

// ❌ Galat
import { PrismaClient } from "@prisma/client";
const prisma = new PrismaClient();
```

### 2. Always Disconnect in Scripts
```typescript
// Test scripts mein hamesha disconnect karo
async function test() {
  try {
    // ... your code
  } finally {
    await prisma.$disconnect();
  }
}
```

### 3. Handle Connection Errors Gracefully
```typescript
try {
  const users = await prisma.user.findMany();
} catch (error) {
  if (error instanceof Prisma.PrismaClientKnownRequestError) {
    // Prisma-specific error handling
    console.error("Database error:", error.message);
  } else {
    // General error handling
    console.error("Unknown error:", error);
  }
}
```

### 4. Use Transactions for Multiple Operations
```typescript
await prisma.$transaction(async (tx) => {
  // Multiple operations — agar ek bhi fail hua, toh sab rollback ho jayega
  await tx.user.create({ data: { email: "test@example.com", password: "test" } });
  await tx.session.create({ data: { userId: "..." } });
});
```

### 5. Type Safety Use Karo
```typescript
// Prisma automatically types provide karta hai
const user = await prisma.user.findUnique({
  where: { email: "test@example.com" },
  select: {
    id: true,
    email: true,
    isVerified: true,
    // password select mat karo — sensitive data
  },
});

// user ka type automatically infer ho jayega
// { id: string, email: string, isVerified: boolean } | null
```
