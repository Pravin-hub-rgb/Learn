## Phase 3 Overview: Email Verification — Pura Flow

**Bada picture samjho — Phase 3 mein kya hone wala hai:**

### Problem Kya Hai?

Abhi tak (Phase 2 mein) humne registration system banaya hai. User form bharta hai, hum database mein save kar dete hain. Lekin ek problem hai:

**Koi bhi fake email se register kar sakta hai!**

Example:
- Kisi ne form bhara: `abc@xyz.com`, password: `123456`
- Humara system ne bina verify kiye user create kar diya
- Yeh email actually exist bhi nahi karti!

**Result:** Database mein fake users pile up ho rahe hain.

### Solution: Email Verification

Humko yeh confirm karna hai ki user ne jo email di hai, woh actually uski hai. Iske liye hum yeh karenge:

**Step-by-Step Flow:**

1. **User register karta hai**
   - Form bharta hai: email + password
   - Hum database mein user save karte hain
   - `isVerified = false` set karte hain (abhi verify nahi hua)

2. **Token generate aur database mein save**
   - Ek unique token generate karte hain (random string, jaise `abc123xyz789`)
   - **Token ko `VerificationToken` table mein save karte hain** (nayi table banegi Phase 3.3 mein)
   - Table mein store hota hai: `userId`, `token`, `expiresAt` (24 hours)
   - **Yeh important hai:** Token database mein save hota hai taaki baad mein verify kar sakein

3. **Hum verification email bhejte hain**
   - Email mein ek link bhejte hain: `http://localhost:3000/verify?token=abc123xyz789`
   - Yeh link user ke email pe jata hai (Resend se)
   - **Note:** Testing mein localhost, production mein domain hoga

4. **User email check karta hai**
   - User apna email kholta hai
   - Wahan ek email aayi hai humari taraf se
   - Email mein ek "Verify Email" button/link hai

5. **User link pe click karta hai**
   - User "Verify Email" pe click karta hai
   - Browser URL open hota hai: `http://localhost:3000/verify?token=abc123xyz789`
   - **Yeh `/verify` page hai** - ek normal Next.js page (Server Component)
   - Page load hote hi token URL se extract hota hai

6. **Token verify karna (Server Action)**
   - `/verify` page load hote hi ek **Server Action** call hoti hai: `verifyEmail(token)`
   - **API endpoint nahi banana padega** - hum Server Actions use kar rahe hain
   - Server Action:
     - Token ko `VerificationToken` table mein dhundhta hai
     - Check karta hai: token valid hai? expiry nahi hui?
     - Agar valid, toh user ko dhundhta hai (userId se)
     - User ka `isVerified = true` karta hai
     - Token ko delete karta hai (one-time use)
     - Success/error return karta hai

7. **User ab login kar sakta hai**
   - User ko message dikhta hai: "Email verified! Ab login karo"
   - User login page pe jata hai
   - Login karta hai → `isVerified = true` check hota hai → Session banta hai

### Token Kyun Use Karte Hain?

Token ek random string hai (jaise `abc123xyz789`). Iske kuch important features hain:

1. **Unique:** Har user ka alag token hota hai
2. **One-time use:** Ek baar use hone ke baad delete ho jata hai
3. **Expiry:** Token ki expiry hoti hai (24 hours). Agar user ne 24 hours mein verify nahi kiya, toh token kaam nahi karega
4. **Secure:** Token se hum user ko identify kar sakte hain bina email/password ke

### Token Kahan Store Hota Hai?

Token **do jagah** store hota hai:

**1. Database mein** - `VerificationToken` table mein (nayi table banegi):
```
VerificationToken table:
id | userId | token | expiresAt
1  | 1      | abc123xyz789 | 2024-01-02 12:00:00
```

**2. URL mein** - Email link mein `?token=...`:
```
https://app.com/verify?token=abc123xyz789
```

## Why Email Verification? (Kyun Zaroori Hai?)

**Without verification (Current State):**
```typescript
// Koi bhi fake email se register kar sakta hai
// Server Action call hoti hai:
registerUser(formData);
// → User created! ❌
```

**With verification (Phase 3):**
```typescript
// Register hoga, lekin login nahi kar payega
// Server Action call hoti hai:
registerUser(formData);
// → Verification email bheja (jo kabhi verify nahi hoga)

loginUser(formData);
// → Error: "Please verify your email first" ✅
```

**Benefits:**
1. **Spam prevention** — Fake accounts kam honge
2. **Email ownership** — User actually owns the email
3. **Security** — Koi aur ke email se register nahi kar sakta
4. **Clean database** — Sirf verified users

## Email Service Options: Kyun Resend Chuna?

2. **Modern Stack:**
   - Built for React/Next.js developers
   - React-based email templates
   - TypeScript support out of the box

3. **Free Tier:**
   - 100 emails/day (3000/month) — enough for development
   - No credit card required

4. **Easy Setup:**
   - No SMTP configuration
   - No DNS records (for testing)
   - API key = ready to go

5. **Next.js Integration:**
   - Made by Vercel team
   - Works seamlessly with Server Actions
   - Edge-compatible

**Hum Resend use kar rahe hain kyunki:**
- Simple API — kam code, zyada kaam
- Free tier sufficient hai development ke liye
- Next.js/React developers ke liye best experience
- Production-ready when needed

## Unverified Users ka Cleanup Strategy

**Problem:** Agar user verify nahi karta, toh woh database mein pile up hote rahenge!

**Solutions:**

### Option A: Auto-cleanup via Token Expiry (Implemented)
- `VerificationToken` model mein `expiresAt` field hai (24 hours)
- Expired tokens automatically kaam nahi karenge
- User ko dobara register karna padega
- **Pros:** Simple, automatic
- **Cons:** Unverified users abhi bhi database mein rahenge

### Option B: Manual Cleanup (Using in this project)
- Admin panel se unverified users delete kar sakte hain
- Ya manually query run kar sakte hain:

```sql
-- Delete unverified users older than 7 days
DELETE FROM "User" 
WHERE "isVerified" = false 
AND "createdAt" < NOW() - INTERVAL '7 days';
```

**Pros:** Full control
**Cons:** Manual effort required

### Option C: Periodic Cleanup (Best for production)
- Background job jo roz unverified users ko clean kare

```typescript
// lib/cleanup.ts — Syntax example

import prisma from "./prisma";

export async function cleanupUnverifiedUsers() {
  const result = await prisma.user.deleteMany({
    where: {
      isVerified: false,
      createdAt: {
        lt: new Date(Date.now() - 7 * 24 * 60 * 60 * 1000), // 7 days ago
      },
    },
  });

  console.log(`Cleaned up ${result.count} unverified users`);
  return result.count;
}
```

**Cron job example (using `node-cron`):**
```typescript
// app/cleanup/route.ts

import { cleanupUnverifiedUsers } from "@/lib/cleanup";
import cron from "node-cron";

// Roz raat 2 baje cleanup
cron.schedule("0 2 * * *", async () => {
  await cleanupUnverifiedUsers();
});
```

**Pros:** Automatic, scalable
**Cons:** Extra package, complexity

**Hum is project mein Option B use karenge** — simple manual cleanup. Production ke liye Option C implement kar sakte hain.

#### Step 5.2: `sendEmail` Function Banao

*"Ab template ko reusable function mein convert karte hain. Function ko `sendEmail` bolenge. Isme `to`, `subject`, aur `html` parameters lenge."*

**Kyun bana rahe hain `sendEmail` function?**

Soch rahe ho: *"Mujhe email bhejni hai. Template toh kaam kar raha hai. Phir alag function kyun bana raha hoon?"*

**Reason:** Abhi hum sirf test kar rahe hain. Lekin Phase 3 mein:
- User register karega → usko verification email bhejni hogi
- User password reset karega → usko reset email bhejni hogi
- Har baar same code likhne se behtar hai ek reusable function

**Simple analogy:**
```
Bina function ke:
  Har baar email bhejni hai → Resend setup → API call → Error handling
  (Har jagah same code repeat)

Function ke saath:
  sendEmail({ to, subject, html })
  (Ek line mein kaam ho gaya)
```

**Code:**

```typescript
// lib/email.ts

import { Resend } from "resend";

const resend = new Resend(process.env.RESEND_API_KEY);

// Reusable function — parameters ke saath
export async function sendEmail({ to, subject, html }: { to: string; subject: string; html: string }) {
  // Template se copy — parameters ke saath
  await resend.emails.send({
    from: "onboarding@resend.dev",
    to,
    subject,
    html,
  });
}
```

**Explanation:**
- `export async function` — function ko export karte hain taaki doosre files use kar sakein
- `async` — kyunki `send()` asynchronous hai
- `{ to, subject, html }` — object destructuring, teen parameters
- `{ to: string; subject: string; html: string }` — TypeScript types
- `await` — kyunki `send()` promise return karta hai

**Note:** Yeh abhi basic version hai. Aage error handling add karenge.

#### Step 5.3: Return Value Handle Karo

*"Resend ka `send()` method ek object return karta hai jisme `data` aur `error` hota hai. Dono ko handle karna padega."*

```typescript
// lib/email.ts

import { Resend } from "resend";

const resend = new Resend(process.env.RESEND_API_KEY);

export async function sendEmail({ to, subject, html }: { to: string; subject: string; html: string }) {
  // Resend ka response handle karo
  const { data, error } = await resend.emails.send({
    from: "onboarding@resend.dev",
    to,
    subject,
    html,
  });

  // Response check karo
  if (error) {
    console.error("Failed to send email:", error);
    throw error;
  }

  // Success — return karo
  return { success: true, id: data?.id };
}
```

**Explanation:**
- `{ data, error }` — Resend do cheezein return karta hai
- `if (error)` — check karo ki error toh nahi
- `console.error` — error ko console mein dikhao
- `throw error` — error ko upar phenko taaki caller handle kar sake
- `return { success: true, id: data?.id }` — caller ko batane ke liye

#### Step 5.4: Try-Catch Error Handling Add Karo

*"Ab unexpected errors ko handle karne ke liye `try-catch` add karte hain."*

```typescript
// lib/email.ts

import { Resend } from "resend";

const resend = new Resend(process.env.RESEND_API_KEY);

export async function sendEmail({ to, subject, html }: { to: string; subject: string; html: string }) {
  try {
    const { data, error } = await resend.emails.send({
      from: "onboarding@resend.dev",
      to,
      subject,
      html,
    });

    if (error) {
      console.error("Failed to send email:", error);
      throw error;
    }

    return { success: true, id: data?.id };
  } catch (error) {
    console.error("Error sending email:", error);
    throw error;
  }
}
```

**Explanation:**
- `try` block — main code jo error de sakta hai
- `catch` block — unexpected errors handle karta hai
- Dono jagah `throw error` — caller ko error pata chale

#### Final Code:

```typescript
// lib/email.ts

import { Resend } from "resend";

// Resend client initialize karo
const resend = new Resend(process.env.RESEND_API_KEY);

/**

export async function sendEmail({
  to,

  html,

  html: string;

}
```

### Issue 1: Email nahi aaya

**Check:**
- `.env` mein `RESEND_API_KEY` sahi hai?
- API key `re_` se start hoti hai?
- Internet connection chal raha hai?

### Issue 2: "Failed to send email" error

**Reason:** Resend ka free plan mein `onboarding@resend.dev` domain verified nahi hota.

**Solution:**
- Test email apne hi email pe bhejo (jo Resend account se linked hai)
- Ya custom domain verify karo

**Check:**
- `.env` file root directory mein hai? (`package.json` ke saath)
- File ka naam `.env` hai (`.env.local` nahi)?
- Server restart kiya `.env` change karne ke baad?
