## Step 1: Ab Problem Samjho Real Mein

✅ Ab tum login karo:
1.  Login page par sahi email + password daalo
2.  Submit karo
3.  Server par log aa jata hai: `✅ Login 100% successful`
4.  ✅ Bas yahi khatam!

❌ **Ab kya hoga?**
- Browser ko koi idea nahi hai ki user login hai ya nahi
- Page refresh karo → sab kuch wapas default state
- User dashboard par jaane ki koshish karo → wapas login page par aa jaega
- Har request par user ko wapas login karna padega

**Yeh problem hai kyunki HTTP is Stateless Protocol!**

> 🧠 HTTP har request ko alag alag treat karta hai. Server ko nahi pata ki kal jo request aaya tha woh aaj wahi user hai ya nahi. Har baar naya request hai.

---

## Step 2: Hum Kya Kar Sakte Hain?

Humko ek tareeqa chahiye jisse **browser apne aap har request ke saath ek identification bheje** jisse server ko pata chale ki yeh kon hai.

✅ **3 options hai:**

---

### Option 1: Local Storage mein User ID store karo

```javascript
localStorage.setItem("userId", user.id);
```

❌ **Galat hai bilkul:**
- LocalStorage JS se access ho sakta hai
- Koi bhi XSS attack user ID chura sakta hai
- Koi bhi user ID change karke dusre user ke account mein ghussa sakta hai
- 0 security, kabhi mat use karo login ke liye

---

### Option 2: Cookie mein User ID set karo

```typescript
cookies().set("userId", user.id);
```

✅ **Thik hai lekin abhi bhi problem hai:**
- Cookie automatically har request ke saath bheji jati hai
- JS se access nahi hoti agar `httpOnly` flag hai
- ❌ Lekin phir bhi user cookie mein userId change kar sakta hai
- ❌ Koi bhi user apni cookie mein `userId=1` edit karke admin account mein ghuss sakta hai

---

### Option 3: Database Session

✅ **Yeh hi sahi tareeqa hai:**

1.  Jab user login kare toh hum database mein ek naya row banayenge Session table mein
2.  Is row mein hoga: `id`, `userId`, `expiresAt`
3.  Sirf is session ka random ID browser ko cookie mein bhejenge
4.  Har next request par browser yeh session ID cookie ke roop mein bhejega
5.  Server us session ID se database mein user find karega

✅ **Kya fayda hai:**
- Cookie mein sirf random ID hoti hai, user id nahi
- Koi bhi is random ID ko guess nahi kar sakta
- Koi bhi isko edit karke dusre user ke account mein nahi ja sakta
- Hum server par kisi bhi time session ko revoke kar sakte hain
- Hum logout par session ko database se delete kar sakte hain

---

## Step 3: Cookie Only vs Database Session

| Feature | Cookie Only Session | Database Session |
|---|---|---|
| Security | Low | High |
| Revoke Session | Impossible | Possible |
| Logout from all devices | Impossible | Possible |
| Session expiry | Easy | Easy |
| Change password effect | No effect | All sessions logout |
| XSS Risk | High | Low |

✅ **Isliye hum Database backed session use karenge.**
