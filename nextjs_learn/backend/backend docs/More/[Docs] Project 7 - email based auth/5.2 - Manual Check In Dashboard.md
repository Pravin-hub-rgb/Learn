# Doc 5.2 — Manual Check In Dashboard

## Abhi Tak Kya Hua

- Humne blank dashboard page banaya hai
- Abhi koi bhi user isko access kar sakta hai
- Bina login wale bhi aa rahe hai
- ✅ **Aaj hum isme manual check lagayenge**

---

## Aaj Ka Goal (5.2)

✅ Dashboard mein `getSession()` call karenge
✅ Agar user login nahi hai toh login page par redirect karenge
✅ Ab sirf logged in user hi aa sakta hai
✅ ✅ Phir hum iske baad problem dikhayenge ki yeh tareeka accha nahi hai.

---

## Step 1: Dashboard mein check lagao

✅ Ab `app/(protected)/dashboard/page.tsx` file open karo.

Ab isme changes karo:

```tsx
import { getSession } from "@/lib/session";
import { redirect } from "next/navigation";

export default async function DashboardPage() {
  const session = await getSession();

  if (!session) {
    redirect("/login");
  }

  return (
    <div>
      <h1>Dashboard</h1>
      <p>Yeh sirf logged in user hi dekh sakta hai!</p>
      <p>Welcome: {session.email}</p>
    </div>
  );
}
```

---

## Step 2: Ab Test Karo

✅ Ab aise test karo:

1.  Logout karo
2.  Direct `/dashboard` par jao
3.  ✅ Ab tum automatically login page par chala jaoge

4.  Ab login karo
5.  Ab `/dashboard` par jao
6.  ✅ Ab tum dashboard dekh sakte ho

✅ Ab problem to fix ho gayi hai. Sirf logged in user hi aa sakta hai.

---

## 🚨 AB NAYA PROBLEM AA GAYA HAI

> 🧠 Ab socho:
>
> Agar tumhare paas 10 protected pages hai:
> - Dashboard
> - Profile
> - Settings
> - Orders
> - Payments
> - etc.
>
> Toh tumko har ek page par yeh 3 line likhni padegi:
>
> ```tsx
> const session = await getSession();
>
> if (!session) {
>   redirect("/login");
> }
> ```
>
> 10 pages → 10 baar same code.
> 100 pages → 100 baar same code.

---

## Yehi Hai Code Repeat Ka Problem

| Problem | Kya hai? |
|---|---|
| ✅ | Ab sirf logged in user hi aa sakte hai |
| ❌ | Har page par same code repeat ho raha hai |
| ❌ | Kal agar tumhe check mein kuch change karna hai toh 10 pages edit karna padega |
| ❌ | Koi bhi page mein agar tum yeh code bhool jaoge toh woh page unprotected ho jayega |
| ❌ | DRY Principle toot raha hai - Don't Repeat Yourself |

✅ Yeh solution ek page ke liye thik hai. Lekin scalable nahi hai.

---

## Ab Aage Kya Hoga?

Hum ab next phase mein is problem ka solution nikalenge. Hum ek common helper function banayenge jisse hum har jagah ye code repeat nahi karna padega.
