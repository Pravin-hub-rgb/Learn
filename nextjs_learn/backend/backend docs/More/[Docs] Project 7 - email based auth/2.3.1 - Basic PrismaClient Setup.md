# Doc 2.3.1 — Basic PrismaClient Setup: Pehla Attempt

## Abhi Tak Kya Hua
- **Doc 2.1 mein** humne `User` model define kiya — `schema.prisma` mein
- **Doc 2.2 mein** humne migration run ki — `users` table database mein ban gayi
- Database ready hai — ab humein Prisma client use karke database operations karne hain

## Broader Goal
**Aaj kya karenge:**
- `lib/prisma.ts` file banayenge — jahan se hum PrismaClient ko import karenge
- Doosre files mein import karke CRUD operations karenge
- Lekin Prisma v7 mein architecture change ho gaya hai — seedha `@prisma/client` import nahi hota
- Generated path se import karna padta hai

**Kyun alag se `lib/prisma.ts` file?**
- Hum chahte hain ki ek hi jagah Prisma client configure ho
- Phir doosre files mein import karke use kar sakein
- Example: `app/api/auth/register/route.ts` mein `import prisma from "@/lib/prisma"`
- Phir hum likh sakte hain: `prisma.user.create()` ya `prisma.user.findUnique()`

## Step-by-Step Implementation

### Step 1: Goal Samjho — Humein Kya Karna Hai?

Humne migration run kar diya — `users` table ban gayi. Ab humein is table mein data daalna hai, read karna hai, update karna hai, delete karna hai.

**CRUD operations karne ke liye humein chahiye:**
- Koi object jo database se connect ho
- Jisme methods ho — `create()`, `findUnique()`, `update()`, `delete()`
- Jo humein TypeScript ke saath type safety de

**Yeh object hai — `PrismaClient`!**

### Step 2: Prisma v7 Architecture Samjho

**Important:** Prisma v7 mein architecture change ho gaya hai!

- Pehle (v5/v6): `import { PrismaClient } from "@prisma/client"` — seedha import hota tha
- Ab (v7): Generated path se import karna padta hai — `import { PrismaClient } from "../app/generated/prisma/client"`

**Kyun?**
- Humne `prisma.config.ts` mein output path set kiya hai: `"../app/generated/prisma"`
- Prisma client generate hone ke baad usi folder mein store hota hai
- Isliye relative path se import karna padta hai

### Step 3: `lib/prisma.ts` File Banao

`lib/` folder banao (agar nahi hai) aur usme `prisma.ts` file create karo:

**Kya code likhna hai:**
1. `PrismaClient` ko generated path se import karo
2. Uska instance banao
3. Export kar do

```typescript
// lib/prisma.ts
import { PrismaClient } from "../app/generated/prisma/client";

const prisma = new PrismaClient();

export default prisma;
```

### Step 4: Error Aayega!

Jab tum yeh code run karoge, error aayega:

```
PrismaClientInitializationError: PrismaClient requires a driver adapter. 
Please provide one.
```

**Kyun?**
- Prisma v7 ne architecture change kar diya
- Ab Prisma client directly database se connect nahi karta
- Humein ek "driver adapter" use karna padta hai
- Driver adapter database aur Prisma client ke beech mein bridge ka kaam karta hai

**Agla step:** Driver adapter integrate karna — woh hum next doc (2.3.2) mein karenge.

## Common Issues & Solutions

### Issue 1: `Cannot find module '../app/generated/prisma/client'`
**Error:**
```
Cannot find module '../app/generated/prisma/client'
```

**Reason:** Generated client nahi hai ya path galat hai

**Solution:**
```bash
# Prisma client regenerate karo
npx prisma generate

# Check karo ki folder exist karta hai
ls app/generated/prisma/
```

### Issue 2: `PrismaClient requires a driver adapter`
**Error:**
```
PrismaClientInitializationError: PrismaClient requires a driver adapter.
```

**Reason:** Prisma v7 mein driver adapter mandatory hai

**Solution:** Yeh expected hai — next doc (2.3.2) mein hum driver adapter integrate karenge.

### Issue 3: `Cannot find module '@/lib/prisma'`
**Error:**
```
Cannot find module '@/lib/prisma'
```

**Reason:** Path alias `@` configure nahi hai

**Solution:**
- `tsconfig.json` check karo — `@/*` path alias hona chahiye
- Ya direct relative path use karo: `import prisma from "../../lib/prisma"`

- Prisma v7 mein architecture change ho gaya hai
