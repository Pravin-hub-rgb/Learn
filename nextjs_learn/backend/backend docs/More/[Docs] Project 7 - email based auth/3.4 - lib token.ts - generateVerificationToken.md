### Step 2: Random Token Generate Karo

*"Ab random string generate karte hain. Node.js ka built-in `crypto` module use karenge."*

**Developer ka thought process:**

Soch rahe ho: *"Mujhe random string chahiye. Node.js mein `crypto` module hota hai. Usse `randomBytes` use kar sakta hoon."*

```typescript
// lib/token.ts

import { randomBytes } from "crypto";

/**
 * Verification token generate karta hai
 * @param userId - User jiske liye token generate karna hai
 * @returns Generated token
 */
export async function generateVerificationToken(userId: string) {
  // 32 bytes ka random token (64 characters hex string)
  const token = randomBytes(32).toString("hex");
  
  return token;
}
```

**Explanation:**
- `randomBytes(32)` — 32 bytes ka random data generate karta hai
- `.toString("hex")` — hex string mein convert karta hai (64 characters)
- Example output: `a1b2c3d4e5f6...` (64 characters)

**Kyun 32 bytes?**
- 32 bytes = 256 bits of entropy
- Bahut hi secure — guess karna namumkin
- Hex string = 64 characters (manageable length)

### Step 3: Expiry Time Calculate Karo (Important!)

*"Token generate karne ke baad, uski expiry time set karni hoti hai. Yeh step bahut important hai kyunki token ki security isi par depend karti hai."*

**Developer ka thought process:**

Soch rahe ho: *"Mera token toh generate ho gaya. Lekin yeh token kab tak valid rahega? Hamesha ke liye? Nahi! Mujhe expiry set karni hogi. 24 hours ki expiry rakhun — standard hai."*

**Problem:** *"Mujhe 24 hours baad ka time chahiye. Kaise calculate karun?"*

**Solution:** JavaScript ka `Date` object use karenge.

#### Date Kya Hota Hai?

JavaScript mein `Date` object time ko represent karta hai:

```typescript
// Current time
const now = new Date();
console.log(now); // Mon Apr 06 2026 11:30:00 GMT+0530

// Specific time (timestamp in milliseconds)
const specificTime = new Date(1712400000000);
console.log(specificTime); // Some specific date
```

#### `Date.now()` Kya Hai?

`Date.now()` current time ko **milliseconds** mein return karta hai (since January 1, 1970 — Unix Epoch):

```typescript
const currentTime = Date.now();
console.log(currentTime); // 1712400000000 (example)
```

**Example:**
- `Date.now()` = `1712400000000` milliseconds (current time)
- Yeh number har millisecond badhta jata hai

#### 24 Hours Ko Milliseconds Mein Kaise Convert Karein?

```
24 hours = 24 * 60 minutes * 60 seconds * 1000 milliseconds

Step by step:
1 hour = 60 minutes
1 minute = 60 seconds
1 second = 1000 milliseconds

Toh:
24 hours = 24 * 60 * 60 * 1000 milliseconds
         = 86,400,000 milliseconds
```

**Code mein:**
```typescript
const millisecondsIn24Hours = 24 * 60 * 60 * 1000;
console.log(millisecondsIn24Hours); // 86400000
```

#### Expiry Time Calculate Karo

Ab current time mein 24 hours add karte hain:

```typescript
// Current time (milliseconds)
const currentTime = Date.now(); // e.g., 1712400000000

// 24 hours add karo
const expiryTime = currentTime + (24 * 60 * 60 * 1000);
// e.g., 1712400000000 + 86400000 = 1712486400000

// Isko Date object mein convert karo
const expiresAt = new Date(expiryTime);
// e.g., Tue Apr 07 2026 11:30:00 GMT+0530 (24 hours later)
```

**Complete example:**
```typescript
// Abhi: Mon Apr 06 2026 11:30:00
const expiresAt = new Date(Date.now() + 24 * 60 * 60 * 1000);
// expiresAt: Tue Apr 07 2026 11:30:00 (24 hours later)
```

### Step 4: Database Mein Save Karo

*"Ab token aur expiry time ko `VerificationToken` table mein save karte hain."*

**Developer ka thought process:**

Soch rahe ho: *"Mera token generate ho gaya, expiry time calculate ho gayi. Ab in dono ko database mein save karna hai. `VerificationToken` table mein teen fields hain: `userId`, `token`, `expiresAt`. Sab data ready hai."*

```typescript
// lib/token.ts

import { randomBytes } from "crypto";
import { prisma } from "./prisma";

/**
 * Verification token generate karta hai aur database mein save karta hai
 * @param userId - User jiske liye token generate karna hai
 */
export async function generateVerificationToken(userId: string) {
  // Step 1: 32 bytes ka random token generate karo (64 characters hex string)
  const token = randomBytes(32).toString("hex");
  
  // Step 2: 24 hours ki expiry time calculate karo
  const expiresAt = new Date(Date.now() + 24 * 60 * 60 * 1000);
  
  // Step 3: Token ko database mein save karo
  await prisma.verificationToken.create({
    data: {
      userId,      // Kis user ka token hai
      token,       // Random string (64 characters)
      expiresAt,   // 24 hours from now
    },
  });
  
  // Note: Abhi token return nahi kar rahe — Phase 3.5 mein zaroorat padegi tab karenge
}
```

**Flow samjho:**

```
1. Token generate karo
   └─ randomBytes(32).toString("hex")
   └─ Result: "a1b2c3d4e5f6..." (64 characters)

2. Expiry time calculate karo
   └─ new Date(Date.now() + 24 * 60 * 60 * 1000)
   └─ Result: Tue Apr 07 2026 11:30:00 (24 hours later)

3. Database mein save karo
   └─ prisma.verificationToken.create({ data: { userId, token, expiresAt } })
   └─ Result: Token saved in database
```

**Explanation:**
- `prisma.verificationToken.create()` — naya token database mein save karta hai
- `userId` — kis user ka token hai (foreign key)
- `token` — random string (64 characters hex)
- `expiresAt` — 24 hours from now (Date object)

**Kya database mein save hoga:**

```
VerificationToken table:
id | userId | token | expiresAt
1  | user_123 | a1b2c3d4e5f6... | 2026-04-07 11:30:00
```

**Important note:** `expiresAt` ko hum `Date` object mein save kar rahe hain. Prisma automatically isko PostgreSQL ke `TIMESTAMP` type mein convert kar deta hai.

**Problem-First Approach:**

Soch rahe ho: *"Token return kyun nahi kar rahe? Baad mein email link banane ke liye toh chahiye hoga?"*

**Answer:** Yeh humara "problem-first approach" hai!

- **Abhi (3.4):** Sirf token generate aur save kar rahe hain. Abhi koi use nahi hai.
- **Phase 3.5 mein:** Jab `registerUser()` action mein email bhejne ka problem aayega, tab:
  - "Arre! Mujhe token chahiye email link banane ke liye!"
  - Tab `generateVerificationToken()` ko modify karenge to return kare
  - Tab samajh aayega ki return kyun zaroori tha

Yeh approach behtar hai kyunki:
1. Pehle problem aati hai (email link banana hai)
2. Phir solution introduce hota hai (token return karna)
3. Tab samajh aata hai ki solution kyun zaroori tha

### Step 4: Error Handling Add Karo

*"Agar token generate karne mein error aaya toh handle karna padega."*

```typescript
// lib/token.ts

import { randomBytes } from "crypto";
import { prisma } from "./prisma";

/**
 * Verification token generate karta hai aur database mein save karta hai
 * @param userId - User jiske liye token generate karna hai
 */
export async function generateVerificationToken(userId: string) {
  try {
    // 32 bytes ka random token (64 characters hex string)
    const token = randomBytes(32).toString("hex");
    
    // 24 hours ki expiry
    const expiresAt = new Date(Date.now() + 24 * 60 * 60 * 1000);
    
    // Database mein save karo
    await prisma.verificationToken.create({
      data: {
        userId,
        token,
        expiresAt,
      },
    });
  } catch (error) {
    console.error("Error generating verification token:", error);
    throw new Error("Failed to generate verification token");
  }
}
```

### Step 5: Purane Tokens Ko Cleanup Karo (Optional)

*"Agar user ne multiple times register kiya, toh purane tokens delete kar sakte hain."*

```typescript
// lib/token.ts

import { randomBytes } from "crypto";
import { prisma } from "./prisma";

/**
 * Verification token generate karta hai aur database mein save karta hai
 * @param userId - User jiske liye token generate karna hai
 */
export async function generateVerificationToken(userId: string) {
  try {
    // 32 bytes ka random token (64 characters hex string)
    const token = randomBytes(32).toString("hex");
    
    // 24 hours ki expiry
    const expiresAt = new Date(Date.now() + 24 * 60 * 60 * 1000);
    
    // Purane tokens delete karo (agar koi hai)
    await prisma.verificationToken.deleteMany({
      where: {
        userId,
      },
    });
    
    // Naya token database mein save karo
    await prisma.verificationToken.create({
      data: {
        userId,
        token,
        expiresAt,
      },
    });
  } catch (error) {
    console.error("Error generating verification token:", error);
    throw new Error("Failed to generate verification token");
  }
}
```

**Kyun purane tokens delete kar rahe hain?**
- Ek user ka ek hi active token hona chahiye
- Purane tokens se confusion ho sakta hai
- Security — purane tokens expire ho jate hain
