# Doc 2.2 ‚Äî Pehli Migration

## Abhi Tak Kya Hua
- **Doc 2.1 mein** humne `User` model define kiya ‚Äî `schema.prisma` mein
- 5 fields: `id`, `email`, `password`, `isVerified`, `createdAt`
- `isVerified` kyun zaroori hai ‚Äî fake accounts filter karne ke liye
- Ab humare paas schema ready hai ‚Äî lekin database mein abhi tak koi table nahi hai

## Broader Goal
**Aaj kya karenge:**
- `npx prisma migrate dev` chalayenge ‚Äî jo `User` model ke liye database table banayega
- Migration SQL file dekhenge ‚Äî samjhenge ki Prisma ne kya SQL run kiya
- Database verify karenge ‚Äî pgAdmin ya psql mein table check karenge

**Migration kya hota hai?**
- Prisma schema changes ko database mein apply karne ka process
- Har migration ek SQL file generate karta hai
- SQL file dekh kar hum samajh sakte hain ki Prisma ne kya changes kiye
- Migrations version control mein track hote hain ‚Äî pata chalta hai ki kab kya change hua

## Step-by-Step Implementation

### Step 1: Migration command chalao

Terminal mein yeh command chalao:

```bash
npx prisma migrate dev --name init
```

**Command breakdown:**
- `npx prisma migrate dev` ‚Äî Development migration run karta hai
- `--name init` ‚Äî Migration ka naam "init" hai (pehli migration)

**Kya hoga:**
1. Prisma schema check karega ‚Äî `User` model ko dekhega
2. SQL file generate karega ‚Äî `users` table banane ke liye
3. Database mein table banayega
4. Prisma client regenerate karega

### Step 2: Migration SQL file dekho

Migration file kholo ‚Äî yeh file `prisma/migrations/` folder mein banegi:

```
prisma/migrations/
  ‚îî‚îÄ 20240101_init/
      ‚îî‚îÄ migration.sql
```

`migration.sql` file kholo ‚Äî isme SQL code hoga:

```sql
-- CreateTable
CREATE TABLE "users" (
    "id" TEXT NOT NULL,
    "email" TEXT NOT NULL,
    "password" TEXT NOT NULL,
    "isVerified" BOOLEAN NOT NULL DEFAULT false,
    "createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP,

    CONSTRAINT "users_pkey" PRIMARY KEY ("id")
);

-- CreateIndex
CREATE UNIQUE INDEX "users_email_key" ON "users"("email");
```

**Har line ka matlab:**

| SQL Line | Meaning |
|----------|---------|
| `CREATE TABLE "users"` | `users` table banegi (Prisma model name `User` se automatically `users` ban jata hai) |
| `"id" TEXT NOT NULL` | `id` column ‚Äî TEXT type, NULL nahi ho sakta |
| `"email" TEXT NOT NULL` | `email` column ‚Äî TEXT type, NULL nahi ho sakta |
| `"password" TEXT NOT NULL` | `password` column ‚Äî TEXT type, NULL nahi ho sakta |
| `"isVerified" BOOLEAN NOT NULL DEFAULT false` | `isVerified` column ‚Äî BOOLEAN type, default value `false` |
| `"createdAt" TIMESTAMP(3) NOT NULL DEFAULT CURRENT_TIMESTAMP` | `createdAt` column ‚Äî TIMESTAMP type, default current time |
| `CONSTRAINT "users_pkey" PRIMARY KEY ("id")` | Primary key ‚Äî `id` column unique identifier hoga |
| `CREATE UNIQUE INDEX "users_email_key"` | Unique index ‚Äî `email` duplicate nahi ho sakta |

## Common Issues & Solutions

### Issue 1: Migration fail ho gaya
**Error:** `P3005` ya `P3018` ‚Äî database already exists
**Reason:** Pehle se koi purana migration hai
**Solution:**
```bash
# Purana migration reset karo
npx prisma migrate reset

# Dobara migration chalao
npx prisma migrate dev --name init
```

### Issue 2: Database connection error
**Error:** `ECONNREFUSED`
**Reason:** PostgreSQL server nahi chal raha
**Solution:**
```bash
# PostgreSQL start karo
pg_ctl start
```

### Issue 3: Migration file nahi bani
**Reason:** Schema mein error hai
**Solution:**
```bash
# Schema validate karo
npx prisma validate

# Schema format karo
npx prisma format
```
