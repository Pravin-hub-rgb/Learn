# Doc 3.4 — `loading.tsx` vs `<Suspense>` — Kab Kya Use Karo

## Abhi Tak Kya Hua
3.3 mein Suspense apply kiya — movies list alag component mein — header turant dikha — movies baad mein. Dono tools ab familiar hain.

---

## Side By Side Dekho

Dono ne loading handle kiya — par alag tarike se.

**`loading.tsx`:**
- File rakho — kaam ho gaya
- Poora route ke liye — jab bhi us page pe jao — loading dikhe
- Granular control nahi — ya poora page — ya kuch nahi

**`<Suspense>`:**
- Component ko wrap karo manually
- Sirf woh ek component ke liye
- Multiple Suspense — alag alag — ek page pe

---

## Practical Rule — Simple Hai

```
Poora page slow hai?
→ loading.tsx

Page ka ek hissa slow hai?
→ Suspense
```

Real example:

```
/movies  → poori list slow hai → loading.tsx theek hai
/dashboard → kuch fast, kuch slow → Suspense use karo per section
```

---

## Ek Experiment — Dono Saath Kaam Karte Hain

Abhi `/movies` mein dono hain — `loading.tsx` bhi, Suspense bhi.

Kya hoga? Dono clash karenge?

**Verify karo:** `/movies` refresh karo — kya dikhta hai loading mein?

---

`loading.tsx` wali loading **nahi dikhegi** — kyunki `MoviesPage` component ab async nahi — woh turant render hoti hai.

`loading.tsx` tabhi dikha tha jab `page.tsx` khud async thi aur await kar rahi thi.

Ab page sync hai — turant render hoti hai — header dikha — Suspense wala fallback dikha movies ke liye.

```
Pehle (page.tsx async thi):
loading.tsx → page render → movies

Ab (page.tsx sync, Suspense hai):
page render (turant) → header dikha → Suspense fallback → movies
```

---

## Summary

- `loading.tsx` — poora page slow ho tab — simple — file rakho
- `Suspense` — page ka ek hissa slow ho — manually wrap karo
- Dono saath ho sakte hain — clash nahi — apna apna kaam
- `loading.tsx` tabhi trigger hoti hai jab page component itself async aur awaiting ho

## Agla Step

Loading handle ho gayi. Ab agle topic pe: **Error States**. Kya hoga agar fetch fail ho jaaye — network issue — server down — galat URL? Abhi kuch nahi hota — dekhte hain.
