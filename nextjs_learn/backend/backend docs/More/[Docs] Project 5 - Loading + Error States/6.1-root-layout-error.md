# Doc 6.1 — Root Layout Mein Error — Normal `error.tsx` Kaam Nahi Karta — Kyun?

## Abhi Tak Kya Hua
Phase 5 complete — `notFound()` aur `not-found.tsx` samjhe. Ab last phase: Global Error.

---

## Pehle Samjho — `error.tsx` Kaise Kaam Karta Hai

`app/movies/error.tsx` kaam karta hai kyunki — Next.js us route ke **parent** mein error boundary wrap karta hai.

Tree socho:

```
app/layout.tsx          ← Root layout — poori app ka wrapper
└── app/movies/
    ├── error.tsx        ← Error boundary yahan hai
    └── page.tsx         ← Error yahan aai → error.tsx ne catch kiya
```

`error.tsx` **sibling** ki errors catch karta hai — apne level ke page ki.

---

## Problem — Root Layout Mein Error

Agar `app/layout.tsx` mein error aaye — koi usse catch nahi kar sakta.

Kyun? Kyunki koi `error.tsx` usse wrap nahi kar raha — `app/error.tsx` bhi nahi karta — woh layout ke **andar** hota hai — layout ke bahar nahi.

```
app/layout.tsx   ← Error yahan aai
└── app/error.tsx  ← Yeh andar hai — bahar ki error nahi dekh sakta
```

---

## Experiment — Root Layout Mein Error Throw Karo

`app/layout.tsx` kholo — yahan ek throw daalo temporarily:

```tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  throw new Error("Layout crash!")  // ← temporary
  
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

**Verify karo:** Koi bhi page refresh karo — kya hota hai?

---

Development mein Next.js error overlay dikhega. Par `app/movies/error.tsx` ya `app/error.tsx` — **kuch bhi catch nahi kar raha**.

Kyunki layout hi crash ho gayi — koi bhi page render nahi hua — error boundaries bhi nahi.

`throw` hata do `layout.tsx` se wapas normal kar do. ✅

---

## Solution — `global-error.tsx`

Next.js mein ek special file hai: `app/global-error.tsx`

Yeh **root layout ke bahar** wrap hoti hai — isliye layout errors bhi catch kar sakti hai.

Yeh last resort hai — jab koi aur kuch kaam na kare.

---

## Summary

- `error.tsx` siblings ki errors catch karta hai — parent layout ki nahi
- Root `app/layout.tsx` error — koi normal `error.tsx` catch nahi kar sakta
- Solution: `app/global-error.tsx` — layout ke bahar wrap hoti hai

## Agla Step

`global-error.tsx` banayenge — structure dekhenge — kab use karein.
