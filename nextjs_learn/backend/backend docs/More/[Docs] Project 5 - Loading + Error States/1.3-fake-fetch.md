# Doc 1.3 — Fake Fetch Banao — Async Function Se

## Abhi Tak Kya Hua
1.2 mein movies array component mein directly thi — `.map()` se render kiya. Sab hardcoded, sab ek jagah.

---

## Ek Problem Hai — Pehle Dekho

Abhi movies array directly component mein hai. Real app mein aisa nahi hota — data kisi function se aata hai — jo API call karta hai.

Abhi agar hum seedha JSONPlaceholder se fetch karein — toh ek nayi problem aa jaayegi jo hum abhi samajhna nahi chahte.

Toh pehle yeh samajhte hain: **async function kaisa dikhta hai — data return karne ke liye.**

---

## Step 1 — Alag File Banao Fetch Ke Liye

Real projects mein data fetching ka code component mein nahi hota — alag file mein hota hai.

Kyun? Kyunki kal ko dusri jagah bhi same data chahiye — toh function copy nahi karna, sirf import karna.

`app/movies/` folder mein ek naya file banao: `lib.ts`

```
app/
└── movies/
    ├── page.tsx
    └── lib.ts    ← yeh naya banao
```

---

## Step 2 — Pehle Simple Function

`lib.ts` mein pehle bilkul simple function — koi async nahi:

```ts
export function getMovies() {
  return [
    { id: 1, title: "Inception", year: 2010, rating: 8.8 },
    { id: 2, title: "Interstellar", year: 2014, rating: 8.6 },
    { id: 3, title: "The Dark Knight", year: 2008, rating: 9.0 },
  ]
}
```

`export` kyun? — Taaki doosri files mein `import` kar sakein.

**Verify karo:** File save karo — koi error nahi. ✅

---

## Step 3 — Page Mein Import Karo

`app/movies/page.tsx` mein abhi movies array hai component ke upar — woh hata do. Uski jagah `getMovies` import karo:

```tsx
import { getMovies } from './lib'

export default function MoviesPage() {
  const movies = getMovies()
  console.log("Movies:", movies)

  return (
    <div>
      <h1>Movies</h1>
      {movies.map((movie) => (
        <div key={movie.id}>
          <p>{movie.title}</p>
          <p>{movie.year} • ⭐ {movie.rating}</p>
        </div>
      ))}
    </div>
  )
}
```

**Verify karo:** Terminal mein movies print ho rahi hain? Browser mein movies dikh rahi hain? ✅

---

## Step 4 — Ab Async Banao

Real API calls `async` hoti hain — matlab: "yeh kaam time lagega, ruko."

`lib.ts` mein function ko async banao:

```ts
export async function getMovies() {
  return [
    { id: 1, title: "Inception", year: 2010, rating: 8.8 },
    { id: 2, title: "Interstellar", year: 2014, rating: 8.6 },
    { id: 3, title: "The Dark Knight", year: 2008, rating: 9.0 },
  ]
}
```

`async` function hamesha ek **Promise** return karta hai. Promise matlab: "abhi nahi, baad mein milega result."

---

## Step 5 — Page Mein `await` Karo

Ab `getMovies()` async hai — toh page mein `await` karna padega. `await` matlab: "ruko jab tak Promise resolve na ho — phir aage bado."

`await` use karne ke liye function bhi `async` hona chahiye:

```tsx
import { getMovies } from './lib'

export default async function MoviesPage() {
  const movies = await getMovies()
  console.log("Movies:", movies)

  return (
    <div>
      <h1>Movies</h1>
      {movies.map((movie) => (
        <div key={movie.id}>
          <p>{movie.title}</p>
          <p>{movie.year} • ⭐ {movie.rating}</p>
        </div>
      ))}
    </div>
  )
}
```

**Verify karo:** Terminal mein movies aa rahi hain? Browser same dikh raha hai? ✅

Kuch nahi badla visually — par ab structure ready hai real fetch ke liye.

---

## Summary

- Data fetch ka code alag file mein rakha — `lib.ts`
- `export` kyun — doosri files import kar sakein
- `async`/`await` ka basic flow samjha — Promise kya hota hai
- Server Component `async` ho sakta hai — Next.js allow karta hai

## Agla Step

Abhi `getMovies` mein hardcoded data hai. Agla step: **real fetch** — JSONPlaceholder se actual data laayenge. Ek line badlegi `lib.ts` mein — dekho kya hota hai.
