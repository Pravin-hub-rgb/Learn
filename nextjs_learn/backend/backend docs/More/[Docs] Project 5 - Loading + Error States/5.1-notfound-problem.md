# Doc 5.1 — `/movies/999` Kholo — Movie Nahi Hai — Phir Bhi Koi Error Nahi — Yeh Galat Kyun?

## Abhi Tak Kya Hua
Phase 4 complete — `error.tsx` banaya — `error.message` dikhhaya — `reset()` button daala. Phase 5 shuru: Not Found states.

---

## Problem Dekhte Hain Pehle

`app/movies/[id]/page.tsx` abhi sirf ID dikhhata hai. Real app mein — ID se movie fetch karni hogi.

Pehle woh banao. `lib.ts` mein ek nayi function:

```ts
export async function getMovieById(id: string) {
  const response = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`)
  const data = await response.json()
  return data
}
```

---

## Step 1 — Detail Page Mein Use Karo

`app/movies/[id]/page.tsx`:

```tsx
import { getMovieById } from '../lib'

export default async function MovieDetailPage({ 
  params 
}: { 
  params: Promise<{ id: string }> 
}) {
  const { id } = await params
  const movie = await getMovieById(id)
  
  console.log("Movie data:", movie)

  return (
    <div>
      <h1>{movie.title}</h1>
      <p>{movie.body}</p>
    </div>
  )
}
```

**Verify karo:** `/movies/1` kholo — terminal mein movie data — page pe title? ✅

---

## Step 2 — Ab `/movies/999` Kholo

JSONPlaceholder pe sirf 100 posts hain — 999 exist nahi karta.

**Verify karo:** `http://localhost:3000/movies/999` kholo — kya hota hai?

---

Terminal mein dekho — `movie` kya print hua?

```
Movie data: {}
```

Empty object. JSONPlaceholder non-existent ID ke liye empty object deta hai — error nahi — 200 OK.

Page pe: sirf blank — `movie.title` undefined — kuch nahi dikhta.

Yeh **wrong UX** hai. User /999 pe gaya — movie nahi mili — koi feedback nahi — blank page. Hona kya chahiye? **404 page** — "Yeh movie exist nahi karti."

---

## Kyun Error Nahi Aayi?

Kyunki fetch successful tha — HTTP 200 — server ne response diya — sirf data empty tha. `error.tsx` tabhi trigger hota hai jab koi exception throw ho — yahan koi exception nahi.

Yeh ek **data-level not found** hai — **network-level error** nahi.

---

## Solution

`getMovieById` ke baad check karo — agar data empty hai — `notFound()` call karo.

`notFound()` — Next.js ka built-in function — 404 response trigger karta hai — aur `not-found.tsx` page dikhata hai.

Woh banate hain agla doc mein.

---

## Summary

- JSONPlaceholder invalid ID pe empty object deta hai — error nahi
- `error.tsx` tabhi kaam karta hai jab exception throw ho
- Data empty hone pe manually handle karna padta hai
- Solution: `notFound()` function — agla doc mein

## Agla Step

`notFound()` function use karenge — phir `not-found.tsx` banayenge custom 404 page ke liye.
