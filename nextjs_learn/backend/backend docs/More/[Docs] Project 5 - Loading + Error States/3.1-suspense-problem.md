# Doc 3.1 — Naya Problem — Ek Component Slow Hai, Poora Page Kyun Wait Kare?

## Abhi Tak Kya Hua
2.3 mein `loading.tsx` ka scope samjha — parent loading child pe bhi apply hoti hai — child override kar sakta hai. Abhi `/movies` pe loading kaam kar raha hai.

---

## Ek Nayi Situation — Socho

Abhi `/movies` pe sirf movies list hai — woh load hoti hai — loading dikhi — phir list aayi.

Par soch — real app mein `/movies` pe do cheezein hoti hain:

1. **Header section** — "Total movies: 10" — yeh instantly available hai — koi fetch nahi
2. **Movies list** — yeh 3 second leta hai

Abhi kya ho raha hai? — Jab tak movies nahi aayi — **poora page blank** (ya loading). Header bhi nahi dikhta — halankay woh instantly available hai.

Yeh galat lagta hai. Header dikhna chahiye turant — movies waala hissa loading dikhhaye jab tak data aaye.

---

## Step 1 — Problem Create Karo

`app/movies/page.tsx` mein ek header section add karo — jo instantly available ho:

```tsx
import { getMovies } from './lib'

export default async function MoviesPage() {
  const movies = await getMovies()

  return (
    <div>
      <div>
        <h1>Movies</h1>
        <p>Total: {movies.length} movies</p>
      </div>

      <div>
        {movies.map((movie: any) => (
          <div key={movie.id}>
            <p>{movie.title}</p>
            <p>{movie.body}</p>
          </div>
        ))}
      </div>
    </div>
  )
}
```

**Verify karo:** `/movies` refresh karo — 3 second blank — phir sab ek saath aata hai. Header bhi delay ke saath aata hai. ✅ (yeh problem hai)

---

## Kyun Ho Raha Hai Yeh?

Kyunki `await getMovies()` — **poori component ruk jaati hai** jab tak data nahi aata.

```
MoviesPage chali
→ await getMovies() — RUK JADO
→ 3 second...
→ data aaya
→ ab poora return() render hua — header bhi, list bhi
```

Header ke liye koi wait nahi chahiye tha — par woh bhi ruk gaya kyunki dono ek hi component mein hain.

---

## Solution Kya Hoga?

Movies list ko alag component mein nikalna hoga — taaki sirf woh ruke — header turant dikhe.

Phir us slow component ko ek special wrapper mein daalna hoga jo bolega: "yeh component load ho raha hai — baaki page mat roko."

Woh wrapper hai — **Suspense**. Agla doc mein banayenge.

---

## Summary

- Problem clearly dekhi — slow component ki wajah se fast content bhi ruk raha hai
- Kyun — `await` poori component ko rok deta hai
- Solution ka direction — alag component + Suspense wrapper

## Agla Step

Movies list ko alag component mein nikalenge — phir `<Suspense>` wrapper se wrap karenge. Dekho kya hota hai.
