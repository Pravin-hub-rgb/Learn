# Doc 6.2 — `global-error.tsx` — Last Resort

## Abhi Tak Kya Hua
6.1 mein samjha — root layout mein error aaye toh normal `error.tsx` kaam nahi karta — kyunki woh layout ke andar hota hai. Solution: `global-error.tsx`.

---

## `global-error.tsx` Ka Structure

`app/global-error.tsx` banao — root `app/` folder mein:

```
app/
├── global-error.tsx   ← naya — root level
├── layout.tsx
├── not-found.tsx
├── page.tsx
└── movies/
    └── ...
```

---

## Step 1 — Basic `global-error.tsx`

```tsx
'use client'

export default function GlobalError({
  error,
  reset,
}: {
  error: Error
  reset: () => void
}) {
  return (
    <html>
      <body>
        <h2>Kuch bahut bura ho gaya 💥</h2>
        <p>{error.message}</p>
        <button onClick={() => reset()}>Try Again</button>
      </body>
    </html>
  )
}
```

**Kuch important cheezein:**

1. `'use client'` — zaroori hai — same reason jaise `error.tsx` mein
2. `<html>` aur `<body>` — zaroor likho — kyunki `global-error.tsx` root layout ki jagah render hoti hai — layout available nahi hota — toh khud provide karna padta hai
3. Props same hain — `error` aur `reset`

---

## Step 2 — Test Karo

`app/layout.tsx` mein temporarily throw daalo:

```tsx
export default function RootLayout({ children }: { children: React.ReactNode }) {
  throw new Error("Layout ne crash kar diya!")
  
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

**Verify karo:** Koi bhi page refresh karo — `global-error.tsx` ka content dikh raha hai?

---

Development mein Next.js phir bhi apna error overlay dikhata hai upar se — par page ke neeche `global-error.tsx` render ho raha hoga.

**Production mein** — `npm run build` karke `npm run start` se — sirf `global-error.tsx` dikhega — overlay nahi.

`throw` hata do `layout.tsx` se. ✅

---

## Step 3 — Kab Use Karo

`global-error.tsx` **last resort** hai:

```
Normal route error     → app/movies/error.tsx
Movies route error     → app/movies/error.tsx
Root layout crash      → app/global-error.tsx
Kuch aur nahi baccha   → app/global-error.tsx
```

Real app mein root layout crash bahut rare hota hai — par defense ke liye rakho.

---

## Poora Error Hierarchy Ek Jagah

```
app/
├── global-error.tsx     ← Root layout crash — last resort
├── not-found.tsx        ← Global 404
├── layout.tsx
└── movies/
    ├── error.tsx         ← /movies aur children ki errors
    ├── loading.tsx       ← /movies aur children ki loading
    └── [id]/
        ├── not-found.tsx  ← Movie-specific 404
        └── page.tsx
```

---

## Summary

- `global-error.tsx` — root `app/` mein — last resort
- `<html>` aur `<body>` likho — layout replace ho jaati hai
- `'use client'` zaroori — same reason
- Same props — `error` aur `reset`
- Production mein test karo properly
