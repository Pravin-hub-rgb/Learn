# Doc 4.1 — Error Throw Karo — Dekho Kya Hota Hai

## Abhi Tak Kya Hua
Phase 3 complete — loading.tsx aur Suspense dono samjhe. Ab Phase 4: Error States.

---

## Abhi Ka Goal

Error handling ka pehla step — **pehle dekho bina handling ke kya hota hai.**

Problem aane do — phir fix karenge.

---

## Step 1 — Fetch Mein Error Throw Karo

`app/movies/lib.ts` kholo. Delay ke baad — artificially error throw karo:

```ts
export async function getMovies() {
  await new Promise((resolve) => setTimeout(resolve, 1000))
  
  throw new Error("Server down hai!")
  
  // Yeh lines ab kabhi nahi challengi — upar throw hai
  const response = await fetch('https://jsonplaceholder.typicode.com/posts')
  const data = await response.json()
  return data.slice(0, 10)
}
```

`throw new Error("...")` — ek error object banata hai aur function rok deta hai — jaisa `return` rok deta hai par error ke saath.

**Verify karo:** `/movies` refresh karo — kya dikhta hai?

---

Development mein Next.js ek error overlay dikhata hai — red screen — error ka message. Yeh **sirf development mein** hota hai — users ko nahi dikhta.

Production mein — **white screen / blank page** — kuch nahi. User confuse ho jaata hai.

Yeh bahut buri UX hai. Fix karni hai.

---

## Step 2 — Observe Karo — Error Ka Nature

Error terminal mein bhi aata hai — dekho:

```
Error: Server down hai!
    at getMovies (lib.ts:4:9)
    ...
```

Yeh **unhandled error** hai — kisi ne catch nahi kiya — React tak pohoncha — React ne crash kar diya.

---

## Yeh Problem Kyun Hoti Hai

Server Components mein `try/catch` nahi kiya — error directly bahar aa gayi — Next.js ko nahi pata kya kare — crash.

Solution: Next.js mein ek special file hoti hai — `error.tsx` — jo automatically catch karti hai errors — aur user ko ek proper page dikhhati hai.

---

## Summary

- `throw new Error()` — error throw kiya manually
- Bina error handling — development mein red overlay — production mein blank page
- Unhandled error — React crash — user confuse

## Agla Step

`error.tsx` banayenge — automatically errors ko catch karegi — user ko proper message dikhayegi.
