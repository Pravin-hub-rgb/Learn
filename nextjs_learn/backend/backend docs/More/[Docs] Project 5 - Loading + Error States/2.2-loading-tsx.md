# Doc 2.2 — `loading.tsx` Banao — Blank Page Fix Karo

## Abhi Tak Kya Hua
2.1 mein real fetch kiya — 3 second delay daali — browser mein `/movies` refresh karne pe **3 second blank page** aa raha hai. User ko kuch pata nahi chal raha.

---

## Problem Ko Clearly Samjho

Abhi kya ho raha hai:

```
User → /movies khola
Next.js → page.tsx chalaya
page.tsx → getMovies() call kiya
getMovies → 3 second ruka... ruka... ruka...
Phir data aaya → page render hua → user ko dikha
```

Beech mein 3 second — **blank**. Koi feedback nahi.

---

## Next.js Ka Solution — `loading.tsx`

Next.js mein ek convention hai: agar `loading.tsx` file rakho kisi route folder mein — toh **jab tak us route ka data load ho raha hai — loading.tsx show hoga automatically**.

Koi extra code nahi — koi state nahi — sirf file rakho.

---

## Step 1 — `loading.tsx` Banao

`app/movies/` folder mein nayi file banao: `loading.tsx`

```
app/
└── movies/
    ├── page.tsx
    ├── lib.ts
    └── loading.tsx   ← yeh naya
```

Andar sirf yeh:

```tsx
export default function MoviesLoading() {
  return (
    <div>
      <p>Movies load ho rahi hain...</p>
    </div>
  )
}
```

**Verify karo:** `/movies` refresh karo — **3 second tak "Movies load ho rahi hain..." dikhega** — phir actual movies. ✅

---

## Step 2 — Andar Se Kya Ho Raha Hai

Yeh magic lagta hai — par actually Next.js ek cheez use kar raha hai internally: **React Suspense**.

Flow aisa hai:

```
User → /movies
Next.js → "page.tsx async hai — data aane do"
          meanwhile → loading.tsx dikhhao
Data aaya → loading.tsx hatao → page.tsx dikhhao
```

`loading.tsx` basically ek Suspense fallback hai — jo Next.js automatically wrap karta hai.

**Important:** `loading.tsx` sirf **us route** ke liye kaam karta hai — matlab `app/movies/loading.tsx` sirf `/movies` pe kaam karega — baaki routes pe nahi.

---

## Step 3 — Home Page Pe Jao — Loading Nahi Aayegi

Home page pe koi async kaam nahi hai — toh wahan loading dikhegi nahi.

**Verify karo:** `http://localhost:3000` refresh karo — instantly load hota hai — koi loading nahi. ✅

Yeh expected behavior hai.

---

## Step 4 — Loading Ko Thoda Better Banao

Abhi loading sirf text hai. Ek simple spinner add karo — CSS animation se:

```tsx
export default function MoviesLoading() {
  return (
    <div style={{ padding: '20px' }}>
      <p>⏳ Movies load ho rahi hain...</p>
    </div>
  )
}
```

**Verify karo:** Refresh karo — loading state dikh raha hai — phir data? ✅

---

## Summary

- `loading.tsx` — Next.js convention — automatically show hoti hai jab page load ho raha ho
- Koi extra code nahi — sirf file rakho sahi jagah
- Internally React Suspense use hota hai
- Scope — sirf us route ke liye jahan file rakhi hai

## Agla Step

Abhi `loading.tsx` poore `/movies` page ke liye hai — jab bhi kuch bhi load ho. Par kya hoga agar page pe **do sections** hों — ek fast, ek slow? Dono ke liye poora page wait karega? Woh problem next doc mein dekhenge — aur uska solution hoga **Suspense** directly.
