# Doc 4.2 — `error.tsx` Banao — Basic Structure

## Abhi Tak Kya Hua
4.1 mein error throw kiya — bina handling ke — development mein red overlay — production mein blank page. Fix karna hai.

---

## `error.tsx` — Next.js Convention

Bilkul `loading.tsx` ki tarah — `error.tsx` ek convention file hai.

Agar `error.tsx` rakho kisi route folder mein — toh jab bhi us route pe error aaye — yeh file automatically show hogi.

---

## Step 1 — `error.tsx` Banao

`app/movies/` folder mein nayi file: `error.tsx`

```
app/
└── movies/
    ├── [id]/
    │   └── page.tsx
    ├── error.tsx     ← naya
    ├── loading.tsx
    ├── lib.ts
    ├── MoviesList.tsx
    └── page.tsx
```

Pehle bilkul simple:

```tsx
'use client'

export default function MoviesError() {
  return (
    <div>
      <h2>Kuch galat ho gaya!</h2>
      <p>Movies load nahi ho payi.</p>
    </div>
  )
}
```

**Pehle `'use client'` kyun?** — Abhi mat socho — agla doc mein explain karenge. Abhi bas rakho.

**Verify karo:** `/movies` refresh karo — ab red overlay nahi — "Kuch galat ho gaya!" dikhega. ✅

---

## Step 2 — Scope Same Hai Jaise `loading.tsx`

`app/movies/error.tsx` — `/movies` aur uske child routes pe kaam karega.

**Verify karo:** `/movies/1` pe jao — wahan bhi koi error hai? (Nahi honi chahiye — woh page apna data fetch karta hai alag se)

`/movies/1` normal dikhega — sirf `/movies` pe error hai — kyunki sirf wahan `throw` kiya. ✅

---

## Step 3 — Error Ko `lib.ts` Mein Temporarily Rakhne Do

Ab tak `throw new Error(...)` fake tha — artificial. Aglar docs ke liye ise aisa rakhna chahte hain.

```ts
export async function getMovies() {
  await new Promise((resolve) => setTimeout(resolve, 1000))
  throw new Error("Server down hai!")
  // ...
}
```

Theek hai — agle docs mein error ke saath kaam karenge — phir hata denge.

---

## Summary

- `error.tsx` — convention file — automatically error catch karti hai us route pe
- `'use client'` zaroori hai — kyun — agla doc mein
- Scope — same as `loading.tsx` — us route + children

## Agla Step

`'use client'` kyun zaroori hai `error.tsx` mein — andar se samjhenge.
