## Pehle Suspense Ka Concept — Form Se Alag Example Se

Seedha movies mein mat jao — pehle simple example se samjho.

Socho tumhare paas ek restaurant ka page hai:

```
RestaurantPage
├── RestaurantName    ← instant — hardcoded
├── Menu              ← slow — database se fetch
└── Reviews           ← slow — alag API
```

Bina Suspense ke — poora page ruka rahega jab tak Menu aur Reviews dono load na ho jaayein.

Suspense se — RestaurantName turant dikhe — Menu aur Reviews ka apna apna loading ho.

Suspense ek wrapper hai jo bolata hai React ko:

> "Yeh component andar slow hai — jab tak load ho raha hai — yeh fallback dikhhao. Baaki page mat roko."

```tsx
<Suspense fallback={<p>Menu load ho raha hai...</p>}>
  <Menu />   {/* yeh slow hai */}
</Suspense>
```

`fallback` — woh cheez jo Suspense dikhata hai jab andar wala component wait kar raha ho.

---

## Suspense Kaam Kab Karta Hai

Suspense tab kaam karta hai jab andar wala component **async** ho — matlab koi `await` kar raha ho.

```tsx
// Yeh component async hai — Suspense isko handle karega
async function Menu() {
  const items = await fetchMenu()  // slow
  return <div>{items.map(...)}</div>
}

// Page mein:
<Suspense fallback={<p>Loading...</p>}>
  <Menu />
</Suspense>
```

Flow:

```
Page render hua
→ Menu encounter hua — async hai — Suspense ne pakda
→ Fallback dikhhaya — "Loading..."
→ Menu ka data aaya — Suspense ne fallback hataya — Menu dikhhaya
→ Baaki page already dikh raha tha — usse koi farak nahi pada
```

---

## `loading.tsx` vs Suspense — Alag Kab?

| | `loading.tsx` | `<Suspense>` |
|---|---|---|
| **Scope** | Poora route/page | Sirf woh ek component |
| **Control** | Automatic — file rakho | Manual — khud wrap karo |
| **Use karo jab** | Poora page load ho raha ho | Page ka ek hissa slow ho |

Ab movies pe apply karte hain.

---

## Summary

- Suspense — ek wrapper — andar slow component — bahar fallback
- `fallback` prop — kya dikhhao jab wait ho
- Sirf async components ke liye kaam karta hai
- `loading.tsx` = poora page; `Suspense` = component level
