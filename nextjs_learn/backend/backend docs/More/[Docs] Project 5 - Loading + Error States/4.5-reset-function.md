# Doc 4.5 — `reset()` Function — Retry Bina Refresh Ke

## Abhi Tak Kya Hua
4.4 mein `error` prop use kiya — error message UI mein dikhhaya. Abhi error page pe koi action nahi — user sirf padh sakta hai — kuch kar nahi sakta.

---

## Problem — User Kya Kare?

Error aa gayi — user kya kare? Page refresh kare? Nahi pata.

Better UX — ek "Try Again" button — jisse page retry kare — shayad is baar kaam kare.

---

## `reset` Prop Kya Hai

Next.js `error.tsx` ko ek aur prop deta hai — `reset` — ek function hai.

Jab `reset()` call karo — Next.js woh route dobara render karne ki koshish karta hai — bina full page refresh ke.

---

## Step 1 — `reset` Console Mein Dekho

```tsx
'use client'

export default function MoviesError({ 
  error, 
  reset 
}: { 
  error: Error
  reset: () => void 
}) {
  console.log("reset function:", reset)

  return (
    <div>
      <h2>Kuch galat ho gaya!</h2>
      <p>Error: {error.message}</p>
    </div>
  )
}
```

`() => void` — TypeScript type — ek function jo koi argument nahi leta aur kuch return nahi karta.

**Verify karo:** `/movies` refresh karo — browser console mein `reset` kya hai? — Function dikhega. ✅

---

## Step 2 — Button Daalo

```tsx
'use client'

export default function MoviesError({ 
  error, 
  reset 
}: { 
  error: Error
  reset: () => void 
}) {
  return (
    <div>
      <h2>Kuch galat ho gaya!</h2>
      <p>Error: {error.message}</p>
      <button onClick={() => reset()}>
        Try Again
      </button>
    </div>
  )
}
```

`onClick={() => reset()}` — button click pe `reset` function call ho.

**Verify karo:** Error page pe "Try Again" button dikh raha hai? Click karo — kya hota hai?

---

Page dobara render karne ki koshi karega — phir error aayegi — phir error page — kyunki `lib.ts` mein abhi bhi throw hai.

Par button kaam kar raha hai — flow theek hai. ✅

---

## Step 3 — `throw` Hata Do — Normal Flow Restore Karo

`lib.ts` mein se throw hata do — real fetch wapas laao:

```ts
export async function getMovies() {
  await new Promise((resolve) => setTimeout(resolve, 1000))
  
  const response = await fetch('https://jsonplaceholder.typicode.com/posts')
  const data = await response.json()
  return data.slice(0, 10)
}
```

**Verify karo:** `/movies` — movies load ho rahi hain? ✅

Ab "Try Again" click karo error page pe — (error abhi nahi hai toh directly movies dikhengi — par flow samajh aa gayi)

---

## Summary

- `reset` prop — function — page retry karta hai bina full refresh
- `() => void` — TypeScript — function type — no args, no return
- `onClick={() => reset()}` — button pe click handler
- Real app mein — intermittent errors ke liye useful — network flicker etc.
