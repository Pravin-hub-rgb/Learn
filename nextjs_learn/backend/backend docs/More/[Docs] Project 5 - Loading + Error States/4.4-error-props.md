# Doc 4.4 — `error.tsx` Ko Props Milte Hain — `error.message` Use Karo

## Abhi Tak Kya Hua
4.3 mein samjha kyun `'use client'` zaroori hai — Error Boundary client side kaam karta hai. Abhi `error.tsx` mein generic message hai — "Kuch galat ho gaya". Real error message nahi dikha.

---

## Props Kya Milte Hain

Next.js `error.tsx` ko automatically do props deta hai:

1. `error` — actual Error object — jisme `.message` hota hai
2. `reset` — ek function — page retry karne ke liye (agla doc mein)

Abhi `error` prop use karte hain.

---

## Step 1 — Console Mein Pehle Dekho

`error.tsx` mein console.log daalo:

```tsx
'use client'

export default function MoviesError({ error }: { error: Error }) {
  console.log("Error aaya:", error)
  console.log("Error message:", error.message)

  return (
    <div>
      <h2>Kuch galat ho gaya!</h2>
    </div>
  )
}
```

**Yahan `{ error }: { error: Error }` kya hai?**

- `{ error }` — destructuring — props object se sirf `error` nikaala
- `: { error: Error }` — TypeScript — batao ki props mein `error` hoga jo `Error` type ka hai
- `Error` — JavaScript ka built-in error type — jisme `.message`, `.stack` etc hote hain

**Verify karo:** `/movies` refresh karo — browser console mein (F12) kya aata hai?

---

Browser console mein dikhega:

```
Error aaya: Error: Server down hai!
Error message: Server down hai!
```

`error.message` — wahi string jo `throw new Error("Server down hai!")` mein likhi thi. ✅

---

## Step 2 — Message UI Mein Dikhhao

Ab console.log hatao — directly UI mein dikhhao:

```tsx
'use client'

export default function MoviesError({ error }: { error: Error }) {
  return (
    <div>
      <h2>Kuch galat ho gaya!</h2>
      <p>Error: {error.message}</p>
    </div>
  )
}
```

**Verify karo:** `/movies` refresh karo — "Error: Server down hai!" dikh raha hai page pe? ✅

---

## Step 3 — Real Error Bhi Test Karo

`lib.ts` mein throw ki jagah galat URL use karo:

```ts
export async function getMovies() {
  await new Promise((resolve) => setTimeout(resolve, 1000))
  
  const response = await fetch('https://jsonplaceholder.typicode.com/GALAT-URL')
  
  if (!response.ok) {
    throw new Error(`Fetch fail hua: ${response.status}`)
  }
  
  const data = await response.json()
  return data.slice(0, 10)
}
```

`response.ok` — true hai agar status 200-299 — false agar 404, 500 etc.

`!response.ok` — `!` matlab NOT — agar ok nahi hai toh...

`response.status` — HTTP status code — 404 Not Found etc.

**Verify karo:** `/movies` refresh — error message mein status code dikhe? ✅

Theek hai — ab wapas throw wala rakho simple example ke liye agla doc ke liye:

```ts
export async function getMovies() {
  await new Promise((resolve) => setTimeout(resolve, 1000))
  throw new Error("Server down hai!")
  // ...
}
```

---

## Summary

- `error.tsx` ko `error` prop milta hai — Error object
- `error.message` — throw karte waqt jo string di woh
- `{ error }: { error: Error }` — destructuring + TypeScript type
- `response.ok` aur `response.status` — real fetch errors handle karna

## Agla Step

`reset` prop — user bina refresh kiye retry kar sake. Woh next doc mein.
