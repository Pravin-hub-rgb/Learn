# Doc 2.3 — `loading.tsx` Ka Scope — Sirf Us Route Ke Liye

## Abhi Tak Kya Hua
2.2 mein `loading.tsx` banaya — `/movies` pe load hone pe automatically dikhta hai. Home page pe nahi dikhta — kyunki wahan async kaam nahi.

---

## Abhi Ka Goal

Ek cheez clearly samajhni hai: `loading.tsx` ka **scope** kya hai — kahan tak kaam karta hai — kahan nahi.

---

## Experiment 1 — `/movies` Ke Andar Ek Aur Route Banao

Socho — har movie ka alag page chahiye — `/movies/1`, `/movies/2` — jaise ye Netflix pe hoti hai.

Next.js mein dynamic route ke liye folder ka naam `[id]` hota hai — brackets mein. Yeh `id` variable ki tarah kaam karta hai.

Banao:

```
app/
└── movies/
    ├── [id]/
    │   └── page.tsx    ← /movies/1, /movies/2 etc.
    ├── loading.tsx
    ├── lib.ts
    └── page.tsx
```

`app/movies/[id]/page.tsx`:

```tsx
export default async function MovieDetailPage({ 
  params 
}: { 
  params: Promise<{ id: string }> 
}) {
  const { id } = await params
  
  await new Promise((resolve) => setTimeout(resolve, 2000))
  
  return (
    <div>
      <h1>Movie ID: {id}</h1>
    </div>
  )
}
```

`params` — Next.js automatically deta hai URL se — `{ id: "1" }` agar `/movies/1` pe ho.

`params` Next.js 15 mein `Promise` hai — toh `await` karna padta hai.

**Verify karo:** `http://localhost:3000/movies/1` kholo — 2 second baad "Movie ID: 1" dikhega? ✅

## Experiment 2 — Kya `/movies/1` Pe Bhi Loading Dikhegi?

Ab socho — `loading.tsx` `app/movies/` mein hai.

`/movies/1` route `app/movies/[id]/` mein hai — jo `app/movies/` ke **andar** hai.

Toh kya parent ki `loading.tsx` child pe bhi apply hogi?

**Verify karo:** `/movies/1` refresh karo — 2 second tak kya dikhta hai?

---

Haan — **parent ki `loading.tsx` child routes pe bhi apply hoti hai.**

Yeh Next.js ka **nested routing** hai — parent ke baad child ka route aata hai — toh parent ke conventions bhi inherit hote hain.

```
/movies       → app/movies/loading.tsx apply hoti hai
/movies/1     → app/movies/loading.tsx apply hoti hai (inherited)
/movies/1/xyz → app/movies/loading.tsx apply hoti hai (inherited)
```

---

## Experiment 3 — Child Ki Apni `loading.tsx`

Ab `app/movies/[id]/` ke andar apni `loading.tsx` banao:

```tsx
export default function MovieDetailLoading() {
  return (
    <div>
      <p>Movie detail load ho rahi hai...</p>
    </div>
  )
}
```

**Verify karo:** `/movies/1` refresh karo — ab kya dikhta hai loading mein?

---

Child ki apni `loading.tsx` hogi toh **woh override karegi parent wali** — us route ke liye.

```
/movies    → "Movies load ho rahi hain..." (parent loading)
/movies/1  → "Movie detail load ho rahi hai..." (child loading — override)
```

---

## Summary

- `loading.tsx` ka scope — us folder ka route + uske saare child routes
- Child ki apni `loading.tsx` hogi toh parent wali override hogi
- Next.js nested routing — parent conventions inherit hote hain children mein
