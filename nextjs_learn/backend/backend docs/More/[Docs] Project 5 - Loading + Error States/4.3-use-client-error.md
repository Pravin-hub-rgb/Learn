# Doc 4.3 — `error.tsx` Mein `'use client'` Kyun Zaroori Hai

## Abhi Tak Kya Hua
4.2 mein `error.tsx` banaya — kaam kar raha hai — par `'use client'` likha bina explain kiye. Ab samjhte hain kyun.

---

## Pehle Samjho — Server Component vs Client Component

Next.js mein do tarah ke components hote hain:

**Server Component** (default):
- Server pe render hota hai
- Data fetch kar sakta hai directly
- Browser mein koi JS nahi jaati — sirf HTML
- `useState`, `useEffect` nahi use kar sakta

**Client Component** (`'use client'` likho):
- Browser mein render hota hai
- `useState`, `useEffect` use kar sakta hai
- Interactivity ke liye

---

## Error Boundary — Kyun Client Chahiye

React mein errors ko catch karne ka mechanism hai — **Error Boundary**. Yeh ek class-based concept hai jo React internally use karta hai.

Error Boundaries sirf **Client side** pe kaam karti hain — Server pe nahi.

Kyun? Kyunki error boundary ka kaam hai — runtime pe error catch karo — user ko fallback dikhhao. Yeh **browser ka kaam** hai — server ka nahi. Server pe error aaye toh directly crash — boundary nahi hoti.

Next.js `error.tsx` ko internally Error Boundary ki tarah wrap karta hai. Isliye `error.tsx` client component hona zaroori hai.

---

## Experiment — `'use client'` Hatao

`error.tsx` se `'use client'` hatao:

```tsx
// 'use client'  ← yeh hata diya

export default function MoviesError() {
  return (
    <div>
      <h2>Kuch galat ho gaya!</h2>
    </div>
  )
}
```

**Verify karo:** `/movies` refresh karo — kya error aati hai?

---

Terminal mein aisa kuch dikhega:

```
Error: error.tsx must be a Client Component.
Add the "use client" directive to the top of the file.
```

Next.js khud bolta hai — `error.tsx` client component hona chahiye.

`'use client'` wapas daalo. ✅

---

## Summary

- Server Components default hain — browser JS nahi jaati
- Client Components — `'use client'` — browser mein chalte hain
- Error Boundary — React ka mechanism — sirf client side kaam karta hai
- Isliye `error.tsx` mein `'use client'` mandatory hai — Next.js enforce karta hai

## Agla Step

`error.tsx` ko `error` prop milta hai — jo actual error object hai — message etc. Woh use karke better error UI banayenge.
