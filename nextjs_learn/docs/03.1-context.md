# 🟢 Ab Real Version — State Ke Saath

Ab tak hum static value de rahe the.
Real world me theme change bhi hoga.

Toh hume state chahiye.

📁 src/context/ThemeContext.tsx

```tsx
import { createContext } from "react";

interface ThemeContextType {
  theme: string;
  setTheme: React.Dispatch<React.SetStateAction<string>>;
}

export const ThemeContext = createContext<ThemeContextType | null>(null);
```

Yaha kya kiya?

Ab context sirf string nahi.
Object hai.

{ theme, setTheme }

# 🟢 Step 4 — Custom Provider Banana

Ab hum ek wrapper component banayenge.

📁 src/context/ThemeProvider.tsx

```tsx
import { useState } from "react";
import { ThemeContext } from "./ThemeContext";

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState("light");

  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}
```

Ab samjho dhyaan se.

children kya hai?

Jo bhi ThemeProvider ke andar likhenge
wo children ke through yaha render hoga.

Ye wrapper isliye banaya
kyunki state ko reusable banana hai.

# 🟢 Step 6 — Context Use + Change Karna

📁 src/Button.tsx

```tsx
import { useContext } from "react";
import { ThemeContext } from "./context/ThemeContext";

function Button() {
  const context = useContext(ThemeContext);

  if (!context) return null;

  const { theme, setTheme } = context;

  return (
    <button
      className={theme}
      onClick={() => setTheme(theme === "light" ? "dark" : "light")}
    >
      Toggle Theme
    </button>
  );
}
```

Ab kya ho raha hai?

Button theme read bhi kar raha hai
aur change bhi kar raha hai.

Change karte hi
Provider ka state update hoga
Aur jitne bhi components context use kar rahe hain
sab re-render honge.

---

# 🧠 Important Mental Model

Context koi global variable nahi hai.

Ye React ka subscription system hai.

Jo component useContext karega
wo value change hone par re-render hoga.

Jo useContext nahi karega
wo re-render nahi hoga.

---

# ⚠️ Kab Context Use Karna Chahiye?

Use karo jab:

* Theme
* Auth user
* Language
* Global settings

Use mat karo har choti cheez ke liye.

Har state ko context me daal doge
performance issues aa sakte hain.

---

# 🆚 Props vs Context

Props:

* Clear
* Explicit
* Small tree ke liye best

Context:

* Deeply nested data ke liye best
* Global type data
