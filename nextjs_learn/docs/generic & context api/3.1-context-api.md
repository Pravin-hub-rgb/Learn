# 3.1 — Context API 🌍

## Pehle Woh Problem Yaad Kar

Props wali file mein tune dekha tha —

```
App
 ↓
Page
 ↓
Layout
 ↓
Dashboard
 ↓
Button  ← isko theme chahiye
```

`theme` ko Button tak pahunchane ke liye **har component** mein prop pass karna pada — chahe unhe zarurat thi ya nahi. Yahi tha **Props Drilling.**

Context API iska solution hai.

---

## Context Ka Simple Idea

Soch building mein **WiFi** hai.

WiFi ka password har ghar mein wire se nahi diya jaata. Ek central router hai — **jo chahiye, seedha connect ho jaao.**

```
[WiFi Router]  ← central jagah hai data
     ↓↓↓
 Room1  Room2  Room3  ← jo chahiye directly le lo
```

Context bhi yehi karta hai:

```
[Context]  ← central jagah data rakho
    ↓↓↓
Navbar  Button  ProfileCard  ← jo chahiye seedha lo
```

Beech mein kisi ko kuch forward nahi karna.

---

## Teen Steps — Hamesha Yahi Hote Hain

Context use karne ke liye hamesha teen cheezein hoti hain:

```
1. CREATE  — Context banao (ek baar)
2. PROVIDE — Data do (parent mein)
3. CONSUME — Data lo (jahan chahiye)
```

Ab ek ek step — **har line tod ke** samjhenge.

---

## STEP 1 — Context Banao (CREATE) 🏗️

Pehle ek naya file banao:

```
app/
└── context/
    └── ThemeContext.tsx   ← yeh file banao
```

```tsx
'use client'

import { createContext } from 'react'

export const ThemeContext = createContext('light')
```

Ab **har line** tod ke:

---

**Line 1:**
```tsx
'use client'
```
Next.js ko bata raha hai — yeh file browser pe chalegi (server pe nahi). Context interactive hai isliye yeh zaroori hai.

---

**Line 2:**
```tsx
import { createContext } from 'react'
```
`createContext` React ka function hai — use karne se pehle import karna padta hai. Jaise koi tool use karne se pehle box se nikalna padta hai.

---

**Line 3:**
```tsx
export const ThemeContext = createContext('light')
```

Isko aur todenge:

```
export          ← doosri files mein use kar sako isliye
const           ← variable banana
ThemeContext    ← variable ka naam — tune rakha
=               ← assign kar raha hai
createContext   ← React ka function — context banata hai
('light')       ← default value — agar koi Provider na mile toh 'light' use hoga
```

**`createContext('light')` kya karta hai?**

Ek "dabba" banata hai. Abhi usme `'light'` hai by default. Baad mein Provider isme actual data dalega.

---

## STEP 2 — Data Do (PROVIDE) 🎁

Ab jo bhi components Context ka data use karenge — unhe ek **Provider** ke andar wrap karna padta hai.

Provider = WiFi Router. Jo router ke range mein hai — woh data le sakta hai.

### ⚠️ Next.js Mein Ek Important Cheez — Server vs Client

Next.js mein do tarah ke components hote hain:

| Server Component | Client Component |
|-----------------|-----------------|
| Default — server pe chalta hai | `'use client'` likhna padta hai |
| Fast, SEO friendly | Interactive cheezein — useState, onClick, Context |
| `metadata` export kar sakta hai | `metadata` export nahi kar sakta |

**Problem yeh hai:**

`app/layout.tsx` ek **Server Component** hai by default — wahan `metadata`, fonts sab hota hai. Isse hataana nahi chahiye.

Par `ThemeContext.Provider` ek **Client Component** hai — `'use client'` chahiye.

**Dono ek hi file mein nahi aa sakte!**

Agar seedha `layout.tsx` mein `ThemeContext.Provider` lagao — yeh error aayega:

```
Error: Element type is invalid...
```

---

### Solution — `Providers.tsx` Alag File Banao

Iska solution simple hai — Provider ka kaam **alag file** mein karo.

```
app/
├── context/
│   └── ThemeContext.tsx    ← Context
├── components/
│   └── Providers.tsx       ← naya file — Provider ka kaam yahan
└── layout.tsx              ← Server Component — metadata yahan rahega
```

**Step 1 — `app/components/Providers.tsx` banao:**

```tsx
'use client'   // ← yeh Client Component hai

import { ThemeContext } from '../context/ThemeContext'

export default function Providers({ children }: { children: React.ReactNode }) {
  return (
    <ThemeContext.Provider value="dark">
      {children}
    </ThemeContext.Provider>
  )
}
```

Har line tod ke:

```
'use client'    ← yeh file browser pe chalegi — zaruri hai Provider ke liye

Providers       ← ek component banaya jiska kaam sirf Provider lagana hai

{ children }    ← jo bhi andar aayega (poori app) woh yahan se pass hoga

<ThemeContext.Provider value="dark">   ← Context ka data "dark" set kar raha hai
  {children}                           ← poori app andar wrap ho gayi
```

**Step 2 — `layout.tsx` mein `Providers` use karo:**

```tsx
// app/layout.tsx
import type { Metadata } from "next";
import { Geist, Geist_Mono } from "next/font/google";
import "./globals.css";
import Providers from "./components/Providers";  // ← yeh import karo

const geistSans = Geist({
  variable: "--font-geist-sans",
  subsets: ["latin"],
});

const geistMono = Geist_Mono({
  variable: "--font-geist-mono",
  subsets: ["latin"],
});

export const metadata: Metadata = {
  title: "Create Next App",
  description: "Generated by create next app",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en">
      <body className={`${geistSans.variable} ${geistMono.variable} antialiased`}>
        <Providers>        {/* ← ThemeContext.Provider ki jagah yeh */}
          {children}
        </Providers>
      </body>
    </html>
  );
}
```

**Kya hua yahan?**

```
layout.tsx      → Server Component rehta hai ✅ — metadata kaam karega
Providers.tsx   → Client Component hai ✅ — 'use client' wahan hai
```

Dono apna apna kaam kar rahe hain — koi conflict nahi! 🎉

## STEP 3 — Data Lo (CONSUME) 🤲

Ab koi bhi component — chahe kitna bhi neeche ho — seedha Context se data le sakta hai.

```tsx
// app/components/Button.tsx
'use client'

import { useContext } from 'react'
import { ThemeContext } from '../context/ThemeContext'

export default function Button() {
  const theme = useContext(ThemeContext)

  return (
    <button style={{
      background: theme === 'dark' ? '#333' : '#fff',
      color: theme === 'dark' ? '#fff' : '#333',
      padding: '10px 20px',
      border: '1px solid #ddd',
      borderRadius: '6px',
    }}>
      Click Me
    </button>
  )
}
```

Ab **har line** tod ke:

---

**`import { useContext } from 'react'`**

`useContext` React ka ek hook hai — Context se data lene ke liye. Use karne se pehle import karna padta hai.

---

**`import { ThemeContext } from '../context/ThemeContext'`**

Woh dabba import kar raha hai jo Step 1 mein banaya tha. Usi dabbe se data lenge.

---

**`const theme = useContext(ThemeContext)`**

Tod ke:

```
const theme      ← variable — isme Context ka data aayega
=
useContext       ← React ka function — Context se data nikalta hai
(ThemeContext)   ← kaunsa Context? — woh dabba jo tune banaya
```

Matlab — `theme` mein `"dark"` aa gaya — seedha, bina kisi prop ke!

---

**Koi prop nahi aaya. Koi drilling nahi. Seedha Context se liya.** ✅

## Ab Real Banate Hain — User Toggle Kar Sake

Abhi tak `theme` hamesha `"dark"` tha — static. Real apps mein user **toggle** kar sakta hai.

Iske liye Context ke saath `useState` use karenge.

**Yeh thoda zyada code hai — par har ek naya cheez explain karunga.**

---

### `ThemeContext.tsx` update karo

```tsx
'use client'

import { createContext, useState, useContext } from 'react'
```

**`useState` aur `useContext` bhi import kiye** — kyunki ab dono use honge.

---

```tsx
interface ThemeContextType {
  theme: string
  toggleTheme: () => void
}
```

**Yeh interface kya hai?**

Context mein ab sirf ek string nahi — do cheezein hongi:
- `theme` — current theme (string)
- `toggleTheme` — theme change karne ka function

Toh pehle unka **shape define kiya** — `interface` se.

**`() => void` kya hai?**

Yeh function ka type hai. Tod ke:

```
()      ← is function mein koi argument nahi aata
=>      ← "return karta hai" wala arrow
void    ← kuch return nahi karta (bas kaam karta hai)
```

---

```tsx
const ThemeContext = createContext<ThemeContextType | null>(null)
```

Ab yeh line **completely** tod ke:

```
createContext         ← React ka function — context banata hai
             <        ← generic shuru (tune 00.1 mein padha)
              ThemeContextType  ← Context mein yeh shape ka data aayega
                      |         ← OR
                       null     ← ya null bhi ho sakta hai
                           >    ← generic khatam
                            (null)  ← default value — abhi khaali hai
```

**`| null` kyun?**

Abhi Context create ho raha hai — koi data nahi. Khaali hai. Toh TypeScript ko batana pada — "abhi null hai, baad mein `ThemeContextType` aayega."

---

```tsx
export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState<string>('light')

  function toggleTheme() {
    setTheme(theme === 'light' ? 'dark' : 'light')
  }

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}
```

**`toggleTheme` function kya karta hai?**

```
theme === 'light'   ← agar abhi light hai
?  'dark'           ← toh dark kar do
:  'light'          ← warna light kar do
```

**`value={{ theme, toggleTheme }}` kya hai?**

```
value=              ← Provider ko data de raha hai
      {             ← JSX mein JS likhna ho toh curly braces
       {            ← object shuru
        theme,      ← theme variable (current value)
        toggleTheme ← toggleTheme function
       }            ← object khatam
      }             ← JSX wale curly braces khatam
```

---

```tsx
export function useTheme() {
  const context = useContext(ThemeContext)
  if (!context) throw new Error('ThemeProvider ke andar use karo!')
  return context
}
```

**`useTheme()` kyun banaya?**

Har component mein yeh likhna padta:

```tsx
import { useContext } from 'react'
import { ThemeContext } from '../context/ThemeContext'
const context = useContext(ThemeContext)
if (!context) ...
```

Boring aur repetitive. `useTheme()` ek **shortcut** hai — ek line mein sab. Isko **custom hook** kehte hain.

**`if (!context) throw new Error(...)`** kya hai?

Safety check — agar koi `ThemeProvider` ke bahar `useTheme()` use kare toh helpful error milega.

---

### Poori `ThemeContext.tsx` File

```tsx
// app/context/ThemeContext.tsx
'use client'

import { createContext, useState, useContext } from 'react'

interface ThemeContextType {
  theme: string
  toggleTheme: () => void
}

const ThemeContext = createContext<ThemeContextType | null>(null)

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState<string>('light')

  function toggleTheme() {
    setTheme(theme === 'light' ? 'dark' : 'light')
  }

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}

export function useTheme() {
  const context = useContext(ThemeContext)
  if (!context) throw new Error('ThemeProvider ke andar use karo!')
  return context
}
```

### Ab `Providers.tsx` Bhi Update Karo

`ThemeProvider` ab `ThemeContext.tsx` mein hi hai — toh `Providers.tsx` mein wahi use karo:

```tsx
// app/components/Providers.tsx
'use client'

import { ThemeProvider } from '../context/ThemeContext'

export default function Providers({ children }: { children: React.ReactNode }) {
  return (
    <ThemeProvider>
      {children}
    </ThemeProvider>
  )
}
```

`layout.tsx` same rahega — koi change nahi.

## Koi Bhi Component Use Kare

```tsx
// app/components/Button.tsx
'use client'

import { useTheme } from '../context/ThemeContext'

export default function Button() {
  const { theme, toggleTheme } = useTheme()

  return (
    <button
      onClick={toggleTheme}
      style={{
        background: theme === 'dark' ? '#333' : '#fff',
        color: theme === 'dark' ? '#fff' : '#333',
        padding: '10px 20px',
        border: '1px solid #ddd',
        borderRadius: '6px',
        cursor: 'pointer'
      }}
    >
      {theme === 'dark' ? '☀️ Light Mode' : '🌙 Dark Mode'}
    </button>
  )
}
```

## Poora Flow Ek Jagah

```
ThemeContext.tsx    → Context + ThemeProvider + useTheme banaya
       ↓
Providers.tsx       → ThemeProvider ko wrap kiya ('use client' yahan)
       ↓
layout.tsx          → Providers se poori app wrap ki (Server Component rehta hai ✅)
       ↓
Button.tsx          → useTheme() se seedha data liya — koi prop nahi
       ↓
toggleTheme()       → click hua → state badli → poori app re-render
```

## Props vs Context — Kab Kya?

| Situation | Use karo |
|-----------|----------|
| Direct parent → child data | **Props** |
| Chhota tree, 1-2 levels | **Props** |
| Theme, logged-in user, language | **Context** |
| Bahut saare components ko data chahiye | **Context** |
