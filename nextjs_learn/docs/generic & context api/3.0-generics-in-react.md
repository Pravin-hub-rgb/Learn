# 3.0 — Generics React Mein 🔵

## Pehle Yaad Karo

Pichli file mein tune seekha:

- `<>` — function ko type dena
- `T` — placeholder naam
- `|` — OR, do mein se koi ek
- `null` — khaali, kuch nahi

Ab dekhte hain — yeh React mein **kahan kahan already use ho raha tha** — aur tune notice nahi kiya tha. 😄

---

## useState — Tu Pehle Se Generic Use Kar Raha Tha

```ts
const [naam, setNaam] = useState<string>("")
```

Tod ke dekho:

```
useState          ← React ka function
        <string>  ← generic: "is state mein string aayegi"
                ("")  ← default value: shuru mein khaali string
```

Aur yeh sab valid hain:

```ts
useState<string>("")       // string state
useState<number>(0)        // number state
useState<boolean>(false)   // boolean state
```

**Ek hi function — `useState` — alag alag types ke saath.** Yahi generic ka fayda hai.

---

## Kabhi Kabhi TypeScript Khud Samajh Jaata Hai

Yeh bhi valid hai:

```ts
useState("")      // TypeScript khud samjha — string hai
useState(0)       // TypeScript khud samjha — number hai
useState(false)   // TypeScript khud samjha — boolean hai
```

**Toh `<>` likhna hamesha zaroor nahi.**

Par kabhi kabhi TypeScript khud **nahi samajh paata.** Tab likhna padta hai.

Kab? — Jab default value `null` ho:

```ts
useState(null)
// TypeScript confused — yeh null hai, par baad mein kya aayega??
```

Tab batana padta hai:

```ts
useState<string | null>(null)
//       ^^^^^^^^^^^^^
//       "null se shuru, par baad mein string aayegi"
```

---

## `string | null` — Dobara Yaad Karo

```ts
useState<string | null>(null)
```

Matlab:

- Shuru mein → `null` (khaali, koi data nahi)
- Baad mein → `string` aa sakti hai

Real example — user ka naam fetch karna hai API se:

```ts
const [naam, setNaam] = useState<string | null>(null)
//                                              ^^^^
//                                    abhi koi naam nahi

// API se data aaya
setNaam("Ali")  // ab string aa gayi ✅
```

## Interface Ke Saath Generics

Tu `interface` jaanta hai:

```ts
interface User {
  naam: string
  age: number
}
```

Yeh `User` ek **shape** hai — ek type hai.

Isse bhi `<>` mein de sakte hain:

```ts
useState<User | null>(null)
```

Matlab:

- Shuru mein → `null`
- Baad mein → `User` shape ka object aayega

```ts
setUser({ naam: "Ali", age: 25 })  // ✅ User shape match kiya
setUser({ naam: 42, age: 25 })     // ❌ TypeScript error! naam string hona chahiye
```

---

## Ek Chota Example — Sab Saath

```tsx
'use client'
import { useState } from 'react'

interface User {
  naam: string
  age: number
}

export default function UserCard() {
  // Shuru mein koi user nahi — null
  // Baad mein User shape ka object aayega
  const [user, setUser] = useState<User | null>(null)

  function loadUser() {
    // Maan lo API se data aaya
    setUser({ naam: "Ali", age: 25 })
  }

  return (
    <div>
      {user === null ? (
        <button onClick={loadUser}>User Load Karo</button>
      ) : (
        <p>{user.naam} — {user.age} saal</p>
      )}
    </div>
  )
}
```

## Ab Context Ke Liye Ready Ho

Context mein bhi yahi sab hoga:

```ts
createContext<ThemeContextType | null>(null)
```

Wahi pattern:
- `<ThemeContextType | null>` — generic, type bata raha hai
- `(null)` — default value

**Fark sirf itna hai — `useState` ki jagah `createContext` hai.**
