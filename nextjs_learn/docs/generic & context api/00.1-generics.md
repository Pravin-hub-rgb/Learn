# 00.1 — Generics 🧠

## Pehle Ek Problem Dekho

Tu jaanta hai yeh:

```ts
const age: number = 25
const naam: string = "Ali"
```

TypeScript ko bata diya — `age` mein number aayega, `naam` mein string.

Ab maan le tune ek function banaya jo kuch bhi leke wapas de deta hai:

```ts
function wapasKaro(cheez) {
  return cheez
}
```

Ab tune use kiya:

```ts
wapasKaro("Ali")     // string di
wapasKaro(25)        // number di
wapasKaro(true)      // boolean di
```

**TypeScript ka problem:**

TypeScript poochhega — "bhai, is function mein kya aayega? String? Number? Kuch bhi??"

TypeScript ko type pata nahi — toh woh `any` maan leta hai. Aur `any` matlab — **koi bhi cheez chal jaayegi, koi check nahi.** Yeh dangerous hai.

---

## Solution Sochte Hain

Option 1 — Teen alag functions banao:

```ts
function wapasKaroString(cheez: string): string {
  return cheez
}

function wapasKaroNumber(cheez: number): number {
  return cheez
}

function wapasKaroBoolean(cheez: boolean): boolean {
  return cheez
}
```

Kaam karta hai. Par **kitna repetition** hai! Ek hi kaam — teen baar likhna pada.

**Koi better tarika hai?**

Haan. Yahi hai **Generics.**

---

## Generic — Seedha Idea

Generic ka idea simple hai:

> "Function ko batao — **abhi nahi, baad mein** type bataunga. Jab use karunga tab bataunga."

Jaise ek **khaali slot** chhod dete hain — baad mein bharenge.

```ts
function wapasKaro<T>(cheez: T): T {
  return cheez
}
```

Ruk — yeh `<T>` aur `T` kya hai? **Ek ek karke toodte hain.**

---

## `<T>` — Yeh Slot Ka Naam Hai

```ts
function wapasKaro<T>(cheez: T): T {
```

**`<T>`** — function ke naam ke baad likha. Yeh TypeScript ko bata raha hai:

> "Oye TypeScript, main ek slot bana raha hun. Iska naam `T` hai. Jab koi is function ko use karega, tab woh batayega `T` kya hai."

**`T` koi special word nahi hai.** Yeh bas ek **placeholder naam** hai — jaise variable ka naam hota hai. `T` isliye use karte hain kyunki "Type" ka short form hai. Par tu `Cheez`, `X`, `Kuch` — kuch bhi likh sakta hai.

---

## Ab Poori Line Samajh

```ts
function wapasKaro<T>(cheez: T): T {
  return cheez
}
```

Isko tod ke padho:

| Part | Matlab |
|------|--------|
| `function wapasKaro` | function ka naam |
| `<T>` | ek slot bana raha hun, naam `T` |
| `(cheez: T)` | parameter `cheez` ka type wahi hoga jo `T` mein aayega |
| `: T` | return bhi wahi type hoga jo `T` mein aaya |

Matlab — **jab use karunga tab type bataunga, aur wahi type andar bhi hoga, bahar bhi hoga.**

---

## Ab Use Karte Hain

```ts
wapasKaro<string>("Ali")
```

Yahan tune bataya — `T = string`

Toh TypeScript samjha:
- Parameter `cheez` → string hoga
- Return → string hoga

```ts
wapasKaro<number>(25)
```

Yahan tune bataya — `T = number`

Toh TypeScript samjha:
- Parameter `cheez` → number hoga
- Return → number hoga

**Ek hi function — alag alag types ke saath.** Koi repetition nahi. 🔥

---

## `:` aur `<>` Ka Fark — Once and For All

Tu pehle se jaanta hai:

```ts
const naam: string = "Ali"
//          ^^^^^^
//    variable ka type bata rahe hain
```

Yeh `:` hai — **"yeh cheez is type ki hai"** — seedha bata diya.

Ab Generic:

```ts
wapasKaro<string>("Ali")
//        ^^^^^^
//   function ko type de rahe hain
```

Yeh `<>` hai — **"is function ko is type ke saath use karo"** — function ko type parameter de rahe hain.

Simple fark:

```
:      → variable ya return ka type batao
< >    → function ko batao ki kis type ke saath kaam karna hai
```

---

## `|` Kya Hai — OR Wala

Yeh TypeScript mein **"ya toh... ya toh..."** hota hai.

```ts
const cheez: string | number = "Ali"
// cheez mein string bhi aa sakti hai, number bhi
```

```ts
const cheez: string | number = 25
// yeh bhi valid hai ✅
```

Real life mein — ek form field mein phone number ho sakta hai (number) ya naam (string). Dono allowed hain.

```ts
const cheez: string | null = null
// string bhi aa sakti hai, ya null bhi
```

**`null` kya hai?** — "khaali, kuch nahi." Jaise glass mein paani nahi hai.

---

## Ab Teeno Cheezein Ek Saath

Tune yeh wali line dekhi thi jo confuse kar rahi thi:

```ts
createContext<ThemeContextType | null>(null)
```

Ab isko tod ke dekho — **ab samjh aayega:**

```
createContext                 ← ek function hai (React ka)
          <ThemeContextType | null>   ← generic: "andar ya toh ThemeContextType aayega ya null"
                              (null)  ← default value: abhi khaali hai, toh null
```

**`ThemeContextType`** — ek interface ka naam hai (shape define karta hai, tune `00-typescript-basics.md` mein padha tha).

**`| null`** — "ya null bhi ho sakta hai" — kyunki abhi Context khaali hai, koi data nahi.

**`(null)`** — default value de raha hai null — "jab tak Provider na aaye, null raho."

---

## Ek Aur Example — `useState` Bhi Generic Hai!

Tune `useState` use kiya tha. Woh bhi generic function hai:

```ts
useState<string>("")
```

Tod ke dekho:

```
useState       ← React ka function
        <string>   ← generic: "is state mein string aayegi"
                ("")   ← default value: shuru mein khaali string
```

Aur tune kabhi yeh bhi likha hoga:

```ts
useState<number>(0)
useState<boolean>(false)
```

**Tu generic pehle se use kar raha tha — bas pata nahi tha!** 😄

---

## Summary — Ek Baar Sab

| Cheez | Kya hai | Example |
|-------|---------|---------|
| `:` | Variable ya return ka type | `const naam: string` |
| `<>` | Function ko type dena (generic) | `useState<string>` |
| `T` | Placeholder naam — koi bhi naam chalega | `function karo<T>` |
| `\|` | OR — do mein se koi ek | `string \| null` |
| `null` | Khaali — kuch nahi | `useState<string \| null>(null)` |

---

# 00.1 — Generics 🧠

## Pehle Ek Baat Yaad Rakho — TS Ka Funda

TypeScript ka poora point yeh hai:

> **Coding karte samay hi error pakad lo — taaki app chalte waqt koi dikkat na ho.**

Matlab — VS Code mein red underline aa jaaye, na ki user ke browser mein app toot jaaye.

Iske liye TypeScript ko **pehle se batana padta hai** — function kya receive karega, kya return karega.

---

## Jab Type Pehle Se Pata Ho — Simple Hai

Maan le ek function banana hai jo Car ki info le aur kuch kaam kare.

Pehle `Car` ka shape define karo:

```ts
interface Car {
  door: number
  name: string
}
```

Ab function banao — pehle se pata hai `Car` aayega:

```ts
function whichCar({ door, name }: Car) {
  console.log(`${name} mein ${door} doors hain`)
}
```

**Simple. Direct. Type pehle se pata tha — seedha likh diya.**

---

## Problem — Jab Type Pehle Se Pata Na Ho

Ab maan le tu ek aisa function banana chahta hai jo **kisi bhi** vehicle ke saath kaam kare — `Car`, `Truck`, `Van` — kuch bhi.

**Option 1 — Har type ke liye alag function banao:**

```ts
function carKaam(cheez: Car): Car {
  return cheez
}

function truckKaam(cheez: Truck): Truck {
  return cheez
}

function vanKaam(cheez: Van): Van {
  return cheez
}
```

Kaam karta hai — par **kitni repetition!** Ek hi kaam — baar baar likhna pad raha hai.

Aur agar kal `Bike` type aa jaaye — phir ek naya function? 😩

---

## Solution — Generic Aaya Picture Mein! 🔥

Generic ka simple idea:

> **"Abhi type mat batao — jab function use karein tab batao."**

```ts
function vehicle<T>(cheez: T): T {
  return cheez
}
```

Ab yeh function **kisi bhi type** ke saath kaam karega.

---

## `<T>` Kya Hai — Bilkul Simple

`T` ek **placeholder** hai — ek khaali jagah.

Jaise tune variable mein value store ki:

```ts
const x = 5   // x mein value store hui
```

Same tarah `T` mein **type** store hogi — jab use karega tab:

```ts
vehicle<Truck>(...)   // T mein Truck store ho gaya
vehicle<Van>(...)     // T mein Van store ho gaya
```

**`T` koi special word nahi hai.** Tu `Kuch`, `X`, `VehicleType` — kuch bhi likh sakta hai. `T` isliye use karte hain kyunki "Type" ka short form hai — bas convention hai.

---

## Tera Wala Example — Truck aur Van

```ts
interface Truck {
  door: number
  name: string
}

interface Van {
  door: number
  color: string
  tyre: number
}

function vehicle<T>(cheez: T): T {
  return cheez
}
```

Ab use karo:

```ts
vehicle<Truck>({ door: 2, name: "Ford F150" })
// T = Truck — TypeScript jaanta hai andar Truck aayega

vehicle<Van>({ door: 1, color: "Red", tyre: 6 })
// T = Van — TypeScript jaanta hai andar Van aayega
```

**Ek hi function — alag alag types ke saath.** Koi repetition nahi! 🎉

Aur agar galat data diya:

```ts
vehicle<Truck>({ door: 1, color: "Red" })
// ❌ TS Error! Truck mein 'color' nahi hota — 'name' hona chahiye
```

**Yahi TS ka fayda — coding karte waqt hi pakad liya!**

---

## Poori Line Ka Breakdown

```ts
function vehicle<T>(cheez: T): T {
```

| Part | Matlab |
|------|--------|
| `function vehicle` | function ka naam |
| `<T>` | placeholder bana raha hun — naam `T`, baad mein bharunga |
| `(cheez: T)` | jo bhi `T` hoga, `cheez` usi type ka hoga |
| `: T` | return bhi usi type ka hoga jo `T` mein aaya |

---

## `:` Aur `<>` Ka Fark — Once and For All

Tu pehle se jaanta hai:

```ts
const naam: string = "Ali"
//          ^^^^^^
//    yeh variable string type ka hai — seedha bata diya
```

Yeh `:` hai — variable ya function ka type **seedha** bata do.

Generic mein:

```ts
vehicle<Truck>({ door: 2, name: "Ford F150" })
//      ^^^^^
//   function ko bata raha hai — is baar Truck type ke saath kaam karo
```

Yeh `<>` hai — function ko **"abhi"** type bata rahe hain use karte waqt.

```
:      → type seedha batao (variable, return)
< >    → function ko use karte waqt type batao (generic)
```

---

## `|` Kya Hai — OR Wala

```ts
const cheez: string | number = "Ali"
// cheez mein string bhi aa sakti hai, number bhi — dono valid
```

```ts
const cheez: string | null = null
// string bhi aa sakti hai, ya null (khaali) bhi
```

**`null` kya hai?** — "khaali, kuch nahi." Jaise glass mein paani nahi.

---

## Ab Woh Confusing Line Dekho — Khud Samajh Aayegi

```ts
createContext<ThemeContextType | null>(null)
```

Tod ke:

```
createContext           ← React ka function — context banata hai
             <          ← generic shuru — type batao
              ThemeContextType   ← context mein yeh interface wala data aayega
                        |        ← OR
                         null    ← ya null bhi ho sakta hai (shuru mein khaali hai)
                             >   ← generic khatam
                              (null)  ← default value — abhi koi data nahi, toh null
```

**Exactly wahi pattern hai** jo tune `vehicle<Truck>(...)` mein dekha — bas `vehicle` ki jagah `createContext` hai, aur `Truck` ki jagah `ThemeContextType | null` hai.

---

## `useState` Bhi Generic Hai — Tu Pehle Se Use Kar Raha Tha!

```ts
useState<string>("")
```

Same pattern:

```
useState       ← React ka function
        <string>   ← generic: "is state mein string aayegi"
                ("")   ← default value: shuru mein khaali string
```

**Tu generics pehle se use kar raha tha — bas naam nahi pata tha!** 😄

---

## Summary — Ek Baar Sab

| Situation | Kya karo | Example |
|-----------|----------|---------|
| Pehle se pata hai type | Seedha likh do | `function whichCar({}: Car)` |
| Pehle se pata nahi | Generic use karo | `function vehicle<T>(cheez: T)` |
| Use karte waqt type batao | `<>` mein likho | `vehicle<Truck>(...)` |
| Do mein se koi ek type | `\|` use karo | `string \| null` |
| Khaali, koi data nahi | `null` | `useState<string \| null>(null)` |
